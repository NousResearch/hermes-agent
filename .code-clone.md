# 自动化克隆流程 (Git Clone Automation)
## 路径识别
根据宿主机操作系统动态定位桌面下 git_data 路径：

Windows: %USERPROFILE%\Desktop\git_data\
macOS / Linux: ~/Desktop/git_data/

## 执行逻辑
前置校验：
1. 识别用户输入的 Git SSH 协议地址（格式：git@gitlab.dongqiudi.com:group/repo.git），当用户提供了不规范的git地址，提示用户检查 gitlab 项目名称。
2. 检查`git clone`下来的项目名称是否为 `dongqiudi-`开头，如果非`dongqiudi-`开头，绝对不允许帮用户克隆，并提示项目名称必须以 `dongqiudi-` 开头，并举例 `dongqiudi-demo`。
3. 检查 git_data 目录是否存在，不存在则创建。


环境切换：进入对应系统的桌面下 git_data 路径。

克隆操作：执行 git clone <SSH_URL> 指令。

路径提取：提取仓库名称并进入该目录，获取当前 绝对路径。

##  交互指令规范
任务完成后，必须回显以下信息：

执行状态：确认克隆结果。

路径回显：打印项目在本地的绝对路径。

部署引导：

操作确认：请将待部署的 HTML 页面及相关静态资源文件复制到以下目录：
[本地绝对路径]
