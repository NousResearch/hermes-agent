## Summary

- add `status` as a compatibility alias for `hermes kanban stats`
- route `/kanban status` through the existing stats handler
- document the alias in `/kanban` slash help

## Problem

`/kanban status` currently fails argument parsing because `status` is not a valid
kanban action, even though users reasonably expect it to behave like a board
summary command.

## Solution

Expose `status` as an alias of the canonical `stats` subcommand in the CLI
parser, dispatch table, and slash-command help text.

## Verification

- `run_slash("help")` includes `status`
- `run_slash("status")` returns the board stats output and starts with `By status:`
