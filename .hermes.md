# Hermes Agent Compact Project Context

This file is the Hermes-specific fast-path router for work inside `~/.hermes/hermes-agent`. The loader uses controlled fusion: `.hermes.md` loads first, then the AGENTS instruction chain loads root → current directory when present. Compatibility files such as `CLAUDE.md` or `.cursorrules` are references unless the source ledger says they were loaded.

## Repo-specific deltas

- Keep changes surgical: inspect first, edit the smallest safe scope, then verify with evidence.
- Use `get_hermes_home()` / profile-aware helpers for Hermes paths; do not hardcode `~/.hermes` in source code unless documenting the default path.
- Do not change tool schemas, system-prompt assembly, memory, gateway, provider routing, prompt-loader behavior, or HASOS runtime injection without targeted tests.
- Keep HASOS runtime compact: load the local runtime-ready slice, not raw packs, long templates, copied external prompts, or full skill bundles.

## Key files

- `run_agent.py` — AIAgent conversation loop and runtime integration.
- `agent/prompt_builder.py` — system prompt, memory/skills/project-context assembly, HASOS runtime guidance.
- `model_tools.py`, `toolsets.py`, `tools/registry.py` — tool discovery, dispatch, and toolset membership.
- `hermes_cli/config.py` — default config and migrations.
- `gateway/` — messaging gateway and platform adapters.
- `tests/` — pytest suite; prefer targeted tests first.

## Focused verification

Use the repo venv when present because system `python3` may be too old:

```bash
venv/bin/python3.11 -m py_compile agent/prompt_builder.py run_agent.py
venv/bin/python3.11 -m pytest tests/agent/test_prompt_builder.py tests/run_agent/test_run_agent.py -q -o addopts=
```

For broader source changes, expand to affected module tests or `scripts/run_tests.sh` before any commit or release.
