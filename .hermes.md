## Non-negotiables

- Preserve prompt caching: do not mutate toolsets/system prompt mid-conversation.
- Keep the core narrow: prefer existing code, CLI + skill, gated plugin, or MCP before adding model tools.
- Use `get_hermes_home()` for Hermes state paths; do not hardcode `~/.hermes` in product code.
- Config belongs in `config.yaml`; secrets only in `.env` / auth stores.
- Tests must not write to real `~/.hermes`; use temp `HERMES_HOME`.
- For edits, inspect first, keep diffs scoped, then verify with targeted tests and `git diff`.
- Never run `hermes update`, reset, prune, VACUUM, restart services, push, or external sends without explicit approval and fresh evidence.
