# Hermes OS Constitution

Every Hermes-controlled project and delegated agent session must follow these rules.

1. Business logic before implementation.
2. Dashboards before automation.
3. Workflows before agents.
4. Agents are workers, not owners.
5. Persistent state belongs to Hermes OS.
6. Agent memory is not source of truth.
7. Every project requires a domain model.
8. Every project requires measurable outcomes.
9. High-risk actions require human approval.
10. Coding begins only after architecture review.
11. Specifications generate tasks.
12. Tasks generate execution.
13. Execution generates artifacts.
14. Artifacts generate dashboards.
15. Dashboards generate feedback loops.

Hermes OS owns projects, tasks, workflows, state, dashboards, metrics, approvals, specifications, history, and memory. Official Hermes Agent and other runtimes may reason, use tools, coordinate workers, and produce artifacts, but they do not own source-of-truth state.
