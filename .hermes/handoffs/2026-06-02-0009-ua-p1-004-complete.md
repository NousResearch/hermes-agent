- Existing ledgers are append-only: existing content is preserved and compact sections are appended.
- EOF whitespace is normalized before append so existing content ends with exactly one newline before the UA section.
- Runtime readiness summaries are recorded when present:
  - `verification_status`
  - blockers capped to top 3
- Target cleanliness summaries are recorded when present:
  - `target_cleanliness_status`
  - `unexpected_changes_count`
- Project-state sections record artifact bundle path only, not full JSON blobs.
- Append exceptions are non-fatal to UA runs.
- Manifests now include:
  - `project_state_append_status`: `not_attempted`, `success`, or `failed`
  - `project_state_append_error`: short sanitized message on failure, otherwise null
