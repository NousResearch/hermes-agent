## 2024-05-12 - The High Cost of `copy.deepcopy` on Message Histories
**Learning:** `copy.deepcopy(api_messages)` is an O(N) operation where N is the size of the conversation history. Because AI applications pass increasingly large message histories (containing MBs of string text from long files, command outputs, or tool responses) back and forth, `deepcopy` can silently introduce hundreds of milliseconds of latency per inference call. In an agent loop, this adds up quickly. A shallow copy combined with targeted duplication of only the altered nested structures is an order of magnitude faster and maintains safety.
**Action:** Replace `copy.deepcopy()` with shallow dict copies (`[msg.copy() for msg in api_messages]`) when modifying specific fields (like `content` or `cache_control`) in message histories.
