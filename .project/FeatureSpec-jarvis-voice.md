## 2) แผนงานแบ่ง Phase (id ใช้ต่อใน Comply/Relay ได้เลย)

### P0 — ปิดเทสต์ค้าง (งานคนล้วน ~5 นาที · ไม่มีโค้ดใหม่)
- P0-I1: เจ้าของดับเบิลคลิก jarvis-start.command → คุย 3 ประโยคยาว → ลองพูดแทรก → กดปุ่มข้างเมาส์สลับเปิด/ปิด
- ตรวจ: คำตอบจบทุกประโยค (tier 5) · log มี ⏱ ทุกเทิร์น · ปุ่มเมาส์ขึ้น 🖱 ใน log · รีบูต 1 รอบแล้วตื่นเอง
- เกณฑ์ผ่าน: เจ้าของยืนยันในแชต · ไม่ผ่าน → หยุดทั้งแผน วินิจฉัยจาก log ก่อน

### P1 — เข้าระบบ git + สวิตช์ปุ่มเมาส์
- P1-I1 (Codex · ไม่ async): เข้า git อย่างระวัง — **ห้าม `git add .`** · ลำดับ: audit `git status` → branch `feature/jarvis-v2-shell` → `.gitignore` ระบุ path เต็ม: `scripts/jarvis-voice/.env` · `scripts/jarvis-voice/.venv-gemini/` · `scripts/jarvis-voice/.hermes/` · `scripts/jarvis-voice/__pycache__/` → `git add` เฉพาะ pathspec ไฟล์ .py/.sh/.command/README + `.project/FeatureSpec-jarvis-voice.md` → เกณฑ์ผ่าน: `git ls-files` ไม่มี .env/.venv/.hermes/ไฟล์นอกสาย · **ระวัง stash "task_d248bfc4" ของสายแม่บท — ห้ามแตะ**
- P1-I2 (Opus): ~~ปุ่มเมาส์~~ **เสร็จแล้ว 2026-07-06** (log 🖱 ทุกปุ่มพิเศษ · unknown=toggle · middle ไม่ผูก · debounce 0.3 วิ · JARVIS_MOUSE_TOGGLE=0 ปิดได้ · Grok+Codex agree) — เหลือ tier 5 ใน P0-I1
- ความเสี่ยง: โปรแกรมยี่ห้อเมาส์ดักปุ่มก่อน → log เงียบ = ปิดโปรแกรมยี่ห้อแล้วลองใหม่

### P2 — โหมดอัจฉริยะ: พิมพ์แทน-หรือ-คุย ตามช่องที่โฟกัส (แทน Wispr Flow)
- **P2-I0 ด่านพิสูจน์ก่อนสร้าง (Opus — บังคับทำก่อน ห้ามข้าม · Grok#1+Codex#3):** โมเดล native-audio รับเฉพาะ response_modalities=["AUDIO"] — ต้อง probe พิสูจน์: (a) วิธีปิดเสียงตอบตอนโหมดพิมพ์ (เปิด input transcription + ทิ้งเสียงใน recv หรือ ต่อสายใหม่ config เฉพาะโหมด) (b) รูปทรง event ของ transcript จริง (c) ไม่พัง half-duplex · เกณฑ์: transcript ไทยถูก + ลำโพงเงียบ 100% ตอนโหมดพิมพ์ · ห้ามเริ่ม I1-I4 ก่อนผ่าน
- **P2-Gate สิทธิ์เครื่อง (งานคน · Grok#5):** ยืนยัน Accessibility + ไมค์ให้ Python ที่ plist ชี้ · ทดสอบ paste 1 ครั้งผ่าน launchd จริง
- P2-I1 (Opus): `focused_text_field()` ผ่าน AXUIElementCreateSystemWide (role: AXTextField/AXTextArea/AXComboBox) + เทสต์เดี่ยว · **AXSecureTextField (ช่องรหัสผ่าน) = ห้ามพิมพ์ ถือเป็นโหมดคุย** (Codex#3)
- P2-I2 (Opus — async): เปิด input transcription ตามทางที่ P2-I0 พิสูจน์ · เก็บ transcript ต่อเทิร์น
- P2-I3 (Opus): ตัวพิมพ์ — clipboard + Cmd+V (pynput) · คืน clipboard เดิมเสมอแม้ error (try/finally)
- P2-I4 (Opus): กติกาเลือกโหมด — โฟกัสช่องพิมพ์ = พิมพ์แทน · นอกช่อง = คุย · ขึ้นต้น "จาร์วิส" = บังคับโหมดคุยเสมอ
- ตรวจ (Codex#5): พิมพ์ลง TextEdit + ช่องเว็บ Chrome ถูก ≥9/10 (tier 5) · **เทสต์ถดถอย: ปุ่มลัด+ปุ่มเมาส์ยังใช้ได้ · โหมดคุยไม่เพี้ยน · ต่อสายใหม่หลัง 20 นาทีรอด · เสียงย้อนไม่ตัดคำตอบ** · ปิด Wispr Flow ชั่วคราวตอนทดสอบ (ยังไม่ยกเลิกบัญชี) · Grok ตรวจโค้ดทุก issue
- เกณฑ์ปิดเฟส: เจ้าของใช้แทน Flow ได้ 1 งานจริง

### P3 — พจนานุกรม (ศัพท์เฉพาะ + แก้คำผิดบ่อย)
- P3-I1 (Codex/Grok): `vocab.yaml` — `terms:` (ศัพท์เฉพาะ: Asana, Opus, linux-nat, ชื่อลูกค้า) + `fixes:` (คำผิด→คำถูก: กี่มวง→กี่โมง)
- P3-I2 (Opus): ฉีด terms เข้า system_instruction → ฟังถูกตั้งแต่หู · **คำใหม่มีผลตอนต่อสายรอบถัดไป — เพิ่มคำแล้วให้ระบบต่อสายใหม่เอง 1 ครั้ง** (Grok)
- P3-I3 (Codex/Grok): ตัวแก้คำจาก fixes ก่อนพิมพ์ (เฉพาะโหมดพิมพ์)
- P3-I4 (Opus — async): เพิ่มคำด้วยเสียง "จาร์วิส จำคำว่า X สะกดว่า Y" → เขียน vocab.yaml
- ตรวจ: ชุดคำเคยเพี้ยน 10 คำ ถูก ≥9 (tier 5) · vocab.yaml ไม่พังเมื่อไฟล์ว่าง/ไม่มี · ตรวจสลับค่าย

### P4 — Dashboard หน้าสรุปการใช้งาน (แบบ Wispr Flow · แบ่งงานตาม Grok#2+Codex#4)
- P4-I1a (Opus — async): ตะขอเก็บเหตุการณ์ใน jarvis_app.py — ทุกเทิร์น 1 แถว (เวลา · โหมด · วินาทีเริ่มตอบ · จำนวนคำ) · สัญญาข้อมูลนิ่งก่อน I1b
- P4-I1b (Codex เขียน · Grok ตรวจ): `stats.py` — SQLite `stats.db` + ตัวสรุปรายวัน ตามสัญญาจาก I1a
- P4-I2 (Codex เขียน · Grok ตรวจ): หน้าเว็บในเครื่อง `http://localhost:8899` — การ์ดรวม · กราฟรายวัน · streak · **มาตรวัดโควตา Gemini วันนี้** (นับเอง — Google ไม่มีให้ดึงฟรี) · หน้าจัดการพจนานุกรม (เพิ่ม/ลบคำใน vocab.yaml)
- P4-I3 (Opus — async · Grok#4): คำสั่งเสียง "จาร์วิส เปิดหน้าสรุป" → `open http://localhost:8899` ตรงๆ **ห้ามผ่าน gateway P5** (gateway บล็อก localhost โดยดีไซน์ — ถูกแล้ว อย่าเจาะ)
- ตรวจ: คุย 1 เทิร์นมีแถวใหม่ใน stats.db · curl = 200 (tier 2) · grep เจอตัวเลขจริง (tier 3) · screenshot + เจ้าของเปิด (tier 4-5) · เพิ่มคำผ่านเว็บแล้ว vocab.yaml เปลี่ยนจริง

### P5 — เครื่องมือเพิ่มทีละตัวผ่านด่านปลอดภัย (เป้าเดิมของงาน)

- ตรวจ: แต่ละ tool สั่งจริง 3/3 (tier 5) + **เทสต์ปฏิเสธ: ของนอก allowlist โดนกันจริง + log ไม่มีค่า .env หลุด** (Codex#5) · Grok ตรวจโค้ด

1. เปิดเครื่อง → จาร์วิสตื่นเอง (มีแล้ว) · 2. กดปุ่มลัด/ปุ่มเมาส์ แล้วพูด (มีแล้ว·รอ tier 5) · 3. ถามทั่วไป → ตอบเสียง ~1-3 วิ (มีแล้ว) · 4. ถามยาก → "ขอถามพี่ Opus" → คำตอบลึกเป็นเสียง (P7) · 5. สั่งงานคอม "เปิด TradingView วิเคราะห์ EURUSD" → มือเปิดเว็บ+ตามองจอ+ตอบ (P5+P6) · 6. จิ้มช่องพิมพ์แล้วพูด = พิมพ์แทน (P2 แทน Wispr Flow · ยกเลิก Flow เมื่อพอใจ P2+P3 เท่านั้น) · 7. "เปิดหน้าสรุป" → dashboard (P4) · 8. อนาคต: Lark + เครื่องพนักงาน + สมองลับในบริษัท (P7)

### ผังระบบ 5 ชั้น
คุณ (ไมค์/ปุ่ม) → ตัวคุมจาร์วิสบนเครื่อง (โหมดคุย/พิมพ์ · กันเสียงย้อน · เสียงคั่น · สถิติ) → Gemini Live หูปาก (2.5→ทดสอบ 3.1 · ฟรี→จ่ายไม่แก้โค้ด) → ด่านปลอดภัย gateway (allowlist · ยืนยัน · log) → ชั้นล่าง 2 ฝั่ง: สมองเสริม (Opus · Grok · Local LLM อนาคต — เลือกโดยตัวจ่ายงานตามชนิดงาน+ชั้นความลับ) + มือ (มองจอ/เบราว์เซอร์ · เปิดโปรแกรม/run_shell ยืนยันก่อน · Lark อนาคต) → คำตอบไหลกลับให้ Gemini พูด

## 4) ความเสี่ยงรวมที่ AI ผู้ทำตามแผนต้องรู้
1. เสียงย้อนลำโพง→ไมค์ = ศัตรูหลัก — ฟีเจอร์ใหม่ห้ามพัง half-duplex/turn_taking
2. macOS TCC: สิทธิ์ผูกกับ Python.app (homebrew 3.12) — อัพเกรด brew python = สิทธิ์หลุด + ต้องแก้ path ใน plist
3. Wispr Flow: ปิดชั่วคราวตอนทดสอบ P2 · ยกเลิกจริงเมื่อเจ้าของพอใจ P2+P3 เท่านั้น
4. Gemini session ตาย 10-15 นาที — resumption มีแล้ว ห้ามถอด
5. `.env` (กุญแจ) ห้ามเข้า git · ห้ามเข้า context AI
6. **ไฟล์สาย jarvis เสี่ยงโดนสายแม่บท stash/กวาด** — เกิดแล้ว 1 ครั้ง (2026-07-06 · กู้ครบ) · ทางแก้ถาวร = P1-I1 commit เข้า git โดยเร็ว · stash "task_d248bfc4" ของสายแม่บทห้ามลบ
7. **อุปกรณ์เสียงเสมือน (BoomAudio/LarkAudioDevice) เป็นค่าเริ่มต้นของเครื่อง** — เคยกลืนเสียงตอบทั้งหมดเงียบๆ (2026-07-06) · แก้แล้ว: ชี้ลำโพงด้วย env `JARVIS_OUT_DEVICE`/`JARVIS_IN_DEVICE` (ตั้ง "MacBook Pro Speakers" ใน plist) + เสียงบอกสถานะ (สูง=เปิด ต่ำ=ปิด) เล่นผ่านท่อเดียวกัน · **ตกค้าง:** เสียงถามตอนเงียบ 30 นาที (speak.sh/afplay) ยังวิ่งตามค่าเริ่มต้นระบบ — ถ้า default เป็นอุปกรณ์เสมือนจะไม่ได้ยินคำถาม (ได้ยินแค่บี๊บต่ำตอนมันปิดเอง) · งานต่อ: ย้าย speak.sh เข้าท่อ play_q
