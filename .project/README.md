# .project/ — ลิ้นชักความจำกลางของโปรเจกต์ (Memory Schema v1.2)

> กฎทองข้อเดียว: **เปิดแชทใหม่ อ่านโฟลเดอร์นี้ที่เดียว ต้องทำงานต่อได้ทันที**

## ไฟล์ในลิ้นชักนี้

| ไฟล์ | หน้าที่ | ใครเขียน |
|---|---|---|
| `OverviewProgress.md` | สถานะเดียวของโปรเจกต์ (4 หัวข้อบนสุด: สถานะล่าสุด/งานถัดไป/ข้อห้าม/งานค้าง) | Close Chat |
| `plan.md` | แผนเฟสที่เจ้าของอนุมัติแล้ว (P0, P1, …) | Act-As (ตอนอนุมัติ) |
| `decisions.md` | บันทึกการตัดสินใจสะสม (append อย่างเดียว) | Close Chat |
| `BusinessPlan.md` + `-Full.md` | แผนธุรกิจ 2 ชั้น (Project OS 1/4) | Use BusinessPlan |
| `FeatureSpec.md` (เมื่อสร้าง) | บัญชีฟีเจอร์ real/mock/planned (Project OS 3/4) | Use FeatureSpec |
| `DesignSystem.md` (เมื่อสร้าง) | สรุประบบดีไซน์/CI ประจำโปรเจกต์ (Project OS 4/4) | Use DesignSystem |

ส่วน `.hermes/` = ไฟล์เครื่องจักรเท่านั้น (ตั้งค่าสายพาน relay · ledger) — **ห้ามเขียนความจำทำงานต่อลง `.hermes/` อีก**

## ⚠️ กฎบังคับหลังสร้าง/แก้ไฟล์ในโฟลเดอร์นี้ — "ด่านไฟล์เข้า git จริง"

ทุกครั้งที่สร้างหรือย้ายไฟล์เข้ามา ต้องพิสูจน์ว่า git เก็บจริงก่อนบอกว่าเสร็จ:

```bash
git check-ignore -v .project/*.md   # ต้องไม่เจอสักไฟล์ (exit 1)
git ls-files .project/              # ต้องเห็นไฟล์ครบทุกตัว
```

ถ้าไฟล์โดน `.gitignore` ซ่อน (บาง repo มีกฎกวาด `*.md`) ให้เจาะช่องอนุญาตใน `.gitignore`:

```gitignore
!.project/
!.project/**
```

เหตุผล: ไฟล์ที่ไม่เข้า git จะอยู่แค่เครื่องเดียว — เปิดจากเครื่องอื่น/แชทอื่นแล้วความจำหายเงียบ ๆ ทั้งที่ไฟล์ "ดูมีอยู่" (เกิดจริงมาแล้ว 2 โปรเจกต์ · 2026-07-05)
