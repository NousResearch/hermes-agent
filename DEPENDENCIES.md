# DEPENDENCIES.md

This document is the Rosetta AS-IS dependency inventory for Hermes Agent. Source of truth remains `pyproject.toml`, `uv.lock`, `package.json`, workspace package manifests, and platform-specific lockfiles.

## Python dependency policy

- Core dependencies in `pyproject.toml` are exact-pinned where practical.
- Optional backend dependencies are grouped under `[project.optional-dependencies]` and many are lazy-installed at first use through Hermes runtime helpers.
- Dependency ranges should not be widened casually; comments in `pyproject.toml` document supply-chain and wheel-availability constraints.
- `requires-python` is `>=3.11,<3.14` to avoid unsupported Rust-backed wheels on Python 3.14 until upstream support is available.
- Build backend requires `setuptools>=77.0,<83`.

## Important Python dependency groups

- Core agent/runtime: `openai`, `pydantic`, `httpx`, `requests`, `tenacity`, `rich`, `prompt_toolkit`, `fire`, `pyyaml`, `ruamel.yaml`, `jinja2`, `croniter`, `psutil`, `pathspec`, `Pillow`.
- CLI/dev/test: `pytest`, `pytest-asyncio`, `ty`, `ruff`, `debugpy`, `mcp`, `starlette` through the `dev` extra.
- Messaging/platform extras: Telegram, Discord, Slack, Matrix, Feishu, DingTalk, WeCom, SMS, Teams, Home Assistant.
- Provider/search/media extras: Anthropic, Exa, Firecrawl, Parallel Web, Fal, Edge TTS, Mistral, Bedrock, Azure identity, Google, YouTube.
- Terminal/cloud extras: Modal, Daytona, PTY aliases, MCP, ACP.

## JavaScript dependency policy

- Root `package.json` is private and uses npm workspaces.
- Node must be `>=20.0.0`.
- Workspace dependencies live under `apps/*`, `web`, `ui-tui`, and `ui-tui/packages/*`.
- Root npm scripts provide workspace install and audit commands.
- Root overrides are used for selected dependency safety/compatibility constraints.

## Generated and external tool dependencies

- BMad Method is installed through `npx bmad-method install` and writes generated files under `_bmad/`, `_bmad-output/`, and `.claude/skills/`.
- Claude Code integration is generated by BMad into `.claude/skills/`; Rosetta plugin activation is optional local setup documented in `docs/bmad-rosetta-setup.md`.
- Hermes supports optional MCP servers via `optional-mcps/` and runtime MCP config.

## Verification commands

- Python tests: `scripts/run_tests.sh` or targeted `python -m pytest ...` / `uvx pytest ...` when appropriate.
- Python syntax for generated scripts: `python3 -m py_compile ...` and `/usr/bin/python3` only when the target script must run under system Python.
- Node workspace install/audit: root `npm` scripts in `package.json`.
- Packaging metadata checks live in `tests/test_packaging_metadata.py` and related tests.

## Caution for BMad/Rosetta work

- Do not copy dependency text from BMad generated artifacts into Rosetta AS-IS docs as authoritative future design. Rosetta docs describe current implementation; BMad artifacts describe proposed work.
- Do not let generated BMad scaffolding introduce runtime dependencies into Hermes Agent unless a separate implementation task intentionally changes package manifests.
