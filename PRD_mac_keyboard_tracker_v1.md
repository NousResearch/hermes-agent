# 产品需求文档 (PRD): Keystats (macOS 键盘数据分析工具)

## 1. 文档概述
- **产品名称**: Keystats (暂定)
- **产品定位**: 本地优先的高颜值 macOS 键盘输入数据可视化与分析驻留工具。
- **当前阶段**: Phase 1 (MVP)

## 2. MVP 目标与核心边界 (Scope)
**我们要做的 (In Scope):**
- 精准拦截并统计 macOS 全局键盘输入事件（仅记录键位和时间戳）。
- 绝对的隐私安全：不记录任何按键组合规律（如密码），不记录文本，不发网络请求。
- 提供“今日数据看板”和“标准 ANSI 布局热力图”。
- 简单的 APM (Actions Per Minute) 峰值追踪。

**我们不做的 (Out of Scope for MVP):**
- 多设备/多键盘配置独立统计（Phase 2）。
- 复杂客制化键盘配列（Alice, Ergo 等）热力图（Phase 2）。
- 疲劳警告与系统级弹窗干预（Phase 2）。
- 数据导出为社交媒体长图（Phase 3）。

## 3. 核心功能需求 (Functional Requirements)

### 3.1 底层监听守护进程 (Daemon)
- **输入捕获**: 拦截 `CGEvent`，提取 `keycode`。
- **数据脱敏**: 内存中直接丢弃 Shift/Option 等修饰键带来的组合逻辑，**严禁**实现任何形式的 Keylogger 逻辑（如连词还原）。
- **本地存储**: 每 5 分钟（或达到特定阈值）批量写入本地 SQLite 数据库。数据库路径应位于 `~/Library/Application Support/Keystats/`。
- **系统自启**: 支持随系统登录自动启动，常驻状态栏 (Menu Bar)。

### 3.2 数据看板层 (UI)
- **状态栏入口 (Menu Bar Icon)**: 显示今日累计敲击数，点击展开主面板。
- **今日概览面板**:
  - 今日总敲击次数 (Total Keystrokes Today)。
  - 今日 APM 峰值 (Highest APM Today) 及发生时间。
  - 最常按下的 Top 5 键位。
- **键盘热力图 (Heatmap View)**:
  - 渲染一张标准 ANSI Mac 键盘布局图。
  - 根据按键频次，在按键上覆盖从“冷色(蓝)”到“暖色(红)”的渐变热力渲染。
- **历史回溯 (History View)**:
  - 简单的过去 7 天敲击量柱状图 (Bar Chart)。

### 3.3 权限与引导流 (Onboarding)
- **权限申请页**: 用户首次打开软件时，必须弹出极其清晰的说明，引导用户前往 `系统设置 -> 隐私与安全性 -> 辅助功能 / 输入监听` 开启权限。
- **隐私声明页**: Onboarding 必须包含“隐私自证”卡片（文案：“我们只记录你按了哪个键，不记录你写了什么。数据永远留在你的 Mac 里。”）。

## 4. 非功能性需求 (Non-Functional Requirements)
- **性能**: 守护进程 CPU 占用日常应 < 1%，内存占用应 < 50MB。必须保证绝对不引入任何系统级输入延迟 (Input Lag)。
- **安全性**: 采用纯本地架构，前端 UI 不允许包含任何埋点 SDK（如 Google Analytics），打消极客用户的抓包疑虑。
- **技术栈推荐**: Tauri (Rust + React/TypeScript) + SQLite。

## 5. 关键指标 (Success Metrics)
- 权限转化率：用户下载后成功开启“输入监听”权限的比例。
- 留存率：7 日内是否保持后台常驻。

## 6. 审查员意见 (The Critic's Audit)

### 6.1 审计状态：**REJECT (打回)**
**审计日期**：2026-04-29

### 6.2 核心逻辑漏洞与隐患 (刺点)

1.  **App Store 沙盒政策与核心功能的致命冲突**：
    *   **风险**：PRD 声明符合 Sandbox，但核心功能（全局键盘监听/Accessibility 权限）极难通过 App Store 针对“隐私敏感型”应用的审核。
    *   **要求**：明确分发策略。若走官网下载，需补充公证（Notarization）流程；若走 App Store，需提供功能裁剪/备案方案。

2.  **CGEventTap 的“自杀机制”性能黑洞**：
    *   **风险**：若 Tap 回调处理过长，系统会静默禁用监听。
    *   **要求**：必须明确生产者-消费者模型，确保回调函数非阻塞。

3.  **硬件测量逻辑的“伪科学”风险**：
    *   **风险**：`NSScreen` 无法保证获取所有第三方显示器的精确物理尺寸，且未处理 HiDPI 缩放下的像素/毫米换算。
    *   **要求**：将“自动识别”降级为“自动猜测 + 强制手动校准”。

4.  **国际化布局盲区**：
    *   **风险**：完全未提及日本等地区的 JIS 布局，热力图渲染将出现物理错位。
    *   **要求**：明确支持的键盘布局集或定义降级方案。

5.  **数据存储与算法定义矛盾**：
    *   **风险**：小时级聚合表无法支撑 60 秒滑动窗口的实时 APM 计算。
    *   **要求**：引入实时数据缓冲表或细粒度采样逻辑。

### 6.3 修复方向建议
*   **补充技术架构图**：重点描述多线程协作与数据流。
*   **数学公式化**：将“流状态”等感性描述转化为确定的算法公式。
*   **明确异常流**：定义在 Secure Input（密码框）、系统睡眠、显示器热插拔时的软件表现。
