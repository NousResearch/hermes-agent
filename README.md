# 📂 Project Manager Skill v2.1

**智能项目上下文隔离与状态管理系统**

## 🚀 简介
这是为 AI Agent 设计的“外挂硬盘”系统 v2.1。
Agent 是无状态的，且上下文窗口有限。本 Skill 通过为每个项目创建独立的 `STATUS.md` 文件，并配合智能意图识别和项目索引，解决了以下痛点：
*   **跨会话记忆丢失**：Agent 重启后不知道之前做了什么。
*   **多项目并发混淆**：项目 A 的进度写进了项目 B 的文件。
*   **上下文爆炸**：把所有历史记录都塞进 Prompt 导致 Agent 变慢且变笨。
*   **操作繁琐**：v2.1 引入智能推断与安全写入机制，无需死记硬背命令，支持多实例安全并发。

## 📁 目录结构
所有项目文件统一存放在 `workspace/projects/` 下（或用户指定位置）：

```text
workspace/projects/
├── index.md              # 📇 项目索引（自动维护，支持模糊搜索、自愈）
├── {project-name}/
│   ├── STATUS.md         # 🧠 核心记忆：进度、待办、关键决策（每次对话必读/必写）
│   ├── docs/             # 方案、报告、调研数据
│   ├── src/              # 代码、产出物
│   └── status/           # auto-coding v3 微观工程文件（可选）
└── _archive/             # 📦 已归档项目（已完成/已废弃）
    └── {project-name}/
        └── STATUS.md     # 保留历史记录
```

## 💡 核心原则 (v2.1)
1.  **物理隔离**：每个项目独立文件，绝不混用。
2.  **意图优先**：支持模糊指代（如"上次那个"），Agent 自动推断上下文。
3.  **分级管理**：
    *   **大项目**（>3天）：使用完整模板（包含目标、决策、Todo）。
    *   **小任务**（<3天）：使用轻量模板（仅状态 + 决策 + 下一步），拒绝过度工程。
4.  **并发安全**：写入前必须 Read，防止多个 Agent 实例互相覆盖文件。
5.  **Memory 瘦身**：全局 Memory 仅存用户偏好，项目细节必须落盘。
6.  **归档机制**：定期清理已完工项目到 `_archive`，保持主目录整洁。

## 📝 STATUS.md 结构模板

### A) 完整模板（适用于长期项目）
*   `## 🎯 核心目标`: 一句话定义终极目标。
*   `## 📍 当前状态`: 简述进度。
*   `## 📋 待办事项`: 下一步行动清单。
*   `## 🔑 关键决策`: 为什么这么做（防止重复犯错）。

### B) 轻量模板（适用于临时任务）
*   `## 📍 当前状态`: 一句话进度。
*   `## 🔑 决策`: 记录重要选择（可选）。
*   `## 📋 下一步`: 接下来做什么。

## ⚡ 智能工作流

### 1. 自动意图识别
Agent 会根据对话内容自动判断你的意图，无需特定指令：

| 用户意图 | 典型表达示例 |
|----------|--------------|
| **新建项目** | "新建项目 XX"、"开始做 XX" |
| **恢复项目** | "回到 XX"、"上次那个 XX"、"接着来" |
| **保存状态** | "先停一下"、"我去忙别的" |
| **查看列表** | "有哪些项目"、"看看 XX 进度" |

### 2. 状态更新策略
*   **立即更新**：完成子任务、产生新决策、发现 Bug 时。
*   **不更新**：仅修改注释、思考中未确定方案时。
*   **安全写入**：写文件前先 Read 最新内容，防止并发覆盖。

## ⚠️ 严格约束 (Agent 必读)
1.  **禁止使用 Memory 记录进度**：只记录偏好和经验。
2.  **读取优先**：回答前 **必须先读** `STATUS.md` 或 `index.md`。
3.  **模糊匹配**：用户说"那个"时，主动查索引推断，无法确定时列出最近项目供用户选择。
