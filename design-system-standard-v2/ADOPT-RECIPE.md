# สูตรสำเร็จ · เอา Design System กลางไปใช้กับโปรเจกต์อื่น

> พิสูจน์วงจรจริงแล้วบน OneManFleet (2026-07-06) — หน้าจอใช้ token 83.7% · build ผ่าน · สีตัดผ่าน
> ไฟล์นี้ = "ทำตามนี้แล้วจบ" สำหรับโปรเจกต์ที่ 2-6 · ไม่ต้องเริ่มคิดใหม่ทุกตัว

## สรุปสั้นสำหรับเจ้าของงาน (อ่าน 30 วินาที)

โปรเจกต์แต่ละตัวไม่ต้องสร้าง Design System เอง · แค่ "ดึงชุดกลางไปวาง แล้วเชื่อมให้หน้าจอใช้"
วิธีที่พิสูจน์แล้วว่าปลอดภัยกับเว็บที่รันอยู่: **ไม่รื้อสีทั้งเว็บ** · เอาตัวตรวจไปล็อกไว้ก่อน แล้วค่อยๆ ขยับให้ดีขึ้น
สีแบรนด์ของแต่ละโปรเจกต์ต่างกันได้ (อยู่ในโปรเจกต์) · แต่โครงสร้าง (ระยะ/เงา/ตัวอักษร/สถานะ) ใช้ชุดเดียวกันหมด

## กฎเหล็ก 3 ข้อ (บทเรียนจาก OneManFleet)

1. **ห้ามเอาสีแบรนด์ของโปรเจกต์ไปฝังใน token กลาง** — สีแบรนด์อยู่ชั้นโปรเจกต์ (ci) · แกนกลาง (dna) เป็น OKLCH กลางๆ ใช้ร่วมกัน
   (OneManFleet เผลอทำครั้งแรก แล้วต้องแก้กลับ — อย่าทำซ้ำ)
2. **ห้ามรื้อ globals.css ของเว็บที่รันอยู่รวดเดียว** — ใส่ตัวตรวจ (ds-check) ล็อกคะแนนปัจจุบันไว้ก่อน แล้วค่อยขยับขึ้นทีละงาน
3. **commit งาน DS แยกก้อน** — ระบุชื่อไฟล์ชัด ไม่ปนกับงานอื่นที่ค้างในโปรเจกต์ · ห้าม merge/deploy เอง (เจ้าของกด)
