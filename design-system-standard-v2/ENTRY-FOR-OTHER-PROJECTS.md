# ทางเข้า Design System กลาง — สำหรับ AI จากโปรเจกต์อื่น

> AI ที่ถูกส่งมาอ่านไฟล์นี้: นี่คือชุด Design System กลางขององค์กร — **เป็น "โครงมาตรฐานว่าง ๆ" ที่ AI ต้องเติมสี/แบรนด์ของแต่ละโปรเจกต์เอง** ไม่ใช่ดีไซน์สำเร็จรูปให้ลอก
> ⛔ OneManFleet เป็นแค่ **1 เคสตัวอย่าง**ที่ใช้พัฒนาชุดนี้ขึ้นมา · **ห้ามลอกสี/ดีไซน์/ข้อความของ OneManFleet ไปใช้กับโปรเจกต์อื่น** — ทุกโปรเจกต์สร้างของตัวเองผ่านด่านสี Phase 3
> ใช้กับทุกโปรเจกต์ 30-40 ตัว · อ่านตามลำดับข้างล่าง แล้วทำตามกติกา ห้ามข้าม

> **เริ่มเร็ว (แนะนำ):** ถ้าจะเอา DS ไปใช้กับโปรเจกต์ให้ "จบเร็ว" อ่าน `ADOPT-RECIPE.md` — สูตร 6 ขั้น (DS0-DS5) ที่พิสูจน์จริงบน OneManFleet แล้ว + prompt ก๊อปวาง + เช็กลิสต์ปิดงาน · ส่วนไฟล์อื่นด้านล่างคือรายละเอียดเต็มไว้อ้างอิง

## อ่านตามลำดับ (ทั้งหมดอยู่ในโฟลเดอร์นี้)
1. `checklist/design-system-checklist.md` — มาตรฐาน 77 หัวข้อ 8 ชั้น (H→F→G→A→B/C→E→D) + เกณฑ์ผ่าน
2. `spec/05-project-identity.md` — ชั้น H ต้องทำก่อนทุกอย่าง (อ่านเอกสารโปรเจกต์ · เหตุผลสี · pain→design · สองภาษา · direction check)
3. `spec/02-emotion-matrix.md` — ลูกโซ่ แบรนด์→Persona→แรงจูงใจ→อารมณ์ 6 แกน→token (มีสูตรตัวเลข)
4. `spec/00-standard.md` + `spec/03-expressive-layer.md` + `spec/04-business-conversion.md` — ข้อกำหนดเต็ม
5. `tokens/` — seed token จริง (DTCG · `node build-tokens.mjs` ได้ css+ts · 2 โปรไฟล์ front/admin)
6. `tools/ds-check.py` + `tools/contrast-check.mjs` — ตัวตรวจบังคับก่อนปิดงานหน้าจอ
7. `preview/onemanfleet-ds.html` — **ตัวอย่าง 1 เคส (สี/แบรนด์เป็นของ OneManFleet)** ให้เปิดดูว่าครบทุกชั้นหน้าตาเป็นยังไง · **ห้ามลอกไปใช้ตรง ๆ** — adopt แล้วต้องเปลี่ยนเป็นสี/แบรนด์ของโปรเจกต์เอง (ด่านสี Phase 3) · ตรวจด้วย `tools/brand-leak-check.sh`

## กติกาสำหรับโปรเจกต์ที่มาใช้
- **วิธีใช้ที่ถูก = พิมพ์ `Use Design System` ในโปรเจกต์ของคุณ** — คำสั่งจะอ่านชุดนี้เองแล้วสร้าง token/ตัวตรวจลงโปรเจกต์ให้ (ไม่ใช่ก๊อปไฟล์ HTML ไปแปะ)
- สีแบรนด์ของแต่ละโปรเจกต์ต่างกันได้ (ชั้น ci) แต่โครง semantic/กติกา (ชั้น dna) ต้องเหมือนกันทุกโปรเจกต์
- ทุกงานหน้าจอ: ห้ามปิดงานโดยไม่รัน ds-check + contrast-check · น้ำหนักฟอนต์ใช้ 300/400/500/600/700 · Noto Sans Thai นำ
- ทุกหัวข้อที่ทำ ต้องมีของ "มองเห็นด้วยตา" บนหน้าโชว์ของโปรเจกต์นั้น (กฎเจ้าของ 2026-07-05)
- ห้ามแก้ไฟล์ในโฟลเดอร์กลางนี้จากโปรเจกต์อื่น — เจอปัญหา/อยากเพิ่ม token ให้บันทึกเป็นคำขอแล้วแจ้งเจ้าของ

## ที่อยู่ชุดนี้ (ใช้ได้ทั้ง Mac + VPS)
- **Mac เจ้าของ:** อยู่ในโฟลเดอร์ repo Hermes Agent → `design-system-standard-v2/`
- **VPS / เครื่องอื่น:** ชุดอยู่บน GitHub แล้ว → `git clone --depth 1 https://github.com/rattanasak-ops/hermes-agent.git` แล้วอ้าง `design-system-standard-v2/`
- อย่าฝัง path เครื่องใดเครื่องหนึ่งลงในงานโปรเจกต์ — หา root จาก git หรือ clone ใหม่ · ตัวช่วย: `bash link-brand-ds.sh` (หา master เองหลาย path · ไม่เจอก็ดึงจาก GitHub)
