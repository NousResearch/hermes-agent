# คู่มือใช้มาตรฐาน Design System (v2) — ฉบับเจ้าของงาน

> อธิบายภาษาคน · ไม่ต้องเป็น dev ก็ตามได้

## มาตรฐานนี้คืออะไร
ชุดกติกากลาง 1 ชุด ที่ทำให้ทุกโปรเจกต์ขององค์กร "หน้าตาเป็นระบบเดียวกัน" และตรวจสอบได้ว่าใช้จริง
หัวใจคือ **Token** = คลังค่าสี/ขนาด/ระยะกลาง ที่ทุกหน้าเบิกจากที่เดียว

## 3 ไฟล์ที่ต้องรู้จัก
1. `spec/` = กติกา (อะไรบังคับ อะไรแนะนำ)
2. `spec/02-emotion-matrix.md` = ตัวแปลงบุคลิกแบรนด์เป็นค่าตัวเลข (หัวใจ)
3. `checklist/` = รายการ 50 หัวข้อ ไว้ไล่ติ๊กต่อโปรเจกต์

## ใช้ยังไง (3 ก้าว)
1. พิมพ์ `Use Design System` ในโปรเจกต์
2. AI ถามบุคลิกแบรนด์ + สีหลัก → กรอกตารางอารมณ์ → สร้าง token ให้ทั้งโปรเจกต์
3. ไล่เช็กลิสต์ 50 หัวข้อ · ครบ 🔴 = ตรงมาตรฐาน

## ตรวจว่าโปรเจกต์ใช้ระบบเดียวกันจริงไหม
รันตัวตรวจ (Phase 5) → ไล่หาสี/ขนาดที่ฝังตายตัวไม่ผ่าน token → เจอตัวนอกคอกทันที
= คำตอบของโจทย์ "ทั้งโปรเจกต์ใช้ Design System เดียวกันไหม"

## กันเว็บลูกค้าพัง
- ทดลองโปรเจกต์ไม่ critical ก่อน · เทียบภาพก่อน/หลัง
- เปลี่ยนชื่อ token ต้องคงของเก่าไว้ก่อน (alias)
- ย้อนกลับ = ถอยเวอร์ชัน token ไม่ใช่รื้อทั้งเว็บ
