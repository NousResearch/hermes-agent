# 02 · Emotion Matrix — แปลง "อารมณ์แบรนด์" ให้เป็นค่า Token ที่วัดได้

> เฟส 2 ของแผน v2 · หัวใจของมาตรฐานนี้ · ทำให้ทั้งโปรเจกต์ไปทางอารมณ์เดียวกัน
> กฎเหล็ก: ทุกอารมณ์ต้องออกมาเป็น "ค่าตัวเลข" อย่างน้อย 3 ค่า ห้ามเหลือแค่คำบรรยาย

## ลูกโซ่บังคับ (อ่านซ้ายไปขวา · เพิ่ม Motivation 2026-07-05)

```
แบรนด์ (archetype) → กลุ่มเป้าหมาย → Persona → แรงจูงใจ (Motivation) → อารมณ์ (5 แกนหลัก + Luxe เมื่อ Premium) → **Mood & Tone (F7 สรุปเป็นคำ+do/don't)** → ฟังก์ชัน → ค่า Token (ตัวเลข)
```

ทุกโปรเจกต์ต้องกรอกลูกโซ่นี้ให้ครบ 1 แถว ก่อนสร้าง token · ถ้ากรอกไม่ครบ = ยังไม่ผ่านเฟสนี้

## ส่วน 2.7 · แรงจูงใจผู้ใช้ (Motivation · เพิ่มตามคำสั่งเจ้าของ)

ถามให้ได้ก่อนออกแบบ: **ผู้ใช้มาหน้านี้เพราะอยากได้อะไร และอะไรทำให้เขาตัดสินใจ** (แนว Jobs-to-be-Done)
แรงจูงใจไม่ได้จบที่สี — มันกำหนดลำดับเนื้อหา ปุ่มเรียกร้อง (CTA) และถ้อยคำ:

| แรงจูงใจหลัก | สิ่งที่ต้องเด่น | ผูกลง token/รูปแบบ |
|---|---|---|
| อยากเสร็จเร็ว (งานประจำ) | ทางลัด · ค้นหา · ปุ่มเดียวจบ | density แน่น · CTA เดี่ยวชัด · motion สั้น |
| อยากมั่นใจก่อนจ่าย (งานแพง) | หลักฐาน · ตัวเลข · รางวัล · รีวิว | ตัวเลข tabular · โทนหรู (Luxe 4-5) · หน้าอ้างอิงผลงาน |
| อยากเข้าใจก่อน (ข้อมูลราชการ) | อ่านง่าย · ขั้นตอนชัด | ฟอนต์ ≥16px · สารบัญ · ภาษาคน |
| อยากถูกกระตุ้น (แคมเปญ) | อารมณ์ · ภาพใหญ่ · นับถอยหลัง | gradient/glow · display type · motion ฉาก |

กฎ: ทุกโปรเจกต์ระบุแรงจูงใจหลัก 1-2 ตัว → CTA หลักของทุกหน้าต้องตอบแรงจูงใจนั้น → microcopy ปุ่มเขียนจากฝั่งผู้ใช้ ("ดูราคาแพ็กเกจ" ไม่ใช่ "Submit")

## ส่วน 2 · 5 แกนอารมณ์ → ช่วงค่า Token (ให้คะแนน 1-5 ต่อแกน)

| แกน | 1 (ต่ำ) | 5 (สูง) | Token ที่ผูก + ช่วงค่า |
|---|---|---|---|
| ความทางการ (Authority) | สบายๆ | ทางการจัด | `color.chroma` 5→50 · `radius.base` 16px→2px · `motion.duration` 120ms→350ms |
| ความอบอุ่น (Warmth) | เย็น/กลาง | อุ่นมาก | `color.hue` 210°→30° · `radius.base` 4px→12px |
| พลัง (Energy) | นิ่ง | เร้าใจ | `color.saturation` 20%→90% · `motion.duration` 350ms→120ms · gradient on/off |
| ความชัด (Clarity) | นุ่มนวล | คมชัด | `contrast` 4.5:1→7:1 · `font.weight` 400→600 · `border.width` 1px→2px |
| ความหนาแน่น (Density) | โปร่ง | แน่น | `spacing.scale` 1.0×→0.75× · `row.height` 56px→36px |

## ส่วน 2.5 · สูตรแปลงคะแนน → Token + แก้ค่าที่ขัดกัน (Grok-fix)

พื้นฐานแกนอารมณ์ อิงกรอบ PAD (Pleasure-Arousal-Dominance) + Plutchik ปรับเป็น 5 แกนที่ผูก token ได้

**เกณฑ์ให้คะแนน 1-5 (ตัดสินยังไง):**
- 1 = ไม่มีเลย/ตรงข้าม · 3 = กลาง · 5 = เด่นชัดมาก · ให้ยึด archetype เป็นจุดตั้ง แล้วขยับตามกลุ่มเป้าหมาย

**สูตรเชิงเส้น (ค่า = ต่ำสุด + (คะแนน-1)/4 × ช่วง):**
```
motion.duration = 350 - (Energy-1)/4 × 230      # ms  (พลังสูง = ไวขึ้น)
color.saturation = 20 + (Energy-1)/4 × 70        # %
contrast         = 4.5 + (Clarity-1)/4 × 2.5     # :1
font.weight      = 400 + (Clarity-1)/4 × 200
spacing.scale    = 1.0 - (Density-1)/4 × 0.25     # 1.0→0.75×
row.height       = 56 - (Density-1)/4 × 20        # px
```

**แก้ค่าที่ขัดกัน (radius ถูกดึงทั้ง Authority + Warmth คนละทาง):**
```
radius.base = base(Warmth)  แล้ว clamp ด้วยเพดาน Authority
  base(Warmth) = 4 + (Warmth-1)/4 × 8            # 4→12px
  เพดาน Authority: A5→2px · A4→4px · A3→8px · A≤2→12px
  radius สุดท้าย = min(base(Warmth), เพดาน Authority)
```
> ลำดับความสำคัญเมื่อชนกัน: **Persona floor > Authority > แกนอื่น** (คนสำคัญกว่าสไตล์)

**ระบุให้ชัด (ปิดจุดลอยที่ Grok ถาม):**
- ใช้ color space = **OKLCH** (ตรง theme system เดิมขององค์กร)
- Hue/Chroma คุมเฉพาะ **primary** · semantic (success/warn/danger/info) มาจาก preset ที่ปรับ hue คงระยะ
- ทุกชุดต้องออก **โหมดมืดคู่กัน** + เคารพ `prefers-reduced-motion` (ตัด motion เหลือ 0)
- `gradient` = เปิดเมื่อ Energy ≥ 4 เท่านั้น (ไม่ใช่ on/off ลอย)

## ส่วน 3 · Persona กำหนด "ขั้นต่ำ" (Floor · แตะไม่ได้)

Persona ทับช่วงค่าด้านบนได้เสมอถ้าชนกัน (คนสำคัญกว่าสไตล์)

| Persona | ขั้นต่ำบังคับ |
|---|---|
| ผู้สูงอายุ (60+) | ฟอนต์ ≥ 18px · contrast ≥ 7:1 · target ≥ 44px |
| ผู้พิการสายตา | รองรับโหมดเข้มสูง · ไม่สื่อด้วยสีเดียว · focus ชัด |
| ผู้ใช้มือถือเป็นหลัก | target ≥ 44px · แตะนิ้วโป้งถึง · ตัวอักษรยืดหด |
| เจ้าหน้าที่ทำงานเร็ว (แอดมิน) | density แน่น · คีย์บอร์ดลัด · แถว ≤ 40px |
| ประชาชนทั่วไป (งานรัฐ) | ภาษาคนอ่านง่าย · WCAG 2.2 AA · โหลดเร็ว |

## ส่วน 4 · ฟังก์ชัน → ชุด Component (ไม่ใช่แค่สี)

| ฟังก์ชันหลัก | ชุด component ที่ต้องมี | โปรไฟล์ |
|---|---|---|
| การตลาด/ประชาสัมพันธ์ | hero · CTA · การ์ดเนื้อหา · หน้าราคา · footer | หน้าเว็บ |
| ข้อมูล/บทความ | บทความ · หมวด+ค้นหา · แชร์ · สารบัญ | หน้าเว็บ |
| จัดการข้อมูล | ตาราง (เรียง/กรอง/แก้ในแถว) · ฟอร์ม · แดชบอร์ด · ตัวกรอง | แอดมิน |
| ควบคุมสิทธิ์ | UI รู้สิทธิ์ 4 บทบาท · 5 สถานะหน้าจอ | แอดมิน |

## ส่วน 5 · เกณฑ์ผ่าน (Validation · ต้องครบทุกข้อ)

1. กรอกลูกโซ่ครบ 1 แถว (แบรนด์→เป้าหมาย→persona→อารมณ์→ฟังก์ชัน→ค่า)
2. ทุกแกนอารมณ์แปลงเป็นค่าตัวเลขจริงในไฟล์ token (ไม่เหลือคำบรรยายลอย)
3. ค่าที่ได้ไม่ต่ำกว่า Persona floor
4. ทุกคู่สีผ่าน WCAG 2.2 AA (ข้อความ ≥ 4.5:1)
5. มีทั้งโปรไฟล์ที่เกี่ยวข้อง (หน้าเว็บ/แอดมิน)

## ส่วน 6 · ตัวอย่างจริง 4 โปรเจกต์ (เทียบให้เห็นภาพ)

| โปรเจกต์ | ต้นแบบ | อารมณ์ (A/W/E/C/D) | ผลลัพธ์ token เด่น |

| DRA (กรมการศาสนา) | ผู้ปกครอง+ผู้ดูแล | A5 W3 E2 C4 D2 | chroma ต่ำ · radius 6px · motion ช้า · contrast 7:1 |
| Content Thailand | นักสำรวจ+ตัวตลก | A2 W3 E5 C3 D2 | สีสด · gradient · motion 150ms · การ์ดหนัง |

## ส่วน 7 · Mood & Tone (F7 · เพิ่ม v3.1)

กลั่นผล archetype + อารมณ์เป็นคำ mood 5-8 คำ แล้วทำตาราง do/don't สำหรับภาพ สี ฟอนต์ และถ้อยคำ
แยกคำแนะนำต่อ touchpoint ที่ใช้จริง ได้แก่ เว็บ แอดมิน และเอกสาร
เป้าหมายคือให้คนนอกทีมอ่านแล้วเห็นภาพเดียวกัน และนำไปใช้เบรียฟช่าง นักออกแบบ หรือ AI ต่อได้
