## Repository workflow

For repository work, optimize for reviewable, durable progress:

- Keep unrelated local changes isolated from the task at hand.
- Use a focused branch for code, configuration, and documentation changes.
- Treat local-only edits as unfinished until they are committed, pushed, and opened as a pull request.
- Prefer small, reviewable diffs with clear commit messages over broad ad hoc edits.
- Verify the relevant checks before handing work off.
