# ภาพรวมระบบ 360 องศา

> เอกสารนี้คือแผนที่กลางของทั้งระบบ ทุกโปรเจกต์และทุกเครื่องมือต้องอ้างอิงไฟล์นี้
> เขียนเป็นภาษาคน ให้เจ้าของงานตัดสินใจต่อได้ ไม่ใช่ภาษาเครื่องมือ

## 1. โจทย์จริง (ทำไมต้องมีระบบนี้)

เปลี่ยนงานที่พี่นัททำกับ AI ทุกวัน ให้กลายเป็นสินทรัพย์ที่ขายได้และสร้างแบรนด์ได้เอง
โดยพี่นัทแทบไม่ต้องลงแรงซ้ำ ตรงกับ 3 จุดสำคัญใน Profile:

- ไม่ชอบงานลงแรงและงานเอกสาร
- ไม่อยากโชว์หน้า ไม่อยากให้โยงกลับมาที่ตัวจริง
- ต้องการรายได้แบบทำซ้ำได้ (recurring) มุ่งตลาดต่างประเทศ (global)

**นิยามความสำเร็จ:** พี่นัทกดอนุมัติวันละไม่กี่นาที แล้วได้บทความ + โพสต์ + สไลด์
คุณภาพสูงออกมาสม่ำเสมอ โดยไม่หลุดข้อมูลบริษัทหรือลูกค้า

## 2. บทบาท 3 โปรเจกต์ (ไม่ทับกัน)

| โปรเจกต์ | บทบาท | เก็บอะไร | ที่อยู่ |
|---|---|---|---|
| **Hermes Agent** | สมอง + ความจำกลาง (ศูนย์รวบรวมและสั่งการ) | ความรู้ดิบจากทุกแชท, Profile, ทะเบียนโปรเจกต์, การตัดสินใจ | `Tech Tools/Hermes Agent/` |
| **Content Factory** | โรงงานอัตโนมัติ | เครื่องมือและ Agent ทั้งหมดที่แปลงความรู้เป็นเนื้อหา | `SaaS Project/Master Content Factory/` |
| **Idea2Logic** | หน้าร้าน + คลังผลงาน | บล็อก, portfolio, แบรนด์แบบไม่โชว์หน้า, ภาษาอังกฤษ | `Private Project/Idea2Logic/` |

คำแปลศัพท์: ศูนย์รวบรวมและสั่งการ = ที่เดียวที่เห็นภาพทั้งหมดและตัดสินใจ ·
portfolio = คลังผลงานที่แสดงให้คนนอกดู

## 3. สายงานหลัก (เล่าเป็นภาษาคน)

```
พี่นัทคุยกับ AI (Codex / Cursor / Claude Code)
   │  จบแต่ละแชท ทิ้งสรุปไว้ 1 ไฟล์
   ▼
[Hermes] ตัวเก็บเกี่ยว ดูดเข้ามา → AI สรุปเป็น "การ์ดความรู้" → เข้ากล่องรอตรวจ
   │  พี่นัทกดอนุมัติจุดเดียว (กันข้อมูลลับ + เลือกว่าต่อบล็อกเก่าหรือสร้างใหม่)
   ▼
[Content Factory] โรงงานผลิต: เขียนบล็อกภาษาอังกฤษ + แคปหน้าจอด้วย Playwright
   │
   ├─→ สไลด์นำเสนอ
   ├─→ โพสต์ LinkedIn + Facebook
   ├─→ พอดแคสต์ (รวมหลายบล็อกเรื่องเดียวกัน)
   └─→ วิดีโอแบบไม่โชว์หน้า (สไลด์ + เสียงบรรยาย)
   ▼
[Idea2Logic] เผยแพร่บล็อก + กระจายไปเว็บนอก (Medium / Dev / Hashnode)
```

คำแปลศัพท์: ตัวเก็บเกี่ยว = โปรแกรมที่ไปดึงประวัติแชทมาเอง · การ์ดความรู้ =
สรุปสั้น 1 หน้าต่อความรู้ 1 เรื่อง · Playwright = เครื่องมือเปิดเว็บแล้วถ่ายภาพหน้าจออัตโนมัติ

## 4. วิธีเก็บข้อมูลแชท (ตัดสินใจแล้ว — แบบผสม)

นำด้วยการดูดจากไฟล์ log อัตโนมัติ เสริมด้วยช่องวางมือ:

- **อัตโนมัติ (หลัก):** Claude Code และ Codex เก็บประวัติแชทเป็นไฟล์ในเครื่องอยู่แล้ว
  ระบบดูดมาสรุปเองได้ ไม่ต้องก๊อปวาง
- **วางมือ (สำรอง):** Cursor และเครื่องมืออื่นที่ดึงยาก ให้มีช่อง "วางสรุปแชท" ไว้ใช้มือ

เหตุผล: ลดงานลงแรงของพี่นัท เริ่มได้ทันทีโดยไม่ต้องรอต่อ API ของเจ้าไหน

## 5. หลักการกลาง (ทุกเฟสต้องเคารพ)

1. **ภาษาอังกฤษสำหรับ output ที่เผยแพร่** (ตลาด global) แต่เอกสารภายในเป็นไทย
2. **แบบไม่โชว์หน้า** ห้ามโยงกลับตัวจริง บริษัท หรือลูกค้า (ดู `privacy-gate-rules.md`)
3. **พี่นัทอนุมัติจุดเดียว** ก่อนเผยแพร่ทุกชิ้น (ตรงจุดอ่อนเรื่องวินัย ต้องมีด่านคุม)
4. **มาจากงานจริง** ไม่ปั้นคอนเทนต์ลอยๆ (จุดที่คู่แข่งลอกยาก)
5. **ทุกชิ้นต้องโยงกลับ source เดิม** (ร่องรอยว่าเอามาจากแชทไหน โปรเจกต์ไหน)

## 7. ทีม Role ที่ใช้ (7 คน)

1. Chief Architect — คุมภาพรวม ตัดสินใจขั้นสุดท้าย
2. Brand & Business Strategist — รื้อแผนและแบรนด์
3. Knowledge Harvester Engineer — ตัวเก็บเกี่ยวความรู้
4. Content Pipeline Engineer — สายงานผลิตเนื้อหา
5. Blog Pattern Designer — ปรับรูปแบบบล็อก global
6. Privacy & Risk Reviewer — ด่านตรวจข้อมูลลับ
7. Sync & Governance Engineer — ระบบอัปเดตข้ามโปรเจกต์
