# Idea2Logic — จุดที่ต้องรื้อก่อนสร้างของต่อ

> นี่คือโจทย์ตั้งต้นของเฟส 1 ถ้าไม่แก้จุดเหล่านี้ก่อน ทุกอย่างที่สร้างต่อจะผิดทิศ
> เปรียบเทียบ: Business Plan v2.0 เดิมของ Idea2Logic เทียบกับ Profile ล่าสุด

## 1. ความขัดแย้งหลัก (ระดับต้องแก้ทันที)

| เรื่อง | Business Plan v2.0 เดิม | Profile ล่าสุด | ผลถ้าไม่แก้ |
|---|---|---|---|
| ตัวตน | เปิดตัวเป็น "CEO SYNERRY บริหารงาน 20-40 ล้าน/ปี" | ห้ามใช้ชื่อบริษัท ห้ามโยงตัวตน เน้นไม่โชว์หน้า | แบรนด์โยงกลับตัวจริงทันที เสียจุดยืน |
| ตลาด | เน้นไทยเป็นหลัก (ผู้บริหารไทย) | เน้น global เป็นหลัก ภาษาอังกฤษ | เนื้อหาผิดภาษาและผิดกลุ่ม |
| มูลค่างาน | ใช้ "20-40 ล้าน/ปี" เป็นจุดขายความน่าเชื่อถือ | ห้ามใช้มูลค่าโครงการในที่สาธารณะ | หลุดข้อมูลลับ เสี่ยงสูง |
| ช่องทาง | LinkedIn ที่พี่นัทโพสต์เองในชื่อจริง | ใช้แบรนด์/เพจของแบรนด์ ไม่ใช่ตัวบุคคล | โยงตัวตนอีก |
| จุดขาย Trading | ชู "เทรดจริง ได้-เสียหลักล้าน" เป็นตัวตน | ตัวตนส่วนตัวห้ามเปิด | เสี่ยงภาพลักษณ์ |

## 2. สิ่งที่ยังใช้ได้ (ไม่ต้องทิ้ง)

- โครงสร้างบล็อกคุณภาพสูง (BlogSkill v8) — เอามาปรับเป็นภาษาอังกฤษ global
- แนวคิด "ให้ผลงานพูดแทน ไม่อวด ไม่โม้" — ตรงกับ Profile (รับได้เฉพาะภาพ builder/operator)
- การเน้นเนื้อหามาจากงานจริง ไม่ปั้นลอยๆ — เป็นจุดที่คู่แข่งลอกยาก
- โมเดลรายได้หลายทาง (product + recurring) — สอดคล้องเป้าหมายรายได้

## 3. สิ่งที่ต้องสร้างใหม่ในเฟส 1

1. **ชื่อแบรนด์ global ใหม่** แทน `Lui` (พี่นัทขอให้เสนอชื่อให้เลือก)
   พร้อมเหตุผล บุคลิกแบรนด์ และข้อควรระวัง
2. **จุดยืนใหม่แบบไม่โชว์หน้า** — เป็น AI Lab / operator เบื้องหลัง ไม่ใช่ตัวบุคคล
3. **กลุ่มเป้าหมายใหม่** — ผู้บริหาร/เจ้าของธุรกิจต่างชาติที่อยากเอา AI ไปใช้จริง
4. **หลักฐานความน่าเชื่อถือแบบปลอดภัย** — demo dashboard, หน้าจอที่เบลอข้อมูล,
   case study แบบไม่ระบุตัวตน (ผ่านด่าน privacy gate)
5. **โมเดลรายได้ที่แตกเป็นตัวเลข** — pricing, จำนวนลูกค้า, recurring เพื่อไปถึง
   300K → 500K → 1M บาทต่อเดือน

## 4. หลักการตัดสินเฟส 1
