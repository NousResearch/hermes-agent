# เฟส 2 — ตัวเก็บเกี่ยวความรู้จากแชท AI

> ตัวเก็บเกี่ยว = โปรแกรมที่ไปดึงประวัติแชทมาเอง แล้วให้ AI สรุปเป็นการ์ดความรู้
> นี่คือหัวใจของระบบ 360 ที่พี่นัทอยากได้มากที่สุด

## 1. ของจริงที่สำรวจแล้ว (2026-06-01)

| แหล่งแชท | ที่เก็บไฟล์ | จำนวน | วิธีดึง |
|---|---|---:|---|
| Claude Code | `~/.claude/projects/**/*.jsonl` | 1,176 ไฟล์ | ดูดอัตโนมัติ |
| Codex | `~/.codex/sessions/` + `archived_sessions/` | 418 ไฟล์ | ดูดอัตโนมัติ |
| Cursor | ฐานข้อมูลภายใน (workspaceStorage) | 128 พื้นที่ | ช่องวางมือ (ดึงยาก) |

## 2. สายงานเก็บเกี่ยว (5 ขั้น)

```
ขั้น 1  ดูด: สแกนไฟล์ใหม่/ที่ยังไม่เคยอ่านจาก Claude Code + Codex
   ▼
ขั้น 2  กรองหยาบ: ทิ้งแชทสั้น/ไม่มีสาระ เก็บเฉพาะแชทที่มีองค์ความรู้ ประสบการณ์
        แนวคิด หรือข้อสรุป
   ▼
ขั้น 3  สรุป: AI อ่านแชท แล้วเขียนเป็น "การ์ดความรู้" 1 ใบ ตามแบบฟอร์มมาตรฐาน
   ▼
ขั้น 4  ตรวจข้อมูลลับ: ส่งผ่านด่าน privacy gate (ดู privacy-gate-rules.md)
        ถ้าเจอข้อมูลลับ ขึ้นธงแดง รอพี่นัทตัดสิน
   ▼
ขั้น 5  เข้ากล่องรอตรวจ: การ์ดไปอยู่ใน knowledge-inbox/ รอพี่นัทกดอนุมัติ
        ก่อนส่งต่อให้โรงงาน Content Factory ผลิตเป็นบล็อก
```

## 3. ที่เก็บ (อยู่ใน Hermes = สมอง/ความจำกลาง)

```
docs/360-content-system/
└── knowledge-inbox/
    ├── _new/          การ์ดที่เพิ่งสรุป รอพี่นัทตรวจ
    ├── _approved/     การ์ดที่อนุมัติแล้ว รอส่งให้โรงงานผลิต
    ├── _flagged/      การ์ดที่ติดธงข้อมูลลับ ต้องแก้ก่อน
    └── _state/        บันทึกว่าไฟล์ไหนเคยอ่านแล้ว (กันสรุปซ้ำ)
```

เหตุผลที่อยู่ Hermes ไม่ใช่ Content Factory: ตัวเก็บเกี่ยวและการ์ดความรู้คือ
"ความจำกลาง" ส่วนโรงงานแปลงการ์ดเป็นบล็อก/สไลด์/โพสต์ คือหน้าที่ของ Content Factory
(แยกหน้าที่ตาม `360-overview.md`)

## 4. กันการสรุปซ้ำ

แต่ละไฟล์ที่อ่านแล้วจะถูกบันทึกลายนิ้วมือ (ชื่อไฟล์ + เวลาแก้ไขล่าสุด) ใน `_state/`
รอบถัดไปจะข้ามไฟล์เดิม อ่านเฉพาะแชทใหม่หรือแชทที่อัปเดต

## 5. การจับคู่ "ต่อบล็อกเก่า หรือ สร้างใหม่" (เชื่อมเฟส 3)

ทุกการ์ดจะมีช่อง "หัวข้อหลัก" และ "คำสำคัญ" เมื่อสรุปการ์ดใหม่ ระบบเทียบกับการ์ด
และบล็อกเก่า ถ้าหัวข้อใกล้กันมาก จะเสนอ "ต่อจากบล็อกเก่า" ถ้าต่างชัดจะเสนอ "สร้างใหม่"
การตัดสินสุดท้ายเป็นของพี่นัทตอนกดอนุมัติ

## 6. งานที่จะทำต่อ (ขั้นถัดไปในเฟส 2)

1. เขียนตัวเก็บเกี่ยวจริง (proof) ที่อ่าน Claude Code transcript ล่าสุด → สรุปการ์ด
   ตัวอย่าง 1-2 ใบให้พี่นัทดูของจริง
2. ตั้งกล่องรอตรวจ (knowledge-inbox) จริง
3. ต่อด่าน privacy gate เข้ากับขั้นสรุป

> ตัวเก็บเกี่ยวจะอ่านไฟล์ประวัติแชทจริงในเครื่อง (อ่านอย่างเดียว ไม่แก้ของเดิม)
> ต้องขออนุญาตพี่นัทก่อนรันรอบแรก
