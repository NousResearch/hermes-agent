# กฎด่านตรวจข้อมูลลับ (Privacy Gate)

> นี่คือจุดที่พลาดแล้วเสียหายหนักที่สุดของทั้งระบบ
> ทุกชิ้นที่จะเผยแพร่ออกนอก ต้องผ่านด่านนี้ก่อน 100% ห้ามข้าม

## 1. ทำไมต้องมีด่านนี้

Profile ของพี่นัทระบุชัด: ตลาดไทยทุกเรื่องเสี่ยงกระทบภาพลักษณ์ จึงเลือกทำตลาด
ต่างประเทศแบบไม่เปิดตัวจริง ปัญหาคือถ้าแชร์ความรู้แล้วคนเดาได้ว่าเป็นใคร
ทำบริษัทอะไร ก็จะเสียจุดยืนทันที ด่านนี้มีไว้กันเรื่องนั้น

## 2. ห้ามเด็ดขาด (ระดับวิกฤต)

| ห้ามเผยแพร่ | ตัวอย่าง |
|---|---|
| ชื่อบริษัทจริง | ชื่อบริษัทพี่นัท, ชื่อบริษัทในเครือ |
| ชื่อลูกค้าจริง | ชื่อหน่วยงาน, ชื่อองค์กรที่ว่าจ้าง |
| มูลค่าโครงการ | ตัวเลขมูลค่างานจริง เช่น 10-20 ล้าน |
| ข้อมูลระบุตัวตน | ชื่อจริง, ชื่อเล่น, รูปหน้า, จังหวัด/ที่อยู่ |
| ความลับทางเทคนิค | internal IP, server URL, credential, API key |
| ข้อมูลภายในบริษัท | ชื่อระบบภายใน, ชื่อทีมงานจริง, กระบวนการลับ |

## 3. ใช้แทนได้ (ปลอดภัย)

| ของจริง | ใช้แทนด้วย |
|---|---|
| ชื่อลูกค้า | "องค์กร A", "a mid-size enterprise" |
| มูลค่าโครงการ | "งานขนาดใหญ่", ตัดตัวเลขออก หรือใช้ช่วงกว้างที่ไม่ระบุ |
| ชื่อบริษัทพี่นัท | ชื่อแบรนด์ faceless ที่จะตั้งใหม่ (ดูเฟส 1) |
| หน้าจอจริง | หน้าจอที่เบลอข้อมูลระบุตัวตนแล้ว หรือข้อมูลจำลอง |

## 4. รายการคำที่ตัวตรวจต้องค้นหา (ก่อนเผยแพร่)

ตัวตรวจอัตโนมัติต้องค้นหาและเตือนเมื่อเจอ:

- เลข IP (รูปแบบ ตัวเลข.ตัวเลข.ตัวเลข.ตัวเลข)
- คำว่า "บริษัท", "Co.,Ltd", "Corporation" ที่ตามด้วยชื่อเฉพาะ
- ตัวเลขเงินที่มีหน่วย "ล้าน", "บาท", "THB" ในบริบทมูลค่างาน
- ชื่อจริงและชื่อเล่นของพี่นัท
- คำว่า key, token, password, secret, credential
- ชื่อโดเมนภายใน (localhost ยกเว้น, แต่โดเมนจริงต้องเตือน)

> รายการนี้ขยายเพิ่มได้ เมื่อเฟส 2 และ 4 สร้างตัวตรวจจริง จะแปลงเป็นไฟล์รายการคำ
> ที่เครื่องอ่านได้

## 5. ขั้นตอนด่านตรวจ (ก่อนเผยแพร่ทุกชิ้น)

1. ตัวตรวจอัตโนมัติค้นหารายการในข้อ 4 → ถ้าเจอ ขึ้นธงแดง หยุดไว้
2. ถ้ามีหน้าจอ ตรวจว่าเบลอข้อมูลระบุตัวตนแล้ว
3. AI สรุปให้พี่นัทดูว่า "ชิ้นนี้มีความเสี่ยงโยงตัวตนไหม"
4. พี่นัทกดอนุมัติ → ถึงเผยแพร่ได้
5. บันทึกว่าอนุมัติเมื่อไร ใช้กับชิ้นไหน (ร่องรอย)

## 6. กฎทอง

ถ้าไม่แน่ใจว่าชิ้นไหนเสี่ยงโยงตัวตน → หยุดไว้ก่อน ถามพี่นัท ห้ามเดาแล้วปล่อย
