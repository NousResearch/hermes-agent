# v2.10 Phase 6: Web Console Session UI

## Goal

Add v2.10 API endpoints and frontend components to expose workspace, session, and adapter health data in the Hermes dashboard.

## Scope

Implemented:
- GET `/api/v2.10/workspaces` — list all workspaces
- GET `/api/v2.10/workspaces/{workspace_id}` — get workspace by ID
- GET `/api/v2.10/sessions` — list sessions, optionally filtered by workspace
- GET `/api/v2.10/sessions/{session_id}` — get session by ID
- GET `/api/v2.10/adapters/health` — aggregate adapter health snapshot
- Frontend types (`V210Workspace`, `V210Session`, `V210AdaptersHealthResponse`, etc.)
- Frontend API methods (`v210ListWorkspaces`, `v210GetWorkspace`, `v210ListSessions`, `v210GetSession`, `v210AdaptersHealth`)
- `AdapterHealthCard` component on OperationsPage — shows registered adapters and their health
- `V210WorkspaceContext` component on SessionsPage — shows workspace count, session count, and adapter mode

## Not implemented

- Mac App Shell (design concept only)
- Streaming cards, approval buttons, notification cards
- Discord productionization
- `/sessions/:session_id` detail page with v2.10 context
- Adapter management actions (register/unregister from UI)
