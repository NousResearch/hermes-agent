# Hermes Quality Stack Issues

This file is the durable checklist for the quality-stack rollout. Percentages
are generated by `scripts/hermes_quality_stack_comply.py`; this file explains
what each issue means and how future AI workers should verify it.

## Phase 0 - Baseline and Safety

| Issue | Work | Done means | Verification |
|---|---|---|---|
| P0-01 | Baseline repo/runtime state | Repo instructions, `.hermes/context.md`, and dependency policy are readable before changes. | `git status --short --branch`; inspect `pyproject.toml`; read `.hermes/context.md`. |
| P0-02 | Local runtime verification path | The repo has a runnable test entry point. | `scripts/run_tests.sh ...` returns exit code 0 for targeted tests. |
| P0-03 | VPS verification path | The team workspace has a documented VPS check path. | `ssh ... systemctl --user is-active hermes-dashboard.service`; `curl http://127.0.0.1:9119`. |
| P0-04 | Rollback path | Every new quality-stack layer has a removal/disable route. | Read `docs/quality-stack/implementation-plan.md`. |
