# Retrospectives

Updated: 2026-04-18

这个目录用于存放 Hermes 的复盘文档或复盘模板。

## 目标

复盘不是写感想，而是把最近工作转化成长期资产：
- memory
- skills
- docs
- plans
- scripts
- cron

## 每次复盘至少看什么

1. 最近几次 session 在重复做什么
2. 有没有用户重复说明同一偏好/约束
3. 有没有已经验证成功但尚未沉淀成 skill 的流程
4. 有没有脚本或 cron 可以把重复劳动自动化
5. 有没有高风险动作不该自动化

## 推荐输出格式

### Summary
- 本轮观察到的重复模式
- 最值得固化的能力
- 当前最大风险

### Conversions
- 新增 memory：
- 新增/patch 的 skill：
- 新增文档：
- 新增计划：
- 新增 cron：

### Next Actions
- 近期最值得推进的 1~3 个动作

## 命名建议

可按日期保存，例如：
- `2026-04-18-weekly.md`
- `2026-04-20-skill-gap-review.md`

## 判断标准

如果一次复盘结束后，没有生成任何可复用资产，说明复盘质量通常不够好。
