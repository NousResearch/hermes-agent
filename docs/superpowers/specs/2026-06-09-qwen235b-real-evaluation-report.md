## 评估背景

**真实巡检流程**:
1. 定时巡检脚本（如heartbeat.py）定期运行
2. 巡检脚本运行多个checker，收集数据
3. 如果发现异常，输出`{"wakeAgent": true}`门控信号
4. Agent被唤醒，开始分析异常
5. Agent根据skill去获取更多信息
6. 逐步深入分析，找到根本原因

**评估目标**:
1. 验证Qwen235B模型是否能从巡检报告中识别异常
2. 评估模型是否能根据skill获取更多信息
3. 测试模型是否能逐步深入分析问题
4. 验证模型是否能给出合理的排查建议
