# Video Studio 具名资产库匹配设计

## 目标

让 Hermes Desktop 的 Video Studio 能显式选择一个已配置的具名视频资产库，例如 `beef-noodle`，并基于文案完成镜头检索、候选预览、人工确认和时间线创建。

本功能必须保持三条边界：

- 每次进入 Video Studio 都要求用户手动选择资产库，不自动选择、不记忆上次选择。
- MoneyPrinterTurbo 的本地上传素材库与 Obsidian 具名资产库互相独立。
- 具名资产库是边缘 capability，通过现有 API 调用，不进入 Hermes Agent Core。

## 用户流程

1. 用户进入 Video Studio。
2. 页面加载可用资产库列表，但不加载任何资产库内容。
3. 用户手动选择一个资产库。
4. 页面加载该库的状态、素材和镜头。
5. 用户完成或修改视频文案，并将文案拆成若干片段。
6. 用户可以对单个文案片段点击“匹配素材”，也可以点击“自动匹配全部文案”。
7. 每段文案展示 3 至 5 个候选镜头。
8. 用户人工确认、替换或重新匹配每段的最终镜头。
9. Video Studio 使用确认的镜头创建具名资产库时间线。
10. 后续配音、字幕、音乐与合成继续走现有 MoneyPrinterTurbo 流程。

## 页面交互

### 资产库选择

- Video Studio 初始状态的 `selectedLibraryId` 必须为空。
- 页面只在初始加载时调用资产库列表接口。
- 未选择资产库时显示“请先选择资产库”。
- 未选择资产库时，刷新资产、匹配素材和创建时间线操作不可用。
- 用户切换资产库时，清空上一资产库的素材、候选镜头、已确认镜头和时间线状态，防止跨库混用。

### 单段匹配

- 每个文案片段提供“匹配素材”操作。
- 查询文本以该段最终文案为主，可附加用户填写的素材关键词。
- 每段最多展示 5 个候选镜头。
- 候选卡片显示关键帧、描述、时间段、质量分、置信度、匹配分和原素材路径。
- 用户选择一个候选镜头作为该段最终镜头。
- 用户可以重新匹配或替换已经选择的镜头。

### 全篇匹配

- “自动匹配全部文案”逐段调用同一匹配逻辑。
- 自动匹配只产生候选和默认建议，不绕过人工确认。
- 单段失败不清空其他段的成功结果。
- 页面汇报成功段数与失败段数。

### 时间线

- 只有所有准备进入时间线的镜头都属于当前选定资产库时才允许创建。
- 创建时间线请求必须携带 `libraryId`。
- 后端按需物化被选中的镜头，不提前复制全部源视频。
- 切换资产库后，旧时间线草稿不得继续使用。

## 前端结构

采用独立控制器与小组件，不继续扩大 Video Studio 主组件的职责。

### `useNamedVideoLibrary`

负责：

- 加载资产库列表。
- 保存当前手动选择。
- 加载当前库状态、素材和镜头。
- 处理切库清理。
- 提供扫描、刷新和错误状态。

### 文案匹配模块

负责：

- 将最终文案拆成稳定片段标识。
- 执行单段匹配。
- 执行全篇匹配。
- 保存每段候选与人工确认结果。
- 创建具名资产库时间线。

### Video Studio 主组件

只负责：

- 渲染资产库选择器。
- 组合文案、素材匹配和现有合成流程。
- 将已确认镜头交给时间线创建接口。

## API 调整

扩展现有 `videoLibraryClient`，不新增核心工具。

需要支持：

- `listLibraries()`
- `getLibraryStatus(libraryId)`
- `scanLibrary(libraryId, dryRun)`
- `listAssets(libraryId)`
- `listClips(libraryId, { query, assetId, tag, limit })`
- `analyzeAsset(libraryId, assetId)`
- `replaceClipTags(libraryId, clipId, tags)`
- `createTimeline(libraryId, clipIds, aspect, script)`

所有针对具名资产库的读写请求必须明确携带 `libraryId`。无 `libraryId` 的现有接口继续服务于桌面上传的通用素材库，保持向后兼容。

后端需要确保 `replaceClipTags` 与其他请求一样通过 `_request_service(library_id)` 选择正确数据库，不能误写通用素材库。

## 数据隔离

- `beef-noodle` 的数据库继续位于 Obsidian 资产库的 `.hermes-assets/index.sqlite`。
- Video Studio 通用上传素材继续位于 Hermes 隔离 Home 和 MoneyPrinterTurbo 白名单目录。
- 前端切库必须清空旧库的内存状态。
- 后端从配置解析具名库路径，不接受任意客户端根目录。
- 原视频不移动、不改名、不修改。

## 错误处理

- 资产库列表失败：显示错误，不提供伪造的默认库。
- 资产库失效：保留选择值并显示不可用状态，禁止匹配。
- 某段匹配失败：只标记该段失败，保留其他段结果。
- 候选为空：显示“未找到合适镜头”，允许修改文案或关键词后重试。
- 切库：立即清空旧候选，避免错误提交旧镜头 ID。
- 时间线创建失败：保留人工确认结果，允许重试。

## 测试

### 客户端单元测试

- 资产库 API 路径正确。
- `library_id` 查询参数正确编码。
- 分析、标签和时间线请求携带 `libraryId`。

### 控制器测试

- 初始状态不自动选择资产库。
- 选择后才加载资产和镜头。
- 切库清空旧候选和人工选择。
- 单段匹配保存候选。
- 全篇匹配允许部分失败。
- 未选择资产库时拒绝匹配。

### Video Studio 组件测试

- 显示手动资产库选择器。
- 未选择时禁用匹配操作。
- 展示候选关键帧和评分。
- 人工替换镜头生效。
- 创建时间线使用当前资产库。

### 后端测试

- 具名资产库素材和镜头查询命中正确数据库。
- 标签修改不会写入通用数据库。
- 时间线只物化当前具名资产库的镜头。

### 桌面实测

1. 使用 `hermes-dev-desktop start` 启动正确开发版。
2. 打开 Video Studio，确认初始没有选中资产库。
3. 手动选择 `beef-noodle`。
4. 确认加载 6 个唯一素材和 24 个镜头。
5. 对一段文案执行单段匹配并人工选择镜头。
6. 对多段文案执行自动匹配全部并检查候选。
7. 创建时间线，确认请求和产物属于 `beef-noodle`。
8. 切换或清空资产库，确认旧候选不会继续使用。

## 不在本次范围

- 向量数据库或 embedding 检索。
- 自动监控素材目录。
- Obsidian 人工标签反向同步。
- 修改最终视频渲染算法。
- 自动发布成片。
