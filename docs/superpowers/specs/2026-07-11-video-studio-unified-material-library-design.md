# Video Studio 统一素材库入口设计

## 背景与问题

Video Studio 当前同时展示三种容易被理解成“素材库”的区域：

- `本地素材`：MoneyPrinterTurbo `storage/local_videos` 白名单及本次合成选择。
- `视频素材库`：Hermes `$HERMES_HOME/video-library` 通用上传库及切镜分析 UI。
- `Obsidian 具名资产库`：按客户或项目配置的长期资产库，例如 `beef-noodle`。

三者的物理存储和数据库互相隔离，不会直接写坏彼此的数据，但会造成重复复制、重复分析、同名文件覆盖风险和明显的用户认知冲突。产品目标是让用户只看到一个“素材库”入口；MoneyPrinterTurbo 白名单退化为不可见的内部合成缓存。

本设计取代此前“通用上传素材库与具名资产库长期并列展示”的产品假设。底层通用素材库接口暂时保留用于旧数据迁移和兼容，不再作为 Video Studio 主流程入口。

## 目标

1. Video Studio 只展示一个“素材库”工作区。
2. 用户每次进入仍须手动选择一个具名资产库，不自动恢复上次选择。
3. 添加文件、选择素材目录、扫描、分析、文案匹配和人工选片都从当前资产库上下文发起。
4. `storage/local_videos` 仅作为 MoneyPrinterTurbo 合成白名单和短期缓存，不再呈现为第二个素材库。
5. 现有通用视频素材库数据不删除，提供显式、可审计的一次性迁移。
6. 用真实牛肉面资产库完成桌面端演示和回归测试，并把最终逻辑更新到 Video Studio 与视频素材库逻辑地图。

## 统一后的用户流程

1. 用户打开 Video Studio。
2. 页面加载可用具名资产库，但初始不选择任何库。
3. 用户手动选择资产库，例如 `牛肉面资产库 · beef-noodle`。
4. 用户选择以下一种素材入口：
   - 添加一个或多个视频文件到当前资产库；
   - 选择或管理当前资产库允许扫描的素材目录；
   - 扫描已配置目录中的新增素材。
5. `video-analysis-expert` / `local-video-asset-indexer` 完成去重、切镜、关键帧、语义分类、固定标签、自由标签、质量状态、SQLite 入库和 Obsidian Markdown 投影。
6. 用户完成或修改文案，执行单段匹配或全篇匹配。
7. 页面展示候选关键帧、描述、标签、质量、置信度和源文件路径。
8. 用户人工确认最终镜头。
9. 系统创建具名资产库时间线，按需物化确认镜头，并自动复制到 MoneyPrinterTurbo 白名单缓存。
10. “本次已选镜头”汇总最终会传入 `video_materials` 的文件；用户继续配音、字幕、音乐和合成。

## 页面设计

### 单一“素材库”卡片

原 `本地素材`、`视频素材库` 和 `Obsidian 具名资产库` 三块合并为一个工作区，顺序如下：

1. **资产库选择**
   - 下拉选择当前具名资产库。
   - 显示资产库根目录、素材数、镜头数、失败数和低置信度数。
   - 未选择资产库时，所有导入、扫描、匹配和时间线操作禁用。
2. **素材管理**
   - `添加素材文件`：将视频导入当前具名资产库，不再导入通用库。
   - `选择素材目录`：配置或选择当前库的允许扫描目录；必须经过桌面路径授权和后端 source root 校验。
   - `扫描新增素材`：先 dry-run，再由用户确认执行真实扫描；重复内容按 SHA-256 跳过。
   - `刷新`：刷新当前库状态和资产列表。
3. **文案匹配与候选镜头**
   - 延续现有单段匹配、全篇匹配和人工确认能力。
4. **本次已选镜头**
   - 仅展示人工确认并准备进入本次合成的镜头。
   - 不提供独立上传入口，不把 MoneyPrinter 缓存伪装成素材库。

### MoneyPrinter 缓存

`external/MoneyPrinterTurbo/storage/local_videos` 继续存在，因为 MoneyPrinterTurbo 只允许读取白名单目录。它是内部实现细节：

- 用户确认镜头后，系统按需物化并复制到缓存。
- `form.localMaterials` 只保存本次合成需要的缓存文件标识。
- 页面不再列出整个缓存目录，也不允许用户把它当长期资产库管理。
- 缓存文件采用内容哈希或资产/镜头 ID 命名，避免同名原文件静默覆盖。

## 数据与安全边界

- 具名库根目录与 source roots 只从当前 Hermes Profile 的 `video_libraries` 配置解析。
- 客户端不能提交任意资产库根目录。
- 原视频默认采用 linked 模式，不移动、不改名、不删除。
- 选择素材目录属于资产库配置变更，必须通过明确的桌面选择动作和后端路径校验。
- timeline 只能引用当前具名资产库内已物化的镜头。
- 复制到 MoneyPrinter 缓存前必须确认源文件属于当前时间线。
- 切换或清空资产库时，必须清空候选、人工确认、时间线和“本次已选镜头”。

## 旧数据迁移

旧通用库位于 `$HERMES_HOME/video-library`，不能自动删除或静默合并。

首次进入新版素材库工作区时，如果检测到旧库资产，显示一次性迁移提示：

1. 用户选择一个现有具名资产库，或创建/配置一个迁移目标库。
2. 页面先展示将迁移的唯一资产数、重复内容数和目标根目录。
3. 用户确认后，后端按内容哈希导入目标库；目标库已有内容跳过。
4. 迁移记录包含源 asset ID、目标 library ID、目标 asset ID、状态和错误。
5. 迁移完成后隐藏旧入口，但保留旧库文件；删除旧库不在本次范围。
6. 用户可以选择“暂不迁移”，之后从素材库管理入口重新发起。

## 组件与职责

### `UnifiedMaterialLibraryPanel`

- 组合资产库选择、素材管理、状态、匹配、候选和本次已选镜头。
- 不直接处理 API 细节。

### `useNamedVideoLibrary`

- 继续负责资产库列表、手动选择、状态、素材、镜头、扫描、匹配和切库清理。
- 扩展当前库文件导入、目录选择状态和旧库迁移状态。

### `MoneyPrinterSelectionBridge`

- 将确认镜头按需物化并复制到白名单缓存。
- 生成稳定、无同名覆盖风险的缓存文件名。
- 更新本次 `video_materials`，不向用户暴露完整缓存目录。

### 旧通用库接口

- 保留现有无 `library_id` 的 assets/clips 接口供迁移和兼容测试。
- Video Studio 新主流程不得再调用通用库导入、分析和列表接口。
- 新主流程所有素材库请求必须携带 `libraryId`。

## 错误处理

- 未选择资产库：禁止导入、扫描、匹配和创建时间线。
- 文件位于允许 source roots 之外：拒绝 linked 导入，并引导用户先选择/配置目录。
- 部分文件分析失败：保留成功资产，逐项显示失败阶段。
- 缓存复制失败：保留人工确认和时间线，不提交不完整的本次素材集合。
- 迁移部分失败：保存迁移进度，支持安全重试，不重复导入成功内容。
- 切库：同步清空旧库候选、确认和本次素材，禁止跨库混用。

## 真实案例演示

使用现有 `beef-noodle` 资产库作为验收案例：

- source roots：`/Users/ruoyu/Documents/Obsidian Vault/视频自动化剪辑/work/2026-07-11-牛肉面-语义匹配-v2/material`
- Obsidian 根目录：`/Users/ruoyu/Documents/Obsidian Vault/01-内容生产/牛肉面资产库`
- 当前基线：11 个源路径、6 个唯一资产、24 个 ready 镜头、0 失败。

演示步骤：

1. 启动 `hermes-dev-desktop start` 并打开 Video Studio。
2. 验证页面只有一个“素材库”工作区，不再出现独立的本地素材和通用视频素材库入口。
3. 初始不选择资产库，所有相关操作禁用。
4. 手动选择 `beef-noodle`，确认基线状态。
5. 执行增量扫描，确认 11 个路径全部 skipped、0 failed。
6. 输入两段牛肉面文案，分别验证单段匹配和全部匹配。
7. 人工确认至少两个镜头，创建时间线。
8. 确认镜头自动进入隐藏的 MoneyPrinter 缓存并写入本次 `video_materials`。
9. 生成一条可预览的视频任务，确认最终 MP4 可播放。
10. 清空资产库选择，确认候选、确认和本次已选镜头归零。

## 回归测试

### 后端

- 具名库导入、扫描、查询、标签和 timeline 均命中正确数据库。
- 无 `library_id` 的旧接口仍可读取旧数据，但新主流程不调用。
- 目录越权、符号链接逃逸、跨库 clip ID 和 timeline 路径被拒绝。
- 缓存文件稳定命名，同名不同内容不会覆盖。
- 迁移可重试、可部分成功、按 SHA-256 去重。

### Desktop

- 页面只渲染一个素材库入口。
- 初始状态不自动选择资产库。
- 上传/目录/扫描必须依赖当前 `libraryId`。
- 不再调用通用库 `importAsset(sourcePath)`。
- 切库清空所有旧库选择状态。
- 人工确认后才生成本次素材列表。
- 创建时间线后正确更新 `video_materials`。
