# Video Studio 可控的一键成片设计规格

**状态：** 待用户复核

**日期：** 2026-07-12

**方案：** B — 可控的一键成片

## 1. 目标

将 Hermes 桌面版 Video Studio 从“分别调用文案、语音、素材、合成能力的工具页”，升级为一条可控、可检查、可追溯的一键成片流程：

1. 用户输入主题、关键词、卖点、目标时长和视频风格。
2. AI 生成可编辑文案，并同时生成逐段镜头意图。
3. 用户确认文案后，系统才允许进入可能产生费用的语音和成片阶段。
4. 系统先取得真实音频时长，再按语句时间轴自动匹配镜头。
5. 用户只需选择一个素材来源；系统自动选镜头、生成字幕、配乐并提交合成。
6. 成片后允许替换个别镜头并局部重做，不要求用户从头生成。

本方案的核心不是去掉所有控制，而是让默认路径自动完成，同时把文案确认、素材来源和付费动作保留为明确的用户决策点。

## 2. 不在本期范围内

- 不实现对爆款参考视频的完整导演风格复刻。
- 不要求引入 CLIP 向量数据库或大规模多模态检索基础设施。
- 不默认混用多个本地资产库。
- 不在用户选择本地资产库时静默回退到 Pexels、Pixabay 或 Coverr。
- 不包含自动发布到抖音、小红书等平台。
- 不改变 Hermes Agent Core 的工具集；能力继续位于桌面业务流程、现有 capability API 和 MoneyPrinterTurbo 适配层。

## 3. 产品原则

### 3.1 一个创作入口，一个素材来源

Video Studio 只保留一个“素材来源”设置：

- 本地素材库：选择一个已建库、已切片、已打标签的命名资产库。
- Pexels、Pixabay、Coverr：选择一个在线供应商。

旧的“上传本地视频”能力归入本地素材库入口，作为“导入并分析素材”的动作，不再成为第二套独立的成片素材选择逻辑。

### 3.2 自动选择是默认，人工候选是修订工具

系统必须先自动完成镜头分配。候选镜头列表用于成片后的替换或高级用户检查，不作为一键成片前的必经步骤。

### 3.3 来源指向性是硬约束

- 选择命名本地素材库时，所有镜头必须来自该库。
- 选择在线供应商时，所有自动下载镜头必须来自该供应商。
- 没有候选镜头时应给出清晰警告，并按用户已选择的同源降级策略处理；不得跨来源补图。

### 3.4 音频时间轴先于镜头时间轴

镜头时长以最终旁白或用户上传音频的真实时间戳为依据，不能只使用文案字数估算。这样可以减少尾帧、字幕错位、语音被截断和视频长度漂移。

## 4. 用户流程

### 阶段一：创意输入

用户填写：

- 主题，例如“兰州牛肉面为什么让人念念不忘”；
- 关键词和卖点；
- 目标时长；
- 视频风格和语气；
- 可选的受众与行动号召。

系统生成 `ScriptPlan`，其中同时包含可朗读文案和每段文案对应的镜头意图。

### 阶段二：文案与镜头计划确认

页面按段展示：

- 文案正文；
- 镜头角色，例如环境建立、人物动作、产品特写、氛围、结尾品牌；
- 必需标签、偏好标签和排除标签；
- 预计时长。

用户可修改文案、调整顺序或重新生成某一段。用户点击“确认文案”后，系统冻结一个版本化的制作快照。修改已确认文案会使后续音频和镜头计划失效，需要重新确认。

### 阶段三：声音与音乐

用户选择：

- MiniMax 音色或用户上传音频；
- 语速、音量；
- 背景音乐或不使用音乐；
- 音乐音量及旁白期间的自动压低强度。

页面支持试听。正式生成前执行声音配置检查，禁止把不合法的 `voice_id` 发送给供应商。

### 阶段四：素材来源与生成

用户只需选择一个来源：

- 命名本地素材库；或
- Pexels / Pixabay / Coverr。

点击“检查并生成”后，系统依次执行：

1. 制作前检查；
2. 生成旁白或读取用户音频；
3. 取得逐段音频时间戳；
4. 在指定来源内检索并排序镜头；
5. 自动创建时间线、字幕、音乐和转场；
6. 缓存所需片段；
7. 向 MoneyPrinterTurbo 提交合成；
8. 显示任务进度和成片预览。

### 阶段五：局部修订

成片完成后，每个镜头可执行：

- 查看选中原因和素材来源；
- 从同来源候选中替换；
- 锁定镜头；
- 仅重新生成未锁定镜头和最终合成。

## 5. 状态模型

单次创作会话使用显式状态，避免按钮可用性和真实数据不同步：

```text
draft_input
  -> script_ready
  -> script_confirmed
  -> audio_ready
  -> preflight_passed
  -> rendering
  -> complete

任意执行阶段 -> failed -> 可在已冻结制作快照上重试
```

约束：

- `script_confirmed` 之前不得创建正式付费 TTS 或合成任务。
- 文案变更会回退到 `script_ready`。
- 音色或用户音频变更会回退到 `script_confirmed`。
- 素材来源变更会使镜头分配失效，但不必重新生成旁白。
- 重试必须使用同一份制作快照，除非用户明确修改配置。

## 6. 核心数据合同

```ts
interface ScriptPlan {
  version: 1
  subject: string
  keywords: string[]
  targetDurationSeconds: number
  style: string
  segments: ScriptSegment[]
}

interface ScriptSegment {
  id: string
  text: string
  intent: string
  shotRole:
    | 'establishing'
    | 'person_action'
    | 'product_closeup'
    | 'process'
    | 'atmosphere'
    | 'brand_outro'
  requiredTags: string[]
  preferredTags: string[]
  negativeTags: string[]
  estimatedDurationSeconds: number
}

interface AudioTimingSegment {
  segmentId: string
  startSeconds: number
  endSeconds: number
  text: string
}

interface ShotAssignment {
  segmentId: string
  libraryId: string | null
  provider: 'local' | 'pexels' | 'pixabay' | 'coverr'
  assetId: string
  clipId: string
  sourcePath: string
  sourceStartSeconds: number
  sourceEndSeconds: number
  timelineStartSeconds: number
  timelineEndSeconds: number
  score: number
  reason: string[]
  warnings: string[]
}
```

`ShotAssignment` 必须保存精确来源，最终时间线不能只留下缓存后的匿名文件名。

## 7. 组件边界

### 7.1 Creation Session Controller

位于 Video Studio 桌面业务层，负责：

- 状态机；
- 制作快照；
- 阶段间失效规则；
- 一键生成编排；
- 重试与局部修订。

页面组件只负责展示和触发动作，不继续承担整个制作流程的控制逻辑。

### 7.2 Script Plan Generator

复用现有 LLM capability，但要求结构化返回 `ScriptPlan`。解析失败时保留原始生成内容并允许重试，不得直接进入渲染。

### 7.3 Production Preflight

以纯函数加少量 capability 探测组成，返回结构化结果：

```ts
interface PreflightResult {
  ok: boolean
  blockers: PreflightIssue[]
  warnings: PreflightIssue[]
  estimatedPaidActions: string[]
}
```

### 7.4 Audio Timing Adapter

负责将 MiniMax TTS 或用户音频统一为音频文件与 `AudioTimingSegment[]`。若供应商不返回逐句时间戳，优先逐段合成后拼接；备选方案是使用现有字幕对齐能力生成时间戳。

### 7.5 Shot Matcher

在桌面侧编排现有本地素材库搜索和在线供应商搜索，不进入 Hermes Agent Core。匹配器接收冻结的脚本计划、真实音频时间轴、指定来源和视频规格，返回完整镜头分配及原因。

### 7.6 Timeline and Render Adapter

复用现有素材缓存、时间线创建和 MoneyPrinterTurbo 任务 API。适配层接收已经确定的镜头计划，不再在渲染阶段重新随机选素材。

## 8. 自动镜头匹配算法

### 8.1 硬过滤

候选必须满足：

- 属于用户指定的命名素材库或在线供应商；
- 本地片段状态为 `ready`；
- 原视频路径仍然可访问；
- 片段时长能够覆盖所需时间，或可安全循环/裁切；
- 画幅可用于目标视频，竖屏项目优先竖屏或可安全中心裁切素材；
- 不包含排除标签。

任何派生输出目录都不得被重新当作原始素材扫描，包括：

- `02_精选镜头`；
- `03_关键帧`；
- `04_素材分析`；
- `timelines`；
- `.hermes-assets`。

### 8.2 排序信号

通过硬过滤的候选按以下信号排序：

1. 镜头意图和固定分类标签的匹配度；
2. 必需标签覆盖率；
3. 偏好标签与自然语言描述匹配度；
4. 现有素材库检索分数；
5. 分析质量和置信度；
6. 片段时长与音频段时长的适配度；
7. 目标画幅适配度；
8. 素材复用惩罚；
9. 相邻镜头相似惩罚。

计划 B 不以向量检索为前置条件。相邻相似度先使用资产 ID、镜头角色、标签重叠、关键帧感知哈希或已有画面特征实现；CLIP 向量检索保留到后续导演引擎阶段。

### 8.3 多样性规则

- 有其他合格候选时，相邻镜头不得来自同一资产。
- 同一资产在全片中的使用次数受上限控制。
- 连续两个镜头不能同时为高度相似的产品特写，除非脚本计划明确要求。
- 建立镜头、人物动作、产品特写和品牌结尾应按脚本意图形成节奏，而非只按关键词最高分重复选择。

### 8.4 同源降级

当某段没有完全匹配候选时：

1. 放宽偏好标签；
2. 放宽镜头角色但保留必需标签；
3. 使用同一来源中综合得分最高的通用镜头并记录警告；
4. 同一来源完全没有可用镜头时阻止生成，提示补充素材或切换来源。

系统不能自动跨库或联网补充素材。

## 9. 音频、字幕与音乐

- 文案确认后才生成正式旁白。
- 旁白时间戳决定镜头边界和字幕时间轴。
- 默认关闭素材原声；用户明确选择保留时才混入。
- 背景音乐自动裁切或循环到成片长度。
- 旁白出现时自动压低背景音乐，片头片尾可恢复。
- 音乐生成或下载失败时，若用户允许“无音乐继续”，则带警告继续；否则阻止提交。
- 音频与视频最终时长差不得超过 0.5 秒或总时长的 5%，取较小值。

## 10. 制作前检查

点击正式生成前必须检查：

- 文案已确认且没有空段；
- 目标时长与文案时长在可接受范围；
- MiniMax 音色 ID 字符合法、凭据存在且试听通过，或用户音频存在；
- 背景音乐配置有效；
- 选中的素材来源唯一且配置完整；
- 本地素材库存在，并有可用片段；
- 链接模式下的原始视频路径仍可读取；
- 每段至少有一个候选，或已经生成明确的同源降级警告；
- 输出目录可写；
- MoneyPrinterTurbo 服务健康；
- 页面向用户列出将发生的付费动作。

阻断项必须处理后才能生成；警告项可以由用户确认后继续。

## 11. 页面结构

Video Studio 主页面收敛为四个连续区域：

1. **创意输入**：主题、关键词、卖点、时长和风格。
2. **文案与镜头计划**：逐段编辑、重新生成、确认。
3. **声音与音乐**：音色、试听、音乐和混音设置。
4. **素材与生成**：唯一素材来源、制作前检查、一键生成、进度和预览。

每个阶段只有一个主操作按钮，并根据状态显示“生成文案”“确认文案”“检查并生成”“重试合成”。现有调试级参数放入“高级设置”，手工候选镜头放入成片后的“替换镜头”。

## 12. 持久化与可追溯性

每个任务保存不可变制作快照，至少包括：

- `ScriptPlan` 及内容哈希；
- 声音、音乐和混音参数；
- 素材来源与命名素材库 ID；
- 音频时间戳；
- 完整 `ShotAssignment[]`；
- 缓存文件与原始素材的映射；
- MoneyPrinterTurbo 任务 ID；
- 创建时间、版本和失败阶段。

任务列表和预览页应能回答“这一句文案用了哪个片段、为什么选它、原文件在哪里”。

## 13. 错误处理与恢复

- 文案生成失败：保留输入和上次成功文案。
- 结构化解析失败：展示解析错误，不创建音频或渲染任务。
- TTS 失败：停在声音阶段，可更换音色后重试。
- 素材匹配不足：展示具体缺失的镜头意图和标签。
- 缓存失败：不得写入半成品时间线；清理本次临时文件。
- 合成失败：保留制作快照、音频、镜头计划和缓存，允许只重试合成。
- 应用重启：从制作快照恢复到最近一个完整阶段。

## 14. 测试策略

### 14.1 单元测试

- 状态迁移和失效规则；
- `ScriptPlan` 解析与校验；
- `voice_id` 本地校验；
- 来源硬过滤；
- 匹配计分、同源降级和多样性惩罚；
- 音频时间戳到镜头时间线的转换；
- 制作前检查的阻断项与警告项。

### 14.2 组件测试

- 文案未确认时生成按钮不可用；
- 修改文案后音频和镜头计划失效；
- 页面只出现一个素材来源入口；
- 本地素材库和在线供应商互斥；
- 替换镜头只显示同来源候选；
- 失败后可在原制作快照上重试。

### 14.3 集成测试

在临时 `HERMES_HOME` 和临时素材库中运行真实模块导入及 API 合同测试：

- 创建命名素材库并索引测试视频；
- 生成结构化脚本计划；
- 使用固定测试音频生成真实时间轴；
- 自动匹配镜头并缓存；
- 创建 MoneyPrinterTurbo 任务；
- 验证任务记录中的来源和时间线没有跨库。

### 14.4 桌面端真实验收

必须使用当前开发版 `hermes-dev-desktop start`，而不是生产版或旧桌面包。人工操作一遍完整流程并保留日志、任务 ID、时间线和成片证据。

## 15. 牛肉面案例验收

使用已经索引的命名资产库，其中原始素材来自：

`/Users/ruoyu/Desktop/牛肉面`

输入一段至少包含以下三个意图的文案：

1. 前厅顾客吃面；
2. 员工端碗或上餐；
3. 成品牛肉面特写。

验收必须同时满足：

- 文案未经用户确认时不能开始正式合成；
- 每个意图至少产生一个自动镜头分配；
- 选中的本地镜头全部来自指定命名素材库；
- 时间线保留资产 ID、片段 ID、原路径、入点和出点；
- 有替代候选时，相邻镜头不重复同一资产；

- 旁白、字幕和视频时间轴对齐；
- 背景音乐没有盖住旁白；
- 最终输出可播放的 H.264/AAC MP4；

还需执行一个在线来源回归案例，验证选择 Pexels 后无需本地素材库，并且不会读取本地资产库作为隐式补充。

## 16. 分阶段交付

### 交付 1：流程收敛

- 创建会话状态机与制作快照；
- 合并素材入口；
- 增加文案确认门；
- 完成制作前检查。

### 交付 2：结构化文案与镜头匹配

- 生成 `ScriptPlan`；
- 增加意图、固定标签、质量和多样性联合排序；
- 保存选择原因与来源。

### 交付 3：音频优先时间线

- 统一 MiniMax TTS 与用户音频的时间戳；
- 根据真实旁白时长创建镜头、字幕和音乐时间线；
- 完成时长一致性检查。

### 交付 4：局部修订与真实验收

- 同来源替换镜头和锁定镜头；
- 局部重新生成；
- 完成牛肉面本地素材库及在线来源回归测试。

## 17. 兼容策略

- 现有文案、关键词、语音、音乐和合成 capability 继续复用。
- 现有自动时间线和素材缓存能力由新的制作控制器调用，避免同时存在两条独立生成路径。
- 旧草稿在首次打开时转换为 `draft_input`，不自动触发任何付费动作。
- 高级用户已有的参数继续保留，但移动到高级设置，不影响默认一键流程。
- 如果旧任务缺少来源追溯字段，只读展示为“旧任务”，不猜测其原始素材来源。

## 18. 完成定义

只有在以下条件全部成立时，方案 B 才算完成：

1. 桌面页面只有一个素材来源入口。
2. 用户确认文案后可通过一个主按钮完成预检、音频、匹配、时间线和合成。
3. 本地素材库模式能够自动选中已经分析、切片和打标签的素材，无需人工逐镜头选择。
4. 所有素材来源、时间点和匹配原因可追溯。
5. 音频时间轴真实驱动镜头和字幕。
6. 牛肉面真实案例与至少一个在线来源案例通过回归测试。
7. 开发版 Hermes 桌面端能够预览最终可播放成片，并能替换单个镜头后重新合成。
