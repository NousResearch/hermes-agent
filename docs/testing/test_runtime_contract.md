# Canonical test-runtime contract

`scripts/run_tests.sh` is the source of truth for local Hermes test execution.
In an isolated linked worktree it has the following contract:

1. The repository root is the Git worktree containing the caller's current
   directory, provided it contains `scripts/run_tests_parallel.py`. The script
   location is only a fallback for callers outside a worktree. This prevents a
   runner reached through an editable or primary checkout from selecting that
   checkout as the source root.
2. The test environment is rebuilt with `env -i`. `PYTHONPATH` is explicitly
   set to the selected worktree root (plus the deliberate live-guard plugin
   path, when present), and `PYTHONNOUSERSITE=1` prevents user-site packages
   from changing import resolution. Inherited `PYTHONPATH`, `PYTHONHOME`, and
   other import configuration are not propagated.
3. Python bytecode generated by the runner and its pytest children is directed
   to a `mktemp` directory through `-X pycache_prefix` and
   `PYTHONPYCACHEPREFIX`. The directory is removed when the runner exits.
4. Pytest's own cache and `test_durations.json` remain worktree-relative. They
   therefore belong to the selected linked worktree, never to the primary
   checkout. The duration file is optional runtime state and is ignored by
   Git; CI may upload it explicitly when needed.
5. The wrapper returns the exact non-zero status of the per-file runner. A
   failure cannot be converted into success by cleanup of the temporary
   bytecode directory.

The boundary is intentionally dependency-light and uses only Bash, Git,
Python's standard library, and the already-provisioned test environment. It
does not change worker, dispatcher, provider, production, or dependency
behavior.

## Security notes

- Runner root selection uses `git rev-parse` without a shell and accepts the
  result only when the expected runner exists under that worktree.
- Runtime paths are generated by `mktemp`; no inherited variable selects an
  arbitrary bytecode destination.
- The environment is allowlisted before pytest starts, so an inherited
  editable path cannot shadow the worktree source.
- The duration writer receives the selected root from the runner implementation;
  it does not accept a caller-controlled output path.
