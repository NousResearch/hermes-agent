## คะแนนรวมแบบซื่อสัตย์
**ทำจริง+ทดสอบแล้ว ≈ 75% ของทั้งระบบ** — เครื่องมือครบ + ทนขึ้น (RESULT/JSON) + ตัวจำ root ถาวร (promote) + shortcut ประตูเดียว + ปิดงานค้างย่อยหมด (Use Summary, status JSON)
ที่เหลือ ~25% = **งานที่ AI บนเครื่องนี้ทำแทนไม่ได้จริง ๆ (ถูกระบบความปลอดภัยกันไว้ ถูกต้องแล้ว):**
1. รันเครื่องมือบน VPS + โน้ตบุ๊ก 15 คน (ต้องอยู่บนเครื่องนั้น — SSH 16 เครื่องแทนไม่ได้)
2. นำ hook/agent เข้า ~/.claude (ติดด่าน relay · ต้องเจ้าของปลด)
3. merge PR เข้า main + ติดตั้ง cron จริง (เป็นการฝังตัวรันถาวร/outward · ระบบกันไม่ให้ AI ทำเอง · เจ้าของรันคำสั่งที่เตรียมให้)

## สิ่งที่ "สร้าง + ทดสอบจริง" แล้ว (tier 4)
- `hermes_std.py` sync/init — คืนโซนกลาง + เก็บโซน project (ทดสอบผ่าน)
- `hermes_scan.py` — จับไฟล์หาย/เกิน 100MB/โซนกลางถูกแก้ (good=HEALTHY, bad=BROKEN)
- `curse_track.py` — จับคำด่า ไทย/อังกฤษ + เตือนซ้ำเกิน 3 + เด้งกลับเมื่อด่าซ้ำหลังแก้ + ตาราง HTML
- `safe_apply.py` — กัน sync ทำ project พัง: รันเกณฑ์ project ก่อน/หลัง · พังแล้วถอยกลับเอง (ทดสอบ 2 ทาง: APPLIED_OK + BROKE_ROLLED_BACK)
- `hermes_rollout.py` — แจกหลาย project ทีเดียวจากทะเบียน projects.txt (ทดสอบ init 2 project ผ่าน)
- `hermes_analyze.py` — สมองวิเคราะห์คำด่าเป็นรอบ: ชี้ "แก้แล้วยังด่าซ้ำ=root ผิด" + ตัวที่ต้อง cross-check (ทดสอบผ่าน)
- `wire_agents.py` — ผูก agent ทุกตัวให้อ่านมาตรฐานกลาง (ทดสอบ 29 agent · รันซ้ำไม่เติมซ้ำ)
- `install_schedulers.py` — สร้างไฟล์ตั้งเวลา launchd/crontab (สแกนทุกวัน + วิเคราะห์ทุก 3 ชม.)
- **Pilot จริง** — รับ project เก่า (CLAUDE.md มีเนื้อหาเดิม + ไฟล์โค้ด) เข้าระบบ: โซนกลางขึ้นบน · ของเดิมอยู่ครบ · โค้ดไม่ถูกแตะ · เกณฑ์ผ่านก่อน/หลัง = APPLIED_OK
- แม่แบบไฟล์ควบคุม AI 2 โซน ครบทุกตัว + ai-tool-coverage + file-health-policy + gate-manifest + governance
