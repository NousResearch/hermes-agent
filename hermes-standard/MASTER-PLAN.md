# MASTER-PLAN — มาตรฐานกลาง Hermes (เอกสารแม่บท · กันมั่ว/กันลืม)

> ไล่จากต้นแชตทั้งหมด · อัปเดต 2026-06-27 · ถ้ามีคำสั่งใหม่ ต้องมาอัปเดตไฟล์นี้ก่อนเสมอ

## A. เป้าหมายสูงสุด (ปลายทาง · ห้ามหลุด)
สร้าง **AI เสมือนพนักงานรายแผนก (Agent) + เทรน Skill** บน Hermes คุมผ่าน VPS + โน้ตบุ๊กเจ้าของและพนักงานทุกคน
ให้พนักงานเรียกถาม/ใช้ช่วยงานได้ง่าย · ยืดหยุ่นกว่าการพิมพ์ shortcut ตายตัว
ของเดิมที่มีแล้ว: agent ธุรกิจ 20 ตัว / 6 แผนก + conductor-router + system agent 9 + shortcut 21 ตัว

## C. ความต้องการเฉพาะ (เงื่อนไขที่ห้ามพลาด)
1. ใช้กับ 30-40 project · บน VPS + โน้ตบุ๊กเจ้าของ + โน้ตบุ๊กพนักงานทุกคน · มาตรฐานเดียวกัน
2. **ไฟล์ควบคุม AI ทุกตัว (.cursorrules, .cursor/rules, CLAUDE.md, AGENTS.md/codex, GEMINI.md, QWEN.md) ต้องคุมจากส่วนกลางเท่านั้น**
3. เปิด project ใหม่ ต้องเข้าร่วมมาตรฐานได้ทันที
4. ของเก่าที่ไม่ชัด → ยกเลิก/รื้อใหม่ (แต่เก็บเข้ากรุก่อน ไม่ลบทันที)
5. คุณภาพสูงสุด + ปรับปรุงต่อได้เรื่อย ๆ

- โครงระบบ = "มาตรฐานกลาง Hermes" 5 ชั้น (แหล่งกลาง → ตัวแจก git → ด่านในเครื่อง → ตัวเฝ้า → วงจรเรียนรู้)
- ผ่าน cross-check GPT-5.5 = fix-then-proceed · รับ 3 จุดแก้: (ก) ของเก่าไม่ลบทันที archive ก่อน (ข) ตัวเฝ้าต้องบันทึก เจ้าของเครื่อง+เวอร์ชัน+สิทธิ์+ผลแก้ (ค) hermes init รันซ้ำไม่ทับงานคน · เพิ่มไฟล์สัญญา `hermes.project.yaml`

  - กลางล็อกเต็ม = ด่าน hook (project แก้ไม่ได้)

  - เฉพาะ project = Overview/Business/Feature/Design + .hermes + hermes.project.yaml
