# hermes-standard — มาตรฐานกลาง Hermes (ต้นฉบับเดียวคุมทุก project)

นี่คือ "บ้านกลาง" ของกฎและไฟล์ควบคุม AI ทั้งหมด · แก้ที่นี่ที่เดียว แล้วแจกลงทุก project (30-40 ตัว) บน VPS + โน้ตบุ๊กทุกคน ผ่าน git

## ทำไมต้องมี
ก่อนหน้านี้กฎกระจัดกระจาย ปนข้าม project · AI โกหก/ลืม/ทำงานนอกคำสั่ง · ไม่มีต้นฉบับเดียวที่บังคับได้จริง
ต้นฉบับนี้แก้ตรงนั้น: มาตรฐานชุดเดียว + ด่านบังคับ + ไฟล์บริบทกันลืม

## โครง
- `VERSION` — เลขรุ่นมาตรฐาน (ล็อกให้ทุกเครื่องตรงกัน)
- `rules/central-block.md` — เนื้อกฎกลาง (ตัวจริงที่ถูกฝังในไฟล์ควบคุม AI ทุกตัว)
- `templates/` — แม่แบบไฟล์ที่ทุก project ต้องมี
- `registry/legacy-inventory.md` — ทะเบียนของเก่า: เก็บ / รื้อ / เข้ากรุ
- `gates/` — โค้ดด่านบังคับ (ทำจริงคลื่น 2)

## 3 ระดับการควบคุม
1. กลางล็อกเต็ม — ด่าน hook · project แก้ไม่ได้
2. ผสม — `CLAUDE.md` `AGENTS.md` `.cursorrules` `GEMINI.md` `QWEN.md` · มีโซนกลาง (ล็อก) + โซน project (แก้ได้)
3. เฉพาะ project — `OverviewProgress.md` `BusinessPlan.md` `FeatureSpec.md` `DesignSystem.md` · ส่วนกลางให้แค่หัวข้อ เนื้อใน project เขียนเอง

## หลักการแยกโซน (ไฟล์ผสม)
ทุกไฟล์ผสมมี 2 โซนคั่นด้วยเครื่องหมาย · คำสั่ง sync เขียนทับเฉพาะโซน `HERMES-CENTRAL` เท่านั้น · โซน `PROJECT` ไม่แตะ → คุมจากกลางได้จริง + project ปรับเองได้ + รันซ้ำไม่ทับงานคน
