# ผูก Agent + Skill เข้ากับราก (req 1, 2, 4, 5)

> ปลายทางของทั้งระบบ · Agent/Skill รันบน "ราก" (มาตรฐานกลาง + ด่าน) ที่วางไว้

## หลักการ
1. **Agent ทุกตัวอยู่ใต้กฎกลาง** — agent 20 ตัวเดิม (6 แผนก) ต้องอ่านโซนกลางเดียวกัน → ไม่โกหก/ไม่นอกคำสั่ง เหมือนกันหมด
2. **เรียกใช้ง่าย (req-4)** — ประตูเดียว: พนักงานพิมพ์คำถาม → ตัวจ่ายงาน (conductor-router เดิม) เลือกแผนก + เลือก AI → ตอบ · ไม่ต้องจำชื่อ agent
3. **ยืดหยุ่นกว่า shortcut (req-5)** — agent "คิดเองตามสถานการณ์" ใต้กรอบกฎกลาง · shortcut เป็นแค่ทางลัดของงานที่ทำซ้ำบ่อย ไม่ใช่ทางเดียว
4. **Skill ต่อแผนก (req-2)** — แต่ละแผนกมีโฟลเดอร์ skill ที่โตได้ · skill ใหม่ = เพิ่มไฟล์ ไม่กระทบ agent อื่น

## ลำดับทำจริง (หลังรากเสร็จ 100%)
1. ใส่ block "อ่านมาตรฐานกลาง" ในนิยาม agent ทุกตัว (ผ่าน sync)
2. ต่อ conductor-router ให้ทำงานจริง (รับคำถาม → ส่งแผนก)
3. ทำประตูเดียวให้พนักงานเรียก (พิมพ์ถามที่เดียว)
4. วางโครง skill ต่อแผนก + วิธีเพิ่ม skill

## เงื่อนไขเริ่ม
ทำได้เมื่อ "ราก" (ด่านตรวจความจริง + มาตรฐานกลาง) ผ่าน 100% ก่อน · ตอนนี้รากยัง ~45-50% → ยังไม่เริ่มผูก agent
