# AI Tool Coverage — เครื่องมือ AI แต่ละตัว คุมด้วยไฟล์ไหน (req-25)

> ครอบคลุม AI ทุกตัวที่เจ้าของใช้ · บอกชัดว่าตัวไหนอ่านไฟล์ควบคุมไหน

| เครื่องมือ AI | ไฟล์ควบคุม (โซนกลางคุม) | หมายเหตุ |
|---|---|---|
| Claude Code | `CLAUDE.md` + `.claude/hooks/` | มีไฟล์เฉพาะตัว |
| Codex | `AGENTS.md` (= "memory codex" ที่เจ้าของพูดถึง) | Codex ใช้ AGENTS.md เป็นไฟล์จำ/คำสั่ง (ยืนยันจาก ~/.codex/AGENTS.md) |
| Cursor | `.cursorrules` + `.cursor/rules/*.mdc` | มี 2 แบบ ทำครบแล้ว |
| Gemini CLI | `GEMINI.md` | มีไฟล์เฉพาะตัว |
| Qwen | `QWEN.md` หรือ `AGENTS.md` | อ่าน AGENTS.md ได้ด้วย |
| CrossCode | `cross-code.md` | ทำแม่แบบแล้ว |
| Grok | `AGENTS.md` (ฉีดโซนกลางตอนเรียกผ่าน relay) | **ไม่มีไฟล์ประจำ project ของตัวเอง** → คุมโดยป้อนโซนกลาง/AGENTS.md ตอนสั่งงาน |
| GPT (ทั่วไป) | `AGENTS.md` (ฉีดโซนกลางตอนเรียก) | เหมือน Grok · ไม่มีไฟล์ประจำ → คุมตอนเรียกผ่าน relay |

## หลักการสำหรับ AI ที่ไม่มีไฟล์ประจำ project (Grok / GPT)
เครื่องมือที่ไม่อ่านไฟล์ในโฟลเดอร์ project เอง จะถูกคุมด้วยการ **ฉีดเนื้อโซนกลาง (central-block) เข้าไปในคำสั่งตอนเรียกใช้** ผ่านตัวกลาง (relay) · ไม่ปล่อยให้รันลอยนอกกฎกลาง
