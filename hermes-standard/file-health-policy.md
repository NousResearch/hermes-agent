# File Health Policy — กติกาสุขภาพไฟล์ (req-26, req-27)

> ตัวเฝ้าศูนย์กลาง (คลื่น 3) ใช้กติกานี้ตอนสแกนทุกวัน · **ไม่ลบไฟล์เองอัตโนมัติ**

## เกณฑ์ขนาดไฟล์
- ปกติ: ไฟล์ ≤ 100 MB
- เกิน: ไฟล์ > 100 MB = ผิดปกติ ต้องจัดการ

## ถ้าไฟล์เกิน 100 MB ทำอะไร (ตามลำดับ · ไม่ลบเอง)
1. **เตือนเจ้าของไฟล์** — ขึ้นในรายงานสแกน ว่าเครื่องไหน/project ไหน/ไฟล์อะไร/ใหญ่เท่าไร
2. **ถ้าเป็นไฟล์ที่สร้างใหม่ได้ (build/cache/log)** → แนะนำใส่ `.gitignore` + เอาออกจาก git (กัน repo บวม)
3. **ถ้าเป็นข้อมูลจำเป็น (วิดีโอ/รูป/ชุดข้อมูล)** → แนะนำย้ายไปที่เก็บนอก git (object storage / โฟลเดอร์ data แยก) แล้วทิ้งลิงก์ไว้แทน
4. **ถ้าไม่แน่ใจ** → มาร์ค "รอเจ้าของตัดสิน" ไม่แตะ
5. ทุกการกระทำต้องให้เจ้าของไฟล์กดยืนยันก่อน · ระบบไม่ลบ/ไม่ย้ายเอง

## ไฟล์ผิดปกติอื่นที่สแกนจับ (คลื่น 3)
- ไฟล์มาตรฐานที่ต้องมีแต่หาย (เทียบกับ `required_files` ใน hermes.project.yaml)
- โซนกลาง (HERMES-CENTRAL) ถูกแก้มือ / รุ่นไม่ตรงกับส่วนกลาง
- ไฟล์ความลับหลุด (key/token) อยู่ผิดที่
