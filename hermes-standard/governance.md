# Governance — สิทธิ์ + ความปลอดภัย + รุ่น + ถอยกลับ (req 56-65)

> ตอบจุดที่ Codex จับว่าขาด · บางข้อทำด้วย git/scan ที่มีแล้ว บางข้อเป็นกติกา

## req-56 · สิทธิ์ใครแก้อะไร
- **เจ้าของ/แอดมิน** = แก้ `hermes-standard/` (โซนกลาง + กฎ + ด่าน) ได้
- **พนักงาน** = แก้ได้เฉพาะ "โซน project" และไฟล์เฉพาะ project · ห้ามแตะโซนกลาง
- บังคับจริงด้วย: โซนกลางมีป้าย "ห้ามแก้มือ" + `hermes_scan.py` จับการแก้ + (บน git) ตั้ง protected branch / code owners ที่ repo กลาง

## req-57 · ร่องรอยว่าใครแก้กฎกลาง
- ทุกการแก้ `hermes-standard/` ผ่าน git commit = มี ใคร/เมื่อไร/ข้อความเหตุผล อยู่แล้ว
- ดูร่องรอย: `git log -- hermes-standard/`

## req-58 · แจกให้รุ่นตรงกันทุกเครื่อง
- รุ่นอยู่ที่ไฟล์ `VERSION` ไฟล์เดียว · แจกด้วย `git pull` + `hermes_std.py sync`
- เครื่องไหนยังไม่ sync = ใช้รุ่นเก่า → ตรวจเจอด้วย scan (ข้อถัดไป)

## req-59 · รุ่นไม่ตรงทำยังไง
- `hermes_scan.py` เทียบโซนกลางในแต่ละ project กับส่วนกลาง · ต่าง = ขึ้น "โซนกลางถูกแก้/รุ่นไม่ตรง" (HIGH)
- ทางแก้: รัน `hermes_std.py sync` ทับโซนกลางให้ตรง · (อนาคต: เพิ่มบล็อกถ้ารุ่นห่างมาก)

## req-60 · ถอยกฎกลับ (rollback)
- `hermes-standard/` อยู่บน git → ถอยด้วย `git revert`/`git checkout <รุ่นก่อน> -- hermes-standard/` แล้ว `sync` ใหม่ = ทุก project กลับกฎเก่าทันที

## req-61 · ความลับ
- ห้ามเก็บ key/token ใน repo · `hermes_scan.py` ตั้งให้จับไฟล์ความลับที่อยู่ผิดที่ (คลื่นถัดไปเพิ่มรูปแบบ)
- log คำด่า (events.jsonl) เก็บในเครื่อง/ที่เก็บภายใน ไม่ push ขึ้น repo สาธารณะ

## req-62 · คนที่ไม่ได้ใช้ Hermes / หลุดจากระบบ
- ถ้าใช้ editor ที่อ่านไฟล์ project ได้ (Cursor/Claude/Codex) → ยังได้กฎกลางจากไฟล์ในโฟลเดอร์ แม้ไม่รัน Hermes
- เครื่อง/คนที่ไม่ลงทะเบียนเข้า scan = มองไม่เห็น → ต้องมี "ทะเบียนเครื่อง/คน" ให้ตัวเฝ้าไล่ครบ (คลื่นถัดไป)

## req-63 · ทุกข้อต้องอ้างแชต ไม่เดา
- `REQUIREMENTS.md` ทุกข้อดึงจากข้อความเจ้าของในแชต · ห้ามเพิ่มข้อจากความจำลอย ๆ

## req-64 · สถานะละเอียด
- ใช้ 4 สถานะ: ต้องทำ / กำลังทำ / ทำแล้ว(ทดสอบผ่าน) / รอเจ้าของยืนยัน · บันทึกใน `COMPLY.md`

## req-65 · เกณฑ์ปิดงาน (ก่อนบอกว่า "ครบ")
ปิดได้เมื่อครบทั้ง 3: (1) สร้าง + ทดสอบจริงเห็นพฤติกรรม (tier 4) · (2) Codex ตรวจครบไม่ตก · (3) เจ้าของยืนยัน
