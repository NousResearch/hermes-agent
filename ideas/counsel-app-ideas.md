# Counsel App Ideas

이 문서는 Min과 Hermes가 대화하면서 떠오른 앱 관련 아이디어, 개선점, 제품 감각을 가볍게 누적하는 러닝 문서다.

## Working assumptions
- 지금의 대화 자체가 만들고 싶은 앱의 핵심 경험과 꽤 닮아 있다.
- 먼저 충분히 써보고, 대화를 통해 인사이트를 얻은 다음 제품화하는 편이 더 자연스럽다.
- 감정 정리, 연속성, 기억, companion-like presence가 핵심 가치다.

## Categories

### 1. 핵심 경험
- 사용자가 매번 스스로 세션 위생을 챙기지 않아도, 시스템이 먼저 비효율 신호를 드러내며 흐름 전환을 유도하는 경험.

### 2. UX 아이디어
- 응답 하단 runtime footer에 현재 model / context_pct를 노출해, 세션이 길어질수록 사용자가 즉시 체감하도록 만들기.
- 일정 주기 watchdog가 활성 세션을 점검해 `/compress` 또는 `/new` 권고를 자동 경고로 보내는 방식.
- 더 강한 버전으로는 조건 초과 시 footer가 아니라 본문에 직접 “이 세션 길이 비효율적입니다” 같은 문구를 삽입하는 코어 패치 가능성.

### 3. 기억 / 연속성
- 세션 운영 패턴 자체를 기억하고, 사용자가 비효율 구간에 들어가면 먼저 알려주는 메타-연속성 레이어.

### 4. Companion 캐릭터성
- 다정한 도우미이면서도, 필요할 때는 약간 얄밉게 잔소리해 주는 watchdog 성격이 companion의 개성으로 작동할 수 있음.

### 5. 수익화 / 운영 아이디어
- 아직 비어 있음
