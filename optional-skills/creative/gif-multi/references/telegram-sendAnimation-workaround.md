# Telegram sendAnimation Workaround

## Problem

Hermes' `MEDIA:<path>` delivery mechanism sends MP4 files via the platform adapter's `send_video()` method. On Telegram, `sendVideo` displays content as a regular video (with play button and controls), **not** as an animated GIF.

## Workaround

For Telegram, bypass Hermes MEDIA delivery and call the Bot API directly:

```bash
# Get bot token
BOT_TOKEN=$(grep TELEGRAM_BOT_TOKEN ~/.hermes/.env | grep -v "^#" | cut -d= -f2-)

# Send as Animation (GIF)
curl -s -X POST "https://api.telegram.org/bot${BOT_TOKEN}/sendAnimation" \
  -F chat_id=<CHAT_ID> \
  -F message_thread_id=<THREAD_ID> \
  -F animation=@/path/to/gif.mp4
```

- `chat_id`: Telegram user/group ID
- `message_thread_id`: required when sending to a topic; omit for root DM
- The file must be a valid MP4 (as generated by gif-multi)
- Add `-F caption="text"` to include a caption with the animation

## Why This Happens

The gateway's `_deliver_media_from_response()` in `gateway/run.py` matches file extensions against `_VIDEO_EXTS` and calls `adapter.send_video()`. It does not use `sendAnimation`, which is the Telegram-specific method for GIF-style content.

## When to Use

- GIFs on Telegram that need to auto-play and loop like a proper reaction GIF
- Always prefer this over MEDIA for Telegram GIF delivery
