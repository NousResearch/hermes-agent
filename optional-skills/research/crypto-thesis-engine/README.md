# Crypto Thesis Engine

> A self-improving Hermes Agent skill that transforms raw crypto market data into institutional-quality investment thesis reports — and gets smarter with every analysis.

## 🎯 What This Is

This is **not** another price bot. This is a **learning research analyst** skill for [Hermes Agent](https://github.com/NousResearch/hermes-agent) that:

- 📊 **Fetches** live market data from CoinGecko (free API, no key needed)
- 🧮 **Computes** derived metrics (momentum signals, dilution risk, liquidity grades)
- 🧠 **Thinks** like a research analyst — derives meaning from data, doesn't just repeat numbers
- 📚 **Learns** from outcomes — track correct/wrong predictions to improve over time
- 🎯 **Scores confidence** based on historical accuracy across tokens and categories
- 🔍 **Detects patterns** — surfaces similar past analyses before making new predictions

## 🚀 Commands

| Command | Description |
|---------|-------------|
| `/crypto-thesis-engine analyze <token>` | Full investment thesis with confidence score |
| `/crypto-thesis-engine compare <t1> <t2>` | Side-by-side comparison of 2-5 tokens |
| `/crypto-thesis-engine report <token> --format brief` | Quick executive summary |
| `/crypto-thesis-engine scan <category>` | Scan a sector for opportunities |
| `/crypto-thesis-engine learn <token> <correct\|wrong>` | **🆕** Teach the system whether an analysis was accurate |
| `/crypto-thesis-engine history <token>` | View past analyses and outcomes |
| `/crypto-thesis-engine stats` | **🆕** View performance dashboard |

## 🧠 Learning System

The skill maintains persistent memory in `~/.hermes/thesis-memory/memory.json`:

### How it works

1. **Analyze** — Every analysis is automatically recorded with price, momentum, sentiment, categories
2. **Learn** — User tells the system if the analysis was correct or wrong
3. **Improve** — Future analyses compute confidence scores based on:
   - Token-specific accuracy history (35% weight)
   - Category accuracy (e.g., "How good are we at L1 tokens?") (25% weight)
   - Overall global accuracy (15% weight)
   - Data completeness (10% weight)
   - Recent related analysis accuracy (15% weight)
4. **Pattern Awareness** — Before analyzing, the system checks for similar past analyses (same category, similar market cap tier) and surfaces their outcomes

### Example Flow

```bash
# Analyze Solana
/crypto-thesis-engine analyze solana
# → Report includes: "🎯 Confidence: 72% (MODERATE)"
# → Report includes: "🔁 Similar: ETH (correct), AVAX (wrong)"

# Later, price confirms the thesis
/crypto-thesis-engine learn solana correct

# Next time you analyze an L1 token, confidence is higher
/crypto-thesis-engine analyze near
# → "🎯 Confidence: 78% (MODERATE) — Layer-1 track record: 3/4 (75%)"
```

- **Hermes Agent** with `terminal` toolset enabled

## 📊 What Gets Analyzed

### Raw Data (from CoinGecko)
- Price, market cap, volume, FDV
- Circulating/total/max supply
- Price changes across 6 timeframes
- ATH/ATL with dates
- Community metrics

### Derived Metrics (computed by script)
- MCap/FDV ratio → dilution risk assessment
- Volume/MCap ratio → liquidity classification
- Multi-timeframe momentum signals
- Supply dynamics and inflation headroom

### AI Analysis (generated by Hermes)
- **Confidence Score** — Data-driven confidence based on learning history
- **Overview** — What is this project and where does it fit?
- **Bull Case** — 3-5 data-supported reasons it could go up
- **Bear Case** — 3-5 counterarguments
- **Risk Matrix** — 7 risk categories rated LOW/MEDIUM/HIGH
- **Catalysts** — Upcoming events with confirmed/speculated markers
- **Comparable Analysis** — Peer comparison with implied pricing
- **Similar Past Analyses** — Pattern awareness from previous analyses
- **Time Horizon Commentary** — Short/medium/long-term outlooks
- **Verdict** — Synthesized opinion (not a recommendation)
