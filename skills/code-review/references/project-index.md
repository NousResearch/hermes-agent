## gambit-group/dac-monorepo
- runtime: node (NestJS backend + Nuxt 3 frontend)
- package_manager: pnpm
- test: No test framework detected for api-enterprise-v2 or web-enterprise-v2 packages
- build: Not tested (monorepo, likely `pnpm build`)
- lint: Not detected
- framework: NestJS (backend), Nuxt 3 + Vue 3 (frontend)
- last_seen: 2026-06-04
- notes: Monorepo with multiple apps under `apps/`. Backend uses MongoDB (Mongoose). Frontend uses Nuxt 3 composables pattern. `useFetch` body must be wrapped in `computed()` for reactivity. No test framework detected for `api-enterprise-v2` or `web-enterprise-v2` тАФ test execution will always be skipped for PRs touching these packages. **Key pattern:** `isLocalEnv()` is copy-pasted across 11 services' `helpers.ts` (not shared). Guards in `api-fireblocks-webhook`, `api-alchemy-webhook`, and `api-enterprise-v2` use `isLocalEnv()` to bypass IP whitelisting тАФ changes to `isLocalEnv` affect both pino-pretty transport AND IP guard bypass (dual-use). `api-alchemy-webhook` has a divergent implementation (only checks `'local'`, no `'dev'` match). PR #312 fixed K8s dev crash by narrowing `'dev'` тЖТ `'dev.'` prefix. Frontend apps (`web-admin-v2`, `web-enterprise-v2`) have their own `isLocalEnv` in `env.utils.ts` that explicitly checks `APP_ENVIRONMENT` for 'dev' тАФ not affected by backend changes but same vulnerable pattern for cookie security. Route restructuring pattern: when moving pages between routes, access control inline checks (wallet access redirects) are often dropped тАФ always diff old vs new page for auth guards. Sidebar link visibility does not equal route protection. **Confirmed 2026-06-04 (PR #310):** Asset wallet page moved from `/` to `/asset-wallets` тАФ old inline wallet-access redirect (checked policy templates API, redirected to `/doc`) was dropped. No route middleware protects the new route. Middleware files: `auth-session.ts` (login check only), `admin-only.ts` (root/super_admin), `role-permission-redirect.ts` (requires `definePageMeta({ permission })`). None cover wallet-access gating. **Nuxt 3 `useFetch` deduplication:** Using a `key` option in `useFetch` deduplicates network requests across consumers in the same render cycle. `useUserWalletAccess` composable (PR #311) uses `key: 'user-wallet-policy-templates'` to share wallet policy template data. **Transaction approval wallet filtering:** Backend accepts `assetWalletIds` query param (validated by class-validator `@IsArray`/`@IsString`/`@Transform`). The `$or` тЖТ `$and` wrapping in `transactions.service.ts` correctly combines role-based and wallet-based filters.
