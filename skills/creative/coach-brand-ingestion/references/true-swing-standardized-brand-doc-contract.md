# True Swing standardized brand-doc contract

Use this as the concrete reference shape for brand-ingestion outputs that are intended to feed:
1. premium lesson recap packets, and/or
2. branded slide-template workflows.

## Required section order
1. Brand Snapshot
2. Identity
3. Color System
4. Typography
5. Visual Style
6. Documents / Application
7. Source Notes

## Why this matters
A weaker summary-style PDF is not sufficient. The downstream recap/slide workflow needs a deterministic visual source-of-truth artifact, not just tasteful prose.

## Required visible proof in the rendered artifact
The final rendered PDF/proof must visibly show:
- cover/title treatment
- source-backed identity mark/logo/icon
- named color swatches with visible fills and hexes
- typography hierarchy with examples
- application guidance for recap + slide use
- provenance / source-notes section

## Identity guidance
- Prefer source-backed official marks shown on the public site or provided assets.
- Do not let a made-up support motif (for example, an improvised monogram) replace the real identity system.
- If a support motif is used only as a compositional helper, label it clearly as secondary/exploratory.

## Render QA gate
Before calling the PDF complete, confirm in the rendered artifact itself:
- logo/wordmark/icon is visible
- color swatches render as visible blocks
- no silent asset drops from SVG/file-URI/export behavior
- the packet is at least as strong in artifact class as the accepted True Swing model

## NO-GO examples
- clean memo with brand prose but no visible identity system
- named palette without rendered swatches
- source HTML contains assets but exported PDF drops them
- improvised monogram dominates while source-backed official mark is absent or minimized
