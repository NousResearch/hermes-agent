# Ebrain Detail Page Builder

한국 마켓 상품 링크를 입력하면 제품 정보와 이미지를 분석해 스마트스토어·쿠팡용 상세페이지 제작 흐름으로 연결하는 Hermes Agent 스킬입니다.

이 README는 외부 수강생, 강의 참여자, 운영자가 이 스킬의 목적과 사용 흐름을 빠르게 이해할 수 있도록 정리한 안내 문서입니다.

## 한 줄 요약

도매매·도매꾹·스마트스토어·쿠팡 등 한국 마켓 상품 링크를 Hermes Agent에 전달하면, 상품 분석부터 상세페이지 기획, 한국어 카피, 이미지 생성 프롬프트, QA, 최종 상세페이지 합본까지 이어지는 제작 워크플로우를 실행합니다.

## 언제 사용하나요?

다음과 같은 상황에서 사용합니다.

- 도매꾹 또는 도매매 상품 링크로 스마트스토어용 상세페이지를 만들고 싶을 때
- 공급사 상세 설명을 그대로 복사하지 않고 B2C 소비자 관점으로 다시 구성하고 싶을 때
- 쿠팡/스마트스토어에 올릴 대표 이미지와 세로형 상세페이지 이미지를 만들고 싶을 때
- 상품 분석, 구매 이유, 섹션 구성, 카피, 이미지 프롬프트, QA 기준을 한 번에 정리하고 싶을 때
- 이커머스 강의나 실습에서 “상품 소싱 → 상세페이지 제작” 전체 흐름을 보여주고 싶을 때

## 핵심 결과물

최종 납품 기준은 아래 2장입니다.

1. 썸네일/대표 이미지 1장
2. 세로로 합쳐진 전체 상세페이지 이미지 1장

섹션별 이미지는 중간 작업물입니다. 수정, QA, 재합본을 위해 보관하지만 최종 납품물로 여러 장을 따로 보고하지 않습니다.

## 기본 작업 흐름

1. 상품 링크 접수
2. 상품명, 옵션, 구성품, 주요 특징, 주의사항 등 사실 정보 추출
3. 원본 제품 이미지와 상세 이미지 확인
4. 일반 소비자 B2C 구매 관점으로 재해석
5. 스마트스토어·쿠팡 상세페이지 섹션 구성
6. 섹션별 한국어 카피 작성
7. 이미지 생성용 프롬프트 작성
8. ChatGPT/OpenAI OAuth/OAuth2 이미지 생성 환경 또는 설정된 이미지 생성 도구로 섹션 이미지 생성
9. 생성 이미지의 제품 형태, 한글 오탈자, 과장 표현, 플랫폼 리스크 QA
10. 통과한 섹션 이미지를 세로로 합쳐 전체 상세페이지 이미지 제작
11. 썸네일 1장과 전체 상세페이지 1장을 최종 산출물로 정리

## 예시 요청문

Hermes Agent에서 아래처럼 요청할 수 있습니다.

```text
이 도매꾹 상품 링크를 기준으로 스마트스토어·쿠팡용 B2C 상세페이지를 만들어줘.

상품 URL: <상품 링크>

최종 결과물은
1. 대표 이미지 1장
2. 세로 합본 상세페이지 이미지 1장
으로 정리해줘.

확인되지 않은 인증, 배터리 용량, 사용 시간, 후기 수, 1위/최저가 표현은 쓰지 말고,
도매/판촉 문구는 일반 소비자가 1개 구매하는 관점으로 바꿔줘.
```

## 이미지 생성 실행 방식

이미지 생성은 특정 API 키를 README에 고정하지 않습니다. 우선순위는 다음과 같습니다.

1. Hermes/OpenClaw 환경에서 ChatGPT/OpenAI OAuth/OAuth2 기반 이미지 생성이 가능하면 그 런타임으로 GPT Image / ChatGPT Image 섹션 이미지를 생성합니다.
2. 별도 Hermes 이미지 생성 도구가 설정되어 있으면 승인된 섹션 프롬프트를 그 도구에 전달합니다.
3. 직접 이미지 생성이 불가능한 환경에서는 제품별 프롬프트팩을 먼저 만들고, 사용자가 ChatGPT Image에 직접 붙여 넣어 생성할 수 있게 합니다.

핵심은 “어떤 렌더러를 쓰느냐”보다 각 섹션 이미지가 제품 원형, 정확한 한국어, 근거 있는 표현, 플랫폼 리스크 기준을 통과하는지입니다.

## 제품별 섹션 설계 예시

이 스킬은 특정 강의자, 특정 교육자료, 특정 파일럿 상품명을 전제로 하지 않습니다. 상품 링크와 소스 이미지를 기준으로 제품에 맞는 섹션 수와 순서를 새로 설계합니다.

예시 흐름:

```text
상품 링크
→ 상품명/모델명/구성품/인증/배송/주의사항 확인
→ 도매·판촉 문구를 B2C 소비자 구매 문구로 변환
→ 스마트스토어·쿠팡용 상세페이지 섹션 구성
→ GPT Image / ChatGPT Image용 한국어 이미지 프롬프트 작성
→ 섹션별 이미지 생성 및 QA
→ 전체 상세페이지 이미지로 세로 합본
```

섹션 구조는 아래처럼 제품별로 필요한 모듈만 선택합니다. 항상 17개로 고정하지 않습니다.

```text
00_main_thumbnail_product_only.png
01_hero_stopper.png
02_problem_or_use_context.png
03_product_solution.png
04_key_benefit_summary.png
05_feature_proof_or_detail.png
06_use_case_scene.png
07_components_or_options.png
08_trust_checkpoints.png
09_faq_or_notice.png
10_final_cta.png
```

## 도매/판촉 문구를 B2C 문구로 바꾸는 기준

도매꾹·도매매 상품 페이지에는 대량구매, 판촉물, 인쇄 가능, OEM, 특판 같은 표현이 자주 포함됩니다. 이 스킬은 해당 표현을 그대로 상세페이지에 넣지 않고 일반 소비자 구매 상황에 맞게 바꿉니다.

예시:

| 도매/판촉 관점 | B2C 상세페이지 관점 |
| --- | --- |
| 대량구매 가능 | 필요할 때 바로 쓰는 개인용 제품 |
| 판촉 사은품 | 여름 외출·출퇴근·산책용 아이템 |
| 인쇄 가능 | 제품 외관과 구성품 확인 |
| 단가 중심 | 사용 상황과 구매 이유 중심 |
| 공급사 정보 중심 | 고객이 궁금해하는 사용 장면 중심 |

## QA 체크리스트

상세페이지를 최종 제출하기 전에 아래 항목을 확인합니다.

- 제품 형태, 색상, 버튼, 구성품이 원본과 크게 달라지지 않았는가?
- 이미지 안의 한국어 문구가 정확한가?
- 깨진 한글, 이상한 글자, 임의 문구가 들어가지 않았는가?
- 확인되지 않은 인증, 배터리 용량, 사용 시간, 성능 수치가 들어가지 않았는가?
- 후기 수, 판매 1위, 최저가, 만족도 같은 근거 없는 표현이 들어가지 않았는가?
- 의료/효능, 완전 안전, 무조건, 100% 같은 위험 표현이 들어가지 않았는가?
- 쿠팡 대표 이미지는 흰 배경 제품 단독 구성 기준을 지키는가?
- 스마트스토어 상세페이지는 모바일에서 읽기 쉬운가?

## 주의사항

- 확인되지 않은 성능, 인증, 사용 시간, 배터리 용량, 후기, 순위, 최저가 표현은 만들지 않습니다.
- 쿠팡 대표 이미지는 흰 배경 제품 단독 구성을 우선합니다.
- 스마트스토어 상세 이미지는 모바일 가독성과 짧은 한국어 문구를 우선합니다.
- 제품 원본 형태, 버튼, 색상, 구성품은 임의로 바꾸지 않습니다.
- 인증, 안전성, 의료/효능, 국내 최저가, 1위 같은 민감 표현은 반드시 검수합니다.
- 원본 상품 이미지와 상세 이미지의 사용 권한은 판매자 또는 운영자가 별도로 확인해야 합니다.
- 특정 마켓, 공급사, 브랜드와의 공식 제휴처럼 오해될 수 있는 표현은 사용하지 않습니다.

## 수강생 실습 방식

수강생은 아래 순서로 실습하면 됩니다.

1. 사용할 상품 링크를 준비합니다.
2. 판매 채널을 정합니다. 기본값은 스마트스토어와 쿠팡입니다.
3. 개인 고객이 1개 구매하는 상황을 기준으로 제품의 구매 이유를 정리합니다.
4. 상세페이지 섹션은 제품에 맞게 설계합니다. 섹션 수는 고정하지 않고 상품 특성에 맞춰 조정합니다.
5. 섹션별 이미지를 생성한 뒤 한글 오탈자와 제품 왜곡 여부를 검수합니다.
6. 최종적으로 대표 이미지 1장과 전체 상세페이지 이미지 1장만 제출합니다.
