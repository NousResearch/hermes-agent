Hermes Agent — System Architecture

Two-zone topology: Host Machine (macOS) + Cloud Services  |  Local execution — no sandbox

Host Machine (Keith's Mac)
  • Hermes process runs here (Python, macOS)
  • ~/.hermes/ — config, skills, memory, sessions
  • Gateway, CLI, cron scheduler — all host-side
  • Agent loop + prompt builder execute locally
  • terminal/file/code tools run DIRECTLY here
  • Full access to filesystem, git, localhost, SSH
Local Execution (the upgrade)
  • terminal(), read_file(), write_file() run on macOS
  • patch(), search_files(), execute_code() run on macOS
  • Persistent filesystem — projects survive across sessions
  • Can access ~/projects, git repos, local dev servers
  • Can SSH to remote servers, run Docker, manage deploys
  • Replaced Modal sandbox — zero isolation overhead
Cloud Services
  • LLM: Nous Portal → Claude Opus 4.6 (current)
  • Browser: Browserbase (Nous managed)
  • Web: Firecrawl search + extract (Nous managed)
  • Image: FAL generation (Nous managed)
  • TTS: Edge TTS (currently active)
  • 20+ LLM providers available with credential rotation
The Agent Loop
  • Runs on host — up to 90 turns per conversation
  • Each turn: build prompt → call LLM → check for tools
  • Tool calls dispatched LOCALLY or to cloud services
  • Results appended to context → next LLM call
  • Context compression triggers near token limit
  • Subagents: up to 3 parallel, each with own session
