# Seedance Script Designer 新人使用手册

本手册说明 `seedance-script-designer` 这个 Skill 的整体架构、工作流程、模块分工、输出标准、验证方法和维护注意事项。目标是让第一次接触这个 Skill 的人，能够在读完一遍后理解它如何工作、什么时候调用、产出什么，以及修改时不能破坏哪些规则。

## 1. Skill 定位

`seedance-script-designer` 是一个面向 **即梦 Seedance 2.0 视频创作** 的剧本拆解、电影级分镜、关键帧表、视频提示词和生产资料设计 Skill。

它不是普通写作助手，也不是单纯的提示词模板。它的核心任务是把以下输入转换成可生产的视频设计资料：

- 完整剧本或片段剧本
- 模糊的视频创意
- 已有分镜表或旧表格
- 需要续写、延长或衔接的已有视频
- 音乐驱动、无台词或少台词的视频段落
- 需要生成九宫格关键帧分镜图、视频提示词、音频同步表或素材规划的需求

默认工作语言是中文。除非用户明确要求英文或其他语言，所有解释、表格和提示词设计都优先用中文。

## 2. 触发场景

当用户提到以下关键词或需求时，应该使用这个 Skill：

- 即梦、Seedance、Seedance 2.0
- 视频提示词、AI 视频、图生视频、文生视频
- 分镜、分镜表、电影级分镜、九宫格、3x3 关键帧
- 剧本拆解、短剧、角色设计、场景图、道具图
- 音频同步、台词、旁白、音效、音乐卡点
- 首尾帧、视频延长、一镜到底
- 旧表格规范化、关键帧表升级

如果用户只问一般写作、普通翻译、代码开发或与 Seedance 视频生产无关的问题，不应该强行套用这个 Skill。

## 3. 总体架构

当前 Skill 目录结构如下：

```text
seedance-script-designer/
  SKILL.md
  README.md
  references/
    story-structure-method.md
    shot-breakdown-method.md
    output-schema.md
    storyboard-annotation-rules.md
    quick-modes.md
    release-checklist.md
  assets/
    README.md
    asset-manifest-template.md
  examples/
    minimal-12col-keyframes.md
    asset-manifest-example.md
    audio-sync-example.md
  scripts/
    README.md
    convert-keyframe-table.mjs
    validate-keyframe-table.mjs
    check-dist-freshness.mjs
    verify-package-contents.mjs
```

每一层的职责不同：

- `SKILL.md` 是入口规则，决定什么时候调用、整体工作顺序、核心输出标准。
- `references/` 是方法论和输出规范，负责把复杂规则拆成可维护的细则。
- `assets/` 是素材命名和 @ 引用规划说明，不承载大体积生产素材。
- `examples/` 是示例输出，让新人和后续代理可以照着格式写。
- `scripts/` 是轻量工具，用于表格导出、表格校验、打包新鲜度检查和包内容检查。
- `dist/seedance-script-designer.skill` 是打包后的分发文件，位于项目根目录的 `dist/` 下。

## 4. 核心文件分工

### 4.1 `SKILL.md`

这是 Skill 的主入口，包含：

- frontmatter：`name` 和 `description`
- 触发关键词
- 主工作流
- 非标准输入处理规则
- 必要输出规则
- 提示词规则
- 电影级分镜集成规则
- 12 列关键帧表规则
- 缺失信息处理规则
- 参考文件索引

修改 Skill 行为时，优先修改 `SKILL.md`。但如果是细则类规则，例如 3x3 时间表、按需输出模式、发布检查流程，应优先修改 `references/` 下的对应文件，避免把主入口写得过长。

### 4.2 `references/story-structure-method.md`

负责剧作层判断。它要求在做镜头之前，先判断：

- 主角是谁
- 主角想要什么
- 阻力是什么
- 场景是否有价值变化
- 转折点在哪里
- 场景是否只是氛围段、过场或可压缩段

这个文件的核心原则是：**不要先堆画面，要先判断戏剧功能。**

如果一个场景没有推动角色欲望、冲突、关系或价值变化，应标记为可压缩、合并、蒙太奇处理或删除候选。

### 4.3 `references/shot-breakdown-method.md`

负责分镜拆解和 Seedance 视频提示词方法。它包含：

- S-A-C-S-C 提示词结构
- 元素分层方法
- @ 引用和素材使用逻辑
- Level 1 宏观节奏
- Level 2 3x3 精细关键帧时间轴
- 3x3 连续性矩阵
- 12 列关键帧表到视频提示词的映射
- 首尾帧、视频延长、局部编辑、一镜到底、音乐和口型同步规则
- 迭代问题定位表

其中最重要的是两层时间系统：

| 层级               | 用途                         | 说明                                                     |
| ------------------ | ---------------------------- | -------------------------------------------------------- |
| Level 1 宏观节奏   | 用于 4-15 秒片段的大节奏规划 | 0-3s 建立，3-6s 推进，6-9s 揭示，9-12s 回收，12-15s 结尾 |
| Level 2 3x3 时间轴 | 用于九宫格关键帧表           | 9 个关键帧覆盖完整 15 秒，精确到每格时间                 |

### 4.4 `references/output-schema.md`

负责完整生产包的输出骨架。它不是每次都要完整输出，而是完整模式下的章节模板。

现在的原则是：

- 不在这里维护按需模式表
- 按需模式统一看 `references/quick-modes.md`
- 只有用户明确要求完整生产包、电影级全流程、全套交付时，才输出完整 18 节

### 4.5 `references/quick-modes.md`

这是按需输出的唯一权威文件。它定义六种模式：

| 模式           | 用途                                       |
| -------------- | ------------------------------------------ |
| 快速分镜模式   | 用户只要某段分镜、3x3 关键帧或旧表格规范化 |
| 音频模式       | 用户只要台词、旁白、音乐、音效、TTS/VO     |
| 视频提示词模式 | 用户只要图生视频或文生视频指令             |
| 素材规划模式   | 用户只要角色、场景、道具或 @ 素材规划      |
| 修订模式       | 用户修改已有分镜、表格或提示词             |
| 完整生产包模式 | 用户明确要求全流程、完整剧本分镜包         |

维护注意：不要在 `AGENTS.md` 或 `output-schema.md` 里复制这张表。模式规则只维护这一份。

### 4.6 `references/storyboard-annotation-rules.md`

负责九宫格分镜图和关键帧标注规则。它规定：

- 默认使用 12 列关键帧表作为文本源
- 默认九宫格分镜图要干净，不压大段文字
- 不默认加字幕、气泡、长标签
- 只有用户明确要求“标注图”时，才在画面内加精简标注
- 标注图也要保留原始未标注版本
- 默认 15 秒 3x3 时间轴

### 4.7 `references/release-checklist.md`

负责发布前检查。它规定：

- 项目副本和全局 Skill 副本要同步
- 12 列关键帧表示例要能通过校验
- dist 包不能比源文件旧
- `.skill` 包内容要和预期清单一致
- 打包文件不能漏掉 references、assets、scripts、examples

## 5. 工作流程

标准工作流分为 8 步。

### 第一步：判断是否应该调用 Skill

先看用户需求是否属于 Seedance 视频生产。如果用户只是要普通文章、普通翻译或代码，不进入这个流程。

如果用户需求属于分镜、关键帧、视频提示词、音频同步、素材规划或旧表格规范化，就调用此 Skill。

### 第二步：确认视觉风格

视觉风格是必要信息。用户没给风格时，要问一个简短问题。

可选风格示例：

- 电影写实
- 动漫二次元
- 日漫风
- 清透光影感
- 国风水墨
- 赛博朋克
- 黏土定格
- 像素风

如果用户要求模仿在世艺术家或具体工作室风格，不直接写“模仿某某”，要转写成中性视觉特征，例如色彩、光影、线条、构图、材质、镜头语言。

### 第三步：识别输入类型

常见输入类型和处理方式如下：

| 输入类型       | 处理方式                                                      |
| -------------- | ------------------------------------------------------------- |
| 完整剧本       | 做剧作结构拆解，再进入分镜和生产包                            |
| 模糊想法       | 先整理成前提、主角、欲望、阻力、利害关系和 3-5 个节拍         |
| 已有视频续集   | 保留上段视频身份、服装、光影、节奏和空间方向，按视频延长处理  |
| 纯音乐或无台词 | 台词、旁白列可留空，重点写音乐节奏、状态/音效、动作卡点和运镜 |
| 旧分镜表       | 识别原字段，升级成 12 列关键帧表                              |
| 局部修改       | 用修订模式，只输出受影响行，不重写全部                        |

### 第四步：选择输出模式

不要默认输出完整生产包。先读 `references/quick-modes.md`，选择最小能满足用户需求的模式。

如果用户说“完整”“全套”“电影级全流程”“完整剧本分镜包”，才使用完整生产包模式。

### 第五步：做剧作结构判断

进入镜头前，必须判断：

- 场景的戏剧功能是什么
- 主角此刻想要什么
- 阻力来自谁或什么
- 这场戏的价值变化是什么
- 是否存在转折点
- 是否只是氛围段或过场

不要只写“画面很好看”。每个镜头最好服务于一个动作、一个情绪节拍或一个信息揭示。

### 第六步：建立视觉连续性

在做长剧本、完整分镜包或需要多段连续生成时，要先建立统一设定：

- 角色身份层：五官、年龄、体型、发型、服装、色彩、标志道具
- 环境层：场景结构、时代、光影、天气、色彩语法
- 道具层：形状、材质、尺寸、磨损、剧情功能
- 情绪/电影语言层：镜头压力、运动节奏、光影变化、声音风格

这个步骤的目的不是堆设定，而是避免后续九宫格、视频提示词和分段生成时角色漂移、服装漂移、场景漂移。

### 第七步：生成 12 列关键帧表

当前默认关键帧表是 **12 列**：

| 序号 | 关键帧 | 时间 | 镜头 | 运镜 | 转场 | 动作 | 情绪/细节 | 台词 | 旁白 | 状态/音效 | 英文 |
| ---- | ------ | ---- | ---- | ---- | ---- | ---- | --------- | ---- | ---- | --------- | ---- |

字段含义：

| 字段      | 含义                                                                      |
| --------- | ------------------------------------------------------------------------- |
| 序号      | 01-09，九宫格阅读顺序                                                     |
| 关键帧    | 标记重点帧，如 `⭐情绪`、`⭐动作`、`⭐台词`、`⭐旁白`、`⭐视觉`、`⭐转折` |
| 时间      | 该格对应时间范围                                                          |
| 镜头      | 景别或构图，如宽景、中景、近景、特写                                      |
| 运镜      | 摄像机运动，如固定机位、缓慢推进、拉远、摇臂上升、跟随                    |
| 转场      | 进入当前格的剪辑方式，如淡入、切、叠化、淡出                              |
| 动作      | 可见行为，只写画面动作                                                    |
| 情绪/细节 | 表情、呼吸、眼神、手部压力、服装摆动等表演细节                            |
| 台词      | 只放角色说话，格式如 `小女孩说：妈妈，王后会疼吗？`                       |
| 旁白      | 只放画外音或旁白                                                          |
| 状态/音效 | 只放非语言信息，如低钟、灰烬飘落、呼吸声、女声吟唱、木轮声                |
| 英文      | 台词/旁白英文；音效用 `[SFX: ...]` 或 `[Ambience: ...]`                   |

重要规则：

- 不再使用 `参考资产` 列。
- 用户通常会用一次性生成的九宫格关键帧分镜图作为整体参考。
- 如果用户需要 @ 引用，他会单独指定；此时放在素材规划或视频提示词里，不放进 12 列关键帧表。
- 台词、旁白、状态/音效必须分列，不能混回一个 `台词/旁白/状态` 字段。
- 每 9 帧通常标 2-3 个重点帧，不要每一格都加星。

默认 15 秒九宫格时间轴：

| 序号 | 时间       |
| ---- | ---------- |
| 01   | 0.0-1.5s   |
| 02   | 1.5-3.0s   |
| 03   | 3.0-4.8s   |
| 04   | 4.8-6.2s   |
| 05   | 6.2-7.8s   |
| 06   | 7.8-9.5s   |
| 07   | 9.5-11.2s  |
| 08   | 11.2-12.7s |
| 09   | 12.7-15.0s |

### 第八步：生成视频提示词或交付包

如果用户需要视频生成指令，把关键帧表转换成 Seedance 视频提示词。

推荐使用 S-A-C-S-C 结构：

| 结构        | 含义                                 |
| ----------- | ------------------------------------ |
| Subject     | 主体是谁或是什么                     |
| Action      | 主体做什么                           |
| Camera      | 镜头、运镜、景别、角度               |
| Style       | 视觉风格、光影、色彩、质感           |
| Constraints | 连续性约束、负面约束、不要出现的问题 |

视频提示词必须写清：

- 基于哪些关键帧
- 时长
- 主体和动作
- 运镜和转场
- 情绪和表演细节
- 台词、旁白、状态/音效
- 是否需要口型同步
- 是否使用首尾帧、视频延长或参考音频
- 连续性约束和负面约束

## 6. 九宫格分镜图规则

九宫格不是 9 张孤立美图，而是一个连续性矩阵。

每个 15 秒九宫格应满足：

- 01-09 按时间推进
- 画面有因果、反应、揭示或转折
- 包含宽景、中景、近景/特写变化
- 角色五官、服装、道具、光影和空间方向连续
- 每格只承载一个主动作和一个情绪节拍
- 不默认在画面内加字幕、气泡或大段说明

干净九宫格默认要求：

- 3x3 grid
- 2K 最小宽度
- 黑色面板边框
- 无台词气泡
- 无字幕
- 无大段标签
- 文字信息保留在 Markdown 表格里

只有用户明确要求“标注图”时，才在图内加入精简标注，例如：

```text
序号 / 时间 / 镜头 / 动作 / 台词或旁白或状态
```

## 7. 素材与 @ 引用规则

素材引用不是默认塞进关键帧表的字段，而是独立规划。

本地规划可用简写：

| 本地简写 | 用途                             |
| -------- | -------------------------------- |
| `@图1`   | 图片、角色、场景、道具、首尾帧   |
| `@视频1` | 动作、运镜、节奏、视频延长       |
| `@音频1` | 音乐、音效、台词、旁白、口型同步 |

注意：即梦平台上传后常显示为 `@图片1`。本地可以用 `@图1` 跟踪，但粘贴到平台提示词时，应按平台显示改成 `@图片1`。

素材使用原则：

- 不要一个镜头塞太多参考。
- 只引用对身份、场景、道具、运镜、节奏或声音有实际作用的素材。
- 如果素材超过 12 个，要标 P0/P1/P2 优先级。
- 3x3 关键帧表默认不写 `参考资产`。
- 需要 @ 引用时，放在素材规划、资产清单或最终视频提示词里。

## 8. 项目文件归档规则

本工作区按剧本项目归档。以后不管是生成文本分镜包、图片分镜表、九宫格关键帧图，还是角色、场景、道具资产图，都必须优先放进对应的 `00剧本\<剧本项目>\` 目录。

不要默认把新产物放到项目根目录的 `image\`，也不要依赖历史上改名过的图片目录。历史目录可以作为查找旧素材的线索，但不是新的生产归档标准。

### 8.1 标准目录结构

标准结构如下：

```text
00剧本\
  01初吻\
    原稿.md
    分镜1.md
    seedance_01-34_分镜总表.md
    image\
      初吻\
        分镜图关键帧\
        资产\

  02灰烬恋人_\
    灰烬恋人_3分钟Seedance分镜包.md
    image\
      灰烬恋人\
        分镜图关键帧\
        资产\
```

### 8.2 文本分镜包位置

文本类产物直接放在剧本项目根目录：

```text
00剧本\<剧本项目>\<剧本名>_分镜包.md
00剧本\<剧本项目>\<剧本名>_3分钟Seedance分镜包.md
00剧本\<剧本项目>\<剧本名>_视频提示词.md
```

例如：

```text
00剧本\02灰烬恋人_\灰烬恋人_3分钟Seedance分镜包.md
```

### 8.3 图片分镜和关键帧位置

九宫格分镜图、关键帧图、首尾帧图放在：

```text
00剧本\<剧本项目>\image\<剧本名>\分镜图关键帧\
```

例如：

```text
00剧本\02灰烬恋人_\image\灰烬恋人\分镜图关键帧\
```

推荐命名：

```text
01_00-00-00-15_灰烬山路_无标注九宫格.png
02_00-15-00-30_黑铁王冠与灰烬契约_标注九宫格.png
03_00-30-00-45_流亡王子归城_首帧.png
04_00-45-01-00_新娘也是祭品_尾帧.png
```

### 8.4 角色、场景、道具资产位置

角色资产、场景资产、道具资产、风格参考图放在：

```text
00剧本\<剧本项目>\image\<剧本名>\资产\
```

例如：

```text
00剧本\02灰烬恋人_\image\灰烬恋人\资产\
```

推荐命名：

```text
角色_艾莉娅_三视图_v01.png
角色_凯兰_情绪表情_v01.png
场景_灰烬神庙_内殿_v01.png
道具_黑铁王冠_v01.png
```

### 8.5 新项目确认规则

开始新的剧本或图片批次前，如果用户没有指定目录，先问一句：

```text
这次要归档到哪个 00剧本\<剧本项目> 文件夹？我会把文本放在该项目根目录，把分镜图放到 image\<剧本名>\分镜图关键帧，把资产图放到 image\<剧本名>\资产。
```

如果用户已经给出明确路径，例如：

```text
00剧本\02灰烬恋人_\image\灰烬恋人\分镜图关键帧
```

就直接使用这个路径，不要另建相似目录。

### 8.6 常见误区

- 不要把最终图片只留在默认生成目录。
- 不要用随机生成 ID 当最终文件名。
- 不要把新剧本图片默认放进历史改名目录。
- 不要把根目录 `image\` 当成默认生产目录，除非用户明确指定。
- 原始生成文件可以保留，但项目内语义命名副本才是交付文件。

## 11. 常见错误

### 错误 1：把 `参考资产` 加回关键帧表

不要加。当前标准是 12 列，不含 `参考资产`。

如果用户需要 @ 引用，单独写素材规划或视频提示词。

### 错误 2：台词、旁白、状态混在一列

错误示例：

```text
台词小女孩说：妈妈，王后会疼吗？
旁白：她只做了一天王后。
状态：低钟，灰烬飘落。
```

正确做法是分列：

| 台词                         | 旁白               | 状态/音效    |
| ---------------------------- | ------------------ | ------------ |
| 小女孩说：妈妈，王后会疼吗？ |                    | 人群低声议论 |
|                              | 她只做了一天王后。 | 低钟延音     |

### 错误 3：每次都输出完整 18 节

不要默认全量输出。先看 `quick-modes.md`，按需输出。

### 错误 4：只写画面，不写戏剧功能

分镜不是纯画面堆叠。每个镜头最好能回答：

- 这个镜头推进了什么？
- 角色在做什么？
- 情绪发生了什么变化？
- 观众知道了什么新信息？

### 错误 5：九宫格像 9 张独立海报

九宫格必须有连续性。要有空间、动作、情绪和时间推进。

### 错误 6：平台 @ 名称写错

本地可写 `@图1`，但即梦平台可能需要 `@图片1`。粘贴到平台前按平台显示名称调整。

### 错误 7：修改脚本后不重打包

修改 Skill 源文件后，`dist/seedance-script-designer.skill` 不会自动更新。必须重打包，并运行：

```powershell
node .\seedance-script-designer\scripts\check-dist-freshness.mjs .\seedance-script-designer .\dist\seedance-script-designer.skill
node .\seedance-script-designer\scripts\verify-package-contents.mjs .\dist\seedance-script-designer.skill
```

### 错误 8：把生成图片放到错误目录

不要默认放到根目录 `image\`，也不要放到历史改名目录。当前标准是：

- 文本分镜包：`00剧本\<剧本项目>\`
- 分镜图/关键帧图：`00剧本\<剧本项目>\image\<剧本名>\分镜图关键帧\`
- 角色/场景/道具资产：`00剧本\<剧本项目>\image\<剧本名>\资产\`

如果不确定剧本项目名，先问用户，不要自作主张创建新位置。
