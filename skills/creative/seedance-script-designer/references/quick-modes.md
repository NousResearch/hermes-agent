# Quick Output Modes

Use this reference to determine which sections of `output-schema.md` to produce based on what the user actually needs. Do not force a full 18-section package unless the user explicitly asks for one.

## Mode Decision Table

| 模式 | 触发信号 | 必须输出 | 可跳过 |
|---|---|---|---|
| 快速分镜模式 | 只要某段分镜、3x3 关键帧、旧表格规范化 | 风格确认 + 分段节奏与批次表 + 12 列关键帧表 | 角色三视图、场景图、道具图、音频同步表 |
| 音频模式 | 只要台词、旁白、音乐、音效、TTS/VO | 台词与音乐对应 + 音频同步表 | 分镜镜头表、角色三视图、场景图、道具图 |
| 视频提示词模式 | 只要图生视频/文生视频指令 | 视频生成指令包 + 素材引用规划 | 剧作结构、场景图、角色三视图 |
| 素材规划模式 | 只要角色、场景、道具、@素材规划 | 元素分层资产清单 + 素材引用规划 | 分镜镜头表、音频同步表、视频提示词 |
| 修订模式 | 修改已有分镜、表格、提示词 | 只输出受影响行和修改说明 | 一切未受影响的部分 |
| 完整生产包模式 | 完整剧本分镜包、电影级全流程，用户明确要求完整输出 | 全套 18 节 output-schema | — |

## Mode-Specific Rules

### 快速分镜模式

- 直接输出目标段落的 12 列关键帧表，不需要先输出完整剧作结构。
- 如果用户给的是旧表格（列数不足 12 列或仍带 `参考资产` 旧列），识别原列结构，补全缺失列，保留原始内容，并默认移除 `参考资产` 列。
- 补全缺失列时，用保守假设填写：缺运镜则填「固定机位」，缺转场则填「切」，缺英文则翻译中文台词。

### 音频模式

- 输出台词与音乐对应表 + 音频同步表，允许时间重叠。
- 类型字段严格使用：`音乐`、`音效`、`台词`、`旁白`，不合并。
- 如果用户提供了参考音频，说明 `@音频X` 的用途（口型同步 / 音乐节奏参考 / 音效引用）。

### 视频提示词模式

- 每个 15 秒段落默认拆成 3 个 5 秒视频指令：
  - 镜头 1（帧 01-03）：建立、入场、空间压迫
  - 镜头 2（帧 04-06）：动作推进、冲突升级、揭示
  - 镜头 3（帧 07-09）：情绪回收、反应、转场或标题
- 每条视频指令必须包含：基于关键帧、时长、主体与可选参考素材、运镜与转场、动作与情绪、台词/旁白/音效、连续性与负面约束。
- 提示词格式优先使用 S-A-C-S-C。

### 素材规划模式

- 输出元素分层资产清单（角色身份层、环境层、道具层、情绪/电影语言层）。
- 输出素材引用规划表（资产 / 类型 / 用途 / 绑定对象 / 使用方式）。
- 如果图片、视频、音频总数超过 12 个，标注优先级并说明哪些可以省略。

### 修订模式

- 先定位问题层级（见 `shot-breakdown-method.md` Section 15 迭代问题定位表）。
- 只输出受影响的行或段，附修改说明。
- 不重新生成整个表或整个段落，除非用户明确要求。

### 完整生产包模式

- 按 `output-schema.md` 的全套 18 节顺序输出。
- 优先使用关键段落先行策略：先验证段（开场+转折+高潮），再补齐段，再修订段。
- 每节完成后可加确认点，但如果用户说「一次输出全部」则跳过中间等待。

## Notes

- 如果用户没有说明模式，根据输入内容主动判断最合适的模式，并在回复开头简短说明你选择的模式。
- 如果判断有歧义（例如输入又像分镜需求又像视频提示词需求），优先问一个简短问题确认。
- 快速模式下风格确认仍然是必要信息。如果用户没给风格，先问，不要跳过。
- 需要格式参照时，优先读取 `examples/minimal-12col-keyframes.md`；需要交付前检查时，运行 `scripts/validate-keyframe-table.mjs`。
