## Coauthor Pair Sorting Bug (CRITICAL)

`get_edge_type()` uses `tuple(sorted([id1, id2]))` for the coauthor pair lookup, but the `coauthor_pairs` set was defined with IDs in their natural UUID order. This caused Hadar Shemtov ↔ Peter Oh to be classified as `colleague` instead of `coauthor` because `sorted(['a9de...', 'a1fe...'])` produces `('a1fe...', 'a9de...')` which didn't match the unsorted tuple in the set.

**Fix:** Always define `coauthor_pairs` with sorted tuples, or use `frozenset` instead of `tuple`:

```python
coauthor_pairs = set()
for id1, id2 in [(yury_id, hadar_id), (yury_id, peter_id), (hadar_id, peter_id)]:
    coauthor_pairs.add(frozenset([id1, id2]))  # Order-independent

# In get_edge_type:
pair = frozenset([id1, id2])
if pair in coauthor_pairs:
    return 'coauthor', '...'
```

**Post-correction verification is mandatory.** After edge creation, verify edge counts by type match expected math:
- coauthor: 3 pairs = 6 directed
- colleague: C(7,2) - 3 = 18 pairs = 36 directed (Google+DeepMind, minus coauthors)
- former_colleague: 7+7 = 14 pairs = 28 directed (Behzadi, Ulaby)
- shared_field: 45 - 3 - 18 - 14 = 10 pairs = 20 directed (Meta cross-org)
