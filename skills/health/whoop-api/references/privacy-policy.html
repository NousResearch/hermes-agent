Whoop API Integration — Privacy Policy

Last updated: May 2026

This is a personal, non-commercial project that connects to the Whoop API for individual fitness data retrieval. It is not a product, service, or business.

Data We Access

This integration requests read-only access to the following Whoop data, belonging solely to the account owner:

  • Recovery scores and HRV
  • Strain and cycle data
  • Sleep records
  • Workout history
  • Body measurements
  • Basic profile information

How Data Is Used

  • Data is retrieved via the Whoop API for the account owner's personal use only.
  • Data is stored locally on the account owner's device.
  • No data is sold, shared, transferred, or disclosed to third parties.
  • No data is aggregated, anonymized, or used for research.

Data Storage and Security

  • All data is stored on the account owner's personal machine.
  • OAuth tokens are stored in the operating system's secure credential store (macOS Keychain) or a local file with restricted permissions (owner-read-only).
  • No data is transmitted to any server, cloud service, or third party.
  • No cookies, tracking, or analytics are used.

Data Retention

Data is retained locally until the account owner deletes it. OAuth tokens can be revoked at any time through the Whoop developer dashboard.

Third-Party Services

This integration only communicates with api.prod.whoop.com (the official Whoop API). No other third-party services are involved.

Changes
