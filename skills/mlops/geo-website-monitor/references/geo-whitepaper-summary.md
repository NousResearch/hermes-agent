# GEO白皮书 - 品牌官网GEO优化技术手段与基础设置总结

## 1. 技术基础设施优化
- HTTPS配置：现代网站基础，影响用户信任和数据传输安全，对AI抓取和信源判断有积极影响
- robots.txt合理设置：避免错误屏蔽核心页面（知识库、案例页、服务页等），确保AI爬虫（GPTBot、Google-Extended、PerplexityBot等）能正常抓取
- 完整的Sitemap：包含首页、栏目页、服务页、案例页、知识库文章、FAQ页面等核心URL，保持动态更新
- 稳定的服务器环境：保证页面长期、稳定、快速访问，避免因服务器问题影响AI抓取
- URL稳定性：避免频繁更换URL路径，保持页面地址长期稳定
- 移动端适配：确保移动端排版正常、按钮可点击、内容完整显示

## 2. 结构化数据与Schema优化
- Schema.org标记：从"SEO加分项"变成"GEO生存必需品"，使用JSON-LD格式实现
- 核心Schema类型：
  - Organization（首页设置）
  - Person（作者、团队成员）
  - Article/BlogPosting
  - Service
  - Product
  - FAQPage
  - LocalBusiness（地理位置相关企业）
- 关键属性：使用@id建立稳定的实体标识符，使用sameAs链接权威外部身份（社交媒体、百科等），避免仅部署验证所需的最低字段，补充完整属性信息，保持数据标记与业务变化同步更新
- Schema实施策略：先审核网站实体缺失、标记缺失情况，建立实体图、页面模板图、模式源图，通过统一的图连接所有内容，在模板级别部署，验证、监控并根据业务变化更新

## 3. 内容结构优化
- 关键页面建设：
  - 首页：清晰说明企业定位、核心业务、服务对象、服务流程和代表案例
  - 关于我们：企业背景、团队经验、发展历程、服务理念、资质信息、联系方式
  - 服务中心：每项服务独立成页，说明服务定义、适用对象、服务流程、交付内容、评估指标
  - 案例中心：真实案例，说明客户行业、问题背景、执行过程、周期变化、结果数据
  - 知识库：持续发布行业知识、方法指南、问题解答等内容
  - FAQ页面：覆盖用户常见问题，答案直接、客观、清晰
- 内容呈现方式：
  - 关键信息使用HTML文本呈现，避免仅放在图片中
  - 正文多使用"是什么、为什么、怎么做、如何判断、常见问题、注意事项"这类结构
  - 结论优先、问句式标题、关键段落加摘要句

## 4. AI专属优化配置
- llms.txt文件：放在网站根目录，作为大模型协议，引导AI爬虫优先读取核心页面，是robots.txt的补充而非替代
- E-E-A-T原则：经验（Experience）、专业性（Expertise）、权威性（Authoritativeness）、可信度（Trustworthiness）

## 5. 第三方信源一致性
- 官网、行业平台、媒体文章、百科资料、企业信息页对企业的描述应保持一致
- 避免多源信息冲突导致AI判断混乱

## 6. 检测与持续优化
- 关注AI引用的长期趋势，而非单次结果
- 定期监测品牌在各AI平台的引用状况
- 根据数据调整内容策略和投放方向
