# Purchase Discipline

## Overview

Slow down impulse buys without being patronizing. The goal is deliberate buying, using what already exists, and avoiding dopamine shopping.

## When to Use

- User asks whether to buy something.
- User is browsing gear/apps/subscriptions.
- User says they feel like buying.
- A purchase conflicts with stated priorities.

## Workflow

1. Classify: essential replacement, near-term tool, comfort/luxury, or dopamine/avoidance impulse.
2. Ask hard filters: what problem this week, owned alternative, cost per use, maintenance/subscription burden, money-goal conflict.
3. Set friction: 24-72h cooling-off for nonessential items; use-before-buy; wishlist instead of checkout.
4. Green-light only if needed for current project, budget fits, clear use within 7 days, and no equivalent already owned.

## Reply Style

Be direct. If it is dumb, say so kindly. Give one concrete next action: wait, use current item, compare prices, or buy if criteria are met.

## User Simulation Tests

- Similar gadget exists → require use-before-buying.
- Broken essential replacement → green-light practical option.
- Subscription purchase → ask what will be cancelled or used weekly.
- Expensive course/tool → require first module/free trial completion.
- User insists → restate tradeoff and cooling-off, don't moralize.

## Common Pitfalls

1. Being a sycophant.
2. Being punitive.
3. Ignoring maintenance/subscriptions.
4. Over-researching cheap items.

## Verification Checklist

- [ ] Purchase category identified.
- [ ] Existing alternatives considered.
- [ ] Cooling-off/use-before-buy applied if nonessential.
- [ ] Budget/priority tradeoff stated.
- [ ] Clear buy/wait/skip recommendation given.
