# Cyber VC Analyst v1.2.0

Date: 2026-07-03

## What changed

- Split the workflow concept into a cyber VC skill family while keeping
  `cyber-vc-analyst` as the compatibility entrypoint.
- Added direct runtime entry shapes for `cyber-vc-company`, `cyber-vc-theme`,
  `cyber-vc-compare`, `cyber-vc-triage`, and `cyber-vc-competitors`.
- Added shared references for workflow phases, research depths, and resumable
  `research-state` artifacts.
- Extended the output templates with compare, triage, and competitors modes.
- Added competitors schema and fixtures.
- Updated the Slack-facing structured normalization so direct skill commands
  and optional `quick | standard | deep` depth prefixes produce stable prompts.

## Why this release exists

This release borrows the best structural ideas from modular startup-skill
repos without weakening the cyber-specific evidence model or Slack workflow.

## Known limits

- Resumable `research-state` is implemented as a documented workflow contract,
  not a separate core runtime subsystem.
- The direct skill family shares the same underlying vault and ROS guidance; it
  does not create separate core tools or plugins.
