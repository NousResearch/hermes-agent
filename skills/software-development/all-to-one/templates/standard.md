# A2O Standard Template

```md
# All To One: {project/task name}

## 1. One-Sentence Result
{What is done and current state. Must include evidence tag.}

## 2. Background and Goal
- Original goal:
- Constraints:
- Final route:
- Routes not taken:

## 3. Final System State
- Environment:
- Key components:
- Working now:
- Not working / unverified:
- Red lines / do not touch:

## 4. Real Timeline
| Phase | Action / Symptom | Judgment at the Time | Later Confirmed | Result |
|---|---|---|---|---|

## 5. Key Changes
### Files / Code
- `{path}`: {what changed}; reason: {why}

### Config / System
- `{service/config}`: {what changed}; reason: {why}

### Data / External State
- `{database/EFI/docker/etc}`: {state change}

### Removed / Deprecated
- `{old thing}`: {why no longer used}

## 6. Bugs, Pitfalls, and Root Causes
| Problem | Symptom | Root Cause | Fix | How to Recognize Next Time | Evidence |
|---|---|---|---|---|---|

## 7. Plain-English Principles
{Explain the underlying mechanism in language a human can repeat.}

## 8. Verification Record
### Verified
- {command/screenshot/test} → {result} [verified/screenshot/observed]

### Unverified / Blocked
- {item} → {reason} [unverified/blocked]

## 9. Risks and Future Improvements
- Risks:
- Improvements:
- Not recommended:

## 10. Resume in 5-10 Minutes
1. First read:
2. Then run:
3. Normal signal:
4. Abnormal branch:
5. Do not repeat:
```
