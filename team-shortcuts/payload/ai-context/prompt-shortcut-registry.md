# Prompt Shortcut Registry

This is the shared prompt shortcut contract for Codex, Qwen, Claude Code, Gemini, Cursor, and future AI tools.

## Runtime Rule

When the user invokes a shortcut, the AI must read the mapped prompt file and apply it to the current task. Do not paraphrase, invent, or partially apply the shortcut.

Lifecycle — each step reads the previous step's output via the `.hermes/` files and reuses the same ids, so work hands off without loss:

```
New (open + read memory/token)
  → Act-As (plan into phases + ids → write .hermes/plan.md on approval)
    → Comply (break issues under the same ids + % + evidence)
      → Continue (execute phase by phase + ledger)
        → Close (verify, write memory, emit decision token)
```

Design invariants that make every shortcut work on the VPS, on every project, and on each staff laptop: capability-based gates (check only what the repo has), no hardcoded paths/IPs (read from project config; `$ROOT` differs local vs VPS), per-staff routing, and a version lock between paired prompts. Two safety notes: merge→main / production deploy / prod migration are OUT of auto by default (`ALLOW_AUTO_PROD=OFF`); and the deploy-verify step lists both GitHub (`gh`) and GitLab (`glab`) commands because real project CI runs on GitLab.
