# Prompt Shortcut Registry

This is the shared prompt shortcut contract for Codex, Qwen, Claude Code, Gemini, Cursor, and future AI tools.

## Runtime Rule

When the user invokes a shortcut, the AI must read `skills/prompt-shortcuts/references/worktree-lifecycle-contract.md` and then read the mapped prompt file. Apply both to the current task. Do not paraphrase, invent, or partially apply the shortcut. WTL v1 takes precedence over older fixed-folder rules.

## Worktree Lifecycle v1 — Shortcut Integration

- 30/30 Shortcuts see the same contract through the central skill/registry.
- Writable shortcuts require a `WTL_READY` task worktree and must not create/switch worktrees directly.
- Read-only shortcuts record path/branch/SHA; they do not need a writer lease until a fix is approved.
- Main lifecycle routes: New Chat=`open/status/enter`, Flow Guardian=`status/doctor`, AI Relay/Pair=single writer + read-only reviewer, Continue=`enter/status`, Close=`close/cleanup dry-run`, Save Git=`status + pushed evidence`, Merge=`close --merged`, Move Folder=no-touch/delegate, Review Chat=report lifecycle.
- Integration routes: Act-As declares task/worktree roles; Comply links issue/evidence; OverviewProgress stores lifecycle; QA QC and SonarQube separate scan from fix; Hermes/Viber Structure install the contract; Viber Audit checks compliance.
