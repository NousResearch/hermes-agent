# AI Relay Catalog — ทะเบียน AI + บัญชี (active v0.5)

> เป้าหมาย: เป็นฐานเดียวให้ Use AI Relay อ่าน ว่ามี AI อะไรบ้าง เรียกยังไง เหมาะงานไหน ราคาเท่าไร ใครเป็นตัวสำรองของใคร และมีกี่บัญชี (ID) ให้สลับเมื่อชนขีดจำกัด
> version pin: catalog ≥ v0.5 ↔ use-ai-relay ≥ v2.7 · คำเตือน: สถานะเครื่องมือฝั่ง VPS ที่ติด ⚠️/❌ ในตาราง §1 ยังไม่ยืนยัน ห้ามเชื่อจนทดสอบ VPS จริง

## 3. ลำดับตัวสำรอง (fallback chain) — เมื่อชนขีดจำกัด/พัง

หลักการ: ตัวที่ใช้อยู่ใช้ไม่ได้ → สลับไปตัวถัดไปในสายของชนิดงานนั้น → คลาวด์หมดทุกตัว → ตกมาที่ Ollama ในเครื่อง (ฟรี)

ตัวอย่างสาย (ปรับได้ใน accounts.yaml):
- งานคิด/วิเคราะห์/วางแผน/ตรวจ/ตัดสิน: **Opus 4.8 ตัวเดียว** (Fable ถอดออกแล้ว 2026-07-06) · งานยากจริง = Opus คิด + ขอ Grok/Codex ตรวจแผนข้ามค่ายก่อนสรุป
- งานเขียนโค้ด: Grok หรือ Codex (สมองเลือกตามความเหมาะ · ดู §7) → Gemini → Ollama (qwen3/deepseek-r1 ในเครื่อง)
- งานตรวจ (ต้องคนละค่ายกับคนเขียน): Codex เขียน → Grok ตรวจ · Grok เขียน → Codex ตรวจ (สลับให้ไม่ซ้ำค่ายผู้เขียนเสมอ)

**Fable/Faber/Fiber 5 ถอดออกแล้ว (เจ้าของสั่ง 2026-07-06 · ประหยัดโควต้า):**
- ไม่มี adapter `fable` · ไม่มีด่านกันเครื่อง · ไม่มีด่านคะแนนความยากเพื่อยกงานให้ Fable
- สมองหลักทุกเครื่อง (โน้ตบุ๊กเจ้าของ + โน้ตบุ๊กพนักงาน + VPS) = **Opus 4.8** เหมือนกันหมด
- ถ้าจะเปิด Fable คืนในอนาคต ต้องเป็นงานใหม่ที่เจ้าของอนุมัติชัดเจน ไม่ใช่ค่าเริ่มต้นของ AI Relay

## 5. วิธีเพิ่ม AI / ID ใหม่ (เจ้าของทำเองได้)

1. **เพิ่มชนิด AI ใหม่** → เติม 1 แถวในตาราง §1 (ชื่อ/วิธีเรียก/เหมาะงาน/ราคา)
2. **เพิ่มบัญชี (ID) ใหม่** → เติม 1 บรรทัดใน `.hermes/ai-relay/accounts.yaml` (ไฟล์เฉพาะเครื่อง ไม่ขึ้นสาธารณะ)
3. **จัดลำดับสำรอง** → ใส่ในสาย §3 ว่าตัวใหม่นี้เป็นสำรองของใคร อันดับเท่าไร

> รูปแบบ `accounts.yaml` (ตัวอย่างโครง · ไม่มีค่าจริง · เก็บเฉพาะเครื่อง):
> ```yaml
> claude_code:
>   - id: ccode-1   # ป้ายชื่อ ไม่ใช่รหัสจริง
>   - id: ccode-2
>   - id: ccode-3
> codex:
>   - id: codex-1
> opus:
>   - id: ccode-1            # สมองหลัก · ใช้บัญชี Claude ชุดเดียวกัน (Fable ถอดออกแล้ว)
> cooldown_minutes: 10       # ตัวไหนพัง/ชนโควต้าติดกัน 3 ครั้งใน 5 นาที พักตัวนั้นชั่วคราว
> # เติม AI/ID ใหม่ใต้นี้
> ```

## 6. ระดับงาน (ทุกงานคิดใช้ Opus 4.8 · Fable ถอดออกแล้ว)

เดิมงานเกรด "ยาก" ยกให้ Fable 5 · ตอนนี้ถอด Fable ออกแล้ว (2026-07-06) → **ทุกงานคิดใช้ Opus 4.8**

| ระดับงาน | ตัวอย่าง | สมองที่ใช้ |
|---|---|---|
| ยาก | แตะ ≥3 ระบบ · เกี่ยวความปลอดภัย/สิทธิ์/เงิน/ข้อมูลส่วนตัว · ออกแบบระบบใหม่ · พลาดแล้วย้อนยาก | Opus 4.8 + ขอ Grok/Codex ตรวจแผนข้ามค่ายก่อนสรุป |
| กลาง | งานโค้ดทั่วไป | Opus 4.8 |
| ง่าย | แก้ข้อความ/ไฟล์เดียว/มี test คลุมแล้ว | Opus 4.8 (คิดสั้น) + coder ถูกสุด |

## 7. ตารางเส้นทางตามชนิดงาน (เลือก coder อัตโนมัติ)

แนวคิดจาก `musistudio/claude-code-router` (35.5k ดาว) — จับคู่ "ชนิดงาน → ตัวที่ใช้" ไม่ใช่ถามทุกครั้ง · เจ้าของสั่งข้ามตารางนี้ได้เสมอ

**รูปแบบทำงานหลัก (ข้อกำหนดเจ้าของ 2026-07-04):**

1. **งานคิด (วิเคราะห์ / วางแผน / จัดแผน / จ่ายงาน / แบ่งเฟส / ลงรายละเอียด):** Opus 4.8 เป็นสมองหลัก + ให้ **Grok และ Codex ช่วยตรวจแผนคนละมุม (cross-check)** เพื่อยืนยันว่าตรวจละเอียดแล้ว → Opus สรุปแผนสุดท้าย
2. **งานเขียนโค้ด:** เลือก **Codex หรือ Grok** ตัวใดตัวหนึ่ง — สมองที่วิเคราะห์เป็นคนตัดสินว่าตัวไหนเหมาะกับงานนั้น
3. **งานตรวจ/รีวิวหลังเขียน:** ต้องเป็น **คนละตัวกับคนเขียนเสมอ** — Codex เขียน → Grok ตรวจ · Grok เขียน → Codex ตรวจ

| ชนิดงาน | ตัวที่ใช้ | หมายเหตุ |
|---|---|---|
| งานคิด / วางแผน / แบ่งเฟส / จ่ายงาน | Opus 4.8 (หลัก) + Grok + Codex ตรวจร่วม | cross-check ก่อนสรุปแผน · สำรองสมอง Opus 4.7 → 4.6 |
| งานเขียนโค้ด | Codex หรือ Grok (สมองเลือก) | เลือกตามความเหมาะกับงาน |
| งานตรวจ/รีวิวหลังเขียน | ตัวตรงข้ามกับคนเขียน (Codex↔Grok) | ห้ามใช้ตัวเดียวกับคนเขียน |

> Gemini / Ollama เป็นสำรองปลายสายเมื่อ Codex+Grok ใช้ไม่ได้ทั้งคู่ (เตือนดัง ๆ ว่าคุณภาพต่ำกว่า · gate ตรวจเท่าเดิม)
