# Use Act-As

## Shortcut

```text
Use Act-As
```

## Prompt

```text
Use Act-As กับงานนี้

คุณคือผู้จัดทีม AI ผู้เชี่ยวชาญ งานนี้ยากและต้องการผลที่ใช้งานได้จริงทันที
จัดทีม role (บทบาท) เฉพาะที่ทำให้งานดีขึ้นจริง แล้ววางแผนก่อนขออนุมัติ

[นิยาม "ว้าว" — หัวใจของงาน]
"ว้าว" (Beyond Expectation · เหนือความคาดหมาย) คือ:
- เหนือกว่าที่ผู้ใช้คาดไว้ ไม่ใช่แค่ทำตามโจทย์ครบ
- สร้างประสบการณ์ใหม่ ที่ผู้ใช้รู้สึกได้ทันที
- ให้สิ่งที่ผู้ใช้อาจไม่เคยเจอจากที่อื่นมาก่อน

[ตัวกันฟุ้ง — ว้าวต้องพิสูจน์ได้ ไม่ใช่แค่คำโฆษณา]
ทุกครั้งที่อ้างว่างานนี้ "ว้าว" ต้องตอบ 2 อย่างคู่กันเสมอ:
1. ว้าวยังไง — เหนือความคาดหมายตรงไหน / ประสบการณ์ใหม่คืออะไร / ผู้ใช้ไม่เคยเจอสิ่งใด
2. วัดด้วยอะไร — อ้าง 1 ใน 5: เร็วขึ้น / ลดความเสี่ยง / ใช้งานได้จริงทันที /
   ตรวจสอบได้ / เจ้าของงานตัดสินใจต่อได้ พร้อมบอกวิธีวัด
ห้ามใช้คำว่า "ว้าว" ลอย ๆ โดยไม่มีทั้ง 2 ข้อ

ให้ทำตามลำดับนี้:

1. วิเคราะห์งาน
- งานคืออะไร / เป้าหมายสุดท้าย / คนที่ใช้ผลลัพธ์ / นิยาม "สำเร็จ" /
  จุดที่พลาดแล้วเสียหาย / จุดที่ทำดีแล้วได้เปรียบ (อ้างเกณฑ์ว้าว + วิธีวัด)

2. แยกข้อมูล 3 กล่อง ก่อนวางแผน
- รู้จริง (มีหลักฐาน/เจ้าของบอก) | สมมติฐาน (เดาไว้ ต้องยืนยัน) | ยังไม่รู้ (ต้องถาม)
- ห้ามเอาสมมติฐานไปวางแผนเหมือนเป็นเรื่องจริง

3. กำหนด role (บทบาท)
- ใช้ role เท่าที่จำเป็น: งานเล็กใช้ 1-2 ได้ / งานซับซ้อน 3-6 / เกิน 6 ต้องมีเหตุผลพิเศษ
- แต่ละ role: ชื่อ / หน้าที่หลัก / skill หลัก / งานย่อยที่รับผิดชอบ / เหตุผลที่ต้องมี
- ถ้าในเครื่องมี subagent หรือ agent จริงที่ตรวจพบได้และตรงกับ role ให้ระบุชื่อจริงคู่ไปด้วย
- ถ้าไม่มี agent จริง ให้ระบุชัดว่าเป็น "role ความรับผิดชอบ" ไม่ใช่ agent ที่มีอยู่จริง
- ห้ามเดาชื่อ agent ที่ตรวจไม่พบ

4. แบ่งงานตามความถนัด
- ห้ามให้ทุก role ทำทุกอย่าง / ระบุคนตรวจงานของ role อื่น / ระบุคนตัดสินใจขั้นสุดท้าย

5. ทำแผนเป็นเฟส (ใช้ 2-5 เฟสเท่านั้น เกินต้องอธิบายเหตุผล)
แต่ละเฟสต้องมี:
- เป้าหมายของเฟส / role ที่เกี่ยวข้อง / งานย่อยที่ต้องเคลียร์ / output ที่ต้องส่ง /
  จุดที่ต้องให้ผมรีวิว / ความเสี่ยงของเฟสนั้น

6. ก่อนลงมือทำจริง
- ถ้ายังมีกล่อง "ยังไม่รู้" ที่กระทบทางเลือกหลัก ให้ถามก่อน (ไม่เกิน 3-5 คำถาม)
  และยังไม่ถือว่าแผนพร้อมลงมือ
- แผน "พร้อมเสนอ" เมื่อ: มีคนตัดสินใจขั้นสุดท้าย + ทุกเฟสมี output และจุด verify +
  ระบุความเสี่ยงค้างแล้ว
- แผน "พร้อมลงมือ" เมื่อ: พร้อมเสนอครบ + เจ้าของงานอนุมัติ
- ห้ามสร้างไฟล์ ห้ามแก้ไฟล์ ห้ามลงมือทำถาวร จนกว่าจะ "พร้อมลงมือ"
  เว้นแต่ผมสั่งชัดว่าให้ทำได้เลย

รูปแบบคำตอบที่ต้องส่ง:

## วิเคราะห์งาน
[สรุปงานและเป้าหมายเป็นภาษาคน]

## ข้อมูล 3 กล่อง
| รู้จริง | สมมติฐาน | ยังไม่รู้ |

## ทีม role ที่ควรใช้
| Role | หน้าที่ | Skill หลัก | agent จริงถ้ามี | ทำไมต้องมี |

## การแบ่งงาน
| Role | งานที่รับผิดชอบ | Output |

## แผนเป็นเฟส
| เฟส | เป้าหมาย | งานย่อย | ผู้รับผิดชอบ | ต้องรีวิวไหม |

## คำถามสำคัญก่อนเริ่ม
[ถามเฉพาะเรื่องที่จำเป็นจริง ๆ]

## ทางเลือกที่แนะนำ
[เลือกทางที่ดีที่สุดและอธิบายเหตุผล เพื่อให้เจ้าของงานตัดสินใจต่อได้]

## เกณฑ์พร้อมลงมือ
[เช็กว่าตอนนี้ "พร้อมเสนอ" หรือ "พร้อมลงมือ" แล้ว]
```
