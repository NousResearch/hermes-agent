ขั้นตอน:

0.3 เช็คซีรีส์ก่อน (บังคับ · แก้โรค "แชทใหม่ไม่รู้บริบทโปรเจกต์")
   - เปิดคลังใบเรื่อง: `$HERMES_OBSIDIAN_ROOT/content-series/` (local: `/Users/rattanasak/ObsidianVault/HermesAgent/content-series/`)
   - งานนี้เป็นตอนต่อของซีรีส์ที่มีอยู่? → อ่าน Series Card ของซีรีส์นั้น: จะได้ บริบทโปรเจกต์ฉบับสาธารณะ + นามแฝงที่ใช้ + ตอนล่าสุด + สถานะเรื่อง → ตอนใหม่ = ตอนล่าสุด + 1
   - เรื่องยาวเรื่องใหม่ (คาดว่ามีหลายตอน)? → เสนอสร้าง Series Card ใหม่ (แบบฟอร์ม: ชื่อซีรีส์สาธารณะ + รหัสย่อ + โปรเจกต์คืออะไร + เป้าหมาย + นามแฝง + ตารางตอน) ให้เจ้าของอนุมัติก่อนสร้าง
   - เรื่องเดี่ยวจบในตัว? → ข้ามได้ ระบุว่า "standalone"
   - กติกา: Series Card เก็บเฉพาะข้อมูลที่ผ่านด่านความลับแล้ว (เขียนครั้งเดียว ใช้ทุกตอน) · หลังส่งงานเข้าโรงงานสำเร็จ ต้องกลับมาอัปเดตตารางตอนใน Card ทุกครั้ง

0.5 ถามแนวเรื่องก่อน (บังคับ · ห้ามเดามุมเอง) — ถามเจ้าของ 5 ข้อ รอคำตอบก่อนขุดแชท:
   (1) ปัญหาที่จะเล่า (คนอ่านเจ็บเรื่องนี้) · (2) เราทำอะไรมาแก้ · (3) Value/ประโยชน์ที่คนอ่านได้ (รูปธรรม) · (4) ว้าวตรงไหน + พิสูจน์ด้วยอะไร (ว้าวลอยไม่ได้) · (5) คนอ่านเอาไปใช้ต่อยังไง + เพื่อใคร
   เจ้าของตอบไม่ครบ = ร่างให้แล้วขอยืนยัน ไม่เดาเงียบ
   โครงเล่าเรื่องบังคับทุกชิ้น: ปัญหา → เราทำอะไร → Value → ว้าว(พิสูจน์ได้) → เอาไปใช้ได้ · เพราะ One Man Fleet วางตัวเป็นห้องทดลองที่ให้ของมีค่า (influencer แบบพิสูจน์ได้ ไม่ใช่กูรูโชว์หน้า)
   โมเดลช่องทาง (hub-and-spoke): บล็อก One Man Fleet = ฮับเนื้อหาเต็ม · โซเชียล/ช่องอื่น = ตัวสั้นล่อคลิก + ลิงก์กลับบล็อกเสมอ
   ความยาวบล็อกฮับ: ไม่น้อยกว่า 3-4 หน้า A4 · ภาษาง่ายระดับ solopreneur/คุณยายอ่านรู้เรื่อง · เล่ามืออาชีพ มีที่มาที่ไป อธิบายเป็นสเต็ป
   ภาพประกอบ (บังคับอย่างน้อย 1 ชิ้น/เรื่อง): วาด diagram workflow เป็น SVG/HTML สีแบรนด์ 1 ชิ้น ที่ใช้ซ้ำได้ทั้ง "ฮีโร่บล็อก" และ "ทัมบ์เนล/ภาพประกอบโซเชียล" · แต่ละจุดบอก ทำอะไร/แก้ปัญหา/ได้ value

1. อ่านแชท/เนื้อหา **ทั้งหมดจริง** แล้วสรุปด้วย 5W1H (ภาษาคน)
   - [ด่านพิสูจน์อ่านครบ · บังคับ · กันอ่านลวก] ก่อนสรุป ต้องแปะ "หลักฐานว่าอ่านครบ" 3 จุด: อ้างอิงสั้น ๆ (1 บรรทัด/จุด) จาก **ต้นเรื่อง · กลางเรื่อง · ท้ายเรื่อง** ของแชท — ถ้าอ้างได้ไม่ครบ 3 จุด = ยังไม่อ่านครบ ห้ามไปต่อ ให้กลับไปอ่านก่อน
   - แชทยาวมาก = สรุปเป็นช่วง ๆ (ต้น/กลาง/ท้าย) ก่อนรวบ ห้ามข้ามช่วงกลางไปจบท้าย
   - What เรื่องอะไร · Why ทำไป/แก้ปัญหาอะไร · Who เกี่ยวกับใคร/เพื่อใคร · When บริบทเวลา · Where ที่ไหน/ระบบไหน · How ทำอย่างไร
   - ดึง "แง่มุมที่เล่าเป็นคอนเทนต์ได้" 2-5 มุม (บทเรียน/วิธีคิด/ผลลัพธ์)

2. ด่านกันความลับ (บังคับ · ยึดกฎความเป็นส่วนตัวของเจ้าของ)
   - ตัดทิ้งเด็ดขาด: ชื่อบริษัท · ชื่อลูกค้า · มูลค่าโครงการ · รายได้ · IP/server/credential · อะไรที่ทำให้เดาโปรเจกต์จริงได้
   - เขียน privacy-report สั้น ๆ ว่าตัดอะไรออกบ้าง · ถ้าตัดแล้วเนื้อหาเหลือน้อยจนเล่าไม่ได้ → บอกเจ้าของ ไม่แต่งเติม

3. ปั้น Content Master แบบลึก — บังคับ 7 ช่อง A-G (ห้ามสรุปย่อ · ต้องลึกพอปั้นบล็อกยาว + โพสต์หลายชิ้นได้แบบอินๆ)
   A. เรื่องเต็ม — เล่าเป็นเรื่อง (ปัญหา → ทางแก้ → ความตึง → ผล) ไม่ใช่หัวข้อ
   B. จังหวะสำคัญ (story beats) 6-12 อัน — แต่ละอันต้องเป็นคอนเทนต์ได้ 1 ชิ้น
   C. ตัวเลข/หลักฐานจริง — สิ่งที่ทำให้คอนเทนต์น่าเชื่อ
   D. ความตึง/ดราม่า — จุดที่ทำให้คนอ่านอิน
   E. ประโยคติดหู — เอาไปเป็นพาดหัว/แคปชั่นได้เลย
   F. มุมคอนเทนต์ — map 1 แชท → บล็อกยาว + โพสต์สั้นหลายชิ้น + วิดีโอ
   G. ห้ามพูดถึง — รายการกรองความลับ (ชื่อคน/บริษัท/ระบบภายใน/มูลค่า)
   เกณฑ์ผ่าน: ถ้า A-G ตื้นจนปั้นคอนเทนต์ยาวไม่ได้ = ยังไม่ผ่าน ต้องขุดแชทเพิ่ม ห้ามปล่อยฉบับบาง

   ช่องส่งโรงงาน (กลั่นจาก A-G · map ช่องรับจริงของ Master Content Factory):
   - title: หัวเรื่อง (จาก E/A) · rawText: แก่นเนื้อหากลั่นจาก A+C (สะอาด ตัดความลับแล้ว) โรงงานเอาไปแตกต่อ
   - brandHint: "One Man Fleet" + เสียงแบรนด์ 1 บรรทัด · goal: เป้าหมายชิ้นนี้
   - suggested_types/channels: จากค่าจริงของโรงงาน (ห้ามเดา VALID_CHANNELS/recipe) · lang_targets th+en (วิญญาณเดียว) · pillars
   - แนบ A-G ทั้งชุดไปกับซอง (โรงงาน/ปลายทางใช้เป็นเหมืองวัตถุดิบ ไม่ใช่แค่ title/rawText)

3B. ปั้น **บล็อกเต็ม** (นี่คือผลลัพธ์หลักที่เจ้าของรีวิว · ไม่ใช่แค่ซอง)
   - [ซีรีส์] ถ้าเป็นตอนในซีรีส์: หัวบล็อกติดป้าย "ชื่อซีรีส์ · ตอนที่ N" + **ความเดิม 2-3 บรรทัด + ลิงก์ตอนก่อน** (คนหลงมาอ่านตอนนี้ก่อนต้องตามทัน) · ห้ามใส่ตัวหารจำนวนตอนรวม (เช่น 3/5) เพราะงานจริงไม่รู้ว่าจบกี่ตอน
   - เขียนบล็อกฉบับเต็มจาก A-G ให้ **ยาว A4 2-3 หน้า** (ประมาณ 900-1,500 คำ) · ตรงเสียง One Man Fleet (we/the fleet · calm, sharp, evidence-first · ไม่ hype)
   - โครงบังคับ 5 จังหวะ: ปัญหา → เราทำอะไร → Value → ว้าว(พิสูจน์ได้) → เอาไปใช้ต่อ
   - ภาษาระดับ solopreneur/คุณยายอ่านรู้เรื่อง · มีที่มาที่ไป อธิบายเป็นสเต็ป · ค่า default = อังกฤษ (ถ้าเจ้าของสั่งไทยก่อนก็ได้ แต่ต้องเป็นแบรนด์เดียวกัน)
   - เสนอจุดวางภาพ 1 จุด (diagram workflow สีแบรนด์) แต่ยังไม่ต้องวาดจริงในสถานีนี้ — ไปวาดตอนปลายทางบล็อก OMF

3C. [ด่านคุณค่า · บังคับ · กันปล่อยของบาง] ก่อนเอาบล็อกให้เจ้าของ ต้องผ่านเช็กลิสต์วัดได้ทุกข้อ — ตกข้อไหน = ยังไม่ปล่อย กลับไปขุด/เขียนใหม่:
   - [ ] มี pain ที่คนอ่านเจ็บจริง (ไม่ใช่ปัญหาลอย)
   - [ ] มี value รูปธรรม คนอ่านเอาไปใช้ได้ (บอกได้ว่าใช้ยังไง/เพื่อใคร)
   - [ ] "ว้าว" มีของพิสูจน์ (ตัวเลข/ตัวอย่าง/ก่อน-หลัง) ไม่ใช่คำโฆษณา
   - [ ] เนื้อยาวพอ A4 2-3 หน้า และไม่มีย่อหน้าน้ำล้วน
   - [ ] ผ่านด่านความลับ (ไม่มีชื่อบริษัท/ลูกค้า/มูลค่า)
   - [ ] เสียงตรงแบรนด์ (ไม่มี "ผม/ฉัน" แบบมีตัวตน · ไทย=อังกฤษเป็นแบรนด์เดียว)
   สรุปผลด่านเป็นตาราง ✅/❌ ให้เจ้าของเห็น ไม่ปิดบังข้อที่ตก

4.5 เจ้าของรีวิว (บังคับ · draft-first) — โชว์บล็อกเต็ม + ตารางด่านคุณค่า ให้เจ้าของ อ่าน/แก้/ตีกลับ (reject) ในแชท · แก้ตามที่สั่งแล้ววนด่านคุณค่าใหม่ · ยังไม่ส่งโรงงานจนกว่าเจ้าของ OK

5. ส่งต่อ (หลังเจ้าของอนุมัติ)
   - ส่ง **บล็อกเต็ม (จาก 3B)** เป็น `rawText` เข้าช่องรับจริงของโรงงาน `POST /api/v1/local/brief` (body: title + rawText[=บล็อกเต็ม] + brandHint + goal · แนบ A-G เป็นวัตถุดิบ) — บทความจะไปโผล่เป็นรายการ **DRAFT** ในหน้าโรงงานให้เจ้าของรีวิว/แก้ต่อได้
   - [ซีรีส์] ตอนในซีรีส์: ฝังรหัสตอนไว้ต้น title เช่น `[OMCF #3] ชื่อตอน` (ระยะสั้น จนกว่าโรงงานมีช่อง seriesId จริง — แผน P13) · โพสต์โซเชียลที่แตกจากตอนนี้ ทุกชิ้นต้องมีป้ายซีรีส์ 1 บรรทัด + ยืนเองได้ + ลิงก์กลับบล็อกฮับ
   - [ซีรีส์] ส่งสำเร็จแล้ว → อัปเดตตารางตอนใน Series Card ทันที (ตอน N · ชื่อ · วันที่ · สถานะ) — ลืมอัปเดต = แชทหน้าหลงบริบทอีก
   - ยิงตรงเฉพาะเมื่อโรงงานเปิดรันอยู่ (`localhost:4000`) · ถ้ายังไม่เปิด = เตรียมซองไว้ บอกเจ้าของให้เปิดโรงงานก่อน ไม่ยิงมั่ว
   - งานผลิตหนัก (แตกหลายแพลตฟอร์ม/หลายภาษา) ส่งต่อผ่าน Use AI Relay ได้ (Claude วางแผน/ตรวจ · coder ถูกกว่าเขียน)
   - งานลงบล็อก One Man Fleet ให้ต่อ Use Blog Auto (ปลายน้ำเว็บ OMF) · จบ session ใช้ Use Close Chat

รูปแบบคำตอบ:
