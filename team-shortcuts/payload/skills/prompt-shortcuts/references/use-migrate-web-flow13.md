# **ขั้นที่ 1 · M0 — ยืนยันเมนูเก่ากับ owner ก่อนเริ่ม 🚦  \[หมวด: เตรียม P\]**

**Journey:**

ก่อนจะแตะอะไรทั้งนั้น AI ต้องกลับไปหาเว็บเก่าจริง หา URL (ที่อยู่หน้าเว็บ) ของเมนูที่จะทำ แล้วส่งลิงก์กดได้ให้ owner ดูของจริงก่อน พร้อมหน้าย่อยทั้งหมดของเมนูนั้น จากนั้น **รอให้ owner กดเข้าไปดูแล้วยืนยัน** ว่าเมนูนี้คืออันไหน มีอะไรบ้าง ที่ต้องทำแบบนี้เพราะที่ผ่านมา AI ชอบ "เดา" แล้วสร้างหน้าขึ้นเอง ทั้งที่เมนูเก่าบางอันเป็นแค่ลิงก์เปล่าไม่มีสาระ ถ้าเมนูเก่ากำกวมหรือว่างเปล่า AI ห้ามคิดเอง ต้องถาม owner ว่าเมนูใหม่ควรมีอะไร

ก่อนทำงานทุกครั้งจะต้องมีการอ่านไฟล์ Token Design System ของ Project นั้น ๆ. และต้องคอยตรวจสอบว่าหน้าที่สร้างขึ้นมีอะไรที่ไม่ตรงกับ Design System ที่จะทำให้เว็บไซต์ไม่เป็นหนึ่งเดียวกัน.

และหลังจากทำ Project นั้นๆ เสร็จแล้วจะต้อง ทำการตรวจสอบเนื้องานทั้งหมดว่ามีลูกเล่นหรือ Design ใหม่เพิ่มเติมขึ้นมาจากเดิมที่ Design System ยังไม่มี หรือมีการเปลี่ยนแปลงที่ดีกว่า Design System. จะต้องถาม User เพื่อ Confirm ว่าจะให้เอาข้อมูลใหม่ไปอัปเดตใน Design System หรือเปล่าทุกครั้ง. 

จะต้องคอยตรวจสอบว่างานที่ทีมทำงานแต่ละหน้า แต่ละส่วน แต่ละ flow จะต้องสอดคล้องกับ Design System. ถ้ามีการเพิ่มมาใหม่ ที่จะต้องใช้กับหลาย ๆ ส่วนในโครงการ ต้องให้อัปเดต Design System โดยการถาม user ก่อน.

ให้ระวังเรื่องของ AI รับคำสั่งผิด แล้วไม่พัฒนาต่อเนื่อง. ไปสร้างไฟล์ใหม่ที่ไม่เกี่ยวข้องกับ Project เพราะฉะนั้น ให้ตรวจสอบ Header, Footer และ Element ต่างๆ ที่เกี่ยวข้องกับ Project ว่า AI ไม่ได้ทำงานมั่ว เพราะ Project นี้ Shortcut นี้ใช้หลายคนในการทำงานร่วมกัน.

ไม่มี Pagination  ใน Workflow น่าจะมีปัญหาเรื่อง Pagination.

ในระบบงานทุกตัวจะต้องใช้ Elasticsearch. ใครที่ทำเกี่ยวกับ Search จะต้องดูแลเรื่องของ Elasticsearch ว่าทำงานได้จริงและสามารถค้นได้แบบ full function ของ Elasticsearch.

การทำงานแต่ละหน้า AI จะต้องช่วยคิดวิเคราะห์ว่าภาพที่จะนำมาใช้จะวางในส่วนไหน เพื่อให้ดีไซน์ที่ออกมามี layout ที่สวยงามเหมือนการอ่าน magazine ตอบโจทย์พฤติกรรมของผู้ใช้ที่คนอ่านจากซ้ายไปขวา มองจากบนลงล่าง และอย่าลืม ถ้าความยาวของหน้านั้นเกินสอง A4 จะต้องมีภาพคั่นสายตาที่สอดคล้องกับเนื้อหาทุก ๆ คั่นหน้าของ A4.

และให้ระวังกรณีที่ข้อมูลยาวมากเกินไป ควรจะนำเสนอ user ว่าทำเป็นแท็บไหม เพื่อที่จะทำให้ผู้ใช้กดได้ง่าย? และกรณีที่เป็นแท็บถ้ากดแท็บไหน แล้วจะต้องมี script ที่จะดึงข้อมูลแท็บไปไว้ด้านบนของหน้าจอ เพื่อสะดวกกับผู้ใช้งาน.

กรณีถ้าเป็นภาพประกอบ ให้ลองดูว่า ถ้าจะทำให้ว้าวขึ้น ควรจะทำเป็นวิดีโอแบ็กกราวนด์ส่วนไหนไหม? ให้ดูตามความเหมาะสม เพราะจะมีปัญหากับเรื่องขนาดของไฟล์หรือการโหลดช้า. ด้วยต้องวิเคราะห์ว่าหน้านั้นเหมาะที่จะใช้ภาพประกอบเป็นวิดีโอ background จริง ๆ.

ปัญหาของเว็บลูกค้าส่วนใหญ่จะมีภาพประกอบที่เป็นอินโฟกราฟิกที่ไม่สวยงาม หรือภาพไดอะแกรมที่ไม่สวยงาม. ให้ทำการวิเคราะห์และ confirm กับผู้ใช้ว่าจะแก้ไขหรือออกแบบใหม่ และให้ AI ออกแบบใหม่ตาม Design System. โดยภาพไดอะแกรมจะต้องดูง่าย เข้าใจง่าย เป็นภาษา user ที่อ่านแล้วเข้าใจ.

หลังการวิเคราะห์เว็บไซต์เก่าจะต้องช่วย user ในการบอกจุดอ่อนและแนะนำว่า AI จะออกแบบยังไงให้สวยงาม, น่าใช้, ใช้ง่าย, เข้าใจง่าย.และตอบโจทย์เป้าหมายของผู้ใช้ในหน้านั้นว่าเขาต้องการอะไร Emotion ยังไง และว้าวยังไง

# **ขั้นที่ 8 · M5 — Admin คุมหน้าได้จริง \+ แก้เห็นหน้าจริง (WYSIWYG/Preview)  \[หมวด: ลงมือ D\]**
