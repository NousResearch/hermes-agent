คู่มือ Prompt Shortcut สำหรับทีม

พิมพ์ชื่อ Shortcut ในแชท AI (Cursor / Claude Code / Codex) แล้ว AI จะทำตามขั้นตอนมาตรฐานให้

อัปเดต 2026-06-24 · ครอบคลุม 18 Shortcut ที่ใช้งานจริง

Shortcut คืออะไร — คำสั่งสั้น ๆ ที่พิมพ์ให้ AI เพื่อเรียก "วิธีทำงานมาตรฐาน" ที่ตรวจสอบแล้ว แทนที่จะอธิบายขั้นตอนยาว ๆ ทุกครั้ง · เรียกได้ทั้งภาษาไทยและอังกฤษ เช่น Use Comply = แตกเฟส

คู่ที่ใช้บ่อยสุด — เปิดงาน Use New Chat (AI อ่านว่างานถึงไหน) → ปิดงาน Use Close Chat (AI บันทึกความจำให้แชทหน้า) · คู่นี้แก้ปัญหา AI ลืมงานข้ามแชท
เริ่ม / ปิด / ต่อ แชท

Use New Chat v1.6

เรียกด้วย: Use New Chat · เปิดแชทใหม่ · เริ่ม New Chat

ใช้เมื่อ: เริ่มแชทใหม่/มาทำงานต่อ · อยากให้ AI รู้ว่างานถึงไหน อยู่ branch ไหน ก่อนเริ่ม (กันทำผิดที่)

Flow
  1. อ่านความจำล่าสุด (handoff + session log) → สรุป 3 บรรทัด: process ขั้นไหน / งานล่าสุดถึงไหน / ต้องทำต่ออะไร
  2. ตรวจของจริง: project path / branch / ไฟล์ค้าง / remote / VPS-service (มีอะไรตรวจอันนั้น)
  3. ฟีเจอร์ใหม่ + อยู่ branch รวม → หยุด บังคับสร้าง branch ใหม่ก่อนแก้โค้ด
  4. เช็กการจองงาน (กันแก้ทับเพื่อนร่วมทีม)
  5. ออกผลเป็น READY / BLOCKED / NEED_OWNER_INPUT
แก้ปัญหาเดิม: AI ลืมบริบท + ทำงานบน branch ผิดจนพัง

Use Close Chat v1.0

เรียกด้วย: Use Close Chat · ปิดแชท · จบแชท

ใช้เมื่อ: จะปิดแชท/เลิกงานรอบนี้ · อยากให้ AI บันทึกว่าทำอะไรไป เพื่อแชทหน้าจำได้

Flow (ถาม 5 ข้อก่อนปิด)
  1. commit แล้วยัง — มีไฟล์ค้าง = ยังปิดสะอาดไม่ได้
  2. ต้อง push / merge / แจ้งคนมีสิทธิ์ merge ไหม (ชี้ไป Save Git / Merge to Production)
  3. แต่ละงานทำจริงไหม — แยก "ตรวจแล้วมีหลักฐาน" กับ "แค่บอกว่าเสร็จ"
  4. AI ลืมอะไรไหม — ทวนคำสั่งทั้งแชท
  5. เขียนความจำถาวร (handoff สั้น + session log ละเอียด) แล้วปิดด้วย CLOSED_CLEAN / CLOSED_WITH_PENDING / NEED_OWNER_ACTION
คู่กับ Use New Chat → ปิดเขียนความจำ เปิดอ่านความจำ

Review Chat v2.1

เรียกด้วย: Review Chat · รีวิวแชท · สรุปส่งต่อ

ใช้เมื่อ: อยากตรวจว่างานในแชทนี้เชื่อถือได้ไหม + เตรียมข้อความต่อแชทใหม่

Flow
  1. ไล่งานทั้งแชท: เสร็จ/ค้าง/ทำผิด + แยก "ตรวจจากหลักฐานจริง" กับ "สรุปจากบทสนทนา"
  2. เสนอจะอัปเดตไฟล์ไหน (ยังไม่เขียนจนกว่าอนุมัติ)
  3. โชว์ "หลักฐานที่ยังขาด" (ยังไม่ได้ตรวจอะไร)
  4. ทำข้อความเปิดแชทใหม่พร้อมใช้

Use Continue v3.1

เรียกด้วย: Use Continue · ทำต่อ · ไม่ต้องรอผม

ใช้เมื่อ: อยากให้ AI ทำงานต่อเองอัตโนมัติ ปิดทุกเฟสให้ครบ

Flow — ระดับอิสระ 3 ชั้น
  1. ชั้น 1 ทำเองได้ (อ่าน/แก้โค้ดใน scope/รันเทส)
  2. ชั้น 2 รายงานแล้วทำต่อ (ลง dependency/แก้ config ที่ไม่ใช่ production)
  3. ชั้น 3A งานเสี่ยง (push/merge/deploy/migration) → ทำได้เองแต่ต้องผ่านด่านตรวจอัตโนมัติก่อน
  4. ชั้น 3B ของย้อนยากสุด (ใช้เงิน/ส่งอีเมล/เปิด secret/ลบถาวร) → ต้องขออนุมัติคนเสมอ
ปิดทุกเฟส 100% (ตรวจจริง) ก่อนข้าม · ไม่ขยายงานนอกเป้าหมาย
วางแผน / บทบาท

Use Act-As v2.1

เรียกด้วย: Use Act-As · กำหนดบทบาท · เรียกทีมผู้เชี่ยวชาญ

ใช้เมื่อ: งานยาก/ซับซ้อน อยากให้ AI จัดทีมผู้เชี่ยวชาญ + วางแผนก่อนลงมือ

Flow
  1. วิเคราะห์งาน + นิยาม "ว้าว" (เหนือคาด/ประสบการณ์ใหม่) ที่วัดได้จริง
  2. แยกข้อมูล 3 กล่อง: รู้จริง / สมมติฐาน / ยังไม่รู้ (กันเดา)
  3. จัดทีม role 3-6 คน ตามความถนัด + ระบุคนตัดสินสุดท้าย
  4. วางแผนเป็นเฟส (2-5) + เสนอทางที่ดีที่สุด · ห้ามแก้ไฟล์จนอนุมัติ

Use Comply v2.1

เรียกด้วย: Use Comply · แตกเฟส · ทำตารางเปอร์เซ็นต์

ใช้เมื่อ: งานใหญ่ อยากกัน AI ลืม/ข้าม/โกหกว่าเสร็จ → แตกเป็นเฟส+เปอร์เซ็นต์ที่ตรวจได้

Flow
  1. แยกเป้าหมาย + scope
  2. แตกเป็น phase และ issue ย่อย พร้อมเกณฑ์ผ่าน + วิธีตรวจ
  3. ให้สถานะมาตรฐาน (verified/blocked/failed...) + % ที่มีสูตร (นับเฉพาะที่ตรวจแล้ว)
  4. รายงานท้ายเฟส + หลักฐานจริง · ห้าม 100% ถ้าไม่ตรวจ
ทำงานหลาย AI

Use AI Relay v1.3

เรียกด้วย: Use AI Relay · สายพาน AI

ใช้เมื่อ: งานโค้ดเยอะ อยากประหยัดเครดิต Claude → ให้ AI ถูกกว่าเขียน Claude ตรวจ

Flow
  1. ถามก่อน: ใครเขียน / ใครตรวจ (คนละค่าย) / งานอะไร ขอบเขตแค่ไหน
  2. ตรวจเครื่องมือพร้อม (preflight) เฉพาะคนเขียนที่เลือก
  3. คนเขียนทำบน branch แยก → ตัวตรวจ scope อัตโนมัติ (กันแตะนอกขอบเขต)
  4. Claude ตรวจ + รันเทสจริง → วนแก้จนผ่าน 100% · บันทึก ledger ทุกครั้ง

Use Scan Feature v1.0

เรียกด้วย: Use Scan Feature · สแกนฟีเจอร์ · บัญชีฟีเจอร์

ใช้เมื่อ: อยากได้บัญชีฟีเจอร์ทั้งหมดของโปรเจกต์ (ของจริง พิสูจน์ได้) เพื่อส่งต่อทีมการตลาด

Flow
  1. สแกน repo ทีละเฟส (หยุดทุก GATE ให้คนอนุมัติ)
  2. ทุกฟีเจอร์ติดสถานะ real/mock/planned + หลักฐาน path:line
  3. ทำ Reality Matrix + Hidden Gems (ของดีที่เจ้าของลืม)
  4. ออกเป็นเอกสารไทยไฟล์เดียว

Use Viber Audit v1.1

เรียกด้วย: Use Viber Audit · ตรวจมาตรฐาน Viber

ใช้เมื่อ: อยากตรวจว่าโปรเจกต์ทำตามมาตรฐาน Viber Enterprise ครบไหม ขาดอะไร

Flow
  1. เลือกโหมด: Quick (ตรวจเร็ว) / Full 360 / Portfolio (หลายโปรเจกต์สรุปก่อน)
  2. ตรวจ artifact + quality gate 0-8 + security แบบมีหลักฐาน
  3. ให้คะแนน % (คะแนนสูงไม่ผ่านถ้า security/release ขาด)
  4. ออก gap เป็น issue พร้อม owner + วิธีแก้

Use Impeccable v1.2

เรียกด้วย: Use Impeccable · ตรวจ UI Slop · แก้ AI Slop

ใช้เมื่อ: หน้าเว็บดูเป็นงาน AI ลวก ๆ อยากให้ตรวจ+ปรับคุณภาพ UI

Flow
  1. ตัดสินก่อน: เป็นงาน frontend ไหม / ติดตั้งเครื่องมือได้ปลอดภัยไหม (ไม่ใช่ = ตรวจด้วยมือ)
  2. แยกปัญหาพัง (blocking) ออกจากหนี้หน้าตา (UI debt) → แก้ตัวพังก่อน
  3. ให้คะแนน 100 (layout/responsive/a11y/design-system/น่าเชื่อถือ) ต่ำกว่า 80 = ยังไม่พร้อม production
  4. แก้แบบ scoped + ตรวจด้วยหลักฐานจริง

Use WOW Resource v1.1

เรียกด้วย: Use WOW Resource · WOW Layout · WOW Design

ใช้เมื่อ: ต้องการดีไซน์เว็บระดับพรีเมียม → ให้ AI คัดทรัพยากรดีไซน์มาปรับให้ตรงงาน

  3. ปฏิเสธตัวที่ generic/flashy เกิน + อธิบายเหตุผล
  4. แปลงเป็นทิศทางเฉพาะโปรเจกต์ (ไม่ก๊อปสคริปต์ตรง)

Use Save Git v2.1

เรียกด้วย: Use Save Git · ก่อน push · ก่อน merge · Merge Gate

ใช้เมื่อ: ก่อนจะ push/merge · อยากให้ตรวจครบก่อนเจ้าของกด merge (ทุกคนใช้ได้)

Flow — ด่านตรวจ
  1. รัน save-git --stage merge-gate (ตรวจ local + MR + CI)
  2. อ่านค่าจาก .savegit.json ของโปรเจกต์ · ขาดข้อมูลสำคัญ = ไม่ปล่อยผ่านเงียบ
  3. คืนคำตัดสินเดียว: SAFE_TO_MERGE / BLOCKED / OWNER_DECISION + ตารางบอกชั้นที่ติด
  4. เจ้าของงานเป็นคนกด merge เอง

Use Merge to Production v1.1

เรียกด้วย: Use Merge to Production · ขึ้น production · Ship to Production

ใช้เมื่อ: รวมงานขึ้น production จริง · เฉพาะคนมีสิทธิ์ merge

Flow
  1. ยืนยันผู้สั่งเป็น merger จริง (จากตัวตน GitLab ไม่ใช่ชื่อที่พิมพ์)
  2. merge-gate → เจ้าของ merge → ship-gate
  3. deploy จาก origin/target เท่านั้น + เตรียม rollback
  4. ยืนยัน production: health เขียว + commit ตรง (เทียบไม่ได้ = ยังไม่ถือว่าเสร็จ)
คนอื่นที่ไม่ใช่ merger ให้ใช้ Use Save Git แทน (ได้แค่ถึงก่อน merge)

Use Summary v1.7

เรียกด้วย: Use Summary · สรุป · สรุปลิงก์ · วิเคราะห์บทความ

ใช้เมื่อ: ส่งลิงก์/บทความให้ AI สรุป+วิเคราะห์ ก่อนตัดสินว่าจะเก็บเข้าระบบความรู้ไหม

Flow
  1. เลือกโหมด (สั้น/ปกติ/ลึก/ติดตั้ง) → คุมความยาว
  2. สรุป + วิเคราะห์ (จริง/ไม่จริง/ยังไม่ยืนยัน) + เหมาะกับงานเราไหม
  3. เสนอทางเลือกให้เจ้าของเลือก · ยังไม่บันทึกจนอนุมัติ
  4. อนุมัติแล้ว → บันทึกเป็นระเบียนที่ติดตามได้ (มีวันครบกำหนดทบทวน)

Use Blog Auto v1.1

  4. ส่งต่อ Content Factory ทำเป็นโพสต์โซเชียล (ร่างก่อน รออนุมัติ)

Use Flow Guardian v1.1

เรียกด้วย: Use Flow Guardian · ตรวจ worktree · กัน AI แก้งานทับกัน

ใช้เมื่อ: ระหว่างทำงาน อยากให้ AI มีวินัยความปลอดภัย กันแก้งานทับเพื่อนร่วมทีม

Flow

  4. ทำเฉพาะ scope → ตรวจจริง → ส่งต่อ handoff

Use Move Folder —
