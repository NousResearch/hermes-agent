# Research Workflows

Use this skill for evidence gathering, academic/technical research, continuous monitoring, market/prediction data, and converting findings into written research artifacts.

## Core research loop

1. Define the question, scope, timeframe, and acceptable source types.
2. Search multiple channels: scholarly indexes, web/docs, feeds, APIs, and local knowledge bases.
3. Extract source metadata and quote/cite primary evidence.
4. Synthesize claims with uncertainty; distinguish facts, interpretations, and recommendations.
5. Preserve reusable notes, scripts, or bibliographies when the research will continue.

## arXiv and scholarly search

- Search by keyword, author, category, or arXiv ID.
- Capture title, authors, date, abstract, URL, and why it is relevant.
- For literature reviews, cluster papers by method/question, not by search result order.

## Blog/feed monitoring

- Use blog/RSS monitoring when the user needs recurring change detection or source tracking.
- Store feeds with clear names and report only new/relevant changes.

## Knowledge-base building

- Use an interlinked markdown wiki for long-running domains.
- Start each session by reading orientation/index files, then add concise pages with links and sources.

## Prediction-market research

- Use market APIs for prices, outcomes, order books, and history.
- Explain market structure and liquidity before interpreting probabilities.

## Research paper writing

- Treat papers as a pipeline: claim → evidence/experiments → related work → figures/tables → reviewer-facing narrative.
- Keep a source/citation workflow and a checklist for venue requirements, ethics, limitations, and reproducibility.
