# Web ダッシュボード

Web ダッシュボードは、Hermes Agent のインストールを管理するためのブラウザベースの UI です。YAML
ファイルを編集したり CLI コマンドを実行したりする代わりに、整理された Web インターフェースから設定の
構成、API キーの管理、セッションの監視ができます。

## クイックスタート

```bash
hermes dashboard
```

これはローカルの Web サーバーを起動し、ブラウザで `http://127.0.0.1:9119` を開きます。
ダッシュボードは完全にお使いのマシン上で動作し、データが localhost の外に出ることはありません。

### オプション

| フラグ | デフォルト | 説明 |
|------|---------|-------------|
| `--port` | `9119` | Web サーバーを実行するポート |
| `--host` | `127.0.0.1` | バインドアドレス |
| `--no-open` | — | ブラウザを自動で開かない |
| `--insecure` | off | localhost 以外のホストへのバインドを許可する（**危険** — API キーをネットワークに公開します。ファイアウォールと強力な認証と組み合わせてください） |
| `--tui` | off | ブラウザ内の Chat タブを公開する（PTY/WebSocket 経由で `hermes --tui` を埋め込み）。代わりに `HERMES_DASHBOARD_TUI=1` を設定してもかまいません。 |

```bash
# カスタムポート
hermes dashboard --port 8080

# すべてのインターフェースにバインド（共有ネットワークでは注意して使用）
hermes dashboard --host 0.0.0.0

# ブラウザを開かずに起動
hermes dashboard --no-open
```

## 前提条件

デフォルトの `hermes-agent` インストールには、HTTP スタックや PTY ヘルパーは同梱されません。
これらはオプションの extra です。**Web ダッシュボード**には FastAPI と Uvicorn（`web` extra）が
必要です。**Chat** タブには、疑似ターミナルの背後で埋め込み TUI を起動するための `ptyprocess`
（POSIX では `pty` extra）も必要です。両方を次のようにインストールします。

```bash
pip install 'hermes-agent[web,pty]'
```

`web` extra は FastAPI/Uvicorn を、`pty` extra は `ptyprocess`（POSIX）または `pywinpty`
（ネイティブ Windows — ただし埋め込み TUI 自体は依然として WSL を必要とします）を取り込みます。
`pip install hermes-agent[all]` は両方の extra を含んでおり、メッセージング／音声などもあわせて
使いたい場合に最も簡単な方法です。

依存関係なしで `hermes dashboard` を実行すると、何をインストールすればよいか教えてくれます。
フロントエンドがまだビルドされておらず `npm` が利用可能な場合、初回起動時に自動的にビルドされます。

## ページ

### Status

ランディングページには、インストールのライブな概要が表示されます。

- **エージェントバージョン**とリリース日
- **ゲートウェイの状態** — 実行中／停止中、PID、接続されているプラットフォームとその状態
- **アクティブセッション** — 直近 5 分間にアクティブだったセッション数
- **最近のセッション** — 直近 20 件のセッションの一覧（モデル、メッセージ数、トークン使用量、会話のプレビュー付き）

Status ページは 5 秒ごとに自動更新されます。

### Chat

**Chat** タブは、完全な Hermes TUI（`hermes --tui` で得られるのと同じインターフェース）を
ブラウザ内に直接埋め込みます。スラッシュコマンド、モデルピッカー、ツール呼び出しカード、markdown
ストリーミング、clarify/sudo/approval のプロンプト、スキンのテーマ設定など、ターミナルの TUI で
できることはすべてここでも同じように動作します。ダッシュボードが実際の TUI バイナリを実行し、その
ANSI 出力を [xterm.js](https://xtermjs.org/) の WebGL レンダラーでピクセルパーフェクトなセル
レイアウトとしてレンダリングしているからです。

**仕組み:**

- `/api/pty` は、ダッシュボードのセッショントークンで認証された WebSocket を開きます
- サーバーは POSIX 疑似ターミナルの背後で `hermes --tui` を起動します
- キー入力は PTY に送られ、ANSI 出力がブラウザにストリーミングされます
- xterm.js の WebGL レンダラーが各セルを整数ピクセルグリッドに描画します。マウストラッキング
  （SGR 1006）、全角文字（Unicode 11）、罫線素片グリフはすべてネイティブにレンダリングされます
- ブラウザウィンドウのリサイズは、`@xterm/addon-fit` アドオンを介して TUI をリサイズします

**既存のセッションを再開する:** **Sessions** タブから、任意のセッションの横にある再生アイコン
（▶）をクリックします。これは `/chat?resume=<id>` にジャンプし、`--resume` で TUI を起動して
完全な履歴を読み込みます。

**前提条件:**

- Node.js（`hermes --tui` と同じ要件。TUI バンドルは初回起動時にビルドされます）
- `ptyprocess` — `pty` extra でインストールされます（`pip install 'hermes-agent[web,pty]'`、
  または `[all]` が両方をカバーします）
- POSIX カーネル（Linux、macOS、または WSL2）。`/chat` のターミナルペインは特に POSIX PTY を
  必要とします。ネイティブ Windows の Python には同等のものがないため、ネイティブ Windows
  インストールでは、ダッシュボードの残り（セッション、ジョブ、メトリクス、設定エディタ）は動作
  しますが、`/chat` タブにはその機能のために WSL2 を使うよう促すバナーが表示されます。

ブラウザのタブを閉じると、PTY はサーバー上でクリーンに回収されます。再度開くと新しいセッションが
起動します。

### Config

`config.yaml` のためのフォームベースのエディタです。150 以上のすべての設定フィールドが
`DEFAULT_CONFIG` から自動検出され、タブ付きのカテゴリに整理されます。

- **model** — デフォルトモデル、プロバイダー、ベース URL、reasoning 設定
- **terminal** — バックエンド（local/docker/ssh/modal）、タイムアウト、シェル設定
- **display** — スキン、ツール進捗、resume 表示、スピナー設定
- **agent** — 最大イテレーション数、ゲートウェイタイムアウト、サービスティア
- **delegation** — サブエージェントの上限、reasoning effort
- **memory** — プロバイダーの選択、コンテキスト注入の設定
- **approvals** — 危険なコマンドの承認モード（ask/yolo/deny）
- その他 — config.yaml の各セクションには対応するフォームフィールドがあります

既知の有効な値を持つフィールド（terminal backend、skin、approval mode など）はドロップダウンとして
レンダリングされます。真偽値はトグルとしてレンダリングされます。それ以外はすべてテキスト入力です。

**アクション:**

- **Save** — 変更を即座に `config.yaml` に書き込みます
- **Reset to defaults** — すべてのフィールドをデフォルト値に戻します（Save をクリックするまで保存されません）
- **Export** — 現在の設定を JSON としてダウンロードします
- **Import** — JSON 設定ファイルをアップロードして現在の値を置き換えます

:::tip
設定変更は、次のエージェントセッションまたはゲートウェイの再起動で有効になります。Web
ダッシュボードは、`hermes config set` やゲートウェイが読み取るのと同じ `config.yaml` ファイルを
編集します。
:::

### API Keys

API キーや認証情報が保存される `.env` ファイルを管理します。キーはカテゴリ別にグループ化されます。

- **LLM Providers** — OpenRouter、Anthropic、OpenAI、DeepSeek など
- **Tool API Keys** — Browserbase、Firecrawl、Tavily、ElevenLabs など
- **Messaging Platforms** — Telegram、Discord、Slack のボットトークンなど
- **Agent Settings** — `API_SERVER_ENABLED` のような秘密でない環境変数

各キーには次が表示されます。

- 現在設定されているかどうか（値のマスキングされたプレビュー付き）
- 用途の説明
- プロバイダーのサインアップ／キー取得ページへのリンク
- 値を設定または更新するための入力フィールド
- 削除するための削除ボタン

高度な、またはめったに使われないキーは、デフォルトではトグルの背後に隠されています。

### Sessions

すべてのエージェントセッションを閲覧して調べます。各行には、セッションのタイトル、ソース
プラットフォームのアイコン（CLI、Telegram、Discord、Slack、cron）、モデル名、メッセージ数、
ツール呼び出し数、最後にアクティブだったのがどれくらい前かが表示されます。ライブセッションは
脈打つバッジでマークされます。

- **Search** — FTS5 を使ったすべてのメッセージ内容に対する全文検索。結果にはハイライトされた
  スニペットが表示され、展開すると最初に一致するメッセージへ自動スクロールします。
- **Expand** — セッションをクリックして完全なメッセージ履歴を読み込みます。メッセージはロール
  （user、assistant、system、tool）別に色分けされ、シンタックスハイライト付きの Markdown として
  レンダリングされます。
- **Tool calls** — ツール呼び出しを含む assistant メッセージは、関数名と JSON 引数を含む折りたたみ
  可能なブロックを表示します。
- **Delete** — ゴミ箱アイコンでセッションとそのメッセージ履歴を削除します。

### Logs

エージェント、ゲートウェイ、エラーのログファイルを、フィルタリングとライブ tailing 付きで表示します。

- **File** — `agent`、`errors`、`gateway` のログファイルを切り替えます
- **Level** — ログレベルでフィルタリングします: ALL、DEBUG、INFO、WARNING、ERROR
- **Component** — ソースコンポーネントでフィルタリングします: all、gateway、agent、tools、cli、cron
- **Lines** — 表示する行数を選択します（50、100、200、500）
- **Auto-refresh** — 5 秒ごとに新しいログ行をポーリングするライブ tailing を切り替えます
- **Color-coded** — ログ行は重大度別に色分けされます（エラーは赤、警告は黄、デバッグは淡色）

### Analytics

セッション履歴から計算される使用量とコストの分析です。期間（7、30、90 日）を選択すると、次が
表示されます。

- **サマリーカード** — 合計トークン（入力／出力）、キャッシュヒット率、見積もりまたは実際の合計
  コスト、日次平均付きの合計セッション数
- **日次トークンチャート** — 1 日あたりの入力トークンと出力トークンの使用量を示す積み上げ棒グラフ。
  ホバー時のツールチップで内訳とコストを表示します
- **日次内訳テーブル** — 各日の日付、セッション数、入力トークン、出力トークン、キャッシュヒット率、
  コスト
- **モデルごとの内訳** — 使用された各モデル、そのセッション数、トークン使用量、見積もりコストを
  示すテーブル

### Cron

エージェントのプロンプトを定期的なスケジュールで実行する cron ジョブを作成・管理します。

- **Create** — 名前（オプション）、プロンプト、cron 式（例: `0 9 * * *`）、配信先（local、Telegram、
  Discord、Slack、email）を入力します
- **Job list** — 各ジョブには、名前、プロンプトのプレビュー、スケジュール式、状態バッジ
  （enabled/paused/error）、配信先、前回の実行時刻、次回の実行時刻が表示されます
- **Pause / Resume** — ジョブをアクティブ状態と一時停止状態の間で切り替えます
- **Trigger now** — 通常のスケジュール外でジョブを即座に実行します
- **Delete** — cron ジョブを完全に削除します

### Skills

スキルとツールセットを閲覧、検索、切り替えします。スキルは `~/.hermes/skills/` から読み込まれ、
カテゴリ別にグループ化されます。

- **Search** — 名前、説明、カテゴリでスキルとツールセットをフィルタリングします
- **Category filter** — カテゴリのピルをクリックして一覧を絞り込みます（例: MLOps、MCP、Red
  Teaming、AI）
- **Toggle** — スイッチで個々のスキルを有効／無効にします。変更は次のセッションで有効になります。
- **Toolsets** — 別のセクションに、組み込みツールセット（ファイル操作、Web ブラウジングなど）が、
  そのアクティブ／非アクティブの状態、セットアップ要件、含まれるツールの一覧とともに表示されます

:::warning セキュリティ
Web ダッシュボードは、API キーやシークレットを含む `.env` ファイルを読み書きします。デフォルトでは
`127.0.0.1` にバインドされ、ローカルマシンからのみアクセスできます。`0.0.0.0` にバインドすると、
ネットワーク上の誰もがあなたの認証情報を閲覧・変更できるようになります。ダッシュボード自体に認証は
ありません。
:::

## `/reload` スラッシュコマンド

このダッシュボードの PR は、インタラクティブ CLI に `/reload` スラッシュコマンドも追加します。
Web ダッシュボードから（または `.env` を直接編集して）API キーを変更した後、アクティブな CLI
セッションで `/reload` を使うと、再起動せずに変更を反映できます。

```
You → /reload
  Reloaded .env (3 var(s) updated)
```

これは `~/.hermes/.env` を実行中のプロセスの環境に読み直します。ダッシュボードから新しい
プロバイダーキーを追加して、すぐに使いたい場合に便利です。

## REST API

Web ダッシュボードは、フロントエンドが消費する REST API を公開します。自動化のために、これらの
エンドポイントを直接呼び出すこともできます。

### GET /api/status

エージェントバージョン、ゲートウェイの状態、プラットフォームの状態、アクティブセッション数を返します。

### GET /api/sessions

直近 20 件のセッションをメタデータ付き（モデル、トークン数、タイムスタンプ、プレビュー）で返します。

### GET /api/config

現在の `config.yaml` の内容を JSON として返します。

### GET /api/config/defaults

デフォルトの設定値を返します。

### GET /api/config/schema

すべての設定フィールドを記述するスキーマ（型、説明、カテゴリ、該当する場合は選択肢）を返します。
フロントエンドはこれを使って、各フィールドに対して正しい入力ウィジェットをレンダリングします。

### PUT /api/config

新しい設定を保存します。ボディ: `{"config": {...}}`。

### GET /api/env

既知のすべての環境変数を、その設定済み／未設定の状態、マスキングされた値、説明、カテゴリとともに
返します。

### PUT /api/env

環境変数を設定します。ボディ: `{"key": "VAR_NAME", "value": "secret"}`。

### DELETE /api/env

環境変数を削除します。ボディ: `{"key": "VAR_NAME"}`。

### GET /api/sessions/\{session_id\}

単一のセッションのメタデータを返します。

### GET /api/sessions/\{session_id\}/messages

セッションの完全なメッセージ履歴を、ツール呼び出しとタイムスタンプを含めて返します。

### GET /api/sessions/search

メッセージ内容に対する全文検索。クエリパラメータ: `q`。一致するセッション ID をハイライトされた
スニペットとともに返します。

### DELETE /api/sessions/\{session_id\}

セッションとそのメッセージ履歴を削除します。

### GET /api/logs

ログ行を返します。クエリパラメータ: `file`（agent/errors/gateway）、`lines`（行数）、`level`、
`component`。

### GET /api/analytics/usage

トークン使用量、コスト、セッションの分析を返します。クエリパラメータ: `days`（デフォルト 30）。
レスポンスには日次の内訳とモデルごとの集計が含まれます。

### GET /api/cron/jobs

設定されたすべての cron ジョブを、その状態、スケジュール、実行履歴とともに返します。

### POST /api/cron/jobs

新しい cron ジョブを作成します。ボディ: `{"prompt": "...", "schedule": "0 9 * * *", "name": "...", "deliver": "local"}`。

### POST /api/cron/jobs/\{job_id\}/pause

cron ジョブを一時停止します。

### POST /api/cron/jobs/\{job_id\}/resume

一時停止した cron ジョブを再開します。

### POST /api/cron/jobs/\{job_id\}/trigger

スケジュール外で cron ジョブを即座にトリガーします。

### DELETE /api/cron/jobs/\{job_id\}

cron ジョブを削除します。

### GET /api/skills

すべてのスキルを、その名前、説明、カテゴリ、有効状態とともに返します。

### PUT /api/skills/toggle

スキルを有効または無効にします。ボディ: `{"name": "skill-name", "enabled": true}`。

### GET /api/tools/toolsets

すべてのツールセットを、そのラベル、説明、ツール一覧、アクティブ／設定済みの状態とともに返します。

## CORS

Web サーバーは CORS を localhost のオリジンのみに制限します。

- `http://localhost:9119` / `http://127.0.0.1:9119`（本番）
- `http://localhost:3000` / `http://127.0.0.1:3000`
- `http://localhost:5173` / `http://127.0.0.1:5173`（Vite 開発サーバー）

カスタムポートでサーバーを実行する場合、そのオリジンが自動的に追加されます。

## 開発

Web ダッシュボードのフロントエンドに貢献する場合は次のようにします。

```bash
# Terminal 1: バックエンド API を起動
hermes dashboard --no-open

# Terminal 2: HMR 付きの Vite 開発サーバーを起動
cd web/
npm install
npm run dev
```

`http://localhost:5173` の Vite 開発サーバーは、`/api` リクエストを `http://127.0.0.1:9119` の
FastAPI バックエンドにプロキシします。

フロントエンドは React 19、TypeScript、Tailwind CSS v4、shadcn/ui スタイルのコンポーネントで
構築されています。本番ビルドは `hermes_cli/web_dist/` に出力され、FastAPI サーバーが静的な SPA
として配信します。

## 更新時の自動ビルド

`hermes update` を実行すると、`npm` が利用可能であれば Web フロントエンドが自動的に再ビルドされます。
これにより、ダッシュボードがコードの更新と同期した状態に保たれます。`npm` がインストールされて
いない場合、更新はフロントエンドのビルドをスキップし、`hermes dashboard` が初回起動時にビルドします。

## テーマとプラグイン

ダッシュボードには 6 つの組み込みテーマが同梱されており、ユーザー定義のテーマ、プラグインタブ、
バックエンド API ルートで拡張できます。いずれもドロップインで、リポジトリのクローンは不要です。

ヘッダーバーから**テーマをライブで切り替え**られます。言語切り替えの横にあるパレットアイコンを
クリックします。選択は `config.yaml` の `dashboard.theme` に永続化され、ページ読み込み時に復元
されます。

組み込みテーマ:

| テーマ | 特徴 |
|-------|-----------|
| **Hermes Teal**（`default`） | ダークティール＋クリーム、システムフォント、ゆったりした余白 |
| **Hermes Teal (Large)**（`default-large`） | default と同じで 18px のテキストとより広い余白 |
| **Midnight**（`midnight`） | 深いブルーバイオレット、Inter ＋ JetBrains Mono |
| **Ember**（`ember`） | 暖かいクリムゾン＋ブロンズ、Spectral セリフ ＋ IBM Plex Mono |
| **Mono**（`mono`） | グレースケール、IBM Plex、コンパクト |
| **Cyberpunk**（`cyberpunk`） | 黒地にネオングリーン、Share Tech Mono |
| **Rosé**（`rose`） | ピンク＋アイボリー、Fraunces セリフ、ゆったり |

独自のテーマを作成したり、プラグインタブを追加したり、シェルスロットに注入したり、プラグイン固有の
REST エンドポイントを公開したりするには、**[ダッシュボードの拡張](./extending-the-dashboard)** を
参照してください。完全なガイドでは次を扱います。

- テーマ YAML スキーマ — palette、typography、layout、assets、componentStyles、colorOverrides、customCSS
