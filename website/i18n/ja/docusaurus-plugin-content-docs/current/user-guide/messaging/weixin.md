# Weixin（WeChat）

Hermesを、Tencentの個人向けメッセージングプラットフォームである[WeChat](https://weixin.qq.com/)（微信）に接続します。アダプターは個人WeChatアカウント向けにTencentの **iLink Bot API** を使用します — これはWeCom（企業向けWeChat）とは別物です。メッセージはロングポーリングで配信されるため、公開エンドポイントやwebhookは不要です。

:::info
このアダプターは **個人WeChatアカウント**（微信）向けです。企業/法人向けWeChatが必要な場合は、代わりに[WeComアダプター](./wecom.md)を参照してください。
:::

:::warning iLinkボットのアイデンティティ — 通常のWeChatグループは機能しないことがある
QRログインは、Hermesを **iLinkボットのアイデンティティ**（例 `a5ace6fd482e@im.bot`）に接続するもので、完全にスクリプト可能な通常の個人WeChatアカウントには接続**しません**。その結果：

- iLinkボットのアイデンティティは、通常の連絡先のように **通常のWeChatグループに招待できない** ことが一般的です。
- iLinkは、ほとんどのボット型アカウントについて、**通常のWeChatグループイベント**（QRログインに使った個人アカウントへの `@` メンションを含む）をゲートウェイに配信**しない**のが通常です。
- QRコードのスキャンに使った個人WeChatアカウントを `@` メンションすることは、iLinkボットを `@` メンションすることと**同じではありません** — ボットは別個のアイデンティティです。
- 下記の `WEIXIN_GROUP_POLICY` / `WEIXIN_GROUP_ALLOWED_USERS` 設定は、iLinkが実際にあなたのアカウントタイプ向けにグループイベントを返す場合にのみ効果があります。返さない場合、ポリシーに関わらずグループメッセージはHermesに届きません。

実際には、ほとんどのデプロイでは、iLinkボットへのDMだけが確実に機能します。設定後にグループ配信が機能しない場合、その制限はiLink側にあり、Hermes側ではありません。`WEIXIN_GROUP_POLICY` が `disabled` 以外に設定されている場合、ゲートウェイは起動時に `WARNING` をログに記録します。
:::

## 前提条件

- 個人WeChatアカウント
- Pythonパッケージ：`aiohttp` と `cryptography`
- Hermesを `messaging` エクストラ付きでインストールすると、ターミナルでのQRレンダリングが含まれます

必要な依存関係をインストールします：

```bash
pip install aiohttp cryptography
# オプション: ターミナルでのQRコード表示用
pip install hermes-agent[messaging]
```

## セットアップ

### 1. セットアップウィザードを実行する

WeChatアカウントを接続する最も簡単な方法は、対話型セットアップです：

```bash
hermes gateway setup
```

プロンプトで **Weixin** を選択します。ウィザードは次を行います：

1. iLink Bot APIからQRコードをリクエストする
2. ターミナルにQRコードを表示する（またはURLを提供する）
3. WeChatモバイルアプリでQRコードをスキャンするのを待つ
4. スマートフォンでログインを確認するよう促す
5. アカウント認証情報を `~/.hermes/weixin/accounts/` に自動保存する

確認が完了すると、次のようなメッセージが表示されます：

```
微信连接成功，account_id=your-account-id
```

ウィザードは `account_id`、`token`、`base_url` を保存するため、手動で設定する必要はありません。

### 2. 環境変数を設定する

最初のQRログインのあと、少なくとも `~/.hermes/.env` にアカウントIDを設定します：

```bash
WEIXIN_ACCOUNT_ID=your-account-id

# オプション: トークンを上書き（通常はQRログインから自動保存される）
# WEIXIN_TOKEN=your-bot-token

# オプション: アクセスを制限
WEIXIN_DM_POLICY=open
WEIXIN_ALLOWED_USERS=user_id_1,user_id_2

# オプション: 従来の複数行分割の振る舞いを復元
# WEIXIN_SPLIT_MULTILINE_MESSAGES=true

# オプション: cron/通知用のホームチャンネル
WEIXIN_HOME_CHANNEL=chat_id
WEIXIN_HOME_CHANNEL_NAME=Home
```

### 3. ゲートウェイを起動する

```bash
hermes gateway
```

アダプターは保存された認証情報を復元し、iLink APIに接続し、メッセージのロングポーリングを開始します。

## 機能

- **ロングポール転送** — 公開エンドポイント、webhook、WebSocketが不要
- **QRコードログイン** — `hermes gateway setup` によるスキャンして接続するセットアップ
- **DMメッセージング** — 設定可能なアクセスポリシー。グループメッセージングは、iLinkが接続されたアイデンティティ向けに実際にグループイベントを配信するかに依存します（iLinkボットアカウントではしばしば配信されません — 上記の警告を参照）
- **メディアサポート** — 画像、動画、ファイル、音声メッセージ
- **AES-128-ECB暗号化CDN** — すべてのメディア転送に対する自動の暗号化/復号
- **コンテキストトークンの永続化** — 再起動をまたいだディスクバックの返信継続性
- **Markdownフォーマット** — 見出し、表、コードブロックを含むMarkdownを保持するため、MarkdownをサポートするWeChatクライアントはネイティブにレンダリングできます
- **スマートなメッセージチャンク分割** — メッセージは上限未満なら単一のバブルにまとまり、過大なペイロードのみが論理的な境界で分割されます
- **入力中インジケーター** — エージェントが処理する間、WeChatクライアントに「入力中…」ステータスを表示します
- **SSRF保護** — 送信メディアURLはダウンロード前に検証されます
- **メッセージの重複排除** — 5分のスライディングウィンドウで二重処理を防止します
- **バックオフ付き自動再試行** — 一時的なAPIエラーから回復します

## 設定オプション

`config.yaml` の `platforms.weixin.extra` の下に設定します：

| キー | デフォルト | 説明 |
|-----|---------|-------------|
| `account_id` | — | iLink BotアカウントID（必須） |
| `token` | — | iLink Botトークン（必須、QRログインから自動保存） |
| `base_url` | `https://ilinkai.weixin.qq.com` | iLink APIのベースURL |
| `cdn_base_url` | `https://novac2c.cdn.weixin.qq.com/c2c` | メディア転送用CDNのベースURL |
| `dm_policy` | `open` | DMアクセス: `open`、`allowlist`、`disabled`、`pairing` |
| `group_policy` | `disabled` | グループアクセス: `open`、`allowlist`、`disabled` |
| `allow_from` | `[]` | DMを許可するユーザーID（dm_policy=allowlist のとき） |
| `group_allow_from` | `[]` | 許可するグループID（group_policy=allowlist のとき） |
| `split_multiline_messages` | `false` | `true` のとき、複数行の返信を複数のチャットメッセージに分割（従来の振る舞い）。`false` のとき、長さ上限を超えない限り複数行の返信を1つのメッセージとして保持。 |

## アクセスポリシー

### DMポリシー

ボットにダイレクトメッセージを送れる人を制御します：

| 値 | 振る舞い |
|-------|----------|
| `open` | 誰でもボットにDMできる（デフォルト） |
| `allowlist` | `allow_from` のユーザーIDのみがDMできる |
| `disabled` | すべてのDMが無視される |
| `pairing` | ペアリングモード（初期セットアップ用） |

```bash
WEIXIN_DM_POLICY=allowlist
WEIXIN_ALLOWED_USERS=user_id_1,user_id_2
```

### グループポリシー

**iLinkが接続されたアイデンティティ向けにグループイベントを配信する場合に**、ボットがどのグループで応答するかを制御します。QRログインのiLinkボットアイデンティティ（例 `...@im.bot`）では、グループイベントは通常まったく配信されないため、このポリシーは効果がないことがあります — ページ上部のiLinkボットの制限に関する警告を参照してください。

| 値 | 振る舞い |
|-------|----------|
| `open` | ボットがすべてのグループで応答する（イベントが配信される場合） |
| `allowlist` | ボットが `group_allow_from` にリストされたグループIDでのみ応答する（イベントが配信される場合） |
| `disabled` | すべてのグループメッセージが無視される（デフォルト） |

```bash
WEIXIN_GROUP_POLICY=allowlist
# 注意: これは変数名に "USERS" が含まれているにもかかわらず、メンバーのユーザーIDではなく
# グループチャットIDのカンマ区切りリストです。設定時にこの点を念頭に置いてください。
WEIXIN_GROUP_ALLOWED_USERS=group_id_1,group_id_2
```

:::note
Weixinのデフォルトのグループポリシーは `disabled` です（デフォルトが `open` のWeComとは異なります）。これは意図的です — 個人WeChatアカウントは多くのグループに参加している可能性があり、iLinkボットのアイデンティティは通常、通常のWeChatグループメッセージをまったく受信できません。`WEIXIN_GROUP_POLICY` を `disabled` 以外に設定すると、ゲートウェイは起動時に `WARNING` をログに記録します。
:::

## メディアサポート

### 受信（受け取る）

アダプターはユーザーからのメディア添付を受信し、WeChat CDNからダウンロードし、復号し、エージェント処理のためにローカルにキャッシュします：

| タイプ | 処理方法 |
|------|-----------------| 
| **画像** | ダウンロード、AES復号され、JPEGとしてキャッシュされる。 |
| **動画** | ダウンロード、AES復号され、MP4としてキャッシュされる。 |
| **ファイル** | ダウンロード、AES復号され、キャッシュされる。元のファイル名は保持される。 |
| **音声** | テキスト書き起こしが利用可能なら、テキストとして抽出される。そうでなければ音声（SILK形式）がダウンロードされキャッシュされる。 |

**引用メッセージ:** 引用（返信先）メッセージのメディアも抽出されるため、エージェントはユーザーが何に返信しているかのコンテキストを得られます。

### AES-128-ECB暗号化CDN

WeChatのメディアファイルは暗号化されたCDNを通じて転送されます。アダプターはこれを透過的に処理します：

- **受信:** 暗号化されたメディアは `encrypted_query_param` URLを使ってCDNからダウンロードされ、メッセージペイロードで提供されるファイルごとのキーを使ってAES-128-ECBで復号されます。
- **送信:** ファイルはランダムなAES-128-ECBキーでローカルに暗号化され、CDNにアップロードされ、暗号化された参照が送信メッセージに含まれます。
- AESキーは16バイト（128ビット）です。キーは生のbase64またはhexエンコードで届くことがあります — アダプターは両方の形式を処理します。
- これには `cryptography` Pythonパッケージが必要です。

設定は不要です — 暗号化と復号は自動的に行われます。

### 送信（送る）

| メソッド | 送信内容 |
|--------|--------------|
| `send` | Markdownフォーマット付きのテキストメッセージ | 
| `send_image` / `send_image_file` | ネイティブの画像メッセージ（CDNアップロード経由） |
| `send_document` | ファイル添付（CDNアップロード経由） |
| `send_video` | 動画メッセージ（CDNアップロード経由） |

すべての送信メディアは、暗号化されたCDNアップロードフローを通ります：

1. ランダムなAES-128キーを生成する
2. AES-128-ECB + PKCS#7パディングでファイルを暗号化する
3. iLink API（`getuploadurl`）からアップロードURLをリクエストする
4. 暗号文をCDNにアップロードする
5. 暗号化されたメディア参照とともにメッセージを送信する

## コンテキストトークンの永続化

iLink Bot APIは、特定の相手に対するすべての送信メッセージで `context_token` をエコーバックすることを要求します。アダプターはディスクバックのコンテキストトークンストアを維持します：

- トークンはアカウント+相手ごとに `~/.hermes/weixin/accounts/<account_id>.context-tokens.json` に保存されます
- 起動時に、以前保存されたトークンが復元されます
- 受信メッセージごとに、その送信者の保存トークンが更新されます
- 送信メッセージには最新のコンテキストトークンが自動的に含まれます

これにより、ゲートウェイの再起動後でも返信の継続性が確保されます。

## Markdownフォーマット

iLink Bot APIを通じて接続されたWeChatクライアントはMarkdownを直接レンダリングできるため、アダプターはMarkdownを書き換えずに保持します：

- **見出し** はMarkdownの見出し（`#`、`##`、…）のまま
- **表** はMarkdownの表のまま
- **コードフェンス** はフェンス付きコードブロックのまま
- **過剰な空行** は、フェンス付きコードブロックの外では2つの改行に折りたたまれる

## メッセージのチャンク分割

メッセージは、プラットフォームの上限内に収まる限り、単一のチャットメッセージとして配信されます。過大なペイロードのみが配信のために分割されます：

- 最大メッセージ長: **4000文字**
- 上限未満のメッセージは、複数の段落や改行を含んでいてもそのまま保たれる
- 過大なメッセージは論理的な境界（段落、空行、コードフェンス）で分割される
- コードフェンスは可能な限り保たれる（フェンス自体が上限を超えない限り、ブロックの途中で分割されない）
- 過大な個別ブロックは、ベースアダプターの切り詰めロジックにフォールバックする
- 複数のチャンクが送信される際、0.3秒のチャンク間遅延がWeChatのレート制限による取りこぼしを防ぐ

## 入力中インジケーター

アダプターはWeChatクライアントに入力中ステータスを表示します：

1. メッセージが届くと、アダプターは `getconfig` API経由で `typing_ticket` を取得します
2. 入力中チケットはユーザーごとに10分間キャッシュされます
3. `send_typing` は入力開始シグナルを送り、`stop_typing` は入力停止シグナルを送ります
4. エージェントがメッセージを処理する間、ゲートウェイは自動的に入力中インジケーターをトリガーします

## ロングポール接続

アダプターはメッセージの受信に（WebSocketではなく）HTTPロングポーリングを使用します：

### 仕組み

1. **接続:** 認証情報を検証し、ポーリングループを開始する
2. **ポーリング:** 35秒のタイムアウトで `getupdates` を呼び出す。サーバーはメッセージが届くかタイムアウトが切れるまでリクエストを保持する
3. **ディスパッチ:** 受信メッセージは `asyncio.create_task` 経由で並行してディスパッチされる
4. **同期バッファ:** 永続的な同期カーソル（`get_updates_buf`）がディスクに保存され、再起動後にアダプターが正しい位置から再開できる

### 再試行の振る舞い

APIエラー時、アダプターはシンプルな再試行戦略を使います：

| 条件 | 振る舞い |
|-----------|----------|
| 一時的なエラー（1〜2回目） | 2秒後に再試行 |
| 繰り返されるエラー（3回以上） | 30秒バックオフしてからカウンターをリセット |
| セッション期限切れ（`errcode=-14`） | 10分間一時停止（再ログインが必要な場合あり） |
| タイムアウト | 即座に再ポーリング（通常のロングポールの振る舞い） |

### 重複排除

受信メッセージは、5分のウィンドウでメッセージIDを使って重複排除されます。これにより、ネットワークの不具合や重複するポーリング応答の際の二重処理を防ぎます。

### トークンロック

1つのWeixinゲートウェイインスタンスのみが、特定のトークンを同時に使用できます。アダプターは起動時にスコープ付きロックを取得し、シャットダウン時に解放します。別のゲートウェイがすでに同じトークンを使っている場合、起動は分かりやすいエラーメッセージとともに失敗します。

## トラブルシューティング

| 問題 | 修正 |
|---------|-----|
| `Weixin startup failed: aiohttp and cryptography are required` | 両方をインストール: `pip install aiohttp cryptography` |
| `Weixin startup failed: WEIXIN_TOKEN is required` | `hermes gateway setup` を実行してQRログインを完了するか、`WEIXIN_TOKEN` を手動で設定 |
| `Weixin startup failed: WEIXIN_ACCOUNT_ID is required` | `.env` に `WEIXIN_ACCOUNT_ID` を設定するか `hermes gateway setup` を実行 |
| `Another local Hermes gateway is already using this Weixin token` | もう一方のゲートウェイインスタンスを先に停止 — トークンごとに1つのポーラーのみ許可される |
| セッション期限切れ（`errcode=-14`） | ログインセッションが期限切れです。`hermes gateway setup` を再実行して新しいQRコードをスキャン |
| セットアップ中にQRコードが期限切れ | QRは最大3回まで自動更新されます。それでも期限切れが続く場合は、ネットワーク接続を確認 |
| ボットがDMに応答しない | `WEIXIN_DM_POLICY` を確認 — `allowlist` に設定されている場合、送信者は `WEIXIN_ALLOWED_USERS` に含まれている必要がある |
| ボットがグループメッセージを無視する | グループポリシーはデフォルトで `disabled` です。`WEIXIN_GROUP_POLICY=open` または `allowlist` を設定 — ただし、QRログインのiLinkボットアイデンティティ（`...@im.bot`）は通常、通常のWeChatグループメッセージをまったく受信できないことに注意。ゲートウェイログにグループメッセージの生の受信イベントが表示されない場合、制限はiLink側にあり、Hermes側ではありません。 |
| メディアのダウンロード/アップロードが失敗する | `cryptography` がインストールされていることを確認。`novac2c.cdn.weixin.qq.com` へのネットワークアクセスを確認 |
| `Blocked unsafe URL (SSRF protection)` | 送信メディアURLがプライベート/内部アドレスを指しています。公開URLのみが許可されます |
| 音声メッセージがテキストとして表示される | WeChatが書き起こしを提供している場合、アダプターはテキストを使用します。これは期待される動作です |
| メッセージが重複して見える | アダプターはメッセージIDで重複排除します。重複が見られる場合、複数のゲートウェイインスタンスが実行されていないか確認 |
| `iLink POST ... HTTP 4xx/5xx` | iLinkサービスからのAPIエラー。トークンの有効性とネットワーク接続を確認 |
| ターミナルのQRコードがレンダリングされない | messagingエクストラ付きで再インストール: `pip install hermes-agent[messaging]`。あるいは、QRの上に表示されたURLを開く |
