# 튜토리얼: 일일 브리핑 봇 구축하기

이 튜토리얼에서는 매일 아침 깨어나 관심 있는 주제를 조사하고 결과를 요약하여 간결한 브리핑을 Telegram이나 Discord로 직접 전달해 주는 개인 브리핑 봇을 구축합니다.

이 튜토리얼이 끝나면 코드를 한 줄도 작성하지 않고 **웹 검색**, **크론 예약**, **위임(delegation)**, 그리고 **메시지 전달** 기능을 결합한 완전히 자동화된 워크플로를 갖추게 됩니다.

:::tip
이 레시피는 웹 검색, 요약, 선택적인 TTS를 모두 Portal 구독 하나로 처리합니다. 가장 빠른 설정 방법은 `hermes setup --portal`입니다. [Nous Portal](/integrations/nous-portal) 문서를 참조하세요.
:::

## 구축할 내용

전체적인 흐름은 다음과 같습니다:

1. **오전 8:00** — 크론 스케줄러가 작업을 트리거합니다.
2. **Hermes 실행** — 프롬프트가 포함된 새 에이전트 세션이 실행됩니다.
3. **웹 검색** — 주제에 관한 최신 뉴스를 수집합니다.
4. **요약** — 깔끔한 브리핑 형식으로 정보를 압축합니다.
5. **전달** — 브리핑을 Telegram이나 Discord로 보냅니다.

이 모든 과정이 자동으로 실행됩니다. 아침 커피와 함께 브리핑을 읽기만 하면 됩니다.

## 사전 요구 사항

시작하기 전에 다음이 준비되어 있는지 확인하세요:

- **Hermes Agent 설치됨** — [설치 가이드](/getting-started/installation) 참조
- **게이트웨이 실행 중** — 게이트웨이 데몬이 크론 실행을 처리합니다:
  ```bash
  hermes gateway install   # 사용자 서비스로 설치
  sudo hermes gateway install --system   # Linux 서버: 부팅 시 시스템 서비스로 설치
  # 또는
  hermes gateway           # 포그라운드에서 실행
  ```
- **Firecrawl API 키** — 웹 검색을 위해 환경 변수에 `FIRECRAWL_API_KEY` 설정
- **메시징 구성됨** (선택 사항이지만 권장) — [Telegram](/user-guide/messaging/telegram) 또는 Discord 홈 채널 설정

:::tip 메시징 앱이 없나요? 문제없습니다.
`deliver: "local"`을 사용하여 이 튜토리얼을 계속 진행할 수 있습니다. 브리핑은 `~/.hermes/cron/output/`에 저장되며 언제든지 읽을 수 있습니다.
:::

## 1단계: 워크플로 수동 테스트

자동화를 시작하기 전에 브리핑이 제대로 작동하는지 확인해 보겠습니다. 채팅 세션을 시작합니다:

```bash
hermes
```

그런 다음 아래 프롬프트를 입력하세요:

```
AI 에이전트 및 오픈 소스 LLM에 대한 최신 뉴스를 검색해 줘.
상위 3개의 뉴스를 링크가 포함된 간결한 브리핑 형식으로 요약해.
```

Hermes가 웹을 검색하고 결과를 읽어 다음과 같은 결과를 생성합니다:

```
☀️ Your AI Briefing — March 8, 2026

1. Qwen 3 Released with 235B Parameters
   Alibaba's latest open-weight model matches GPT-4.5 on several
   benchmarks while remaining fully open source.
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain Launches Agent Protocol Standard
   A new open standard for agent-to-agent communication gains
   adoption from 15 major frameworks in its first week.
   → https://blog.langchain.dev/agent-protocol/

3. EU AI Act Enforcement Begins for General-Purpose Models
   The first compliance deadlines hit, with open source models
   receiving exemptions under the 10M parameter threshold.
   → https://artificialintelligenceact.eu/updates/

---
3 stories • Sources searched: 8 • Generated by Hermes Agent
```

이대로 잘 작동한다면 자동화할 준비가 된 것입니다.

:::tip 형식을 개선해 보세요
원하는 결과물이 나올 때까지 프롬프트를 조정해 보세요. "이모지 헤더 사용"이나 "각 요약은 2문장 이내로 유지"와 같은 지시문을 추가해 보세요. 최종적으로 결정한 프롬프트를 크론 작업에 사용하면 됩니다.
:::

## 2단계: 크론 작업 생성하기

이제 매일 아침 자동으로 실행되도록 예약해 보겠습니다. 두 가지 방법이 있습니다.

크론 작업을 생성하기 전에 Hermes에 전역으로 구성된 기본 모델과 제공자(provider)가 있는지 확인하세요. 특정 작업에 다른 값을 사용하려면 작업 생성 시 명시적인 모델/제공자 오버라이드를 설정하세요.

### 옵션 A: 자연어 사용 (채팅 내에서)

Hermes에게 원하는 것을 말하기만 하면 됩니다:

```
매일 아침 8시에 AI 에이전트 및 오픈 소스 LLM에 관한 최신 뉴스를 웹에서 검색해 줘. 
상위 3개 뉴스를 링크가 포함된 간결한 브리핑으로 요약하고, 친절하고 전문적인 어조를 사용해 줘. telegram으로 전달해.
```

Hermes는 통합 `cronjob` 도구를 사용하여 크론 작업을 생성해 줍니다.

### 옵션 B: CLI 슬래시(Slash) 명령어

더 세밀한 제어를 위해 `/cron` 명령어를 사용하세요:

```
/cron add "0 8 * * *" "AI 에이전트와 오픈 소스 LLM에 대한 최신 뉴스를 웹에서 검색해 줘. 지난 24시간 이내의 기사를 최소 5개 이상 찾아. 가장 중요한 상위 3개의 뉴스를 간결한 일일 브리핑 형식으로 요약해 줘. 각 뉴스에는 명확한 헤드라인, 2문장 요약, 그리고 출처 URL을 포함해. 친절하고 전문적인 어조를 사용해. 이모지 글머리 기호로 포맷팅하고 총 뉴스 개수로 마무리해 줘."
```

### 황금률: 자족적인 프롬프트 (Self-Contained Prompts)

:::warning 핵심 개념
크론 작업은 이전 대화에 대한 기억이나 이전에 설정한 내용에 대한 컨텍스트 없이 **완전히 새로운 세션**에서 실행됩니다. 에이전트가 작업을 수행하는 데 필요한 **모든 정보**가 프롬프트에 포함되어 있어야 합니다.
:::

**나쁜 프롬프트:**
```
내 평소 아침 브리핑을 해줘.
```

**좋은 프롬프트:**
```
AI 에이전트와 오픈 소스 LLM에 대한 최신 뉴스를 웹에서 검색해 줘. 
지난 24시간 이내의 기사를 최소 5개 이상 찾아. 
가장 중요한 상위 3개의 뉴스를 간결한 일일 브리핑 형식으로 요약해 줘. 
각 뉴스에는 명확한 헤드라인, 2문장 요약, 그리고 출처 URL을 포함해. 
친절하고 전문적인 어조를 사용하고 이모지 글머리 기호로 포맷팅해 줘.
```

좋은 프롬프트는 **검색할 내용**, **기사 수**, **형식**, **어조** 등을 구체적으로 명시합니다. 에이전트가 한 번에 작업을 수행하는 데 필요한 모든 것이 담겨 있습니다.

## 3단계: 브리핑 사용자 정의

기본 브리핑이 잘 작동한다면, 더욱 창의적으로 만들어 볼 수 있습니다.

### 다중 주제 브리핑

한 브리핑에서 여러 분야를 다뤄보세요:

```
/cron add "0 8 * * *" "세 가지 주제를 다루는 아침 브리핑을 생성해 줘. 각 주제에 대해 지난 24시간 동안의 최신 뉴스를 검색하고 상위 2개의 뉴스를 링크와 함께 요약해 줘.

주제:
1. AI 및 머신 러닝 — 오픈 소스 모델과 에이전트 프레임워크에 중점
2. 암호화폐 — 비트코인, 이더리움 및 규제 뉴스에 중점
3. 우주 탐사 — SpaceX, NASA 및 상업 우주 비행에 중점

섹션 헤더와 이모지를 포함하여 깔끔한 브리핑 형식으로 포맷팅해 줘. 오늘 날짜와 동기 부여 명언으로 마무리해."
```

### 병렬 조사를 위한 위임(Delegation) 활용

더 빠른 브리핑을 위해 Hermes에게 각 주제를 서브 에이전트에게 위임하도록 지시하세요:

```
/cron add "0 8 * * *" "서브 에이전트에게 조사를 위임하여 아침 브리핑을 생성해. 다음과 같이 세 가지 작업을 병렬로 위임해 줘:

1. 위임: 링크가 포함된 지난 24시간 동안의 상위 2개 AI/ML 뉴스 검색
2. 위임: 링크가 포함된 지난 24시간 동안의 상위 2개 암호화폐 뉴스 검색
3. 위임: 링크가 포함된 지난 24시간 동안의 상위 2개 우주 탐사 뉴스 검색

모든 결과를 수집하여 섹션 헤더, 이모지 포맷팅 및 출처 링크가 포함된 하나의 깔끔한 브리핑으로 병합해. 오늘 날짜를 헤더로 추가해 줘."
```

각 서브 에이전트는 독립적이고 병렬로 검색하며, 메인 에이전트는 모든 것을 하나의 세련된 브리핑으로 결합합니다. 작동 방식에 대한 자세한 내용은 [위임 문서](/user-guide/features/delegation)를 참조하세요.

### 평일 전용 일정

주말에는 브리핑이 필요하지 않나요? 월요일~금요일을 타겟팅하는 크론 표현식을 사용하세요:

```
/cron add "0 8 * * 1-5" "최신 AI 및 기술 뉴스를 검색해 줘..."
```

### 1일 2회 브리핑

아침 개요(overview)와 저녁 요약(recap)을 받아보세요:

```
/cron add "0 8 * * *" "아침 브리핑: 지난 12시간 동안의 AI 뉴스 검색..."
/cron add "0 18 * * *" "저녁 요약: 지난 12시간 동안의 AI 뉴스 검색..."
```

### 메모리를 활용하여 개인적인 컨텍스트 추가

[메모리](/user-guide/features/memory)가 활성화되어 있으면 세션 전체에 유지되는 환경 설정을 저장할 수 있습니다. 하지만 기억하세요 — 크론 작업은 대화형 메모리 없이 새로운 세션에서 실행됩니다. 개인적인 컨텍스트를 추가하려면 프롬프트에 직접 입력하세요:

```
/cron add "0 8 * * *" "당신은 다음 주제에 관심이 있는 시니어 ML 엔지니어를 위해 브리핑을 작성하고 있습니다: PyTorch 생태계, 트랜스포머 아키텍처, 오픈 웨이트 모델, EU의 AI 규제. 오픈 소스와 관련되지 않은 제품 출시나 펀딩 라운드 뉴스는 건너뛰세요.

이 주제들에 대한 최신 뉴스를 검색하세요. 링크가 포함된 상위 3개의 뉴스를 요약하세요. 간결하고 기술적으로 작성하세요 — 이 독자는 기본적인 설명이 필요하지 않습니다."
```

:::tip 페르소나 맞춤 설정
브리핑이 *누구를* 위한 것인지 세부 정보를 포함하면 관련성이 극적으로 향상됩니다. 에이전트에게 당신의 역할, 관심사 및 건너뛸 내용을 알려주세요.
:::

## 4단계: 작업 관리

### 예약된 모든 작업 나열

채팅에서:
```
/cron list
```

또는 터미널에서:
```bash
hermes cron list
```

다음과 같은 결과가 표시됩니다:

```
ID          | Name              | Schedule    | Next Run           | Deliver
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | Morning Briefing  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | Evening Recap     | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### 작업 제거

채팅에서:
```
/cron remove a1b2c3d4
```

또는 대화 형식으로 요청:
```
내 아침 브리핑 크론 작업을 삭제해 줘.
```

Hermes는 `cronjob(action="list")`를 사용하여 해당 작업을 찾고 `cronjob(action="remove")`로 작업을 삭제합니다.

### 게이트웨이 상태 확인

스케줄러가 실제로 실행 중인지 확인하세요:

```bash
hermes cron status
```

게이트웨이가 실행 중이지 않으면 작업이 실행되지 않습니다. 안정성을 위해 백그라운드 서비스로 설치하세요:

```bash
hermes gateway install
# 또는 Linux 서버의 경우
sudo hermes gateway install --system
```

성공적으로 작동하는 일일 브리핑 봇을 구축했습니다. 다음에 살펴볼 주제는 다음과 같습니다:

- **[예약된 작업(Cron)](/user-guide/features/cron)** — 스케줄 형식, 반복 제한, 전달 옵션 등에 대한 전체 레퍼런스
- **[위임(Delegation)](/user-guide/features/delegation)** — 병렬 서브 에이전트 워크플로에 대한 심층 가이드
- **[메시징 플랫폼](/user-guide/messaging)** — Telegram, Discord 또는 기타 전달 대상 설정
- **[메모리(Memory)](/user-guide/features/memory)** — 여러 세션 간의 영구적인 컨텍스트 유지
- **[팁 및 모범 사례](/guides/tips)** — 더 많은 프롬프트 엔지니어링 조언

:::tip 그 밖에 무엇을 예약할 수 있을까요?
브리핑 봇 패턴은 경쟁사 모니터링, GitHub 저장소 요약, 일기 예보, 포트폴리오 추적, 서버 상태 점검, 심지어 매일의 농담 등 어떤 것에든 적용할 수 있습니다. 프롬프트로 설명할 수 있는 것이라면 무엇이든 예약할 수 있습니다.
:::
