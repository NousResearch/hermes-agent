# 슬래시 커맨드 참조 (Slash Commands Reference)

메시징 플랫폼이나 CLI 프롬프트에서 Hermes와 상호작용할 때, 대화 자체 내에서 슬래시 커맨드를 사용하여 세션을 제어하고, 설정을 수정하고, 즉각적인 동작을 트리거할 수 있습니다.

대부분의 커맨드는 현재 대화 스레드의 컨텍스트(활성 세션) 내에서 실행됩니다.

## 세션 관리 (Session Management)

| 커맨드 | 설명 |
|---------|-------------|
| `/new` | 현재 채팅을 재설정하고 새 대화를 시작합니다. 이전 메시지들은 지워지지만, 에이전트가 추출한 메모리(MEMORY.md)는 유지됩니다. 에이전트가 중단되거나 혼란스러워할 때 가장 유용합니다. |
| `/history` | 현재 세션에서 가장 최근 메시지들의 원본 전사(raw transcript)를 출력합니다. 에이전트가 무엇을 "보고 있는지" 확인하는 데 유용합니다. |
| `/stop` | 현재 실행 중인 에이전트 실행을 중단(kill)합니다. 모델이 루프에 빠졌거나 사용자가 실수로 실행 시간이 긴 백그라운드 도구를 트리거한 경우에 사용하세요. |
| `/undo` | 대화 기록에서 에이전트의 마지막 턴(turn)을 제거합니다. 모델이 실수하거나 형편없는 응답을 생성하여 컨텍스트 윈도우를 오염시켰을 때 유용합니다. |

## 시스템 동작 (System Behavior)

| 커맨드 | 설명 |
|---------|-------------|
| `/memory` | 에이전트의 현재 내장 메모리 덤프(MEMORY.md)를 표시합니다. |
| `/flush` | 모델이 실행되기를 기다리지 않고(예: 세션이 끝나거나 만료될 때) 에이전트가 단기 메모리 큐를 장기 스토리지(Honcho/Viking)로 즉시 플러시(flush)하도록 강제합니다. |
| `/model` | 이 대화 스레드에 사용되는 모델을 변경합니다. 대화형 선택기(interactive picker)가 열립니다. |
| `/model <name>` | 대화형 메뉴를 건너뛰고 특정 모델(예: `/model claude-3-5-sonnet-20241022`)로 직접 전환합니다. |
| `/stats` | 사용 중인 모델, 도구, 토큰, 현재 세션에 소요된 시간에 대한 통계를 표시합니다. |
| `/prompt-size` | 현재 시스템 프롬프트 크기의 분석 결과를 토큰 및 바이트 단위로 표시합니다. 에이전트의 컨텍스트 윈도우 한도에 도달한 이유를 디버깅하는 데 유용합니다. |
| `/provider <name>` | 현재 세션의 기본 LLM 제공자(provider)를 전환합니다(예: `/provider anthropic`, `/provider openai`, `/provider openrouter`). |

## 도구 & 스킬 (Tools & Skills)

| 커맨드 | 설명 |
|---------|-------------|
| `/tools` | 현재 활성화된 도구셋(toolsets)과 각 도구셋이 제공하는 도구를 나열합니다. |
| `/tools on <name>` | 특정 도구셋(예: `/tools on shell_execution`)을 활성화합니다. |
| `/tools off <name>` | 특정 도구셋을 비활성화합니다. |
| `/skills` | 설치되어 활성화된 스킬의 목록과 설명을 표시합니다. |
| `/skills off <name>` | 현재 세션에 대해 특정 스킬을 일시적으로 비활성화합니다 (에이전트가 스킬을 과도하게 사용하거나 잘못 사용할 때 유용합니다). |
| `/skills on <name>` | 비활성화된 스킬을 다시 활성화합니다. |
| `/bundles` | 설치된 스킬 번들 목록을 표시합니다. |
| `/<bundle-name>` | 해당 번들에 포함된 모든 스킬을 단일 커맨드로 호출합니다. 번들이 `hermes bundles create backend-dev`로 생성되었다면 `/backend-dev`를 입력하여 에이전트에게 해당 관련 스킬 목록을 불러오도록 지시합니다. |

## 작업 흐름 & 승인 (Workflows & Approvals)

| 커맨드 | 설명 |
|---------|-------------|
| `/approve` | 권한이 필요한 작업을 대기 중인 에이전트의 보류 중인 작업(예: 위험한 쉘 커맨드 실행, 파일 덮어쓰기)을 승인합니다. |
| `/deny` | 보류 중인 작업을 거부합니다. 에이전트에게 거부 이유를 설명하도록 추가 텍스트를 제공할 수 있습니다(예: `/deny 경로가 잘못되었습니다. /tmp/app 대신 /opt/app을 사용하세요.`). |
| `/queue` | 장기 실행 백그라운드 작업, 다운로드 또는 스크립트 실행의 현재 대기열을 표시합니다. |

## 유지보수 (Maintenance)

| 커맨드 | 설명 |
|---------|-------------|
| `/ping` | 게이트웨이가 응답하는지 확인합니다(진단용). |
| `/update` | 게이트웨이에 최신 Hermes 버전을 가져오고(pull) 재시작하도록 지시합니다. CLI 설치에서만 사용할 수 있습니다. |

## 백엔드/게이트웨이 관리 (Backend / Gateway Administration)

이 커맨드들은 글로벌 관리 작업이므로, 설정된 홈/관리자 채널(Admin channel)에서 실행하거나 CLI에서 실행해야 합니다.

| 커맨드 | 설명 |
|---------|-------------|
| `/pair` | 현재 플랫폼(Telegram, Discord 등)의 새 사용자를 승인하기 위해 공유 가능한 1회용 페어링 코드(pairing code)를 생성합니다. 사용자가 새 장치에서 이 코드를 게이트웨이에 보내면 `/approve` 기능 등을 사용할 수 있는 권한이 부여됩니다. |
| `/status` | 모든 활성 연결, 봇의 핑(ping), 시스템 상태, 디스크/메모리 사용량을 출력합니다. |

## 커맨드 작성 (Writing Commands)

슬래시 커맨드는 대부분의 플랫폼에서 지원되지만, 작동 방식이 약간 다를 수 있습니다.

- **Telegram / Discord / Slack:** 백엔드 구문 분석기에 의해 지원됩니다. 커맨드를 입력하기만 하면 게이트웨이가 이를 가로채어 모델에 보내지 않고 핸들러로 라우팅합니다.
- **CLI (`hermes chat`):** 터미널을 통해 입력하면 완전히 동일하게 작동합니다.

### 명령어의 스코프 (Command Scoping)

명령어는 이를 호출한 컨텍스트(메시징 스레드 또는 대화)에 의해 제약을 받습니다. 

Telegram에서 `/model`을 사용하여 모델을 변경하면 해당 변경 사항은 그 특정 Telegram 채팅에만 적용됩니다. 이후에 Discord에서 말을 걸면 Discord 채널은 자체 기본 모델 또는 마지막으로 설정된 모델을 사용합니다. 

단, **글로벌(Global)** 항목은 예외입니다:
- `/update`
- `/pair`

이 명령어들은 전체 게이트웨이 인스턴스에 영향을 미칩니다.

### 커맨드 가로채기 오류 (Command Interception Errors)

에이전트가 "실행 중(running)"일 때(응답을 생성 중이거나 긴 도구 파이프라인을 실행 중일 때) 특정 슬래시 커맨드를 전송하면 봇은 다음과 같이 응답합니다:

> `에이전트가 실행 중입니다 — 완료될 때까지 기다리거나 먼저 /stop을 실행하세요.`

대부분의 커맨드(예: `/model`, `/memory`)는 상태를 안전하게 수정하기 위해 에이전트가 유휴(idle) 상태여야 합니다. 그러나 `/stop`, `/approve`, `/deny` 같은 커맨드는 대기열을 우회하도록 설계되었으므로 에이전트가 사용 중일 때도 즉시 전송되고 가로채기(intercept)됩니다.

## 쉘 훅 통합 (Shell Hooks Integration)

Hermes가 "쉘 훅(Shell Hooks)"으로 구성된 경우, 로컬 쉘의 특정 명령어를 백엔드로 자동 라우팅할 수 있습니다. 

예를 들어, `.zshrc`를 올바르게 설정했다면 `git commit -m "update"`를 입력할 때 Hermes가 커밋 메시지를 교정(proofread)하도록 프롬프트를 표시할 수 있습니다. 이러한 훅은 슬래시 커맨드 인프라의 확장이지만 로컬 터미널에서 투명하게 래핑(wrapped)되어 실행됩니다. 

이러한 내용이 어떻게 작동하는지에 대해서는 [훅(Hooks)](../user-guide/features/hooks.md) 기능을 참조하세요.
