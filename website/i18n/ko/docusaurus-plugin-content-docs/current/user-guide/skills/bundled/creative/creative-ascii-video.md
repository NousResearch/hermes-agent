## 파이프라인 아키텍처

모든 모드는 동일한 6단계 파이프라인을 따릅니다:

```
INPUT → ANALYZE → SCENE_FN → TONEMAP → SHADE → ENCODE
```

1. **INPUT** — 소스 자료 로드/디코딩 (비디오 프레임, 오디오 샘플, 이미지 또는 없음)
2. **ANALYZE** — 프레임별 특징 추출 (오디오 밴드, 비디오 휘도/엣지, 모션 벡터)
3. **SCENE_FN** — 씬 함수가 픽셀 캔버스(`uint8 H,W,3`)에 렌더링합니다. `_render_vf()` + 픽셀 블렌드 모드를 통해 여러 문자 그리드를 합성합니다. `references/composition.md` 참조
4. **TONEMAP** — 백분위수 기반의 적응형 밝기 정규화. `references/composition.md` § Adaptive Tonemap 참조
5. **SHADE** — `ShaderChain` + `FeedbackBuffer`를 통한 후처리. `references/shaders.md` 참조
6. **ENCODE** — H.264/GIF 인코딩을 위해 raw RGB 프레임을 ffmpeg로 파이핑

## 주요 구현 노트

### 밝기 — 선형 승수 대신 `tonemap()` 사용

이것이 가장 큰 시각적 문제입니다. 검은 배경의 ASCII는 본질적으로 어둡습니다. **절대 `canvas * N` 승수를 사용하지 마세요** — 하이라이트가 잘립니다. 적응형 톤맵을 사용하세요:

```python
def tonemap(canvas, gamma=0.75):
    f = canvas.astype(np.float32)
    lo, hi = np.percentile(f[::4, ::4], [1, 99.5])
    if hi - lo < 10: hi = lo + 10
    f = np.clip((f - lo) / (hi - lo), 0, 1) ** gamma
    return (f * 255).astype(np.uint8)
```

파이프라인: `scene_fn() → tonemap() → FeedbackBuffer → ShaderChain → ffmpeg`

씬별 감마: 기본값 0.75, 솔라라이즈 0.55, 포스터라이즈 0.50, 밝은 씬 0.85. 어두운 레이어에는 `overlay` 대신 `screen` 블렌드를 사용하세요.

### 폰트 셀 높이

macOS Pillow: `textbbox()`가 잘못된 높이를 반환합니다. `font.getmetrics()`를 사용하세요: `cell_height = ascent + descent`. `references/troubleshooting.md` 참조.

### ffmpeg 파이프 데드락(Deadlock)

장시간 실행되는 ffmpeg에 절대 `stderr=subprocess.PIPE`를 사용하지 마세요 — 버퍼가 64KB에서 차고 교착 상태가 발생합니다. 파일로 리디렉션하세요. `references/troubleshooting.md` 참조.

### 폰트 호환성

모든 유니코드 문자가 모든 폰트에서 렌더링되는 것은 아닙니다. 초기화 시 팔레트를 검증하세요 — 각 문자를 렌더링하고 빈 출력이 있는지 확인하세요. `references/troubleshooting.md` 참조.

### 클립별 아키텍처

분할된 비디오(인용구, 씬, 챕터)의 경우 병렬 렌더링과 선택적 재렌더링을 위해 각각을 별도의 클립 파일로 렌더링하세요. `references/scenes.md` 참조.

## 크리에이티브 발산 (사용자가 실험적/창의적/독특한 출력을 요청할 때만 사용)

사용자가 창의적이거나, 실험적이거나, 놀랍거나, 틀에 얽매이지 않은 출력을 요청하는 경우, 가장 잘 맞는 전략을 선택하고 코드를 생성하기 전에 단계를 추론해 보세요.

- **강제 연결 (Forced Connections)** — 사용자가 교차 도메인의 영감을 원할 때 ("유기적인 느낌으로 만들어줘", "인더스트리얼 미학")
- **개념적 혼합 (Conceptual Blending)** — 사용자가 결합할 두 가지를 지명할 때 ("바다와 음악이 만나는", "우주 + 서예")
- **우회적 전략 (Oblique Strategies)** — 사용자가 최대한 열려 있을 때 ("나를 놀라게 해봐", "내가 본 적 없는 것")

### 강제 연결
1. 시각적 목표와 무관한 도메인(기상 시스템, 미생물학, 건축, 유체 역학, 직물 짜기 등)을 하나 고르세요.
2. 그것의 핵심적인 시각적/구조적 요소를 나열하세요 (침식 → 점진적 드러남; 유사 분열 → 분열하는 복제; 짜기 → 맞물리는 패턴).
3. 이러한 요소를 ASCII 문자와 애니메이션 패턴에 매핑하세요.
4. 합성 — 문자 그리드에서 "침식"이나 "결정화"는 어떤 모습일까요?

### 개념적 혼합
1. 서로 다른 두 가지 시각적/개념적 공간을 명명하세요 (예: 파도 + 악보).
2. 대응 관계를 매핑하세요 (마루 = 높은 음, 골 = 쉼표, 거품 = 스타카토).
3. 선택적으로 혼합하세요 — 가장 흥미로운 매핑은 유지하고, 억지스러운 것은 버리세요.
4. 혼합 속에서만 존재하는 창발적인 속성을 개발하세요.

### 우회적 전략
1. 하나를 뽑으세요: "실수를 숨겨진 의도처럼 존중하라" / "오래된 아이디어를 사용하라" / "당신의 가장 친한 친구라면 어떻게 할까?" / "결점을 강조하라" / "거꾸로 뒤집어라" / "전체가 아닌 부분만" / "역방향"
2. 현재의 ASCII 애니메이션 과제에 비추어 지시 사항을 해석하세요.
3. 코드를 작성하기 전에 시각적 디자인에 측면적인 통찰력을 적용하세요.
