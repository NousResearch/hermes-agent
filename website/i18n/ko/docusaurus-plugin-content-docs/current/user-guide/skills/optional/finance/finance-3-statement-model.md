# 3재무제표 모델 (3 Statement Model)

# 3재무제표 재무 모델 템플릿 완성

손익계산서(Income Statement), 대차대조표(Balance Sheet), 현금흐름표(Cash Flow Statement) 간의 올바른 연결 고리를 사용하여 통합 재무 모델 템플릿을 완성하고 채웁니다.

## ⚠️ 중요 원칙 — 템플릿을 채우기 전에 읽으세요

**하드코드보다 수식 우선 (협상 불가):**
- 모든 예측 셀, 롤포워드, 연결 및 소계는 반드시 Excel 수식이어야 합니다 — 미리 계산된 값이면 절대 안 됩니다.
- Python/openpyxl을 사용할 때: 수식 문자열(`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`)을 작성하고, 계산된 결과(`ws["D15"] = 12500`)를 작성하지 **마세요**.
- 하드코딩된 숫자를 포함해야 하는 **유일한** 셀은 (1) 과거 실제 데이터, (2) Assumptions 탭의 가정 동인입니다.
- Python에서 값을 계산하고 그 결과를 셀에 기록하고 있는 자신을 발견한다면 — **멈추세요**. 대신 수식을 작성하세요.
- 이유: 모델은 시나리오가 전환되거나 가정이 변경될 때 유연하게 반응해야 합니다. 하드코드는 하위의 모든 무결성 검사를 조용히 손상시킵니다.

**사용자와 함께 단계별 확인:**
1. **템플릿 매핑 후** → 식별한 탭/섹션을 사용자에게 보여주고 셀을 건드리기 전에 확인합니다.
2. **과거 데이터 입력 후** → 사용자에게 과거 블록을 보여주고 값/기간이 원본 데이터와 일치하는지 확인합니다.
3. **손익계산서(IS) 예측 구축 후** → 소계 검사를 실행하고 예상되는 IS를 사용자에게 보여준 후 대차대조표(BS)로 넘어가기 전에 확인합니다.
4. **대차대조표(BS) 구축 후** → 모든 기간에 대한 잔액 확인(자산 = 부채+자본)을 사용자에게 보여주고 현금흐름표(CF)로 넘어가기 전에 확인합니다.
5. **현금흐름표(CF) 구축 후** → 현금 일치(CF 기말 현금 = BS 현금)를 사용자에게 보여주고 마무리하기 전에 확인합니다.
6. **모델 전체를 한 번에 채우고 완성된 형태로 제시하지 마세요** — 각 재무제표에서 멈추고, 작업 내용을 보여주며 오류를 조기에 포착하세요.

## 서식 — 전문적인 파란색/회색 팔레트 (템플릿/사용자가 달리 지정하지 않는 한 기본값)

**색상을 최소화하세요.** 셀 채우기에는 파란색과 회색만 사용하세요. 녹색, 노란색, 주황색 또는 여러 가지 강조 색상을 도입하지 마세요 — 깔끔한 모델은 절제력을 사용합니다.

| 요소 | 채우기 | 글꼴 |
|---|---|---|
| 섹션 헤더 (IS / BS / CF 제목) | 짙은 파란색 `#1F4E79` | 흰색 굵게 |
| 열 헤더 (FY2024A, FY2025E 등) | 밝은 파란색 `#D9E1F2` | 검은색 굵게 |
| 입력 셀 (과거 데이터, 가정 동인) | 밝은 회색 `#F2F2F2` 또는 흰색 | 파란색 `#0000FF` |
| 수식 셀 | 흰색 | 검은색 |
| 교차 탭 링크 | 흰색 | 녹색 `#008000` |
| 검사 행 / 주요 총계 | 중간 파란색 `#BDD7EE` | 검은색 굵게 |

**이게 전부입니다. 파란색 3개 + 회색 1개 + 흰색입니다.** 템플릿 자체의 색상 체계가 있는 경우 대신 템플릿을 따르세요.

글꼴 색상은 셀이 **무엇인지**(입력/수식/링크)를 나타냅니다. 채우기 색상은 **어디에 있는지**(헤더/데이터/검사)를 나타냅니다.

## 모델 구조

### 템플릿 탭 구성 식별

템플릿은 탭 명명 규칙과 구성이 다릅니다. 입력하기 전에 모든 탭을 검토하여 템플릿 구조를 파악하세요. 다음은 일반적인 탭 이름과 전형적인 내용입니다:

| 일반적인 탭 이름 | 찾아야 할 내용 |
|------------------|----------------------|
| IS, P&L, Income Statement | 손익계산서 |
| BS, Balance Sheet | 대차대조표 |
| CF, CFS, Cash Flow | 현금흐름표 |
| WC, Working Capital | 운전자본 일정 |
| DA, D&A, Depreciation, PP&E | 감가상각 및 무형자산상각 일정 |
| Debt, Debt Schedule | 부채 일정 |
| NOL, Tax, DTA | 이월결손금(Net Operating Loss) 일정 |
| Assumptions, Inputs, Drivers | 동인 가정 및 입력 |
| Checks, Audit, Validation | 오류 검사 대시보드 |

**템플릿 검토 체크리스트**
- 템플릿에 어떤 탭이 있는지 확인합니다 (모든 템플릿에 모든 일정이 포함되는 것은 아님).
- 위에 나열되지 않은 템플릿별 특정 탭을 메모합니다.
- 탭 종속성 이해 (예: 주요 재무제표로 연결되는 일정)
- 각 탭에서 입력 셀과 수식 셀의 위치 확인

### 템플릿 구조 이해

템플릿을 채우기 전에 데이터가 올바른 위치에 입력되고 수식이 손상되지 않도록 기존 레이아웃을 숙지하세요.

**행 구조 식별**
- 각 탭의 상단에서 모델 제목 위치 확인
- 섹션 헤더 및 시각적 분리 식별
- $ 백만, %, x 등을 나타내는 단위 행 찾기
- 실제(Actuals) 대 추정치(Estimates) 기간을 구분하는 열 헤더 확인
- 기간 라벨(예: FY2024A, FY2025E) 확인
- (일반적으로 글꼴 색상으로 구분되는) 수식 셀 대비 입력 셀 식별

**열 구조 식별**
- 가장 왼쪽 열에서 라인 항목 라벨 확인
- 예측 연도 이전에 과거 연도가 오는지 검증
- 과거 기간과 예측 기간을 구분하는 시각적 테두리 메모
- 모든 탭에 걸쳐 일관된 열 순서 확인

**이름이 지정된 범위(Named Ranges) 사용**
템플릿은 종종 핵심 입력 및 출력에 대해 이름이 지정된 범위를 사용합니다. 데이터를 입력하기 전에:
- 템플릿의 기존 이름이 지정된 범위를 검토합니다 (Excel에서 수식 → 이름 관리자).
- 일반적인 이름 지정 범위는 다음을 포함합니다: 수익 성장률, 비용 비율, 주요 출력(순이익, EBITDA, 총 부채, 현금), 시나리오 선택기 셀.
- 입력값이 이 이름이 지정된 범위로 공급되는 셀에 입력되는지 확인합니다.

### 예측 기간
- 템플릿은 일반적으로 마지막 과거 연도로부터 5년 앞으로 예측합니다.
- 과거(A) 대 예상(E) 열이 명확하게 구분되어 있는지 확인합니다.
- 열이 회계 연도 표기법(예: FY2024A, FY2025E)을 사용하는지 확인합니다.

## 마진 분석

**참고: 다음 마진 분석은 사용자가 요청하거나 템플릿에서 명시적으로 요구하는 경우에만 수행해야 합니다. 지시가 없으면 이 섹션을 건너뛰세요.**

손익계산서(IS) 탭에서 수익성 마진을 계산하고 표시하여 운영 효율성을 추적하고 피어 비교를 가능하게 합니다.

### 포함해야 할 핵심 마진

| 마진 | 수식 | 측정 대상 |
|--------|---------|------------------|
| 매출총이익률 (Gross Margin) | 매출총이익 / 매출액 | 가격 결정력, 생산 효율성 |
| EBITDA 마진 | EBITDA / 매출액 | 핵심 영업 수익성 |
| 영업이익률 (EBIT Margin) | 영업이익(EBIT) / 매출액 | D&A 후 영업 수익성 |
| 순이익률 (Net Income Margin)| 순이익 / 매출액 | 최종 수익성 |

### 마진이 포함된 손익계산서 레이아웃

각 이익 라인 항목 바로 아래에 마진 백분율을 표시합니다:
- 매출총이익 아래의 Gross Margin %
- 영업이익(EBIT) 아래의 EBIT Margin %
- EBITDA 아래의 EBITDA Margin %
- 순이익 아래의 Net Income Margin %

## 신용 지표 (Credit Metrics)

**참고: 다음 신용 분석은 사용자가 요청하거나 템플릿에서 명시적으로 요구하는 경우에만 수행해야 합니다. 지시가 없으면 이 섹션을 건너뛰세요.**

대차대조표(BS) 탭에서 신용/레버리지 지표를 계산하고 표시하여 재무 건전성, 부채 수용력 및 코버넌트(covenant) 준수 여부를 평가합니다.

### 포함해야 할 핵심 신용 지표

| 지표 | 수식 | 측정 대상 |
|--------|---------|------------------|
| 총부채 / EBITDA | 총부채 / LTM EBITDA | 레버리지 배수 |
| 순부채 / EBITDA | (총부채 - 현금) / LTM EBITDA | 현금을 차감한 레버리지 |
| 이자보상배율 (Interest Coverage) | EBITDA / 이자비용 | 부채 상환 능력 |
| 부채 / 총자본 | 총부채 / (총부채 + 자기자본) | 자본 구조 |
| 부채비율 (Debt / Equity) | 총부채 / 자기자본 | 재무 레버리지 |
| 유동비율 (Current Ratio) | 유동자산 / 유동부채 | 단기 유동성 |
| 당좌비율 (Quick Ratio) | (유동자산 - 재고자산) / 유동부채 | 즉각적인 유동성 |

### 신용 지표 계층 구조 검사

Upside가 가장 강력한 신용 프로필을 보여주는지 검증합니다:
- 레버리지: Upside < Base < Downside (낮을수록 좋음)
- 보상비율: Upside > Base > Downside (높을수록 좋음)
- 유동성: Upside > Base > Downside (높을수록 좋음)

### 코버넌트 준수 추적

부채 코버넌트가 알려져 있는 경우, 실제 지표와 코버넌트 임계값을 비교하는 명시적 준수 검사를 추가합니다.

## 시나리오 분석 (Base / Upside / Downside)

CHOOSE 또는 INDEX/MATCH 수식이 있는 Assumptions 탭에서 시나리오 토글(드롭다운)을 사용합니다.

| 시나리오 | 설명 |
|----------|-------------|
| Base Case | 경영진 가이던스 또는 컨센서스 추정치 |
| Upside Case | 가이던스 이상의 성장, 마진 확장 |
| Downside Case | 추세 미달 성장, 마진 압박 |

**민감도를 측정할 핵심 동인**: 수익 성장률, 매출총이익률, 판관비(SG&A) %, DSO/DIO/DPO, CapEx %, 이자율, 세율.

**시나리오 감사 검사**: 토글이 모든 재무제표를 전환함, 모든 시나리오에서 대차대조표 일치, 현금 결산 일치, 계층 구조 유지 (순이익, EBITDA, FCF, 마진에서 Upside > Base > Downside).

템플릿이 구체적으로 SEC 서류(10-K, 10-Q)에서 데이터를 가져오도록 요구하는 경우, 자세한 추출 지침은 [references/sec-filings.md](https://github.com/NousResearch/hermes-agent/blob/main/optional-skills/finance/3-statement-model/references/sec-filings.md)를 참조하세요. 이 참조는 규제 당국에 제출된 상장 기업 데이터로 템플릿을 채울 때만 필요합니다.

## 모델 템플릿 완료하기

이 섹션은 기존 수식을 보존하고 데이터 무결성을 보장하면서 3재무제표 재무 모델 템플릿을 완성하기 위한 일반적인 지침을 제공합니다.

### 1단계: 템플릿 구조 분석

데이터를 입력하기 전에 템플릿을 철저히 검토하여 구조를 이해하세요:

**입력 셀 vs 수식 셀 식별**
- 수식 셀에서 입력 셀을 구별하는 시각적 단서(글꼴 색상, 셀 음영)를 찾으세요.
- 일반적인 규칙: 파란색 글꼴 = 입력, 검은색 글꼴 = 수식, 녹색 글꼴 = 다른 시트로의 링크
- Excel의 참조하는 셀 추적(수식 → 참조하는 셀 추적)을 사용하여 셀 관계를 이해하세요.
- 주요 입력을 제어할 수 있는 이름이 지정된 범위를 확인하세요 (수식 → 이름 관리자).

**템플릿 흐름 매핑**
- 어느 탭이 다른 탭으로 들어가는지 식별합니다 (예: Assumptions → IS → BS → CF)
- 모든 지원 일정과 주요 재무제표와의 연결 고리를 메모합니다.
- 작성하기 전에 템플릿의 특정 라인 항목과 구조를 문서화합니다.

### 2단계: 수식을 깨지 않고 데이터 채우기

**데이터 입력을 위한 황금률**

| 규칙 | 설명 |
|------|-------------|
| 입력 셀만 편집 | 수식을 의도적으로 교체하는 경우가 아니면 수식이 포함된 셀을 덮어쓰지 마세요 |
| 셀 참조 보존 | 데이터를 복사할 때 원본 서식으로 수식을 덮어쓰지 않도록 '값 붙여넣기(Ctrl+Shift+V)'를 사용하세요 |
| 템플릿의 단위 일치 | 데이터를 입력하기 전에 템플릿이 수천, 수백만 또는 실제 값을 사용하는지 확인하세요 |
| 부호 규약 존중 | 템플릿의 기존 부호 규약(예: 비용을 양수 또는 음수로 표시)을 따르세요 |
| 순환 참조 확인 | 템플릿에서 반복 계산을 사용하는 경우 '반복 계산 사용'이 켜져 있는지 확인하세요 |

**안전한 데이터 입력 프로세스**
1. 입력을 위해 지정된 정확한 셀(일반적으로 강조 표시되거나 레이블이 지정됨)을 식별합니다.
2. 과거 데이터를 먼저 입력한 다음 해당 기간 동안 수식이 올바르게 계산되는지 확인합니다.
3. 예측 계산에 입력되는 가정 동인을 입력합니다.
4. 계산된 출력을 검토하여 수식이 의도한 대로 작동하는지 확인합니다.
5. 수식 셀을 수정해야 하는 경우 변경하기 전에 원래 수식을 문서화하세요.

**미리 작성된 수식 처리**
- 수식이 아직 채우지 않은 셀을 참조하는 경우 모든 입력이 완료될 때까지 일시적인 오류(#REF!, #DIV/0!)를 예상하세요.
- 수식이 예상치 못한 결과를 생성하는 경우 선행 셀을 추적하여 누락되거나 잘못된 입력을 식별하세요.
- 모든 탭에 걸쳐 수식 종속성을 확인하지 않고 행/열을 절대 삭제하지 마세요.

### 3단계: 수식 검증

**수식 무결성 검사**

템플릿 출력에 의존하기 전에 수식이 올바르게 작동하는지 검증하세요:

| 검사 유형 | 방법 |
|------------|--------|
| 참조하는 셀 추적 | 수식 셀 선택 → 수식 → 참조하는 셀 추적을 통해 올바른 입력을 참조하는지 확인 |
| 참조되는 셀 추적 | 주요 입력이 예상 출력 셀로 올바르게 연결되는지 확인 |
| 수식 계산 | 수식 → 수식 계산을 사용하여 복잡한 계산 단계를 추적 |
| 하드코드 확인 | 예측 수식은 하드코딩된 값이 아니라 가정을 참조해야 함 |
| 알려진 값으로 테스트 | 간단한 테스트 값을 입력하여 수식이 예상 결과를 생성하는지 확인 |
| 교차 탭 일관성 | 동일한 수식 논리가 모든 예측 기간에 적용되는지 확인 |

**주의해야 할 일반적인 수식 문제**
- 기간에 걸쳐 복사될 때 잘못된 결과를 유발하는 혼합된 절대/상대 참조
- 외부 파일 또는 삭제된 범위에 대한 깨진 링크 (#REF! 오류)
- 수익이 증가하기 전 초기 기간의 0으로 나누기 (#DIV/0! 오류)
- 순환 참조 경고 (이자 계산을 위해 의도적인 것일 수 있음)
- 예측 열 전체의 일관성 없는 수식 (차이를 찾기 위해 Ctrl+\ 사용)

**교차 탭 연결 확인**
- 여러 탭에 나타나는 값이 복제된 것이 아니라 링크되어 있는지 확인합니다.
- 일정 총계가 주요 재무제표의 해당 라인 항목과 일치하는지 확인합니다.
- 기간 레이블이 모든 탭에서 정렬되어 있는지 확인합니다.

### 4단계: 시트별 품질 검사

템플릿을 채운 후 각 시트에서 다음 검증 검사를 수행하세요:

**손익계산서 (IS) 품질 검사**
- 수익 수치가 과거 기간의 원본 데이터와 일치합니다.
- 모든 비용 항목의 합이 보고된 총계와 일치합니다.
- 소계 (매출총이익, EBIT, EBT, 순이익)가 올바르게 계산됩니다.
- 세금 계산 논리가 적절합니다 (손실을 올바르게 처리함).
- 예측 동인이 Assumptions 탭을 참조합니다 (하드코드 없음).
- 기간별 변화가 방향적으로 합리적입니다.

**대차대조표 (BS) 품질 검사**
- 모든 기간에 대해 자산 = 부채 + 자기자본입니다 (기본 검사).
- 현금 잔액이 현금흐름표 기말 현금과 일치합니다.
- 운전자본 계정이 지원 일정(해당하는 경우)과 일치합니다.
- 이익잉여금이 올바르게 이월됩니다: 기초 이익잉여금 + 순이익 - 배당금 +/- 조정액 = 기말 이익잉여금
- 부채 잔액이 부채 일정(해당하는 경우)과 일치합니다.
- 모든 대차대조표 항목에 적절한 부호가 있습니다 (자산은 양수, 대부분의 부채는 양수).

**현금흐름표 (CF) 품질 검사**
- 영업활동현금흐름(CFO) 상단의 순이익이 손익계산서의 순이익과 일치합니다.
- 비현금 가산 항목(D&A, SBC 등)이 원본 일정/명세서와 일치합니다.
- 운전자본 변동이 올바른 부호를 갖습니다 (자산 증가 = 현금 사용 = 음수).
- CapEx가 유형자산(PP&E) 일정 또는 고정자산 롤포워드와 일치합니다.
- 재무활동이 대차대조표의 부채 및 자기자본 계정 변동과 일치합니다.
- 기말 현금이 대차대조표의 현금과 일치합니다.
- 기초 현금이 이전 기간의 기말 현금과 같습니다.

**지원 일정 품질 검사**
- 기초 잔액이 이전 기간의 기말 잔액과 같습니다.
- 롤포워드 논리가 완전합니다 (기초 + 증가 - 감소 = 기말).
- 일정 총계가 주요 재무제표 라인 항목과 일치합니다.
- 계산에 사용된 가정이 Assumptions 탭과 일치합니다.

### 5단계: 재무제표 간 무결성 검사

개별 시트를 검증한 후, 3재무제표가 올바르게 통합되었는지 확인합니다:

| 검사 | 수식 | 예상 결과 |
|-------|---------|-----------------|
| 대차대조표 대차 일치 | 자산 - 부채 - 자본 | = 0 |
| 현금 일치 | CF 기말 현금 - BS 현금 | = 0 |
| 순이익 링크 | IS 순이익 - CF 기초 순이익 | = 0 |
| 이익잉여금 | 이전 RE + NI - 배당금 - BS 기말 RE | = 0 (필요에 따라 SBC/기타 항목 조정) |

### 6단계: 최종 검토

모델을 완료된 것으로 간주하기 전에:
- 모든 시나리오(해당하는 경우)를 토글하여 각 케이스에서 검사를 통과하는지 확인합니다.
- 모든 #REF!, #DIV/0!, #VALUE!, #NAME? 오류를 검토하고 해결하거나 문서화합니다.
- 모든 입력 셀이 채워져 있는지 확인합니다 (자리 표시자 값 검색).
- 모든 탭에 걸쳐 단위가 일관된지 확인합니다.
- 추가 수정을 하기 전에 깨끗한 버전을 저장합니다.

### 검사 범주

**섹션 1: 통화 일관성**
- Assumptions에 통화가 식별되고 문서화됨
- 모든 탭이 일관된 통화 기호 및 단위를 사용
- 단위 행이 모델 통화와 일치함

**섹션 2: 대차대조표 무결성**
- 자산 = 부채 + 자기자본 (각 기간별)
- 수식: 자산 - 부채 - 자기자본 (반드시 = 0 이어야 함)

**섹션 3: 현금흐름 무결성**
- 현금이 BS와 일치 (CF 기말 현금 = BS 현금)
- 현금 월별 vs 연간: 기말 현금 (월별) = 기말 현금 (연간)
- NI가 IS와 일치 (CF 순이익 = IS 순이익)
- D&A가 일정과 일치
- SBC가 IS와 일치
- ΔAR, ΔInventory, ΔAP가 WC 일정과 일치
- CapEx가 DA 일정과 일치

**섹션 4: 이익잉여금 (Retained Earnings)**
- RE 롤포워드 검사: 이전 RE + NI + SBC - 배당금 = 기말 RE
- 디버깅을 위해 구성 요소 명세서 표시

**섹션 5: 운전자본 (Working Capital)**
- AR, Inventory, AP가 BS와 일치
- DSO, DIO, DPO 합리성 검사 (정상 범위를 벗어난 경우 플래그)

**섹션 6: 부채 일정 (Debt Schedule)**
- 총 부채가 BS와 일치 (유동 + 비유동 부채)
- 이자 계산이 IS와 일치

**섹션 6b: 자본 조달 (Equity Financing)**
- 자본 발행 수익금이 BS Common Stock/APIC 증가분과 일치
- 주식 발행으로 인한 현금 증가 = 자본 계정 증가 (일치해야 함)
- 자본 조달 일치 확인: ΔCommon Stock/APIC (BS) = 자본 발행 (CFF) (반드시 = 0 이어야 함)
- 0년차 자본 일치 확인: 조달된 자본 (0년차) = 기초 자본 (1년차)

**섹션 6c: 이월결손금(NOL) 일정**
- 기초 NOL (1년차 / 설립 시) = 0 (신규 사업은 0에서 시작)
- EBT < 0일 때만 NOL 증가 (손실이 실현되어야 NOL 생성)
- DTA가 BS와 일치 (NOL Schedule DTA = BS 이연법인세자산)
- NOL 활용 ≤ EBT의 80% (2017년 이후 미국 연방 규정)
- NOL 잔액은 음수가 아님 (사용 가능한 것보다 많이 활용할 수 없음)
- EBT < 0일 때만 NOL 생성
- 과세 소득 ≤ 0일 때 세금 비용 = 0

**섹션 7: 시나리오 계층 구조**
- 절대 지표: Upside > Base > Downside (NI, EBITDA, FCF)
- 마진: Upside > Base > Downside (GM%, EBITDA%, NI%)
- 신용 지표: 레버리지의 경우 Upside < Base < Downside (역전됨)

**섹션 8: 수식 무결성**
- COGS, S&M, G&A, R&D, SBC는 수익(Revenue)의 %로 구동 (하드코드 없음)
- 예측 연도 전체에 걸쳐 일관된 수식
- #REF!, #DIV/0!, #VALUE! 오류 없음

**섹션 9: 신용 지표 임계값**
- 코버넌트 임계값에 따라 지표를 녹색/노란색/빨간색으로 표시
- 모든 빨간색 플래그의 요약

### 마스터 검사 수식

모든 섹션 상태를 하나의 마스터 검사로 통합합니다:
- 모든 섹션이 통과한 경우 → "✓ ALL CHECKS PASS"
- 어느 섹션이든 실패한 경우 → "✗ ERRORS DETECTED - REVIEW BELOW"

### 빠른 디버그 워크플로우

Master Status에 오류가 표시될 때:
1. 스크롤하여 빨간색으로 강조 표시된 섹션을 찾습니다.
2. 어느 검사 범주에 오류가 있는지 확인합니다.
3. 원본 탭으로 이동하여 조사합니다.
4. 근본적인 문제를 해결합니다.
5. Checks 탭으로 돌아가서 해결되었는지 확인합니다.


## 데이터 소스 — MCP 우선, 웹 폴백

아래의 많은 구절에서 "S&P Kensho MCP / Daloopa MCP / FactSet MCP 사용"이라고 말합니다. 이는 원래 Cowork 플러그인 컨텍스트의 상용 재무 데이터 MCP입니다. Hermes에서는 다음을 수행합니다:

- **구조화된 재무 데이터 MCP가 구성되어 있는 경우** (Hermes는 MCP를 지원합니다 — `native-mcp` 스킬 참조), 특정 시점의 비교, 선행 거래 및 서류 제출을 위해 이를 우선 사용하세요.
- **그렇지 않은 경우**, 다음으로 폴백합니다:
  - 미국 서류의 경우 SEC EDGAR(`https://www.sec.gov/cgi-bin/browse-edgar`)에 대한 `web_search` / `web_extract`
  - 보도 자료, 실적 발표 자료에 대한 회사 IR 페이지
  - 대화형 데이터 포털에 대한 `browser_navigate`
  - 사용자 제공 데이터 (컨텍스트에 없을 때 명시적으로 요청)
- **절대 위조하지 마세요**. 배수, 선례 또는 서류 제출 번호를 찾을 수 없는 경우 셀을 `[UNSOURCED]`로 플래그 지정하고 사용자에게 표시하세요.
