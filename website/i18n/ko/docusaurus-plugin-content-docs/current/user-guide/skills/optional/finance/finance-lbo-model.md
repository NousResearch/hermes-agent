## 중요 지침 — 먼저 읽으세요

Python/openpyxl을 사용하세요. 수식 문자열(`ws["D20"] = "=B5*B6"`)을 작성한 다음, 전달하기 전에 `excel-author` 스킬의 `recalc.py` 헬퍼를 실행하세요.

### 핵심 원칙
* **모든 계산은 Excel 수식이어야 합니다** - 절대 Python에서 값을 계산하고 결과를 셀에 하드코딩하지 마세요. openpyxl을 사용할 때 `cell.value = 1250`(계산된 결과)가 아닌 `cell.value = "=B5*B6"`(수식 문자열)을 작성하세요. 모델은 입력값이 변경될 때 동적으로 업데이트되어야 합니다.
* **템플릿 구조를 사용하세요** - `examples/LBO_Model.xlsx` 또는 사용자가 제공한 템플릿의 구성을 따르세요. 고유한 레이아웃을 발명하지 마세요.
* **적절한 셀 참조를 사용하세요** - 모든 수식은 적절한 셀을 참조해야 합니다. 다른 셀에서 와야 할 숫자를 직접 입력하지 마세요.
* **부호 규칙 일관성을 유지하세요** - 템플릿이 사용하는 부호 규칙을 따르세요(유출에 음수를 사용하는 경우도 있고, 양수를 사용하는 경우도 있음). 전체적으로 일관성을 유지하세요.
* **섹션별로 작업하고 매 단계마다 사용자와 확인하세요** - 한 섹션을 완전히 마친 후, 구축한 내용을 사용자에게 보여주고 해당 섹션의 확인 체크를 실행한 후 다음 섹션으로 이동하기 **전에** 확인을 받으세요. 전체 모델을 처음부터 끝까지 구축한 후 제시하지 **마세요** — 이후 섹션은 이전 섹션에 의존하므로, 수익 분석을 모두 구축한 후에 자금의 조달 및 운용(Sources & Uses)에서 실수를 발견하면 모든 것을 다시 작업해야 합니다.

### 수식 색상 규칙
* **파란색 (0000FF)**: 하드코딩된 입력값 - 다른 셀을 참조하지 않고 직접 입력된 숫자
* **검은색 (000000)**: 계산이 포함된 수식 - 연산자나 함수를 사용하는 모든 수식 (`=B4*B5`, `=SUM()`, `=-MAX(0,B4)`)
* **보라색 (800080)**: **같은 탭**의 셀에 대한 링크 - 계산 없는 직접 참조 (`=B9`, `=B45`)
* **초록색 (008000)**: **다른 탭**의 셀에 대한 링크 - 교차 시트 참조 (`=Assumptions!B5`, `='Operating Model'!C10`)

### 채우기 색상 팔레트 — 전문적인 파란색 및 회색 (사용자/템플릿에서 달리 지정하지 않는 한 기본값)
* **최소한으로 유지하세요** — 셀 채우기에는 파란색과 회색만 사용하세요. 초록색, 노란색, 빨간색 또는 여러 강조 색상을 도입하지 마세요. 전문적인 LBO 모델은 절제를 사용합니다.
* **기본 채우기 팔레트:**
  * **섹션 헤더** (Sources & Uses, Operating Model 등): 짙은 파란색 `#1F4E79`, 흰색 굵은 텍스트
  * **열 헤더** (Year 1, Year 2 등): 연한 파란색 `#D9E1F2`, 검은색 굵은 텍스트
  * **입력 셀**: 연한 회색 `#F2F2F2` (또는 그냥 흰색) — 파란색 *글꼴*이 신호이며, 채우기는 부차적입니다.
  * **수식/계산된 셀**: 흰색, 채우기 없음
  * **주요 출력** (IRR, MOIC, Exit Equity): 중간 파란색 `#BDD7EE`, 검은색 굵은 텍스트
* **이것이 전체 팔레트입니다.** 3가지 파란색 + 1가지 회색 + 흰색. 템플릿이 자체 색상을 사용하는 경우 템플릿을 대신 따르세요.
* 참고: 위의 파란색/검은색/보라색/초록색 **글꼴** 색상은 입력값, 수식, 링크를 구별하기 위한 것입니다. 이는 여기서의 **채우기** 팔레트와 분리되어 있으며 — 두 가지 모두 함께 작동합니다.

### 숫자 서식 표준
* **통화**: 템플릿에 따라 `$#,##0;($#,##0);"-"` 또는 `$#,##0.0`
* **백분율**: `0.0%` (소수점 1자리)
* **배수**: `0.0"x"` (소수점 1자리)
* **MOIC/상세 비율**: `0.00"x"` (정밀도를 위한 소수점 2자리)
* **모든 숫자 셀**: 우측 정렬

## 템플릿 분석 단계 - 이것을 먼저 수행하세요

수식을 채우기 전에 템플릿을 철저히 검사하세요:

1. **구조 매핑** - 각 섹션이 위치한 곳과 상호 작용 방식을 식별합니다. 어느 섹션이 다른 섹션으로 입력되는지 주목합니다.

2. **타임라인 이해** - 어떤 열이 어떤 기간을 나타냅니까? "Closing" 또는 "Pro Forma" 열이 있습니까? 예측 기간은 언제부터 시작합니까?

3. **입력 셀 대 수식 셀 식별** - 템플릿은 종종 어떤 셀에 입력이 필요한지, 어떤 셀에 수식이 필요한지를 나타내기 위해 색상 코딩, 테두리 또는 음영을 사용합니다. 이러한 규칙을 존중하세요.

4. **기존 레이블 주의 깊게 읽기** - 행 레이블은 예상되는 계산이 무엇인지 정확히 알려줍니다. 추측하지 말고 템플릿이 묻는 내용을 읽으세요.

5. **기존 수식 확인** - 일부 템플릿은 부분적으로 채워져서 나옵니다. 특별히 요구되지 않는 한 작동하는 수식을 덮어쓰지 마세요.

6. **템플릿별 규칙 기록** - 부호 규칙, 소계 구조, 섹션 구성 방식, 서로 다른 구성 요소에 대해 별도의 탭이 있는지 여부 등.

## 일반적인 문제 영역

다음 계산 패턴은 LBO 모델 전반에서 빈번하게 문제를 일으킵니다. 이를 접할 때 특별한 주의를 기울이세요:

### 섹션 균형 맞추기
* 두 섹션이 동일해야 할 때(예: Sources = Uses), 한 항목이 일반적으로 "플러그(plug)"(균형을 맞추는 수치)가 됩니다.
* 어느 항목이 플러그인지 식별하고 그 차이로 계산합니다.

### 세금 계산
* 세금 수식은 관련 소득 행과 세율만 참조해야 합니다.
* 관련 없는 섹션(예: 부채 일정)을 참조해서는 안 됩니다.
* 손실이 절세 효과를 창출하는지 또는 단순히 무시되는지 고려합니다.

### 이자 및 순환 참조
* 이자 계산은 현금 흐름의 영향을 받는 잔액을 참조하는 경우 순환성을 유발할 수 있습니다.
* 순환 참조를 끊으려면 (평균이나 기말이 아닌) **기초 잔액(Beginning Balance)**을 사용합니다.
* 패턴: 이자 → 현금 흐름 → 상환 → 기말 잔액 (이자가 기말 잔액을 사용하면 이것은 다시 되돌아옵니다)

### 부채 상환 / 현금 스윕 (Debt Paydown / Cash Sweeps)
* 여러 부채 트랜치가 존재하는 경우 대개 우선 순위가 있습니다.
* 현금 스윕은 우선 순위 워터폴을 존중해야 합니다.
* 잔액은 음수가 될 수 없습니다 - MAX 또는 MIN 함수를 적절하게 사용합니다.

### 수익률 계산 (IRR/MOIC)
* 현금 흐름은 올바른 부호를 가져야 합니다: 투자 = 음수, 수익 = 양수
* XIRR을 사용하는 경우 해당하는 날짜가 필요합니다.
* IRR을 사용하는 경우 현금 흐름은 연속된 기간에 있어야 합니다.
* MOIC = 총 수익 / 총 투자

### 민감도 분석표 (Sensitivity Tables)
* **홀수 차원 사용** (5×5 또는 7×7) — 절대 4×4 또는 6×6을 사용하지 마세요. 홀수 차원은 진정한 중심 셀을 보장합니다.
* **중심 셀 = 베이스 케이스.** 모델의 실제 가정을 중심으로 행 및 열 축 값을 대칭적으로 구축합니다 (예: 베이스 진입 배수 = 10.0x인 경우, 축 = `[8.0x, 9.0x, 10.0x, 11.0x, 12.0x]`). 그러면 중심 셀의 IRR/MOIC는 모델의 실제 IRR/MOIC 출력과 반드시 일치해야 합니다 — 이것이 표가 올바르게 연결되었다는 증거입니다.
* **중심 셀 강조** — 중간 파란색 채우기 (`#BDD7EE`) + 굵은 글꼴을 사용하여 베이스 케이스가 시각적으로 고정되게 합니다.
* Excel의 DATA TABLE 함수는 openpyxl에서 작동하지 않을 수 있습니다 — 대신 행/열 헤더를 참조하는 명시적인 수식을 작성하세요.
* 각 셀은 다른 값을 보여야 합니다 — 모두 같다면 수식이 올바르게 변하지 않는 것입니다.
* 혼합 참조를 사용합니다 (예: 행 입력의 경우 `$A5`, 열 입력의 경우 `B$4`).

- [ ] 플러그 항목이 균형을 맞추는 수치로 정확하게 계산되었습니다.
- [ ] 여러 섹션에 걸쳐 일치해야 하는 금액이 일관적입니다.

### 손익/운영 예측
- [ ] 수익/탑라인이 동인(drivers)이나 성장률에서 정확하게 구축되었습니다.
- [ ] 모든 원가 및 비용 항목이 적절하게 계산되었습니다.
- [ ] 소계와 총계가 정확히 합산됩니다.
- [ ] 마진과 비율이 합리적입니다.
- [ ] 가정에 대한 링크가 정확합니다.

### 대차대조표 (해당되는 경우)
- [ ] 자산 = 부채 + 자본 (반드시 균형을 이뤄야 함)
- [ ] 모든 항목이 적절한 일정 또는 롤포워드(roll-forwards)에 연결되어 있습니다.
- [ ] 기초 잔액 = 이전 기간 기말 잔액
- [ ] 검증 행(Check row)이 포함되어 있으며 0을 표시합니다.

### 현금 흐름 (해당되는 경우)
- [ ] 정확한 수입 수치에서 시작합니다.
- [ ] 비현금 항목이 적절히 더해지거나 빼졌습니다.
- [ ] 운전 자본(Working capital) 변동이 올바른 부호를 갖습니다.
- [ ] 기말 현금 = 기초 현금 + 순 현금 흐름
- [ ] 현금 잔액이 재무제표 전체에 걸쳐 일관적입니다.

### 보조 일정 (Supporting Schedules)
- [ ] 롤포워드 일정이 균형을 이룹니다 (기초 + 변동 = 기말).
- [ ] 일정이 주요 재무제표에 정확하게 연결되어 있습니다.
- [ ] 계산된 항목이 적절한 동인(drivers)을 사용합니다.
- [ ] 모든 기간이 일관되게 계산되었습니다.

### 부채/자금 조달 일정 (해당되는 경우)
- [ ] 기초 잔액이 조달 내역(sources)이나 이전 기간과 일치합니다.
- [ ] 적절한 잔액(일반적으로 기초)에 대해 이자가 계산되었습니다.
- [ ] 상환이 현금 가용성과 우선 순위를 존중합니다.
- [ ] 기말 잔액은 음수가 될 수 없습니다.
- [ ] 합계가 트랜치를 정확하게 합산합니다.

### 수익률/출력 분석
- [ ] 엑시트/영구 가치(terminal values)가 정확하게 계산되었습니다.
- [ ] 관련된 모든 조정 항목이 포함되었습니다.
- [ ] 현금 흐름 부호가 정확합니다 (투자는 음수, 수익은 양수).
- [ ] IRR/MOIC 수식이 전체 범위를 참조합니다.
- [ ] 결과가 해당 시나리오에서 합리적입니다.

### 민감도 분석표 (해당되는 경우)
- [ ] 그리드 차원이 홀수입니다 (5×5 또는 7×7) — 진정한 중심 셀이 있습니다.
- [ ] 행 및 열 축 값이 베이스 케이스를 중심으로 대칭입니다 (`[base-2Δ, base-Δ, base, base+Δ, base+2Δ]`).
- [ ] 중심 셀의 출력이 모델의 실제 IRR/MOIC와 일치합니다 — 표가 올바르게 연결되었음을 확인합니다.
- [ ] 중심 셀이 강조되어 있습니다 (중간 파란색 채우기 `#BDD7EE`, 굵은 글꼴).
- [ ] 행 및 열 헤더에 적절한 입력 값이 포함되어 있습니다.
- [ ] 각 데이터 셀에 수식이 포함되어 있습니다 (하드코딩 아님).
- [ ] 각 데이터 셀이 다른 값을 표시합니다.
- [ ] 값이 예상되는 방향으로 이동합니다 (더 높은 엑시트 배수 → 더 높은 IRR 등).

### 서식 (Formatting)
- [ ] 하드코딩된 입력값은 파란색(0000FF)입니다.
- [ ] 계산된 수식은 검은색(000000)입니다.
- [ ] 동일 탭 링크는 보라색(800080)입니다.
- [ ] 교차 탭 링크는 초록색(008000)입니다.
- [ ] 모든 숫자는 우측 정렬되어 있습니다.
- [ ] 적절한 숫자 형식이 전체적으로 적용되었습니다.
- [ ] 어떤 셀도 오류 값(#REF!, #DIV/0!, #VALUE!, #NAME?)을 표시하지 않습니다.

### 논리적 건전성 검사
- [ ] 숫자가 합리적인 규모입니다.
- [ ] 추세가 이치에 맞습니다 (예상대로의 성장, 감소, 안정화).
- [ ] 명백히 잘못된 값(양수여야 할 때 음수, 불가능한 백분율 등)이 없습니다.
- [ ] 주요 결과물이 해당 분석 유형에 합리적인 범위 내에 있습니다.

## 피해야 할 일반적인 오류

| 오류 | 잘못되는 부분 | 해결 방법 |
|-------|-----------------|------------|
| 계산된 값을 하드코딩 | 입력값이 변해도 모델이 업데이트되지 않음 | 항상 원본 셀을 참조하는 수식 사용 |
| 복사 후 잘못된 셀 참조 | 수식이 잘못된 셀을 가리킴 | 모든 링크를 검증하고 적절한 $ 고정 사용 |
| 순환 참조 오류 | 모델이 계산할 수 없음 | 이자 유형 계산에 기초 잔액을 사용하여 순환을 끊음 |
| 섹션의 균형이 맞지 않음 | 일치해야 할 합계가 다름 | 한 항목이 플러그(차이로 계산됨)인지 확인 |
| 불가능한 부분에서 음수 잔액 | 가용한 것보다 많이 지급/사용함 | MAX(0, ...) 또는 MIN 함수를 적절히 사용 |
| IRR/수익 오류 | 잘못된 부호 또는 불완전한 범위 | 현금 흐름 부호를 확인하고 수식이 모든 기간을 덮는지 확인 |
| 민감도 분석표가 같은 값 표시 | 수식이 입력값에 따라 변하지 않음 | 셀 참조 확인 - 혼합 참조 필요 ($A5, B$4) |
| 롤포워드가 일치하지 않음 | 기초 ≠ 이전 기말 | 기간 간의 링크 확인 |
| 일관성 없는 부호 규칙 | 더해야 할 것이 빼지거나 반대의 경우 | 전체에 걸쳐 템플릿의 규칙을 일관되게 따름 |

## 사용자와 작업하기 — 섹션별 체크포인트

* **템플릿 구조가 불분명한 경우**, 진행하기 전에 묻습니다.
* **사용자의 요구 사항이 템플릿과 충돌하는 경우**, 선호 사항을 확인합니다.
* **각 주요 섹션을 완료한 후**, 계속하기 전에 멈추고 사용자와 확인합니다:
  - **자금 조달 및 운용(Sources & Uses) 후** → 균형 잡힌 표를 보여주고, 플러그가 맞는지 확인하고, 운영 모델을 구축하기 전에 승인을 받습니다.
  - **운영 모델 / 예측 후** → 예측 손익계산서를 보여주고, 성장률과 마진이 맞는지 확인하고, 부채 일정을 작성하기 전에 승인을 받습니다.
  - **부채 일정(Debt Schedule) 후** → 기초/기말 잔액과 이자를 보여주고, 워터폴 로직을 확인하고, 수익 분석 전에 승인을 받습니다.
  - **수익 분석 (IRR/MOIC) 후** → 현금 흐름 시리즈와 결과물을 보여주고, 부호와 범위를 확인하고, 민감도 분석표 전에 승인을 받습니다.
  - **민감도 분석표 후** → 각 셀이 변하는지 보여주고, 베이스 케이스가 예상된 곳에 위치하는지 확인합니다.
* **확인 중 오류가 발견되면**, 다음 섹션으로 이동하기 전에 수정합니다.
* **작업 내용을 보여줍니다** - 도움이 될 때 주요 수식이나 가정을 설명합니다.
* **각 섹션에서 확인하지 않은 채로 완성된 모델을 절대 제시하지 마세요** — 망가진 IRR에서 역추적하는 것보다 출처에서 잘못된 셀 참조를 잡아내는 것이 더 빠릅니다.
