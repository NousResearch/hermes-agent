# Tutorial: Construa um Bot de Resumo Diário {#tutorial-build-a-daily-briefing-bot}

Neste tutorial, você vai construir um bot de resumo pessoal que desperta todas as manhãs, pesquisa tópicos que interessam a você, resume as descobertas e entrega um resumo conciso diretamente no seu Telegram ou Discord.

Ao final, você terá um fluxo de trabalho totalmente automatizado combinando **busca na web**, **agendamento cron**, **delegação** e **entrega por mensageria** — sem necessidade de código.

:::tip
Esta receita usa busca na web, resumo e TTS opcional — tudo empacotado em uma assinatura do Portal. A configuração mais rápida é `hermes setup --portal`. Veja [Nous Portal](/integrations/nous-portal).
:::

## O que Vamos Construir {#what-were-building}

Aqui está o fluxo:

1. **8h00** — O agendador cron dispara sua tarefa
2. **O Hermes inicia** uma nova sessão de agente com seu prompt
3. **A busca na web** traz as últimas notícias sobre seus tópicos
4. **O resumo** condensa tudo em um formato de resumo limpo
5. **A entrega** envia o resumo para o seu Telegram ou Discord

Tudo isso acontece sem intervenção manual. Você só precisa ler seu resumo com seu café da manhã.

## Pré-requisitos {#prerequisites}

Antes de começar, certifique-se de ter:

- **O Hermes Agent instalado** — veja o [guia de instalação](/getting-started/installation)
- **O gateway em execução** — o daemon do gateway lida com a execução do cron:
  ```bash
  hermes gateway install   # Install as a user service
  sudo hermes gateway install --system   # Linux servers: boot-time system service
  # or
  hermes gateway           # Run in foreground
  ```
- **Uma chave de API do Firecrawl** — defina `FIRECRAWL_API_KEY` no seu ambiente para busca na web
- **Mensageria configurada** (opcional, mas recomendado) — [Telegram](/user-guide/messaging/telegram) ou Discord configurado com um canal principal

:::tip Sem mensageria? Sem problema
Você ainda pode seguir este tutorial usando `deliver: "local"`. Os resumos serão salvos em `~/.hermes/cron/output/` e você pode lê-los quando quiser.
:::

## Passo 1: Teste o Fluxo de Trabalho Manualmente {#step-1-test-the-workflow-manually}

Antes de automatizar qualquer coisa, vamos garantir que o resumo funciona. Inicie uma sessão de chat:

```bash
hermes
```

Depois digite este prompt:

```
Search for the latest news about AI agents and open source LLMs.
Summarize the top 3 stories in a concise briefing format with links.
```

O Hermes vai buscar na web, ler os resultados e produzir algo como:

```
☀️ Your AI Briefing — March 8, 2026

1. Qwen 3 Released with 235B Parameters
   Alibaba's latest open-weight model matches GPT-4.5 on several
   benchmarks while remaining fully open source.
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain Launches Agent Protocol Standard
   A new open standard for agent-to-agent communication gains
   adoption from 15 major frameworks in its first week.
   → https://blog.langchain.dev/agent-protocol/

3. EU AI Act Enforcement Begins for General-Purpose Models
   The first compliance deadlines hit, with open source models
   receiving exemptions under the 10M parameter threshold.
   → https://artificialintelligenceact.eu/updates/

---
3 stories • Sources searched: 8 • Generated by Hermes Agent
```

Se isso funcionar, você está pronto para automatizar.

:::tip Itere no formato
Experimente prompts diferentes até obter uma saída que você adore. Adicione instruções como "use títulos com emoji" ou "mantenha cada resumo com menos de 2 frases". O que você definir vai para a tarefa cron.
:::

## Passo 2: Crie a Tarefa Cron {#step-2-create-the-cron-job}

Agora vamos agendar isso para executar automaticamente todas as manhãs. Você pode fazer isso de duas formas.

Antes de criar tarefas cron, garanta que o Hermes tenha um modelo e provedor padrão configurados globalmente. Se você quiser que uma tarefa específica use valores diferentes, defina overrides explícitos de modelo/provedor por tarefa ao criá-la.

### Opção A: Linguagem Natural (no chat) {#option-a-natural-language-in-chat}

Basta dizer ao Hermes o que você quer:

```
Every morning at 8am, search the web for the latest news about AI agents
and open source LLMs. Summarize the top 3 stories in a concise briefing
with links. Use a friendly, professional tone. Deliver to telegram.
```

O Hermes criará a tarefa cron para você usando a ferramenta unificada `cronjob`.

### Opção B: Comando de Barra na CLI {#option-b-cli-slash-command}

Use o comando `/cron` para mais controle:

```
/cron add "0 8 * * *" "Search the web for the latest news about AI agents and open source LLMs. Find at least 5 recent articles from the past 24 hours. Summarize the top 3 most important stories in a concise daily briefing format. For each story include: a clear headline, a 2-sentence summary, and the source URL. Use a friendly, professional tone. Format with emoji bullet points and end with a total story count."
```

### A Regra de Ouro: Prompts Autocontidos {#the-golden-rule-self-contained-prompts}

:::warning Conceito crítico
Tarefas cron são executadas em uma sessão **completamente nova** — sem memória das suas conversas anteriores, sem contexto sobre o que você "configurou antes". Seu prompt precisa conter **tudo** o que o agente precisa para realizar a tarefa.
:::

**Prompt ruim:**
```
Do my usual morning briefing.
```

**Prompt bom:**
```
Search the web for the latest news about AI agents and open source LLMs.
Find at least 5 recent articles from the past 24 hours. Summarize the
top 3 most important stories in a concise daily briefing format. For each
story include: a clear headline, a 2-sentence summary, and the source URL.
Use a friendly, professional tone. Format with emoji bullet points.
```

O prompt bom é específico sobre **o que pesquisar**, **quantos artigos**, **qual formato** e **qual tom**. É tudo o que o agente precisa de uma vez só.

## Passo 3: Personalize o Resumo {#step-3-customize-the-briefing}

Depois que o resumo básico funcionar, você pode ser criativo.

### Resumos com Múltiplos Tópicos {#multi-topic-briefings}

Cubra várias áreas em um único resumo:

```
/cron add "0 8 * * *" "Create a morning briefing covering three topics. For each topic, search the web for recent news from the past 24 hours and summarize the top 2 stories with links.

Topics:
1. AI and machine learning — focus on open source models and agent frameworks
2. Cryptocurrency — focus on Bitcoin, Ethereum, and regulatory news
3. Space exploration — focus on SpaceX, NASA, and commercial space

Format as a clean briefing with section headers and emoji. End with today's date and a motivational quote."
```

### Usando Delegação para Pesquisa Paralela {#using-delegation-for-parallel-research}

Para resumos mais rápidos, diga ao Hermes para delegar cada tópico a um subagente:

```
/cron add "0 8 * * *" "Create a morning briefing by delegating research to sub-agents. Delegate three parallel tasks:

1. Delegate: Search for the top 2 AI/ML news stories from the past 24 hours with links
2. Delegate: Search for the top 2 cryptocurrency news stories from the past 24 hours with links
3. Delegate: Search for the top 2 space exploration news stories from the past 24 hours with links

Collect all results and combine them into a single clean briefing with section headers, emoji formatting, and source links. Add today's date as a header."
```

Cada subagente pesquisa de forma independente e em paralelo, e depois o agente principal combina tudo em um resumo bem elaborado. Veja a [documentação de Delegação](/user-guide/features/delegation) para mais detalhes sobre como isso funciona.

### Agendamento Somente em Dias de Semana {#weekday-only-schedule}

Não precisa de resumos nos fins de semana? Use uma expressão cron que aponte para segunda a sexta:

```
/cron add "0 8 * * 1-5" "Search for the latest AI and tech news..."
```

### Resumos Duas Vezes por Dia {#twice-daily-briefings}

Obtenha uma visão geral matinal e um recapitulativo noturno:

```
/cron add "0 8 * * *" "Morning briefing: search for AI news from the past 12 hours..."
/cron add "0 18 * * *" "Evening recap: search for AI news from the past 12 hours..."
```

### Adicionando Contexto Pessoal com Memória {#adding-personal-context-with-memory}

Se você tem a [memória](/user-guide/features/memory) habilitada, pode armazenar preferências que persistem entre sessões. Mas lembre-se — tarefas cron são executadas em sessões novas, sem memória conversacional. Para adicionar contexto pessoal, incorpore-o diretamente no prompt:

```
/cron add "0 8 * * *" "You are creating a briefing for a senior ML engineer who cares about: PyTorch ecosystem, transformer architectures, open-weight models, and AI regulation in the EU. Skip stories about product launches or funding rounds unless they involve open source.

Search for the latest news on these topics. Summarize the top 3 stories with links. Be concise and technical — this reader doesn't need basic explanations."
```

:::tip Adapte a persona
Incluir detalhes sobre para quem o resumo é *para* melhora drasticamente a relevância. Diga ao agente seu papel, interesses e o que ignorar.
:::

## Passo 4: Gerencie Suas Tarefas {#step-4-manage-your-jobs}

### Liste Todas as Tarefas Agendadas {#list-all-scheduled-jobs}

No chat:
```
/cron list
```

Ou no terminal:
```bash
hermes cron list
```

Você verá uma saída como:

```
ID          | Name              | Schedule    | Next Run           | Deliver
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | Morning Briefing  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | Evening Recap     | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### Remova uma Tarefa {#remove-a-job}

No chat:
```
/cron remove a1b2c3d4
```

Ou pergunte de forma conversacional:
```
Remove my morning briefing cron job.
```

O Hermes usará `cronjob(action="list")` para encontrá-la e `cronjob(action="remove")` para excluí-la.

### Verifique o Status do Gateway {#check-gateway-status}

Certifique-se de que o agendador está realmente em execução:

```bash
hermes cron status
```

Se o gateway não estiver em execução, suas tarefas não serão executadas. Instale-o como um serviço em segundo plano para maior confiabilidade:

```bash
hermes gateway install
# or on Linux servers
sudo hermes gateway install --system
```
