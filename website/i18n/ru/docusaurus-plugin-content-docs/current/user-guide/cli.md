# Интерфейс CLI

Hermes Agent CLI — это полноценный терминальный интерфейс, а не веб-интерфейс. Он поддерживает многострочное редактирование, автодополнение слэш-команд, историю разговоров, прерывание текущего хода с передачей управления и потоковый вывод инструментов. Он создан для тех, кто живёт в терминале.

:::tip
У Hermes также есть современный TUI с модальными окнами, выделением мышью и неблокирующим вводом. Запускается он командой `hermes --tui` — см. руководство по [TUI](/user-guide/tui).
:::

## Запуск CLI

```bash
# Запустить интерактивный сеанс (по умолчанию)
hermes

# Режим одного запроса (неинтерактивный)
hermes chat -q "Hello"

# С конкретной моделью
hermes chat --model "anthropic/claude-sonnet-4"

# С конкретным провайдером
hermes chat --provider nous        # Использовать Nous Portal
hermes chat --provider openrouter  # Принудительно выбрать OpenRouter

# С конкретными наборами инструментов
hermes chat --toolsets "web,terminal,skills"

# Запустить с заранее загруженными навыками
hermes -s hermes-agent-dev,github-auth
hermes chat -s github-pr-workflow -q "открой черновик PR"

# Возобновить предыдущие сеансы
hermes --continue             # Возобновить самый последний CLI-сеанс (-c)
hermes --resume <session_id>  # Возобновить конкретный сеанс по ID (-r)

# Подробный режим (отладочный вывод)
hermes chat --verbose

# Изолированный git worktree (для параллельной работы нескольких агентов)
hermes -w                         # Интерактивный режим в worktree
hermes -w -q "Fix issue #123"     # Одноразовый запрос в worktree
```

### Строка состояния

Постоянная строка состояния находится над областью ввода и обновляется в реальном времени:

```
 ⚕ claude-sonnet-4-20250514 │ 12.4K/200K │ [██████░░░░] 6% │ $0.06 │ 15m
```

| Элемент | Описание |
|---------|-------------|
| Название модели | Текущая модель (обрезается, если длиннее 26 символов) |
| Счётчик токенов | Использованные токены контекста / максимальный размер контекстного окна |
| Индикатор контекста | Визуальная шкала заполнения с цветовой градацией по порогам |
| Стоимость | Оценочная стоимость сеанса (или `n/a` для неизвестных/нулевых по цене моделей) |
| 🗜️ N | **Счётчик сжатий контекста** — сколько раз текущий сеанс был автоматически сжат. Появляется после первого сжатия. |
| ▶ N | **Активные фоновые задачи** — сколько запросов `/background` ещё выполняется в текущем сеансе. Показывается, когда хотя бы одна задача уже запущена. |
| Продолжительность | Время, прошедшее с начала сеанса |
| ⚠ YOLO | **Предупреждение о режиме YOLO** — показывается, когда включён `HERMES_YOLO_MODE` (либо через `hermes --yolo` при запуске, либо через `/yolo` в середине сеанса). Дублирует предупреждение в баннере, чтобы вы не забывали, что находитесь в режиме автоодобрения. |

Ширина панели подстраивается под размер терминала: полный макет показывается при ширине от 76 колонок, компактный — в диапазоне 52–75, а минимальный (только модель и продолжительность, плюс значок YOLO при активности) — ниже 52.

**Цвета контекста:**

| Цвет | Порог | Значение |
|-------|---------|---------|
| Зелёный | < 50% | Запаса ещё много |
| Жёлтый | 50–80% | Контекст уже заметно заполняется |
| Оранжевый | 80–95% | Почти предел |
| Красный | ≥ 95% | Почти переполнение — стоит вызвать `/compress` |

Для детальной разбивки, включая затраты по категориям (ввод и вывод), используйте `/usage`.

### Отображение при возобновлении сеанса

Когда вы возобновляете предыдущий сеанс (`hermes -c` или `hermes --resume <id>`), между баннером и строкой ввода появляется панель «Предыдущий разговор», где показывается компактная сводка истории. Подробности и настройки см. в разделе [Сеансы — Сводка разговора при возобновлении](sessions.md#conversation-recap-on-resume).

## Горячие клавиши {#keybindings}

| Клавиша | Действие |
|-----|--------|
| `Enter` | Отправить сообщение |
| `Alt+Enter`, `Ctrl+J` или `Shift+Enter` | Новая строка (многострочный ввод). Для `Shift+Enter` нужен терминал, который различает его и `Enter` — см. ниже. В Windows Terminal `Alt+Enter` перехватывается терминалом (полноэкранный режим); используйте `Ctrl+Enter` или `Ctrl+J` вместо него. |
| `Alt+V` | Вставить изображение из буфера обмена, если терминал это поддерживает |
| `Ctrl+V` | Вставить текст и по возможности прикрепить изображения из буфера обмена |
| `Ctrl+B` | Начать или остановить запись голоса, когда включён голосовой режим (`voice.record_key`, по умолчанию `ctrl+b`) |
| `Ctrl+G` | Открыть текущий буфер ввода в `$EDITOR` (vim/nvim/nano/VS Code и т. п.). Сохраните и выйдите, чтобы отправить отредактированный текст как следующий запрос — это удобно для длинных многоабзацных запросов. |
| `Ctrl+X Ctrl+E` | Альтернативное emacs-подобное сочетание для внешнего редактора (то же поведение, что и у `Ctrl+G`). |
| `Ctrl+C` | Прервать агента (двойное нажатие в течение 2 секунд завершает работу принудительно) |
| `Ctrl+D` | Выйти |
| `Ctrl+Z` | Перевести Hermes в фон (только Unix). Чтобы вернуться, выполните `fg` в shell. |
| `Tab` | Принять автоподсказку (ghost text) или автодополнение слэш-команд |

**Предпросмотр многострочной вставки.** Когда вы вставляете многострочный блок, CLI показывает компактный однострочный предпросмотр (`[pasted: 47 lines, 1,842 chars — press Enter to send]`) вместо того, чтобы забивать scrollback целиком. В отправку всё равно уходит полный текст — это лишь улучшение отображения.

**Удаление Markdown из финальных ответов.** CLI убирает лишнюю Markdown-обвязку и стили `**bold**` / `*italic*` из *финальных* ответов агента, чтобы в терминале они выглядели как обычный текст, а не как сырой Markdown. Блоки кода и списки сохраняются. Это не затрагивает платформы шлюза или результаты инструментов — там Markdown остаётся для нативного рендеринга.

## Слэш-команды

Введите `/`, чтобы открыть выпадающий список автодополнения. Hermes поддерживает большой набор слэш-команд, динамических команд навыков и пользовательских быстрых команд.

Примеры из коробки:

| Команда | Описание |
|---------|----------------|
| `/help` | Показать справку по командам |
| `/model` | Показать или изменить текущую модель |
| `/tools` | Показать список доступных инструментов |
| `/skills browse` | Открыть хаб навыков и официальный каталог дополнительных навыков |
| `/background <prompt>` | Запустить запрос в отдельном фоновом сеансе |
| `/skin` | Показать или сменить активную тему CLI |
| `/voice on` | Включить голосовой режим CLI (для записи нажмите `Ctrl+B`) |
| `/voice tts` | Переключить озвучивание ответов Hermes |
| `/reasoning high` | Увеличить уровень рассуждений |
| `/title My Session` | Назвать текущий сеанс |
| `/status` | Показать информацию о сеансе — модель, профиль, токены и длительность — а затем локальный блок «Сводка сеанса» (счётчики последних ходов, самые используемые инструменты, изменённые файлы, последний запрос пользователя и ответ ассистента). Всё считается локально; вызов LLM не нужен. |
| `/sessions` | Открыть интерактивный экран выбора сеанса прямо в классическом CLI; тот же экран выбора используется и в TUI. Введите текст для фильтрации, перемещайтесь стрелками и нажмите Enter, чтобы возобновить. |

Полный список встроенных CLI-команд и сообщений см. в [справочнике слэш-команд](/reference/slash-commands).

Для настройки, выбора провайдера, тихого режима и голосового режима в мессенджерах см. [Голосовой режим](/user-guide/features/voice-mode).

:::tip
Команды не чувствительны к регистру — `/HELP` работает так же, как `/help`. Установленные навыки тоже автоматически становятся слэш-командами.
:::

## Быстрые команды

Вы можете определить собственные команды, которые мгновенно запускают команды оболочки без вызова LLM. Они работают и в CLI, и на платформах мессенджеров (Telegram, Discord и т. д.).

```yaml
# ~/.hermes/config.yaml
quick_commands:
  status:
    type: exec
    command: systemctl status hermes-agent
  gpu:
    type: exec
    command: nvidia-smi --query-gpu=utilization.gpu,memory.used --format=csv,noheader
  restart:
    type: alias
    target: /gateway restart
```

Затем наберите `/status`, `/gpu` или `/restart` в любом чате. Больше примеров см. в [руководстве по конфигурации](/user-guide/configuration).

## Предзагрузка навыков при запуске

Если вы уже знаете, какие навыки должны быть активны в сеансе, передайте их при запуске:

```bash
hermes -s hermes-agent-dev,github-auth
hermes chat -s github-pr-workflow -s github-auth
```

Hermes загружает каждый указанный навык в контекст сеанса до первого хода. Этот же флаг работает и в интерактивном режиме, и в режиме одного запроса.

## Слэш-команды навыков

Каждый установленный навык в `~/.hermes/skills/` автоматически регистрируется как слэш-команда. Имя навыка становится командой:

```
/gif-search funny cats
/axolotl помоги дообучить Llama 3 на моём датасете
/github-pr-workflow создай PR для рефакторинга аутентификации

# Одного имени навыка достаточно, чтобы он загрузился, а агент уточнил задачу:
/excalidraw
```

## Персоны

Выберите предустановленную персону, чтобы изменить тон общения агента:

```
/personality pirate
/personality kawaii
/personality concise
```

Встроенные персоны: `helpful`, `concise`, `technical`, `creative`, `teacher`, `kawaii`, `catgirl`, `pirate`, `shakespeare`, `surfer`, `noir`, `uwu`, `philosopher`, `hype`.

Вы также можете определить собственные персоны в `~/.hermes/config.yaml`:

```yaml
personalities:
  helpful: "You are a helpful, friendly AI assistant."
  kawaii: "You are a kawaii assistant! Use cute expressions..."
  pirate: "Arrr! Ye be talkin' to Captain Hermes..."
  # Добавьте собственные!
```

## Многострочный ввод

Есть два способа вводить сообщения в несколько строк:

1. **`Alt+Enter`, `Ctrl+J` или `Shift+Enter`** — вставляют новую строку
2. **Продолжение через обратный слэш** — завершите строку символом `\`, чтобы продолжить:

```
❯ Write a function that:\
  1. Takes a list of numbers\
  2. Returns the sum
```

:::info
Вставлять многострочный текст можно без ограничений — используйте любую из клавиш перевода строки выше или просто вставьте текст целиком.
:::

### Совместимость `Shift+Enter`

Большинство терминалов по умолчанию отправляют для `Enter` и `Shift+Enter` одну и ту же последовательность байт, поэтому приложения не могут их различить. Hermes распознаёт `Shift+Enter` только тогда, когда терминал отправляет отдельную последовательность через [Kitty keyboard protocol](https://sw.kovidgoyal.net/kitty/keyboard-protocol/) или режим `modifyOtherKeys` у xterm.

| Терминал | Статус |
|---|---|
| Kitty, foot, WezTerm, Ghostty | `Shift+Enter` различается по умолчанию |
| iTerm2 (новые версии), Alacritty, терминал VS Code, Warp | Поддерживается после включения Kitty protocol в настройках |
| Windows Terminal Preview 1.25+ | Поддерживается после включения Kitty protocol в настройках |
| macOS Terminal.app, стандартный Windows Terminal (stable) | Не поддерживается — `Shift+Enter` неотличим от `Enter` |

Если терминал не умеет их различать, `Alt+Enter` и `Ctrl+J` продолжают работать везде. **В Windows Terminal отдельно:** `Alt+Enter` перехватывается самим терминалом (переключает полноэкранный режим) и до Hermes не доходит — для новой строки используйте `Ctrl+Enter` (он приходит как `Ctrl+J`) или напрямую `Ctrl+J`.

## Прерывание агента

Вы можете прервать агента в любой момент:

- **Введите новое сообщение и нажмите Enter**, пока агент работает — текущий ход будет прерван, а ваш новый запрос обработан
- **`Ctrl+C`** — прервать текущую операцию (двойное нажатие в течение 2 секунд завершает работу принудительно)
- Запущенные терминальные команды завершаются немедленно (`SIGTERM`, затем `SIGKILL` через 1 секунду)
- Несколько сообщений, введённых во время прерывания, объединяются в один запрос

### Режим занятости ввода

Ключ конфигурации `display.busy_input_mode` задаёт, что происходит, когда вы нажимаете Enter, пока агент занят:

| Режим | Поведение |
|------|----------|
| `"interrupt"` (по умолчанию) | Ваше сообщение прерывает текущую операцию и обрабатывается сразу |
| `"queue"` | Ваше сообщение тихо ставится в очередь и отправляется следующим ходом после завершения работы агента |
| `"steer"` | Ваше сообщение внедряется в текущий запуск через `/steer` и попадает к агенту после следующего вызова инструмента — без прерывания и без нового хода |

```yaml
# ~/.hermes/config.yaml
display:
  busy_input_mode: "steer"   # или "queue", или "interrupt" (по умолчанию)
```

`"queue"` удобно, когда вы хотите подготовить сообщение на следующий ход и не прервать текущую работу. `"steer"` полезен, когда нужно мягко перенаправить агента в середине задачи — например, сказать «заодно проверь и тесты», пока он ещё редактирует код. Неизвестные значения откатываются к `"interrupt"`.

У `"steer"` есть два автоматических запасных варианта: если агент ещё не начал работу или если к сообщению прикреплены изображения, оно будет вести себя как `"queue"`, чтобы ничего не потерялось.

Изменить режим можно и прямо в CLI:

```text
/busy queue
/busy steer
/busy interrupt
/busy status
```

:::tip Первая подсказка
В самый первый раз, когда вы нажимаете Enter, пока Hermes работает, он выводит однострочную подсказку, объясняющую настройку `/busy` (`"(подсказка) Ваше сообщение прервало текущий запуск…"`). Она показывается только один раз для установки — это состояние фиксируется флагом `onboarding.seen.busy_input_prompt` в `config.yaml`. Удалите этот ключ, чтобы увидеть подсказку снова.
:::

### Приостановка в фоне

В Unix-системах нажмите **`Ctrl+Z`**, чтобы отправить Hermes в фон — как любой другой терминальный процесс. Shell покажет подтверждение:

```
Hermes Agent has been suspended. Run `fg` to bring Hermes Agent back.
```

Введите `fg` в shell, чтобы вернуться в сеанс ровно с того места, где остановились. В Windows это не поддерживается.

## Отображение прогресса инструментов

Пока агент работает, CLI показывает анимированные индикаторы:

**Анимация мышления** (во время API-вызовов):
```
  ◜ (｡•́︿•̀｡) pondering... (1.2s)
  ◠ (⊙_⊙) contemplating... (2.4s)
  ✧٩(ˊᗜˋ*)و✧ got it! (3.1s)
```

**Лента выполнения инструментов:**
```
  ┊ 💻 terminal `ls -la` (0.3s)
  ┊ 🔍 web_search (1.2s)
  ┊ 📄 web_extract (2.1s)
```

Переключайте режимы отображения через `/verbose`: `off → new → all → verbose`. Эту команду можно включить и для платформ мессенджеров — см. [конфигурацию](/user-guide/configuration).

### Длина предпросмотра инструментов

Ключ конфигурации `display.tool_preview_length` задаёт максимальное число символов, которое показывается в строках предпросмотра вызовов инструментов (например, путей к файлам или команд терминала). По умолчанию значение `0`, то есть ограничений нет — выводятся полные пути и команды.

```yaml
# ~/.hermes/config.yaml
display:
  tool_preview_length: 80   # Обрезать предпросмотр вызовов инструментов до 80 символов (0 = без ограничения)
```

Это полезно на узких терминалах или тогда, когда аргументы инструментов содержат очень длинные пути.

## Управление сеансами

### Возобновление сеансов

После выхода из CLI-сеанса Hermes выводит команду для возобновления:

```
Resume this session with:
  hermes --resume 20260225_143052_a1b2c3

Session:        20260225_143052_a1b2c3
Duration:       12m 34s
Messages:       28 (5 user, 18 tool calls)
```

Команды возобновления:

```bash
hermes --continue                          # Возобновить самый последний CLI-сеанс
hermes -c                                  # Краткая форма
hermes -c "my project"                     # Возобновить сеанс по имени (самый свежий в линии наследования)
hermes --resume 20260225_143052_a1b2c3     # Возобновить конкретный сеанс по ID
hermes --resume "refactoring auth"         # Возобновить по заголовку
hermes -r 20260225_143052_a1b2c3           # Краткая форма
```

Возобновление восстанавливает полную историю разговора из SQLite. Агент видит все предыдущие сообщения, вызовы инструментов и ответы — как будто вы никуда не уходили.

Используйте `/title My Session Name` в чате, чтобы назвать текущий сеанс, или `hermes sessions rename <id> <title>` из командной строки. Для просмотра прошлых сеансов используйте `hermes sessions list`.

### Хранилище сеансов

CLI-сеансы хранятся в SQLite-базе Hermes по пути `~/.hermes/state.db`. База хранит:

- метаданные сеанса (ID, заголовок, временные метки, счётчики токенов)
- историю сообщений
- линии наследования между сжатыми и возобновлёнными сеансами
- индексы полнотекстового поиска, которые использует `session_search`

Некоторые адаптеры мессенджеров также хранят рядом с базой транскрипты по платформам, но сам CLI возобновляет сеансы из SQLite-хранилища.

### Сжатие контекста

Длинные разговоры автоматически сворачиваются в сводку, когда приближаются к лимитам контекста:

```yaml
# В ~/.hermes/config.yaml
compression:
  enabled: true
  threshold: 0.50    # Сжимать при достижении 50% от лимита контекста по умолчанию

# В `auxiliary:` задаётся модель для суммаризации:
auxiliary:
  compression:
    model: ""  # Оставьте пустым, чтобы использовать основную чат-модель (по умолчанию), либо зафиксируйте недорогую быструю модель, например "google/gemini-3-flash-preview".
```

Когда срабатывает сжатие, средние ходы сворачиваются в сводку, а первые 3 и последние 20 ходов всегда сохраняются.

## Фоновые сеансы

Запустите запрос в отдельном фоне и продолжайте работать в CLI над другими задачами:

```
/background Analyze the logs in /var/log and summarize any errors from today
```

Hermes сразу подтверждает запуск задачи и показывает ваш запрос:

```
🔄 Background task #1 started: "Analyze the logs in /var/log and summarize..."
   Task ID: bg_143022_a1b2c3
```

### Как это работает

Каждый запрос `/background` создаёт **полностью отдельный фоновый сеанс** в изолированном фоновом процессе:

- **Изолированный разговор** — фоновый агент не знает ничего об истории вашего текущего сеанса. Он получает только тот запрос, который вы передали.
- **Та же конфигурация** — фоновый агент наследует модель, провайдера, наборы инструментов, настройки рассуждений и резервную модель из текущего сеанса.
- **Неблокирующий режим** — ваш основной сеанс остаётся полностью интерактивным. Можно продолжать чат, запускать команды и даже создавать новые фоновые задачи.
- **Несколько задач** — можно запускать несколько фоновых задач одновременно. Каждая получает свой номер.

### Результаты

Когда фоновая задача завершается, в терминале появляется отдельная панель с результатом:

```
╭─ ⚕ Hermes (background #1) ──────────────────────────────────╮
│ Found 3 errors in syslog from today:                         │
│ 1. OOM killer invoked at 03:22 — killed process nginx        │
│ 2. Disk I/O error on /dev/sda1 at 07:15                      │
│ 3. Failed SSH login attempts from 192.168.1.50 at 14:30      │
╰──────────────────────────────────────────────────────────────╯
```

Если задача завершается с ошибкой, вместо этого вы увидите уведомление об ошибке. Если в конфигурации включён `display.bell_on_complete`, по завершении задачи терминал подаст звуковой сигнал.

### Примеры использования

- **Долгие исследования** — `"/background" изучи последние разработки в области квантовой коррекции ошибок`, пока вы работаете над кодом
- **Обработка файлов** — `"/background" проанализируй все Python-файлы в этом репозитории и перечисли возможные проблемы безопасности`, пока вы продолжаете разговор
- **Параллельные расследования** — запустите несколько фоновых задач, чтобы одновременно проверить разные гипотезы

:::info
Фоновые сеансы не попадают в основную историю разговоров. Это самостоятельные сеансы со своим task ID, например `bg_143022_a1b2c3`.
:::

## Тихий режим

По умолчанию CLI работает в тихом режиме, который:
- скрывает подробные логи инструментов
- включает анимацию в стиле kawaii
- делает вывод более чистым и дружелюбным

Для отладочного вывода:
```bash
hermes chat --verbose
```
