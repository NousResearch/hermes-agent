# Обзор возможностей

Hermes Agent включает широкий набор возможностей, выходящих далеко за рамки обычного чата. От постоянной памяти и контекстных файлов до автоматизации браузера и голосового общения — эти функции работают вместе, чтобы сделать Hermes мощным автономным ассистентом.

## Основные

- **[Инструменты и наборы инструментов](/docs/user-guide/features/tools)** — инструменты — это функции, которые расширяют возможности агента. Они сгруппированы в логические наборы, которые можно включать и отключать для каждой платформы. К ним относятся веб-поиск, выполнение команд в терминале, редактирование файлов, память, делегирование и многое другое.
- **[Система навыков](/docs/user-guide/features/skills)** — документы со знаниями, которые агент может загрузить по запросу, когда они действительно нужны. Навыки используют механизм прогрессивного раскрытия, чтобы экономить токены, и соответствуют открытому стандарту [agentskills.io](https://agentskills.io/specification).
- **[Постоянная память](/docs/user-guide/features/memory)** — ограниченный, тщательно отобранный объём памяти, который сохраняется между сеансами. Hermes помнит ваши предпочтения, проекты, окружение и усвоенные сведения через `MEMORY.md` и `USER.md`.
- **[Файлы контекста](/docs/user-guide/features/context-files)** — Hermes автоматически находит и подгружает файлы контекста проекта (`.hermes.md`, `AGENTS.md`, `CLAUDE.md`, `SOUL.md`, `.cursorrules`), которые задают его поведение в вашем проекте.
- **[Контекстные ссылки](/docs/user-guide/features/context-references)** — введите `@`, а затем ссылку, чтобы вставить файлы, папки, `git diff` и URL прямо в сообщение. Hermes разворачивает её на месте и автоматически добавляет содержимое.
- **[Контрольные точки](/docs/user-guide/checkpoints-and-rollback)** — Hermes автоматически создаёт снимки рабочей директории перед изменением файлов, чтобы при необходимости можно было откатиться через `/rollback`.

## Автоматизация

- **[Запланированные задачи (Cron)](/docs/user-guide/features/cron)** — запускайте задачи автоматически по естественному описанию или выражению cron. Задачи могут подключать навыки, доставлять результаты на любую платформу и позволяют ставить их на паузу, возобновлять и редактировать.
- **[Делегирование субагентам](/docs/user-guide/features/delegation)** — инструмент `delegate_task` создаёт дочерние экземпляры агентов с изолированным контекстом, ограниченными наборами инструментов и собственными терминальными сеансами. По умолчанию можно запускать до 3 субагентов параллельно, чтобы вести несколько независимых потоков работы.
- **[Выполнение кода](/docs/user-guide/features/code-execution)** — инструмент `execute_code` позволяет агенту писать Python-скрипты, которые программно вызывают инструменты Hermes, сворачивая многошаговые сценарии в один вызов LLM через изолированное RPC-исполнение.
- **[Хуки событий](/docs/user-guide/features/hooks)** — выполняйте собственный код в ключевых точках жизненного цикла. Хуки шлюза отвечают за логирование, уведомления и вебхуки; хуки плагинов — за перехват инструментов, сбор метрик и защитные барьеры.
- **[Пакетная обработка](/docs/user-guide/features/batch-processing)** — запускайте Hermes Agent на сотнях и тысячах запросов параллельно, чтобы генерировать структурированные траектории в формате ShareGPT для подготовки обучающих данных или оценки качества.

## Медиа и веб

- **[Голосовой режим](/docs/user-guide/features/voice-mode)** — полноценное голосовое взаимодействие в CLI и на мессенджер-платформах. Говорите с агентом через микрофон, слушайте голосовые ответы и ведите живые голосовые диалоги в каналах Discord.
- **[Автоматизация браузера](/docs/user-guide/features/browser)** — полноценная автоматизация браузера с несколькими вариантами бэкенда: Browserbase cloud, Browser Use cloud, локальный Chrome через CDP или локальный Chromium. Открывайте сайты, заполняйте формы и извлекайте информацию.
- **[Визуальный анализ и вставка изображений](/docs/user-guide/features/vision)** — мультимодальная работа с изображениями. Вставляйте изображения из буфера обмена в CLI и просите агента анализировать, описывать или использовать их с любой моделью, поддерживающей vision.
- **[Генерация изображений](/docs/user-guide/features/image-generation)** — создавайте изображения по текстовым запросам через FAL.ai. Поддерживается девять моделей (FLUX 2 Klein/Pro, GPT-Image 1.5/2, Nano Banana Pro, Ideogram V3, Recraft V4 Pro, Qwen, Z-Image Turbo); выбирайте нужную через `hermes tools`.
- **[Голос и TTS](/docs/user-guide/features/tts)** — преобразование текста в речь и транскрибация голосовых сообщений на всех мессенджер-платформах. Доступны десять встроенных провайдеров: Edge TTS (бесплатно), ElevenLabs, OpenAI TTS, MiniMax, Mistral Voxtral, Google Gemini, xAI, NeuTTS, KittenTTS и Piper, а также пользовательские командные провайдеры для любого локального TTS CLI.

## Интеграции

- **[Интеграция MCP](/docs/user-guide/features/mcp)** — подключайтесь к любому MCP-серверу через stdio- или HTTP-транспорт. Получайте доступ к внешним инструментам GitHub, баз данных, файловых систем и внутренних API без написания собственных инструментов Hermes. Поддерживаются фильтрация инструментов на уровне сервера и семплинг.
- **[Маршрутизация провайдеров](/docs/user-guide/features/provider-routing)** — точный контроль над тем, какие ИИ-провайдеры обрабатывают ваши запросы. Оптимизируйте расходы, скорость или качество с помощью списков разрешений и запретов, а также приоритетов.
- **[Резервные провайдеры](/docs/user-guide/features/fallback-providers)** — автоматический переход на резервных LLM-провайдеров, когда основная модель сталкивается с ошибками, включая независимые резервные механизмы для вспомогательных задач вроде vision и сжатия.
- **[Пулы учётных данных](/docs/user-guide/features/credential-pools)** — распределяйте API-запросы между несколькими ключами одного и того же провайдера. Автоматическое переключение при превышении лимитов запросов и сбоях.
- **[Кеширование промптов](/docs/user-guide/configuration#prompt-caching)** — встроенное межсеансовое часовое кеширование префикса для Claude в Anthropic, OpenRouter и Nous Portal. Включено по умолчанию, без настройки.
- **[Провайдеры памяти](/docs/user-guide/features/memory-providers)** — подключайте внешние хранилища памяти (Honcho, OpenViking, Mem0, Hindsight, Holographic, RetainDB, ByteRover, Supermemory) для межсеансовой персонализации и формирования пользовательского профиля поверх встроенной системы памяти.
- **[API-сервер](/docs/user-guide/features/api-server)** — публикуйте Hermes как HTTP-эндпоинт, совместимый с OpenAI. Подключайте любой клиент, совместимый с форматом OpenAI, — Open WebUI, LobeChat, LibreChat и другие.
- **[Интеграция с IDE (ACP)](/docs/user-guide/features/acp)** — используйте Hermes внутри ACP-совместимых редакторов, таких как VS Code, Zed и JetBrains. Чат, активность инструментов, диффы файлов и команды терминала отображаются прямо в редакторе.
- **[Обучение с подкреплением (RL)](/docs/user-guide/features/rl-training)** — генерируйте траектории по сеансам агента для обучения с подкреплением и тонкой настройки модели.

## Настройка

- **[Персона и SOUL.md](/docs/user-guide/features/personality)** — полностью настраиваемая личность агента. `SOUL.md` — основной файл идентичности и первая часть системной инструкции; вы можете подменять встроенные или собственные пресеты `/personality` для каждого сеанса.
- **[Обложки и темы](/docs/user-guide/features/skins)** — настраивайте внешний вид CLI: цвета баннера, лица и глаголы спиннера, подписи блоков ответа, фирменный текст и префикс активности инструментов.
- **[Плагины](/docs/user-guide/features/plugins)** — добавляйте собственные инструменты, хуки и интеграции без изменения основного кода. Есть три типа плагинов: общие плагины (tools/hooks), провайдеры памяти (межсеансовые знания) и контекстные движки (альтернативное управление контекстом). Управление доступно через единый интерактивный интерфейс `hermes plugins`.
