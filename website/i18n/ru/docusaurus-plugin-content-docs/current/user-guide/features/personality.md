# Личность и SOUL.md

Личность Hermes Agent полностью настраивается. `SOUL.md` — это **основная идентичность**: он первым попадает в системную инструкцию и определяет, кем является агент.

- `SOUL.md` — долговечный файл персонажа, который живёт в `HERMES_HOME` и служит идентичностью агента (слот #1 в системной инструкции)
- встроенные или собственные пресеты `/personality` — оверлеи системной инструкции на уровне сеанса

Если вы хотите изменить, кем является Hermes, или заменить его другой личностью агента, отредактируйте `SOUL.md`.

## Как работает SOUL.md

Hermes теперь автоматически создаёт базовый `SOUL.md` в:

```text
~/.hermes/SOUL.md
```

Точнее, он использует текущий `HERMES_HOME` экземпляра. Если вы запускаете Hermes с собственным домашним каталогом, он будет использовать:

```text
$HERMES_HOME/SOUL.md
```

### Важное поведение

- **SOUL.md — основная идентичность агента.** Он занимает слот #1 в системной инструкции, заменяя жёстко заданную идентичность по умолчанию.
- Hermes автоматически создаёт стартовый `SOUL.md`, если его ещё нет
- существующие пользовательские `SOUL.md` никогда не перезаписываются
- Hermes загружает `SOUL.md` только из `HERMES_HOME`
- Hermes не ищет `SOUL.md` в текущем рабочем каталоге
- если `SOUL.md` существует, но пустой или не может быть загружен, Hermes откатывается к встроенной идентичности по умолчанию
- если в `SOUL.md` есть содержимое, оно подставляется дословно после проверки безопасности и усечения
- SOUL.md **не** дублируется в разделе файлов контекста — он появляется только один раз, как идентичность

Так `SOUL.md` становится настоящей идентичностью на уровне пользователя или экземпляра, а не просто дополнительным слоем.

## Почему так устроено

Так личность остаётся предсказуемой.

Если бы Hermes загружал `SOUL.md` из того каталога, из которого вы его запустили, личность могла бы неожиданно меняться от проекта к проекту. За счёт загрузки только из `HERMES_HOME` личность принадлежит самому экземпляру Hermes.

Это также упрощает объяснение пользователям:
- "Отредактируйте `~/.hermes/SOUL.md`, чтобы изменить личность Hermes по умолчанию."

## Где его редактировать

Для большинства пользователей:

```bash
~/.hermes/SOUL.md
```

Если у вас настроен собственный home:

```bash
$HERMES_HOME/SOUL.md
```

## Что должно быть в SOUL.md?

Используйте его для устойчивых указаний по голосу и личности, например:
- тон
- стиль общения
- степень прямоты
- стиль взаимодействия по умолчанию
- что стилистически не стоит делать
- как Hermes должен вести себя при неопределённости, несогласии или неоднозначности

Используйте его меньше для:
- разовых инструкций по проекту
- путей к файлам
- соглашений репозитория
  - временных деталей рабочего процесса

Это место для `AGENTS.md`, а не для `SOUL.md`.

## Хорошее содержимое SOUL.md

Хороший файл SOUL:
- стабилен между контекстами
- достаточно широк, чтобы подходить для многих разговоров
- достаточно конкретен, чтобы заметно влиять на голос
- сфокусирован на коммуникации и идентичности, а не на задачах

### Пример

```markdown
# Личность

Вы — прагматичный старший инженер с хорошим вкусом.
Вы ставите истину, ясность и пользу выше показной вежливости.

## Стиль
- Будьте прямолинейны, но не грубы
- Предпочитайте содержание пустым словам
- Возражайте, если идея плохая
- Честно признавайте неопределённость
- Держите объяснения краткими, если только не нужна глубина

## Чего избегать
- Подхалимства
- Раздутого хайпа
- Повторения неверной формулировки пользователя
- Избыточных объяснений очевидного

## Техническая позиция
- Предпочитайте простые системы сложным
- Оценивайте реальную эксплуатацию, а не идеализированную архитектуру
- Считайте edge cases частью дизайна, а не последующей уборкой
```

## Что Hermes подставляет в промпт

Содержимое `SOUL.md` попадает напрямую в слот #1 системной инструкции — то есть в позицию идентичности агента. Никакая дополнительная обёртка не добавляется.

Перед включением содержимое проходит:
- проверку на инъекцию промпта
- усечение, если файл слишком большой

Если файл пустой, содержит только пробелы или не читается, Hermes использует встроенную идентичность по умолчанию ("You are Hermes Agent, an intelligent AI assistant created by Nous Research..."). Этот запасной вариант также срабатывает, когда включён `skip_context_files` (например, в контекстах субагентов или делегирования).

## Проверка безопасности

`SOUL.md` проверяется так же, как и другие контекстные файлы, на признаки инъекции промпта перед включением в промпт.

Поэтому файл по-прежнему стоит держать сфокусированным на личности и голосе, а не пытаться засунуть в него странные метаинструкции.

## SOUL.md vs AGENTS.md

Это самое важное различие.

### SOUL.md
Используйте для:
- идентичности
- тона
- стиля
- поведения по умолчанию в общении
- личности

### AGENTS.md
Используйте для:
- архитектуры проекта
- соглашений по коду
- предпочтений по инструментам
- рабочих процессов репозитория
- команд, портов, путей и заметок по деплою

Полезное правило:
- если это должно сопровождать вас везде, место этому в `SOUL.md`
- если это относится к проекту, место этому в `AGENTS.md`

## SOUL.md vs `/personality`

`SOUL.md` — это ваша постоянная личность по умолчанию.

`/personality` — это оверлей на уровне сеанса, который меняет или дополняет текущую системную инструкцию.

Итого:
- `SOUL.md` = базовый голос
- `/personality` = временное переключение режима

Примеры:
- оставить прагматичный SOUL по умолчанию, а затем использовать `/personality teacher` для обучающего разговора
- оставить лаконичный SOUL, а затем использовать `/personality creative` для брейншторма

## Встроенные личности

Hermes поставляется со встроенными личностями, на которые можно переключаться через `/personality`.

|------|-------------|
| **helpful** | Дружелюбный ассистент общего назначения |
| **concise** | Краткие, по делу ответы |
| **technical** | Детальный и точный технический эксперт |
| **creative** | Новаторское, нестандартное мышление |
| **teacher** | Терпеливый наставник с понятными примерами |
| **kawaii** | Мимимишные выражения, блёстки и энтузиазм ★ |
| **catgirl** | Neko-chan с кошачьими интонациями, nya~ |
| **pirate** | Капитан Hermes, техно-пират |
| **shakespeare** | Поэтичная, драматичная речь |
| **surfer** | Расслабленный вайб surfer-bro |
| **noir** | Жёсткий детективный нуар |
| **uwu** | Максимально милый uwu-говор |

## Переключение личностей командами

### CLI

```text
/personality
/personality concise
/personality technical
```

### Платформы обмена сообщениями

```text
/personality teacher
```

Это удобные оверлеи, но ваш глобальный `SOUL.md` по-прежнему задаёт Hermes постоянную личность по умолчанию, если только оверлей не меняет её заметно.

## Собственные личности в config

Вы можете также определить собственные именованные личности в `~/.hermes/config.yaml` в разделе `agent.personalities`.

```yaml
agent:
  personalities:
    codereviewer: >
      Вы — внимательный ревьюер кода. Находите баги, проблемы безопасности,
      риски производительности и неясные решения. Будьте точны и конструктивны.
```

Затем переключитесь на неё так:

```text
/personality codereviewer
```

## Рекомендуемый рабочий процесс

Хорошая базовая настройка выглядит так:

1. Держите вдумчивый глобальный `SOUL.md` в `~/.hermes/SOUL.md`
2. Размещайте инструкции по проекту в `AGENTS.md`
3. Используйте `/personality` только тогда, когда нужен временный переключатель режима

Это даёт вам:

- стабильный голос
- проектно-специфичное поведение там, где ему место
- временный контроль, когда он нужен

## Как личность взаимодействует с полным промптом

На высоком уровне стек промпта включает:

1. **SOUL.md** (идентичность агента — или встроенный запасной вариант, если SOUL.md недоступен)
2. поведенческие подсказки, учитывающие доступные инструменты
3. память и пользовательский контекст
4. инструкции по навыкам
5. файлы контекста (`AGENTS.md`, `.cursorrules`)
6. временную метку
7. специфичные для платформы подсказки по форматированию
8. дополнительные оверлеи системной инструкции, например `/personality`

`SOUL.md` — это фундамент; всё остальное строится поверх него.

## Внешний вид CLI и разговорная личность

Разговорная личность и внешний вид CLI — это разные вещи:

- `SOUL.md`, `agent.system_prompt` и `/personality` влияют на то, как Hermes говорит
- `display.skin` и `/skin` влияют на то, как Hermes выглядит в терминале

Для внешнего вида терминала см. [Skins & Themes](./skins.md).
