# Скины и темы

Скины отвечают за **визуальное оформление** Hermes CLI: цвета баннера, лица и глаголы спиннера, подписи блока ответа, фирменный текст и префикс активности инструментов.

Разговорный стиль и визуальный стиль — это разные вещи:

- **Personality** меняет тон и формулировки агента.
- **Skin** меняет внешний вид CLI.

## Смена скина

```bash
/skin                # показать текущий скин и список доступных скинов
/skin ares           # переключиться на встроенный скин
/skin mytheme        # переключиться на пользовательский скин из ~/.hermes/skins/mytheme.yaml
```

Или задайте скин по умолчанию в `~/.hermes/config.yaml`:

```yaml
display:
  skin: default
```

## Встроенные скины

| Скин | Описание | Брендинг агента | Визуальные особенности |
|------|-------------|----------------|------------------|
| `default` | Классический Hermes — золото и kawaii | `Hermes Agent` | Тёплые золотистые рамки, текст цвета cornsilk и kawaii-лица в спиннере. Знакомый баннер с кадуцеем. Чисто и дружелюбно. |
| `ares` | Тема бога войны — киноварь и бронза | `Ares Agent` | Глубокие багряные рамки с бронзовыми акцентами. Агрессивные глаголы спиннера («куём», «маршируем», «закаливаем сталь»). Баннер с ASCII-артом меча и щита. |
| `mono` | Монохром — чистая шкала серого | `Hermes Agent` | Только оттенки серого — без цвета. Рамки `#555555`, текст `#c9d1d9`. Идеально для минималистичных терминалов или записи экрана. |
| `slate` | Холодный синий — для разработчиков | `Hermes Agent` | Королевские синие рамки (`#4169e1`), мягкий голубой текст. Спокойно и профессионально. Собственного спиннера нет — используются стандартные лица. |
| `daylight` | Светлая тема для ярких терминалов с тёмным текстом и холодными синими акцентами | `Hermes Agent` | Предназначена для белых или ярких терминалов. Тёмно-сланцевый текст с синими рамками, бледные статусные поверхности и светлое меню автодополнения, которое остаётся читаемым в светлых профилях терминала. |
| `warm-lightmode` | Тёплый коричнево-золотой текст для светлых фонов терминала | `Hermes Agent` | Тёплые пергаментные тона для светлых терминалов. Тёмно-коричневый текст с акцентами `saddle-brown`, кремовые статусные поверхности. Земляная альтернатива более холодной теме `daylight`. |
| `poseidon` | Тема бога океана — глубокий синий и морская пена | `Poseidon Agent` | Градиент от глубокого синего к морской пене. Спиннеры в морской тематике («прокладываем курс», «прощупываем глубину»). Баннер с ASCII-артом трезубца. |
| `sisyphus` | Сизифова тема — строгая шкала серого с упорством | `Sisyphus Agent` | Светлые оттенки серого и резкий контраст. Спиннеры с мотивами камня («толкаем в гору», «сбрасываем валун», «идём по кругу»). Баннер с ASCII-артом валуна и холма. |
| `charizard` | Вулканическая тема — обожжённый оранжевый и тлеющие угли | `Charizard Agent` | Тёплый обожжённый оранжевый с переходом к цвету углей. Спиннеры на огненную тему («подхватываем поток», «измеряем жар»). Баннер с ASCII-артом силуэта дракона. |

## Полный список настраиваемых ключей

### Цвета (`colors:`)

Управляет всеми цветовыми значениями CLI. Значения задаются в виде шестнадцатеричных строк цвета.

| Ключ | Описание | Значение по умолчанию (`default` skin) |
|-----|-------------|--------------------------|
| `banner_border` | Цвет рамки панели вокруг стартового баннера | `#CD7F32` (bronze) |
| `banner_title` | Цвет заголовка в баннере | `#FFD700` (gold) |
| `banner_accent` | Цвет заголовков разделов в баннере (например, Available Tools) | `#FFBF00` (amber) |
| `banner_dim` | Приглушённый текст в баннере (разделители, второстепенные подписи) | `#B8860B` (dark goldenrod) |
| `banner_text` | Основной текст в баннере (названия инструментов, названия навыков) | `#FFF8DC` (cornsilk) |
| `ui_accent` | Общий акцентный цвет интерфейса (подсветка, активные элементы) | `#FFBF00` |
| `ui_label` | Метки и теги интерфейса | `#4dd0e1` (teal) |
| `ui_ok` | Индикаторы успеха (галочки, завершение) | `#4caf50` (green) |
| `ui_error` | Индикаторы ошибок (сбои, блокировка) | `#ef5350` (red) |
| `ui_warn` | Предупреждения (осторожность, запросы подтверждения) | `#ffa726` (orange) |
| `prompt` | Цвет интерактивного текста запроса | `#FFF8DC` |
| `input_rule` | Горизонтальная линия над областью ввода | `#CD7F32` |
| `response_border` | Рамка вокруг блока ответа агента (ANSI escape) | `#FFD700` |
| `session_label` | Цвет метки сеанса | `#DAA520` |
| `session_border` | Приглушённый цвет рамки идентификатора сеанса | `#8B8682` |
| `status_bar_bg` | Цвет фона для полосы статуса и использования в TUI | `#1a1a2e` |
| `voice_status_bg` | Цвет фона для бейджа статуса голосового режима | `#1a1a2e` |
| `selection_bg` | Цвет фона подсветки при выделении мышью в TUI. Если не задан, используется `completion_menu_current_bg`. | `#333355` |
| `completion_menu_bg` | Цвет фона списка меню автодополнения | `#1a1a2e` |
| `completion_menu_current_bg` | Цвет фона активной строки автодополнения | `#333355` |
| `completion_menu_meta_bg` | Цвет фона вспомогательной колонки меню автодополнения | `#1a1a2e` |
| `completion_menu_meta_current_bg` | Цвет фона активной вспомогательной колонки автодополнения | `#333355` |

### Спиннер (`spinner:`)

Управляет анимированным спиннером, который показывается во время ожидания ответов API.

| Ключ | Тип | Описание | Пример |
|-----|------|-------------|---------|
| `waiting_faces` | list of strings | Лица, которые сменяются в ожидании ответа API | `["(⚔)", "(⛨)", "(▲)"]` |
| `thinking_faces` | list of strings | Лица, которые сменяются во время рассуждений модели | `["(⚔)", "(⌁)", "(<>)"]` |
| `thinking_verbs` | list of strings | Глаголы, показываемые в сообщениях спиннера | `["forging", "plotting", "hammering plans"]` |
| `wings` | list of [left, right] pairs | Декоративные скобки по бокам спиннера | `[["⟪⚔", "⚔⟫"], ["⟪▲", "▲⟫"]]` |

Когда значения спиннера пусты (как в `default` и `mono`), используются жёстко заданные значения по умолчанию из `display.py`.

### Фирменные строки (`branding:`)

Текстовые строки, которые используются по всему интерфейсу CLI.

| Ключ | Описание | Значение по умолчанию |
|-----|-------------|---------|
| `agent_name` | Имя, которое показывается в заголовке баннера и в статусе | `Hermes Agent` |
| `welcome` | Приветствие, которое показывается при запуске CLI | `Welcome to Hermes Agent! Type your message or /help for commands.` |
| `goodbye` | Сообщение, которое показывается при выходе | `Goodbye! ⚕` |
| `response_label` | Подпись в заголовке блока ответа | ` ⚕ Hermes ` |
| `prompt_symbol` | Символ перед приглашением ввода пользователя (голый токен; рендереры добавляют пробел в конце) | `❯` |
| `help_header` | Заголовок вывода команды `/help` | `(^_^)? Available Commands` |

### Прочие ключи верхнего уровня

| Ключ | Тип | Описание | Значение по умолчанию |
|-----|------|-------------|---------|
| `tool_prefix` | string | Символ, который ставится перед строками вывода инструментов в CLI | `┊` |
| `tool_emojis` | dict | Переопределения эмодзи для отдельных инструментов (`{tool_name: emoji}`) | `{}` |
| `banner_logo` | string | ASCII-арт с разметкой Rich (заменяет стандартный баннер HERMES_AGENT) | `""` |
| `banner_hero` | string | Главное баннерное изображение с разметкой Rich (заменяет стандартное изображение кадуцея) | `""` |

## Пользовательские скины

Создавайте YAML-файлы в `~/.hermes/skins/`. Пользовательские скины наследуют недостающие значения от встроенного скина `default`, так что достаточно задать только те ключи, которые нужно изменить.

### Полный шаблон YAML для пользовательского скина

```yaml
# ~/.hermes/skins/mytheme.yaml
# Complete skin template — all keys shown. Delete any you don't need;
# missing values automatically inherit from the 'default' skin.

name: mytheme
description: My custom theme

colors:
  banner_border: "#CD7F32"
  banner_title: "#FFD700"
  banner_accent: "#FFBF00"
  banner_dim: "#B8860B"
  banner_text: "#FFF8DC"
  ui_accent: "#FFBF00"
  ui_label: "#4dd0e1"
  ui_ok: "#4caf50"
  ui_error: "#ef5350"
  ui_warn: "#ffa726"
  prompt: "#FFF8DC"
  input_rule: "#CD7F32"
  response_border: "#FFD700"
  session_label: "#DAA520"
  session_border: "#8B8682"
  status_bar_bg: "#1a1a2e"
  voice_status_bg: "#1a1a2e"
  selection_bg: "#333355"
  completion_menu_bg: "#1a1a2e"
  completion_menu_current_bg: "#333355"
  completion_menu_meta_bg: "#1a1a2e"
  completion_menu_meta_current_bg: "#333355"

spinner:
  waiting_faces:
    - "(⚔)"
    - "(⛨)"
    - "(▲)"
  thinking_faces:
    - "(⚔)"
    - "(⌁)"
    - "(<>)"
  thinking_verbs:
    - "processing"
    - "analyzing"
    - "computing"
    - "evaluating"
  wings:
    - ["⟪⚡", "⚡⟫"]
    - ["⟪●", "●⟫"]

branding:
  agent_name: "My Agent"
  welcome: "Welcome to My Agent! Type your message or /help for commands."
  goodbye: "See you later! ⚡"
  response_label: " ⚡ My Agent "
  prompt_symbol: "⚡"
  help_header: "(⚡) Available Commands"

tool_prefix: "┊"

# Переопределения эмодзи для отдельных инструментов (необязательно)
tool_emojis:
  terminal: "⚔"
  web_search: "🔮"
  read_file: "📄"

# Пользовательские ASCII-баннеры (необязательно, поддерживается разметка Rich)
# banner_logo: |
#   [bold #FFD700] MY AGENT [/]
# banner_hero: |
#   [#FFD700]  Custom art here  [/]
```

### Минимальный пример пользовательского скина

Поскольку всё наследуется от `default`, в минимальном скине нужно менять только то, что действительно отличается:

```yaml
name: cyberpunk
description: Neon terminal theme

colors:
  banner_border: "#FF00FF"
  banner_title: "#00FFFF"
  banner_accent: "#FF1493"

spinner:
  thinking_verbs: ["jacking in", "decrypting", "uploading"]
  wings:
    - ["⟨⚡", "⚡⟩"]

branding:
  agent_name: "Cyber Agent"
  response_label: " ⚡ Cyber "

tool_prefix: "▏"
```

## Hermes Mod — визуальный редактор скинов

[Hermes Mod](https://github.com/cocktailpeanut/hermes-mod) — это созданный сообществом веб-интерфейс для визуального создания и управления скинами. Вместо ручного редактирования YAML вы получаете редактор с управлением мышью и живым предпросмотром.

![Редактор скинов Hermes Mod](https://raw.githubusercontent.com/cocktailpeanut/hermes-mod/master/nous.png)

**Что он умеет:**

- Показывает все встроенные и пользовательские скины
- Открывает любой скин в визуальном редакторе со всеми полями скина Hermes (цвета, спиннер, брендинг, префикс инструментов, emoji инструментов)
- Генерирует текстовый `banner_logo` по текстовому запросу
- Преобразует загруженные изображения (PNG, JPG, GIF, WEBP) в ASCII-арт для `banner_hero` с несколькими стилями рендеринга (braille, ASCII-градиент, блоки, точки)
- Сохраняет результат напрямую в `~/.hermes/skins/`
- Активирует скин, обновляя `~/.hermes/config.yaml`
- Показывает сгенерированный YAML и живой предпросмотр

### Установка

**Вариант 1 — Pinokio (в один клик):**

Найдите его на [pinokio.computer](https://pinokio.computer) и установите в один клик.

**Вариант 2 — npx (самый быстрый способ из терминала):**

```bash
npx -y hermes-mod
```

**Вариант 3 — вручную:**

```bash
git clone https://github.com/cocktailpeanut/hermes-mod.git
cd hermes-mod/app
npm install
npm start
```

### Использование

1. Запустите приложение (через Pinokio или из терминала).
2. Откройте **Skin Studio**.
3. Выберите встроенный или пользовательский скин для редактирования.
4. Сгенерируйте логотип по тексту и/или загрузите изображение для основного баннерного изображения. Выберите стиль рендеринга и ширину.
5. Отредактируйте цвета, спиннер, брендинг и другие поля.
6. Нажмите **Save**, чтобы записать YAML-файл скина в `~/.hermes/skins/`.
7. Нажмите **Activate**, чтобы сделать его текущим скином (обновит `display.skin` в `config.yaml`).

Hermes Mod учитывает переменную окружения `HERMES_HOME`, поэтому он работает и с [профилями](/docs/user-guide/profiles).

## Эксплуатационные замечания

- Встроенные скины загружаются из `hermes_cli/skin_engine.py`.
- Неизвестные скины автоматически откатываются к `default`.
- `/skin` сразу же обновляет активную CLI-тему для текущего сеанса.
- Пользовательские скины в `~/.hermes/skins/` имеют приоритет над встроенными скинами с тем же именем.
- Изменения скина через `/skin` действуют только в рамках текущего сеанса. Чтобы сделать скин постоянным по умолчанию, задайте его в `config.yaml`.
- Поля `banner_logo` и `banner_hero` поддерживают разметку Rich для консоли (например, `[bold #FF0000]text[/]`) для цветного ASCII-арта.
