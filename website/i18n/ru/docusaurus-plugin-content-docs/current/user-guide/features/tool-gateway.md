# Шлюз инструментов Nous

**Одна подписка. Все инструменты включены.**

Шлюз инструментов входит в любую платную подписку [Nous Portal](https://portal.nousresearch.com). Он направляет обращения Hermes к инструментам — веб-поиск, генерацию изображений, преобразование текста в речь и облачную автоматизацию браузера — через инфраструктуру Nous, чтобы вам не приходилось заводить отдельные аккаунты Firecrawl, FAL, OpenAI, Browser Use или кого-то ещё только ради полезного агента.

## Что входит

| | Инструмент | Что вы получаете |
|---|---|---|
| 🔍 | **Веб-поиск и извлечение** | Поиск по сети уровня агентных сценариев и извлечение полных страниц через Firecrawl. О лимитах можно не беспокоиться — масштабирование берёт на себя шлюз. |
| 🎨 | **Генерация изображений** | Девять моделей через одну конечную точку: **FLUX 2 Klein 9B**, **FLUX 2 Pro**, **Z-Image Turbo**, **Nano Banana Pro** (Gemini 3 Pro Image), **GPT Image 1.5**, **GPT Image 2**, **Ideogram V3**, **Recraft V4 Pro**, **Qwen Image**. Модель можно выбирать на каждый вызов отдельно или оставить FLUX 2 Klein по умолчанию. |
| 🔊 | **Преобразование текста в речь** | Голоса OpenAI TTS, подключённые к инструменту `text_to_speech`. Озвучивайте голосовые сообщения в Telegram, генерируйте аудио для пайплайнов, озвучивайте любой текст. |
| 🌐 | **Автоматизация облачного браузера** | Сеансы headless Chromium через Browser Use. `browser_navigate`, `browser_click`, `browser_type`, `browser_vision` — все основные примитивы управления агентом, без аккаунта Browserbase. |

Все четыре инструмента тарифицируются по модели pay-as-you-use и списываются с вашей подписки Nous. Вы можете сочетать их как угодно: использовать шлюз для веба и изображений, а свой ключ ElevenLabs оставить для TTS — или, наоборот, отправлять всё через Nous.

## Зачем это нужно

Чтобы агент действительно *умел что-то делать*, обычно приходится собирать 5+ подписок на API — у каждой свои регистрация, лимиты, биллинг и особенности. Шлюз сводит это к одному аккаунту:

- **Один счёт.** Платите Nous, а мы берём остальное на себя.
- **Одна регистрация.** Не нужно отдельно заводить Firecrawl, FAL, Browser Use или OpenAI audio.
- **Один ключ.** OAuth Nous Portal покрывает все инструменты.
- **Тот же уровень качества.** Те же бэкенды, что и при прямом подключении, только через нашу прослойку.

Свои ключи можно подключать в любой момент — на уровне отдельных инструментов. Шлюз не запирает вас внутри экосистемы, он просто сокращает путь.

## Начало работы

```bash
hermes model          # выберите Nous Portal в качестве провайдера
```

Когда вы выбираете Nous Portal, Hermes предлагает включить Шлюз инструментов. Подтвердите — и всё готово: при следующем запуске все поддерживаемые инструменты будут активны.

Проверить, что именно активно, можно в любой момент:

```bash
hermes status
```

Вы увидите раздел примерно такого вида:

```
◆ Nous Tool Gateway
  Nous Portal     ✓ управляемые инструменты доступны
  Web tools       ✓ активны через подписку Nous
  Image gen       ✓ активна через подписку Nous
  TTS             ✓ активен через подписку Nous
  Browser         ○ активен через ключ Browser Use
```

Инструменты с пометкой `active via Nous subscription` работают через шлюз. Всё остальное использует ваши собственные ключи.

## Доступность

Шлюз инструментов — это функция **платной подписки**. У аккаунтов Nous на бесплатном уровне Portal доступен для инференса, но управляемые инструменты не входят в пакет. Чтобы разблокировать шлюз, [обновите тариф](https://portal.nousresearch.com/manage-subscription).

## Совмещайте как удобно

Шлюз работает по каждому инструменту отдельно. Включайте его только там, где это действительно удобно:

- **Все инструменты через Nous** — самый простой вариант: одна подписка, и на этом всё.
- **Шлюз для веба и изображений, свой TTS** — оставьте ElevenLabs для голоса, а остальное отдайте Nous.
- **Шлюз только для тех сервисов, где у вас нет ключей** — сценарий вроде «я уже плачу за Browserbase, но не хочу заводить Firecrawl» отлично работает.

Переключить инструмент можно в любой момент через:

```bash
hermes tools          # интерактивный выбор для каждой категории инструментов
```

Выберите инструмент и укажите **Nous Subscription** в качестве провайдера (или любой прямой провайдер, который вам нужен). Редактировать конфиг вручную не требуется.

## Использование отдельных моделей изображений

По умолчанию генерация изображений использует FLUX 2 Klein 9B ради скорости. При необходимости можно переопределить модель на конкретный вызов, передав ID модели в инструмент `image_generate`:

| Модель | ID | Лучше всего подходит для |
|---|---|---|
| FLUX 2 Klein 9B | `fal-ai/flux-2/klein/9b` | Быстро, хороший вариант по умолчанию |
| FLUX 2 Pro | `fal-ai/flux-2/pro` | Более высокое качество в семействе FLUX |
| Z-Image Turbo | `fal-ai/z-image/turbo` | Стильные изображения, высокая скорость |
| Nano Banana Pro | `fal-ai/gemini-3-pro-image` | Google Gemini 3 Pro Image |
| GPT Image 1.5 | `fal-ai/gpt-image-1/5` | Генерация OpenAI с текстом и изображением |
| GPT Image 2 | `fal-ai/gpt-image-2` | Последняя модель OpenAI для изображений |
| Ideogram V3 | `fal-ai/ideogram/v3` | Сильное следование промпту и хорошая типографика |
| Recraft V4 Pro | `fal-ai/recraft/v4/pro` | Векторный стиль, графический дизайн |
| Qwen Image | `fal-ai/qwen-image` | Мультимодальная модель Alibaba |

Набор моделей со временем меняется — команда `hermes tools` → `Image Generation` показывает актуальный список.

## Справочник по конфигурации

Большинству пользователей это не нужно: `hermes model` и `hermes tools` покрывают все сценарии через интерактивный интерфейс. Этот раздел нужен тем, кто пишет `config.yaml` вручную или автоматизирует установку.

### Флаг `use_gateway` для каждого инструмента

В конфигурации каждого инструмента есть булев флаг `use_gateway`:

```yaml
web:
  backend: firecrawl
  use_gateway: true

image_gen:
  use_gateway: true

tts:
  provider: openai
  use_gateway: true

browser:
  cloud_provider: browser-use
  use_gateway: true
```

Приоритет такой: `use_gateway: true` направляет трафик через Nous независимо от любых прямых ключей в `.env`. Если `use_gateway: false` или поле отсутствует, Hermes использует прямые ключи при их наличии и только потом откатывается к шлюзу.

### Отключение шлюза

```yaml
web:
  use_gateway: false   # Hermes теперь использует FIRECRAWL_API_KEY из .env
```

Когда вы выбираете не-gateway провайдера через `hermes tools`, Hermes сам обычно очищает флаг за вас.

### Собственный шлюз Nous (для продвинутых сценариев)

Если вы запускаете собственный шлюз, совместимый с Nous, переопределите конечные точки в `~/.hermes/.env`:

```bash
TOOL_GATEWAY_DOMAIN=your-domain.example.com
TOOL_GATEWAY_SCHEME=https
TOOL_GATEWAY_USER_TOKEN=your-token        # обычно подставляется автоматически после входа в Portal
FIRECRAWL_GATEWAY_URL=https://...         # точечное переопределение конкретной конечной точки
```

Эти параметры нужны для собственной инфраструктуры: enterprise-развёртываний, dev-сред или отладочных стендов. Обычным подписчикам их вручную задавать не нужно.

## FAQ

### Работает ли это с Telegram / Discord / другими gateway-мессенджерами?

Да. Шлюз инструментов работает на уровне выполнения инструментов, а не на уровне CLI. Любой интерфейс, который умеет вызывать инструменты — CLI, Telegram, Discord, Slack, IRC, Teams, API-сервер и т. д. — получает выгоду автоматически.

### Что будет, если подписка закончится?

Инструменты, маршрутизируемые через шлюз, перестанут работать, пока вы не продлите подписку или не переключитесь на прямые API-ключи через `hermes tools`. Hermes покажет понятную ошибку с указанием на Portal.

### Можно ли видеть использование или стоимость по каждому инструменту?

Да — [панель Nous Portal](https://portal.nousresearch.com) разбивает использование по инструментам, так что вы видите, что именно формирует счёт.

### Modal (serverless terminal) входит в пакет?

Modal доступен как **дополнительная опция** через подписку Nous, но не входит в базовый пакет Шлюза инструментов. Настраивайте его через `hermes setup terminal` или напрямую в `config.yaml`, если вам нужен удалённый песочничный контур для выполнения shell-команд.

### Нужно ли удалять существующие API-ключи после включения шлюза?

Нет — оставьте их в `.env`. Когда `use_gateway: true`, Hermes пропускает прямые ключи и использует шлюз. Вернёте флаг в `false` — и ключи снова станут источником по умолчанию. Шлюз не создаёт lock-in.
