# Голосовой режим

Hermes Agent поддерживает полноценное голосовое взаимодействие в CLI и на мессенджерах. Говорите с агентом через микрофон, слушайте голосовые ответы и ведите живые разговоры в голосовых каналах Discord.

Если вам нужен практический разбор настройки с рекомендуемыми конфигурациями и примерами реального использования, см. [Использование голосового режима с Hermes](/docs/guides/use-voice-mode-with-hermes).

## Требования перед началом

Перед использованием голосовых функций убедитесь, что у вас есть:

1. **Установленный Hermes Agent** — `pip install hermes-agent` (см. [Установку](/docs/getting-started/installation))
2. **Настроенный LLM-провайдер** — выполните `hermes model` или задайте нужные учётные данные провайдера в `~/.hermes/.env`
3. **Рабочая базовая конфигурация** — запустите `hermes`, чтобы убедиться, что агент отвечает на текст, прежде чем включать голос

:::tip
Каталог `~/.hermes/` и файл `config.yaml` создаются автоматически при первом запуске `hermes`. Вручную создавать нужно только `~/.hermes/.env` для API-ключей.
:::

## Обзор

| Функция | Платформа | Описание |
|--------|----------|----------|
| **Интерактивный голос** | CLI | Нажмите Ctrl+B, чтобы начать запись; агент автоматически определяет тишину и отвечает |
| **Автоматический голосовой ответ** | Telegram, Discord | Агент отправляет голосовую версию ответа вместе с текстом |
| **Голосовой канал** | Discord | Бот подключается к голосовому каналу Discord, слушает речь пользователей и отвечает им голосом |

## Требования

### Пакеты Python

```bash
# Голосовой режим в CLI (микрофон + воспроизведение аудио)
pip install "hermes-agent[voice]"

# Discord + Telegram messaging (включает discord.py[voice] для поддержки голосовых каналов)
pip install "hermes-agent[messaging]"

# Премиальный TTS (ElevenLabs)
pip install "hermes-agent[tts-premium]"

# Локальный TTS (NeuTTS, необязательно)
python -m pip install -U neutts[all]

# Всё сразу
pip install "hermes-agent[all]"
```

| Extra | Пакеты | Нужно для |
|-------|--------|----------|
| `voice` | `sounddevice`, `numpy` | Голосового режима в CLI |
| `messaging` | `discord.py[voice]`, `python-telegram-bot`, `aiohttp` | Ботов Discord и Telegram |
| `tts-premium` | `elevenlabs` | Провайдера ElevenLabs TTS |

Опциональный локальный TTS-провайдер: установите `neutts` отдельно через `python -m pip install -U neutts[all]`. При первом использовании модель скачается автоматически.

:::info
`discord.py[voice]` автоматически устанавливает **PyNaCl** (для шифрования голоса) и **opus bindings**. Это необходимо для поддержки голосовых каналов Discord.
:::

### Системные зависимости

```bash
# macOS
brew install portaudio ffmpeg opus
brew install espeak-ng   # для NeuTTS

# Ubuntu/Debian
sudo apt install portaudio19-dev ffmpeg libopus0
sudo apt install espeak-ng   # для NeuTTS
```

| Зависимость | Назначение | Нужно для |
|------------|------------|----------|
| **PortAudio** | Ввод с микрофона и воспроизведение аудио | Голосового режима в CLI |
| **ffmpeg** | Конвертация аудиоформатов (MP3 → Opus, PCM → WAV) | Всех платформ |
| **Opus** | Голосовой кодек Discord | Голосовых каналов Discord |
| **espeak-ng** | Бэкенд phonemizer | Локального провайдера NeuTTS |

### API-ключи

Добавьте в `~/.hermes/.env`:

```bash
# Speech-to-Text — локальному провайдеру ключ не нужен
# pip install faster-whisper          # Бесплатно, работает локально, рекомендуется
GROQ_API_KEY=your-key                 # Groq Whisper — быстро, есть free tier (cloud)
VOICE_TOOLS_OPENAI_KEY=your-key       # OpenAI Whisper — платно (cloud)

# Text-to-Speech (необязательно — Edge TTS и NeuTTS работают без ключа)
ELEVENLABS_API_KEY=***           # ElevenLabs — премиальное качество
# VOICE_TOOLS_OPENAI_KEY above also enables OpenAI TTS
```

:::tip
Если установлен `faster-whisper`, голосовой режим работает **без API-ключей** для STT. Модель (~150 МБ для `base`) скачивается автоматически при первом использовании.
:::

## Голосовой режим в CLI

Голосовой режим доступен и в **классическом CLI** (`hermes chat`), и в **TUI** (`hermes --tui`). Поведение одинаково — одни и те же слэш-команды, одно и то же определение тишины, тот же потоковый TTS и тот же фильтр ложных срабатываний. Кроме того, TUI пишет отладочные логи сбоев в `~/.hermes/logs/`, чтобы проблемы режима "нажми и говори" на необычных аудиобэкендах не исчезали без следа, а их можно было разобрать по полному стек-трейсу.

### Быстрый старт

Запустите CLI и включите голосовой режим:

```bash
hermes                # Запустить интерактивный CLI
```

Затем используйте внутри CLI следующие команды:

```
/voice          Включить или выключить голосовой режим
/voice on       Включить голосовой режим
/voice off      Выключить голосовой режим
/voice tts      Переключить TTS-вывод
/voice status   Показать текущее состояние
```

### Как это работает

1. Запустите CLI командой `hermes` и включите голосовой режим через `/voice on`
2. **Нажмите Ctrl+B** — прозвучит сигнал (880 Гц), начнётся запись
3. **Говорите** — на экране появится индикатор уровня аудио: `● [▁▂▃▅▇▇▅▂] ❯`
4. **Закончите говорить** — после 3 секунд тишины запись автоматически остановится
5. Прозвучат **два сигнала** (660 Гц), подтверждающие завершение записи
6. Аудио будет преобразовано в текст через Whisper и отправлено агенту
7. Если TTS включён, ответ агента будет озвучен
8. Запись **автоматически запускается снова** — можно говорить дальше, не нажимая ничего

Цикл продолжается, пока вы не нажмёте **Ctrl+B** во время записи (это выходит из непрерывного режима) или пока 3 последовательные записи не распознают речь.

:::tip
Клавишу записи можно изменить через `voice.record_key` в `~/.hermes/config.yaml` (по умолчанию: `ctrl+b`).
:::

### Определение тишины

Двухэтапный алгоритм определяет, когда вы закончили говорить:

1. **Подтверждение речи** — ждёт звук выше RMS-порога (200) не менее 0,3 секунды, терпя короткие провалы между слогами
2. **Обнаружение конца** — после подтверждения речи срабатывает через 3,0 секунды непрерывной тишины

Если в течение 15 секунд речи не обнаружено вообще, запись останавливается автоматически.

И `silence_threshold`, и `silence_duration` можно настроить в `config.yaml`. Также можно отключить сигналы начала/окончания записи через `voice.beep_enabled: false`.

### Потоковый TTS

Когда TTS включён, агент озвучивает ответ **по предложениям** по мере генерации текста — не нужно ждать весь ответ целиком:

1. Буферизует текстовые дельты в законченные предложения (минимум 20 символов)
2. Убирает markdown-разметку и блоки `<think>`
3. Генерирует и проигрывает аудио для каждого предложения в реальном времени

### Фильтр галлюцинаций

Whisper иногда генерирует фантомный текст из тишины или фонового шума («Thank you for watching», «Subscribe» и т. п.). Агент отфильтровывает такие случаи с помощью набора из 26 известных фраз-галлюцинаций на нескольких языках, а также regex-паттерна, который ловит повторяющиеся варианты.

## Голосовые ответы через gateway (Telegram и Discord)

Если вы ещё не настроили ботов для мессенджеров, сначала посмотрите платформенные руководства:
- [Руководство по настройке Telegram](../messaging/telegram.md)
- [Руководство по настройке Discord](../messaging/discord.md)

Запустите gateway, чтобы подключить Hermes к мессенджерам:

```bash
hermes gateway        # Запустить gateway (подключает все настроенные платформы)
hermes gateway setup  # Интерактивный мастер настройки для первого запуска
```

### Discord: личные сообщения и каналы

Бот поддерживает два режима общения в Discord:

| Режим | Как говорить | Нужно упоминание | Настройка |
|------|-------------|--------------|----------|
| **Личные сообщения (DM)** | Откройте профиль бота → «Message» | Нет | Работает сразу |
| **Канал сервера** | Пишите в текстовый канал, где бот присутствует | Да (`@botname`) | Бота нужно пригласить на сервер |

**DM (рекомендуется для личного использования):** просто откройте DM с ботом и пишите — упоминание не нужно. Голосовые ответы и все команды работают так же, как и в каналах.

**Каналы сервера:** бот отвечает только если вы упомянете его через `@` (например, `@hermesbyt4 hello`). Убедитесь, что в списке упоминаемых пользователей выбран **сам бот**, а не одноимённая роль.

:::tip
Чтобы отключить требование упоминания в каналах сервера, добавьте в `~/.hermes/.env`:
```bash
DISCORD_REQUIRE_MENTION=false
```
Или задайте конкретные каналы как режим свободного ответа (без необходимости упоминания):
```bash
DISCORD_FREE_RESPONSE_CHANNELS=123456789,987654321
```
:::

### Команды

Они работают и в Telegram, и в Discord (в DM и текстовых каналах):

```
/voice          Включить или выключить голосовой режим
/voice on       Голосовые ответы только когда вы отправляете голосовое сообщение
/voice tts      Голосовые ответы для ВСЕХ сообщений
/voice off      Выключить голосовые ответы
/voice status   Показать текущее состояние
```

### Режимы

| Режим | Команда | Поведение |
|------|---------|----------|
| `off` | `/voice off` | Только текст (по умолчанию) |
| `voice_only` | `/voice on` | Озвучивает ответ только когда вы отправляете голосовое сообщение |
| `all` | `/voice tts` | Озвучивает ответ на каждое сообщение |

Настройка голосового режима сохраняется между перезапусками gateway.

### Доставка по платформам

| Платформа | Формат | Примечания |
|----------|--------|-----------|
| **Telegram** | Голосовое сообщение (Opus/OGG) | Проигрывается прямо в чате. ffmpeg конвертирует MP3 → Opus, если это требуется |
| **Discord** | Нативное голосовое сообщение (Opus/OGG) | Проигрывается как обычное голосовое сообщение пользователя. Если API голосовых сообщений не сработает, используется вложение-файл |

## Голосовые каналы Discord

Самый иммерсивный сценарий работы с голосом: бот заходит в голосовой канал Discord, слушает речь пользователей, распознаёт её, прогоняет через агента и отвечает в этот же канал.

### Настройка

#### 1. Права бота Discord

Если бот уже настроен для текста (см. [Руководство по настройке Discord](../messaging/discord.md)), нужно добавить ему права для голоса.

Зайдите в [Discord Developer Portal](https://discord.com/developers/applications) → ваше приложение → **Installation** → **Default Install Settings** → **Guild Install**:

**Добавьте к уже существующим текстовым правам следующие разрешения:**

| Разрешение | Назначение | Нужно |
|-----------|------------|------|
| **Connect** | Подключаться к голосовым каналам | Да |
| **Speak** | Проигрывать TTS-аудио в голосовых каналах | Да |
| **Use Voice Activity** | Определять, когда пользователи говорят | Рекомендуется |

**Обновлённое числовое значение прав:**

| Уровень | Число | Что включено |
|-------|-------|--------------|
| Только текст | `274878286912` | Просмотр каналов, отправка сообщений, чтение истории, embeds, attachments, threads, реакции |
| Текст + голос | `274881432640` | Всё выше + Connect, Speak |

**Пригласите бота заново** с обновлённой ссылкой на permissions:

```text
https://discord.com/oauth2/authorize?client_id=YOUR_APP_ID&scope=bot+applications.commands&permissions=274881432640
```

Замените `YOUR_APP_ID` на Application ID из Developer Portal.

:::warning
Если пригласить бота повторно на уже существующий сервер, его права обновятся, но он не будет удалён. Вы не потеряете ни данные, ни конфигурацию.
:::

#### 2. Привилегированные Gateway Intents

В [Developer Portal](https://discord.com/developers/applications) → ваше приложение → **Bot** → **Privileged Gateway Intents** включите все три:

| Intent | Назначение |
|--------|------------|
| **Presence Intent** | Определять статус пользователя online/offline |
| **Server Members Intent** | Преобразовывать имена пользователей из `DISCORD_ALLOWED_USERS` в числовые ID (при необходимости) |
| **Message Content Intent** | Читать содержимое текстовых сообщений в каналах |

**Message Content Intent** обязателен. **Server Members Intent** нужен только если список `DISCORD_ALLOWED_USERS` содержит имена пользователей — если вы используете числовые ID, его можно оставить выключенным. Сопоставление SSRC → user_id в голосовых каналах приходит из Discord SPEAKING opcode по voice websocket и **не требует** Server Members Intent.

#### 3. Кодек Opus

Библиотека кодека Opus должна быть установлена на машине, где работает gateway:

```bash
# macOS (Homebrew)
brew install opus

# Ubuntu/Debian
sudo apt install libopus0
```

Бот автоматически загружает кодек из:
- **macOS:** `/opt/homebrew/lib/libopus.dylib`
- **Linux:** `libopus.so.0`

#### 4. Переменные окружения

```bash
# ~/.hermes/.env

# Discord bot (already configured for text)
DISCORD_BOT_TOKEN=your-bot-token
DISCORD_ALLOWED_USERS=your-user-id

# STT — локальному провайдеру ключ не нужен (pip install faster-whisper)
# GROQ_API_KEY=your-key            # Альтернатива: облачная, быстрая, есть free tier

# TTS — необязательно. Edge TTS и NeuTTS не требуют ключа.
# ELEVENLABS_API_KEY=***      # Премиальное качество
# VOICE_TOOLS_OPENAI_KEY=***  # OpenAI TTS / Whisper
```

### Запуск gateway

```bash
hermes gateway        # Запуск с существующей конфигурацией
```

Бот должен появиться в Discord через несколько секунд.

### Команды

Используйте их в текстовом канале Discord, где присутствует бот:

```
/voice join      Бот подключается к текущему голосовому каналу
/voice channel   Псевдоним для /voice join
/voice leave     Бот отключается от голосового канала
/voice status    Показать режим голоса и подключённый канал
```

:::info
Перед выполнением `/voice join` вы должны уже находиться в голосовом канале. Бот подключается к тому же голосовому каналу, в котором находитесь вы.
:::

### Как это работает

Когда бот подключается к голосовому каналу, он:

1. **Слушает** аудиопоток каждого пользователя отдельно
2. **Определяет тишину** — 1,5 секунды тишины после минимум 0,5 секунды речи запускают обработку
3. **Транскрибирует** аудио через Whisper STT (локально, Groq или OpenAI)
4. **Прогоняет** текст через полный агентный пайплайн (сеанс, инструменты, память)
5. **Озвучивает** ответ обратно в голосовом канале через TTS

### Интеграция с текстовым каналом

Когда бот находится в голосовом канале:

- Расшифровки появляются в текстовом канале: `[Voice] @user: what you said`
- Ответы агента отправляются как текст в канал **и** озвучиваются в голосовом канале
- Текстовый канал — это канал, в котором была выполнена команда `/voice join`

### Защита от эха

Бот автоматически приостанавливает прослушивание собственного аудиовыхода на время TTS-ответа, чтобы не слышать и не переобрабатывать собственную речь.

### Контроль доступа

Только пользователи, перечисленные в `DISCORD_ALLOWED_USERS`, могут взаимодействовать через голос. Аудио остальных пользователей игнорируется.

```bash
# ~/.hermes/.env
DISCORD_ALLOWED_USERS=284102345871466496
```

## Справочник конфигурации

### config.yaml

```yaml
# Voice recording (CLI)
voice:
  record_key: "ctrl+b"            # Клавиша для старта/остановки записи
  max_recording_seconds: 120       # Максимальная длина записи
  auto_tts: false                  # Автоматически включать TTS при старте голосового режима
  beep_enabled: true               # Проигрывать сигналы начала/окончания записи
  silence_threshold: 200           # Уровень RMS (0-32767), ниже которого считается тишина
  silence_duration: 3.0            # Количество секунд тишины перед автостопом

# Speech-to-Text
stt:
  enabled: true                     # set to false to skip auto-transcription —
                                    # the gateway still caches the audio file and
                                    # passes its path to the agent as part of the
                                    # inbound message, useful for custom pipelines
                                    # (diarization, alignment, archival, etc.)
  provider: "local"                  # "local" (free) | "groq" | "openai"
  local:
    model: "base"                    # tiny, base, small, medium, large-v3
  # model: "whisper-1"              # Устарело: используется, когда провайдер не задан

# Text-to-Speech
tts:
  provider: "edge"                 # "edge" (free) | "elevenlabs" | "openai" | "neutts" | "minimax"
  edge:
    voice: "en-US-AriaNeural"      # 322 voices, 74 languages
  elevenlabs:
    voice_id: "pNInz6obpgDQGcFmaJgB"    # Adam
    model_id: "eleven_multilingual_v2"
  openai:
    model: "gpt-4o-mini-tts"
    voice: "alloy"                 # alloy, echo, fable, onyx, nova, shimmer
    base_url: "https://api.openai.com/v1"  # необязательно: переопределяет самостоятельно размещаемый или OpenAI-совместимый endpoint
  neutts:
    ref_audio: ''
    ref_text: ''
    model: neuphonic/neutts-air-q4-gguf
    device: cpu
```

### Переменные окружения

```bash
# Speech-to-Text providers (local needs no key)
# pip install faster-whisper        # Free local STT — no API key needed
GROQ_API_KEY=...                    # Groq Whisper (fast, free tier)
VOICE_TOOLS_OPENAI_KEY=...         # OpenAI Whisper (paid)

# Дополнительные переопределения STT (необязательно)
STT_GROQ_MODEL=whisper-large-v3-turbo    # Override default Groq STT model
STT_OPENAI_MODEL=whisper-1               # Override default OpenAI STT model
GROQ_BASE_URL=https://api.groq.com/openai/v1     # Custom Groq endpoint
STT_OPENAI_BASE_URL=https://api.openai.com/v1    # Custom OpenAI STT endpoint

# Text-to-Speech providers (Edge TTS and NeuTTS need no key)
ELEVENLABS_API_KEY=***             # ElevenLabs (premium quality)
# VOICE_TOOLS_OPENAI_KEY above also enables OpenAI TTS

# Discord voice channel
DISCORD_BOT_TOKEN=...
DISCORD_ALLOWED_USERS=...
```

### Сравнение STT-провайдеров

| Провайдер | Модель | Скорость | Качество | Стоимость | API-ключ |
|----------|--------|---------|----------|----------|---------|
| **Local** | `base` | Быстро (зависит от CPU/GPU) | Хорошее | Бесплатно | Нет |

## Диагностика и устранение проблем

### «No audio device found» (CLI)

Не установлен PortAudio:

```bash
brew install portaudio    # macOS
sudo apt install portaudio19-dev  # Ubuntu
```

### Бот не отвечает в каналах Discord

В каналах сервера бот по умолчанию требует упоминания. Убедитесь, что вы:

1. Вводите `@` и выбираете **bot user** (с `#discriminator`), а не **роль** с тем же именем
2. Или используете DM — там упоминание не нужно
3. Или задаёте `DISCORD_REQUIRE_MENTION=false` в `~/.hermes/.env`

### Бот заходит в голосовой канал, но не слышит меня

- Проверьте, что ваш Discord user ID есть в `DISCORD_ALLOWED_USERS`
- Убедитесь, что вы не заглушены в Discord
- Боту нужен событие SPEAKING от Discord, прежде чем он сможет сопоставить ваш аудиопоток — начните говорить через несколько секунд после входа

### Бот слышит меня, но не отвечает

- Проверьте, доступен ли STT: установите `faster-whisper` (ключ не нужен) или задайте `GROQ_API_KEY` / `VOICE_TOOLS_OPENAI_KEY`
- Проверьте, что LLM-модель настроена и доступна
- Посмотрите логи gateway: `tail -f ~/.hermes/logs/gateway.log`

### Бот отвечает текстом, но не отвечает голосом в канале

- Возможно, провайдер TTS падает — проверьте API-ключ и квоту
- Edge TTS (бесплатный, без ключа) используется как запасной вариант по умолчанию
- Проверьте логи на ошибки TTS

### Whisper выдаёт мусорный текст

Фильтр галлюцинаций автоматически ловит большинство случаев. Если фантомные транскрипты всё ещё появляются:

- Перейдите в более тихую обстановку
- Подстройте `silence_threshold` в конфиге (чем выше, тем менее чувствительно)
- Попробуйте другую модель STT
