# Профили: несколько независимых агентов

Запускайте несколько независимых Hermes-агентов на одной машине — у каждого будет своя конфигурация, API-ключи, память, сеансы, навыки и состояние шлюза.

## Что такое профиль?

Профиль — это отдельный домашний каталог Hermes. У каждого профиля свой каталог со своими `config.yaml`, `.env`, `SOUL.md`, памятью, сеансами, навыками, cron-задачами и базой состояния. Профили позволяют держать разных агентов для разных задач — ассистент для кода, персональный бот, исследовательский агент — и при этом не смешивать их состояние Hermes.

Когда вы создаёте профиль, у него автоматически появляется собственная команда. Создайте профиль `coder`, и у вас сразу появятся `coder chat`, `coder setup`, `coder gateway start` и т. д.

## Быстрый старт

```bash
hermes profile create coder       # создаёт профиль и алиас команды "coder"
coder setup                       # настраивает API-ключи и модель
coder chat                        # начинает работу
```

Вот и всё. Теперь `coder` — отдельный профиль Hermes со своей конфигурацией, памятью и состоянием.

## Создание профиля

### Профиль с нуля

```bash
hermes profile create mybot
```

Создаёт свежий профиль с предустановленными навыками. Запустите `mybot setup`, чтобы настроить API-ключи, модель и токены шлюза.

Если вы хотите использовать этот профиль как исполнителя задач Kanban или чтобы оркестратор Kanban направлял на него работу, передайте `--description "<role>"` прямо при создании — тогда оркестратор будет понимать, в чём сильная сторона профиля:

```bash
hermes profile create researcher --description "Читает исходный код и внешнюю документацию, оформляет выводы."
```

Описание можно также задать или сгенерировать автоматически позже через `hermes profile describe` — подробности механизма оркестрации см. в [руководстве по Kanban](./features/kanban#auto-vs-manual-orchestration).

### Только clone config (`--clone`)

```bash
hermes profile create work --clone
```

Копирует в новый профиль `config.yaml`, `.env` и `SOUL.md` из текущего профиля. Те же API-ключи и модель, но новые сеансы и память. Чтобы изменить API-ключи, отредактируйте `~/.hermes/profiles/work/.env`; чтобы сменить личность, отредактируйте `~/.hermes/profiles/work/SOUL.md`.

### Клонировать всё (`--clone-all`)

```bash
hermes profile create backup --clone-all
```

Копирует **всё** — конфигурацию, API-ключи, личность, все воспоминания, полную историю сеансов, навыки, cron-задачи и плагины. Получается полный снимок профиля. Полезно для резервной копии или для форка агента, у которого уже есть контекст.

### Клонировать из конкретного профиля

```bash
hermes profile create work --clone --clone-from coder
```

:::tip Память Honcho и профили
Когда включён Honcho, `--clone` автоматически создаёт для нового профиля отдельного ИИ-напарника, но рабочее пространство пользователя остаётся общим. У каждого профиля формируются собственные наблюдения и своя идентичность. Подробности см. в [Honcho -- Multi-agent / Profiles](./features/memory-providers#honcho).
:::

## Использование профилей

### Алиасы команд

Каждый профиль автоматически получает командный псевдоним в `~/.local/bin/<name>`:

```bash
coder chat                    # чат с агентом coder
coder setup                   # настройка coder
coder gateway start           # запуск шлюза coder
coder doctor                  # проверка состояния coder
coder skills list             # список навыков coder
coder config set model.default anthropic/claude-sonnet-4
```

Этот командный псевдоним работает со всеми подкомандами `hermes` — под капотом это всего лишь `hermes -p <name>`.

### Флаг `-p`

Можно явно указать профиль для любой команды:

```bash
hermes -p coder chat
hermes --profile=coder doctor
hermes chat -p coder -q "hello"    # работает в любой позиции
```

### Профиль по умолчанию (`hermes profile use`)

```bash
hermes profile use coder
hermes chat                   # теперь работает с coder
hermes tools                  # настраивает tools профиля coder
hermes profile use default    # вернуться назад
```

Задаёт профиль по умолчанию, чтобы обычные команды `hermes` обращались именно к нему. По смыслу это похоже на `kubectl config use-context`.

### Как понять, какой профиль активен

CLI всегда показывает, какой профиль активен:

- **Строка приглашения**: `coder ❯` вместо `❯`
- **Баннер**: при запуске показывает `Profile: coder`
- **`hermes profile`**: показывает текущее имя профиля, путь, модель и статус шлюза

## Профили, рабочие пространства и песочницы

Профили часто путают с рабочими каталогами или песочницами, но это разные вещи:

- **Профиль** даёт Hermes собственный каталог состояния: `config.yaml`, `.env`, `SOUL.md`, сеансы, память, журналы, cron-задачи и состояние шлюза.
- **Рабочее пространство** или **рабочий каталог** — это место, откуда стартуют терминальные команды. Оно настраивается отдельно через `terminal.cwd`.
- **Песочница** ограничивает доступ к файловой системе. Профили **не** превращают агента в песочницу.

На стандартном локальном терминальном бэкенде агент по-прежнему видит ту же файловую систему, что и ваша учётная запись. Профиль не мешает ему читать каталоги за пределами каталога профиля.

Если нужен профиль, который стартует в конкретном каталоге проекта, задайте явный абсолютный `terminal.cwd` в его `config.yaml`:

```yaml
terminal:
  backend: local
  cwd: /absolute/path/to/project
```

Если на локальном бэкенде указать `cwd: "."`, это означает не каталог профиля, а тот каталог, из которого Hermes был запущен.

Ещё несколько важных замечаний:

- `SOUL.md` может направлять модель, но не задаёт границу рабочего пространства.
- Изменения в `SOUL.md` полностью подхватываются в новом сеансе. Уже открытые сеансы могут ещё работать со старой версией системной инструкции.
- Вопрос вроде "в каком каталоге ты находишься?" не является надёжной проверкой изоляции. Если вам нужен предсказуемый стартовый каталог для инструментов, задайте `terminal.cwd` явно.

## Запуск шлюза

У каждого профиля свой шлюз как отдельный процесс и свой токен бота:

```bash
coder gateway start           # запускает шлюз coder
assistant gateway start       # запускает отдельный шлюз assistant
```

### Разные токены бота

У каждого профиля свой файл `.env`. Настройте отдельный токен бота Telegram / Discord / Slack для каждого:

```bash
# Редактируем токены профиля coder
nano ~/.hermes/profiles/coder/.env

# Редактируем токены профиля assistant
nano ~/.hermes/profiles/assistant/.env
```

### Безопасность: блокировка по токену

Если два профиля случайно используют один и тот же токен бота, второй шлюз завершится понятной ошибкой с указанием конфликтующего профиля. Такой сценарий поддерживается для Telegram, Discord, Slack, WhatsApp и Signal.

### Постоянные сервисы

```bash
coder gateway install         # создаёт службу hermes-gateway-coder
assistant gateway install     # создаёт службу hermes-gateway-assistant
```

У каждого профиля своё имя сервиса. Они работают независимо.

## Настройка профилей

У каждого профиля есть свои:

- **`config.yaml`** — модель, провайдер, наборы инструментов и все настройки
- **`.env`** — API-ключи и токены ботов
- **`SOUL.md`** — личность и инструкции

```bash
coder config set model.default anthropic/claude-sonnet-4
echo "You are a focused coding assistant." > ~/.hermes/profiles/coder/SOUL.md
```

Если профиль должен по умолчанию работать в конкретном проекте, задайте ему ещё и `terminal.cwd`:

```bash
coder config set terminal.cwd /absolute/path/to/project
```

## Обновление

`hermes update` один раз подтягивает общий код и автоматически синхронизирует новые встроенные навыки во **все** профили:

```bash
hermes update
# → Код обновлён (12 commits)
# → Навыки синхронизированы: default (up to date), coder (+2 new), assistant (+2 new)
```

Изменённые пользователем навыки никогда не перезаписываются.

## Управление профилями

```bash
hermes profile list           # показать все профили со статусом
hermes profile show coder     # подробности по одному профилю
hermes profile rename coder dev-bot   # переименовать (обновляет псевдоним и службу)
hermes profile export coder   # экспортировать в coder.tar.gz
hermes profile import coder.tar.gz   # импортировать из архива
```

## Удаление профиля

```bash
hermes profile delete coder
```

Это остановит шлюз, удалит systemd/launchd-службу, уберёт псевдоним команды и сотрёт все данные профиля. Для подтверждения потребуется ввести имя профиля.

Используйте `--yes`, чтобы пропустить подтверждение: `hermes profile delete coder --yes`

:::note
Удалить профиль по умолчанию (`~/.hermes`) нельзя. Чтобы удалить всё, используйте `hermes uninstall`.
:::

## Автодополнение по Tab

```bash
# Bash
eval "$(hermes completion bash)"

# Zsh
eval "$(hermes completion zsh)"
```

Добавьте эту строку в `~/.bashrc` или `~/.zshrc`, чтобы автодополнение работало постоянно. После этого автодополнение по Tab умеет подсказывать имена профилей, подкоманды профилей и команды верхнего уровня.

## Как это работает

Профили используют переменную окружения `HERMES_HOME`. Когда вы запускаете `coder chat`, командный псевдоним передаёт `HERMES_HOME=~/.hermes/profiles/coder` ещё до старта Hermes. Поскольку в кодовой базе более 119 файлов вычисляют пути через `get_hermes_home()`, состояние Hermes автоматически привязывается к каталогу профиля: конфигурация, сеансы, память, навыки, база состояния, PID шлюза, журналы и cron-задачи.

Это отдельно от рабочего каталога терминала. Выполнение инструментов стартует из `terminal.cwd` (или из каталога запуска, если на локальном бэкенде используется `cwd: "."`), а не автоматически из `HERMES_HOME`.

Профиль по умолчанию — это просто `~/.hermes`. Никакой миграции не требуется: существующие установки работают точно так же.

## Профили как дистрибутивы

Профиль, который вы собрали на одной машине, можно упаковать как **git-репозиторий** и установить одной командой на другой машине — на своём ноутбуке, на ноутбуке коллеги или в обычной пользовательской среде. Общий пакет включает SOUL, конфигурацию, навыки, cron-задачи и MCP-подключения. Учётные данные, память и сеансы остаются привязаны к конкретной машине.

```bash
# Установить целого агента из git repo
hermes profile install github.com/you/research-bot --alias

# Обновить позже, когда автор выпустит новую версию (ваша память + .env сохраняются)
hermes profile update research-bot
```

Полное руководство см. в **[Дистрибутивы профилей: делитесь полноценным агентом](./profile-distributions.md)** — там есть авторинг, публикация, семантика обновлений, модель безопасности и сценарии использования.
