# here.now

here.now позволяет агентам публиковать сайты и хранить приватные файлы в облачном Drive.

У here.now две основные задачи:

- **Sites**: публиковать сайты и файлы по адресу `{slug}.here.now`.
- **Drives**: хранить приватные файлы агента в облачных папках.

## Создание сайта

```bash
PUBLISH="${HERMES_SKILL_DIR}/scripts/publish.sh"
bash "$PUBLISH" {file-or-dir} --client hermes
```

Команда возвращает live URL, например `https://bright-canvas-a7k2.here.now/`.

Внутри это трёхэтапный процесс: create/update -> upload files -> finalize. Сайт становится доступным только после успешного finalize.

Без API-ключа создаётся **анонимный сайт**, который истекает через 24 часа.
При сохранённом API-ключе сайт становится постоянным.

**Структура файлов:** для HTML-сайтов кладите `index.html` в корень публикуемой директории, а не в подпапку. Содержимое этой директории и становится корнем сайта. Например, публикуйте `my-site/`, где лежит `my-site/index.html`, а не родительскую папку, внутри которой находится `my-site/`.

Можно также публиковать обычные файлы без HTML. Один файл получает богатый авто-предпросмотр (изображения, PDF, видео, аудио). Несколько файлов превращаются в автоматически сгенерированный список каталогов с навигацией по папкам и галереей изображений.

## Обновление существующего сайта

```bash
PUBLISH="${HERMES_SKILL_DIR}/scripts/publish.sh"
bash "$PUBLISH" {file-or-dir} --slug {slug} --client hermes
```

Скрипт автоматически подхватывает `claimToken` из `.herenow/state.json` при обновлении анонимных сайтов. Чтобы переопределить токен, передайте `--claim-token {token}`.

Для обновления с аутентификацией нужен сохранённый API-ключ.

## Работа с Drive

Используйте Drive, когда пользователю нужно приватное облачное хранилище для файлов агента: документов, контекста, памяти, планов, ассетов, медиа, исследований, кода и всего остального, что должно сохраняться, но не публиковаться как сайт.

У каждого вошедшего аккаунта есть Drive по умолчанию с именем `My Drive`.

```bash
DRIVE="${HERMES_SKILL_DIR}/scripts/drive.sh"
bash "$DRIVE" default
bash "$DRIVE" ls "My Drive"
bash "$DRIVE" put "My Drive" notes/today.md --from ./notes/today.md
bash "$DRIVE" cat "My Drive" notes/today.md
bash "$DRIVE" share "My Drive" --perms write --prefix notes/ --ttl 7d
```

Для передачи между агентами используйте токены Drive с ограниченной областью действия. Если вам приходит блок `herenow_drive`, используйте его `token` как `Authorization: Bearer <token>` для `api_base`, учитывайте `pathPrefix`, если он задан, и сохраняйте ETag при записи. `pathPrefix` со значением `null` означает полный доступ к Drive. Если навык доступен, предпочитайте `drive.sh`; в противном случае вызывайте перечисленные операции API напрямую.

## Хранение API-ключа

Скрипт публикации читает API-ключ из следующих источников (первый найденный используется):

1. флаг `--api-key {key}` (только для CI/скриптов — в интерактивной работе избегайте его);
2. переменная окружения `$HERENOW_API_KEY`;
3. файл `~/.herenow/credentials` (рекомендуется для агентов).

Чтобы сохранить ключ, запишите его в файл учётных данных:

```bash
mkdir -p ~/.herenow && echo "{API_KEY}" > ~/.herenow/credentials && chmod 600 ~/.herenow/credentials
```

**ВАЖНО**: после получения API-ключа сохраните его сразу — выполните указанную команду сами. Не просите пользователя делать это вручную. Не передавайте ключ через CLI-флаги (например, `--api-key`) в интерактивных сеансах; предпочтительный способ хранения — файл учётных данных.

Никогда не коммитьте учётные данные или локальные state-файлы (`~/.herenow/credentials`, `.herenow/state.json`) в исходный код.

## Получение API-ключа

Чтобы перейти с анонимного режима (24 часа) на постоянные сайты:

1. Попросите у пользователя адрес электронной почты.
2. Запросите одноразовый код входа:

```bash
curl -sS https://here.now/api/auth/agent/request-code \
  -H "content-type: application/json" \
  -d '{"email": "user@example.com"}'
```

3. Скажите пользователю: "Проверьте почту. Вам должен прийти код входа от here.now — вставьте его сюда."
4. Проверьте код и получите API-ключ:

```bash
curl -sS https://here.now/api/auth/agent/verify-code \
  -H "content-type: application/json" \
  -d '{"email":"user@example.com","code":"ABCD-2345"}'
```

5. Сохраните возвращённый `apiKey` сами (не просите пользователя делать это):

```bash
mkdir -p ~/.herenow && echo "{API_KEY}" > ~/.herenow/credentials && chmod 600 ~/.herenow/credentials
```

## Файл состояния

После каждой операции создания или обновления скрипт записывает `.herenow/state.json` в рабочей директории:

```json
{
  "publishes": {
    "bright-canvas-a7k2": {
      "siteUrl": "https://bright-canvas-a7k2.here.now/",
      "claimToken": "abc123",
      "claimUrl": "https://here.now/claim?slug=bright-canvas-a7k2&token=abc123",
      "expiresAt": "2026-02-18T01:00:00.000Z"
    }
  }
}
```

Перед созданием или обновлением сайтов вы можете проверить этот файл, чтобы найти прежние slug.
Считайте `.herenow/state.json` только внутренним кэшем.
Никогда не представляйте этот локальный путь как URL и никогда не используйте его как источник истины для режима аутентификации, срока действия или claim URL.

## Что говорить пользователю

Для опубликованных сайтов:

- Всегда сообщайте `siteUrl` из текущего запуска скрипта.
- Читайте и учитывайте строки `publish_result.*` из stderr скрипта, чтобы определить режим аутентификации.
- Когда `publish_result.auth_mode=authenticated`: сообщите пользователю, что сайт **постоянный** и сохранён в его аккаунте. Claim URL не нужен.
- Когда `publish_result.auth_mode=anonymous`: сообщите пользователю, что сайт **истекает через 24 часа**. Укажите claim URL (если `publish_result.claim_url` не пустой и начинается с `https://`), чтобы сайт можно было сохранить навсегда. Предупредите, что claim-токены показываются только один раз и восстановить их нельзя.
- Никогда не предлагайте пользователю смотреть `.herenow/state.json`, чтобы узнать claim URL или режим аутентификации.

Для Drive:

- Не описывайте файлы Drive как публичные URL.
- Сообщайте, что содержимое Drive остаётся приватным, если не выдан scoped token.
- При предоставлении доступа другому агенту предпочитайте scoped token с узким `pathPrefix` и коротким TTL.

## Параметры `publish.sh`

| Флаг | Описание |
| ---------------------- | -------------------------------------------- |
| `--slug {slug}`        | Обновить существующий сайт вместо создания |
| `--claim-token {token}`| Переопределить claim token для анонимных обновлений |
| `--title {text}`       | Заголовок просмотра (для не-HTML сайтов) |
| `--description {text}` | Описание просмотра |
| `--ttl {seconds}`      | Задать срок жизни (только для authenticated) |
| `--client {name}`      | Имя агента для атрибуции (например, `hermes`) |
| `--base-url {url}`     | Базовый URL API (по умолчанию: `https://here.now`) |
| `--allow-nonherenow-base-url` | Разрешить отправку auth на нестандартный `--base-url` |
| `--api-key {key}`      | Переопределение API-ключа (предпочтительнее файл учётных данных) |
| `--spa`                | Включить SPA routing (обслуживать index.html для неизвестных путей) |
| `--forkable`           | Разрешить другим форкать этот сайт |
