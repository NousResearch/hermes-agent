# 创意构思

## 使用场景

当用户说"我想构建一些东西"、"给我一个项目创意"、"我无聊了"、"我应该做什么"、"激励我"，或任何"我有工具但没有方向"的变体时使用。适用于代码、艺术、硬件、写作、工具以及任何可以制作的东西。

通过创意约束生成项目创意。约束 + 方向 = 创造力。

## 工作原理

1. **从库中选择一个约束** — 随机，或与用户的领域/情绪匹配
2. **广义解释** — 编码提示可以变成硬件项目，艺术提示可以变成CLI工具
3. **生成3个满足约束的具体项目创意**
4. **如果他们选择一个，就构建它** — 创建项目、编写代码、交付

## 规则

每个提示都尽可能广义地解释。"这包含X吗？" → 是。提示提供方向和轻微约束。没有任一个，就没有创造力。

## 约束库

### 面向开发者

**解决你自己的痛点：**
构建你这周希望存在的工具。少于50行。今天交付。

**自动化烦人的事情：**
你的工作流程中最乏味的部分是什么？把它写成脚本。每天五分钟可以修复的问题，两小时搞定。

**应该存在的CLI工具：**
想想你一直希望能够输入的命令。`git undo-that-thing-i-just-did`。`docker why-is-this-broken`。`npm explain-yourself`。现在构建它。

**除了胶水什么都没有：**
完全使用现有API、库和数据集制作东西。唯一的原创贡献是你连接它们的方式。

**弗兰肯斯坦周：**
拿做X的东西，让它做Y。播放音乐的git仓库。生成诗歌的Dockerfile。发送表扬的cron任务。

**减法：**
在代码库崩溃之前你能移除多少？将工具削减到最小可行功能。删除直到只剩下本质。

**高概念，低执行：**
一个深刻的想法，懒散地执行。概念应该是精彩的。实现应该只需要一个下午。如果花了更长时间，那就是过度思考了。

### 面向创客和艺术家

**公然抄袭一些东西：**
选择你钦佩的东西——一个工具、一件艺术品、一个界面。从头重新创建。学习在于你的版本和他们之间的差距。

**一百万个某物：**
一百万既是很多，也不多。一百万像素是一张1MB的照片。一百万次API调用是一个星期二。一百万个任何东西在规模上都会变得有趣。

**制作一些会消亡的东西：**
一个每天失去一个功能的网站。健忘的聊天机器人。一个通往虚无的倒计时。一种腐烂、杀戮或放手的练习。

**做很多数学：**
生成几何、着色器高尔夫、数学艺术、计算折纸。是时候重新学习arcsin是什么了。

### 面向任何人

**文本是通用接口：**
构建只有文本作为接口的东西。没有按钮，没有图形，只有文字进文字出。文本几乎可以进入任何东西并从任何东西出来。

**从妙语开始：**
想一个会成为有趣句子的东西。逆向工作使其成真。"我教我的恒温器对我进行精神控制" → 现在构建它。

**敌对UI：**
制作有意痛苦使用的东西。要求47个条件的密码字段。每个标签都撒谎的表单。评判你命令的CLI。

**再做一次：**
记住一个旧项目。从头再做一次。不看原版。看看你思考方式改变了什么。

参见`references/full-prompt-library.md`获取跨交流、规模、哲学、转型等30+额外约束。

## 将约束与用户匹配

| 用户说 | 从中选择 |
|--------|----------|
| "我想构建一些东西"（无方向） | 随机 — 任何约束 |
| "我正在学习[语言]" | 公然抄袭、自动化烦人的事情 |
| "我想要奇怪的东西" | 敌对UI、弗兰肯斯坦周、从妙语开始 |
| "我想要有用的东西" | 解决你自己的痛点、应该存在的CLI、自动化烦人的事情 |
| "我想要美丽的东西" | 做很多数学、一百万个某物 |
| "我精疲力竭了" | 高概念低执行、制作会消亡的东西 |
| "周末项目" | 除了胶水什么都没有、从妙语开始 |
| "我想要挑战" | 一百万个某物，减法，再做一次 |

## 输出格式

```
## Constraint: [名称]
> [约束，一句话]

### Ideas

1. **[一句话推销]**
   [2-3句话：你将构建什么以及为什么它有趣]
   ⏱ [周末/一周/一个月] • 🔧 [技术栈]

2. **[一句话推销]**
   [2-3句话]
   ⏱ ... • 🔧 ...

3. **[一句话推销]**
   [2-3句话]
   ⏱ ... • 🔧 ...
```

## 示例

```
## Constraint: 应该存在的CLI工具
> 想想你一直希望能够输入的命令。现在构建它。

### Ideas

1. **`git whatsup` — 显示你离开时发生了什么**
   比较你最后的活动提交和HEAD，总结发生了什么，
   谁提交了，以及哪些PR合并了。像你仓库的早间站会。
   ⏱ 周末 • 🔧 Python, GitPython, click

2. **`explain 503` — 面向人类的HTTP状态码**
   管道任何状态码或错误消息，获取通俗易懂的解释，
   包含常见原因和修复方法。从精选数据库拉取，不是LLM。
   ⏱ 周末 • 🔧 Rust或Go，静态数据集

3. **`deps why <package>` — 为什么这在我的依赖树中**
   将传递依赖追溯到引入它的直接依赖。一次性回答
   "为什么我有47份lodash副本"。
   ⏱ 周末 • 🔧 Node.js，npm/yarn lockfile解析
```

用户选择一个后，开始构建 — 创建项目、编写代码、迭代。
