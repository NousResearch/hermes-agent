# 宝可梦玩家

使用 `pokemon-agent` 包通过无头模拟游玩宝可梦游戏。

## 使用场景
- 用户说"play pokemon"、"start pokemon"、"pokemon game"
- 用户询问宝可梦红、蓝、黄、火红等
- 用户想观看 AI 玩宝可梦
- 用户引用 ROM 文件（.gb、.gbc、.gba）

## 启动流程

### 1. 首次设置（克隆、虚拟环境、安装）
仓库是 GitHub 上的 NousResearch/pokemon-agent。克隆后
设置 Python 3.10+ 虚拟环境。使用 uv（速度更快）
创建虚拟环境并以可编辑模式安装包（含 pyboy 扩展）。
如果 uv 不可用，回退使用 python3 -m venv + pip。

在本机上已设置在 /home/teknium/pokemon-agent
并准备好虚拟环境 — 只需 cd 到该目录并 source .venv/bin/activate。

你还需要一个 ROM 文件。向用户索要。在本机上
roms/pokemon_red.gb 文件已存在于该目录中。
**切勿**下载或提供 ROM 文件 — 始终向用户索取。

### 2. 启动游戏服务器
在 pokemon-agent 目录中激活虚拟环境后，运行
pokemon-agent serve，--rom 指向 ROM，--port 9876。
在后台运行（使用 &）。
要从存档恢复，添加 --load-state 和存档名称。
等待 4 秒启动，然后通过 GET /health 验证。

### 3. 设置实时仪表板供用户观看
通过 localhost.run 使用 SSH 反向隧道，让用户可以在浏览器中查看
仪表板。使用 ssh 连接，将本地端口 9876 转发到 nokey@localhost.run 的远程端口 80。
将输出重定向到日志文件，等待 10 秒，然后在日志中 grep 查找 .lhr.life URL。
向用户提供带有 /dashboard/ 附加的 URL。
隧道 URL 每次都会变化 — 如果重新启动，请向用户提供新的 URL。

## 存档与读取

### 何时保存
- 每隔 15-20 回合游戏
- 在道馆战、劲敌遭遇或危险战斗前**务必**保存
- 在进入新城镇或洞穴之前
- 在任何不确定的操作之前

### 如何保存
POST /save 并附带描述性名称。好的示例：
before_brock、route1_start、mt_moon_entrance、got_cut

### 如何读取
POST /load 并附带存档名称。

### 列出可用存档
GET /saves 返回所有已保存的状态。

### 服务器启动时读取
启动服务器时使用 --load-state 标志自动加载存档。
这比启动后通过 API 加载更快。

## 动作参考
- press_a — 确认、对话、选择
- press_b — 取消、关闭菜单
- press_start — 打开游戏菜单
- walk_up/down/left/right — 移动一格
- hold_b_N — 按住 B 键 N 帧（用于快速跳过文字）
- wait_60 — 等待约 1 秒（60 帧）
- a_until_dialog_end — 反复按 A 直到对话框清除

## 经验教训

### 不断使用视觉
- 每移动 2-4 步截图一次
- 内存状态告诉你位置和 HP，但不会告诉你周围有什么
- 悬崖、栅栏、路牌、建筑门、NPC — 只能通过截图看到
- 向视觉模型提出具体问题："我北边一格是什么？"
- 当卡住时，在尝试随机方向之前始终先截图

### 场景切换需要额外的等待时间
当你走过一扇门或楼梯时，屏幕在地图过渡期间会变黑。
你**必须**等待过渡完成。在任何门/楼梯切换后
添加 2-3 个 wait_60 动作。不等待的话，位置读取
会是过时的，你会认为你还在旧地图中。

### 建筑出口陷阱
当你离开建筑时，你会出现在门的**正前方**。
如果你向北走，你会直接回到建筑里。始终先
向左或向右走 2 格，然后再向你想要的方向前进。

### 对话框处理
第一代文字逐字缓慢滚动。要加速跳过对话框，
按住 B 键 120 帧然后按 A 键。根据需要重复。按住 B
使文字以最大速度显示。然后按 A 推进到下一行。
a_until_dialog_end 动作检查内存中的对话框标志，但此标志
不能捕获所有文字状态。如果对话框似乎卡住了，使用手动
的 hold_b + press_a 模式并通过截图验证。

### 悬崖是单向的
悬崖（小悬崖边缘）只能向下跳（南方），不能向上爬（北方）。
如果被向北方走的悬崖挡住，你必须向左或向右
找到绕过去的缺口。使用视觉来识别缺口的方向。
明确向视觉模型提问。

### 导航策略
- 每次移动 2-4 步，然后截图检查位置
- 进入新区域时立即截图以定位
- 向视觉模型提问"到[目的地]应该走哪个方向？"
- 如果卡住 3 次以上，截图并完全重新评估
- 不要连续输入 10-15 个移动指令 — 你会走过头或卡住

### 逃跑战斗
在战斗菜单中，逃跑在右下角。要从默认光标位置（战斗，左上角）到达：
按向下然后向右将光标移动到逃跑，然后按 A。用 hold_b 包裹以加速跳过文字/动画。

### 战斗（战斗指令）
在战斗菜单中战斗在左上角（默认光标位置）。
按 A 进入技能选择，再按 A 使用第一个技能。
然后按住 B 以加速跳过攻击动画和文字。

## 战斗策略

### 决策树
1. 想要捕捉？ → 削弱然后扔精灵球
2. 不需要的野生宝可梦？ → 逃跑
3. 有属性优势？ → 使用特效技能
4. 没有优势？ → 使用最强的本系加成技能
5. HP 低？ → 切换宝可梦或使用伤药

### 第一代属性表（关键对战）
- 水系击败火系、地面系、岩石系
- 火系击败草系、虫系、冰系
- 草系击败水系、地面系、岩石系
- 电系击败水系、飞行系
- 地面系击败火系、电系、岩石系、毒系
- 超能力系击败格斗系、毒系（第一代中非常强势！）

### 第一代特性
- 特攻 = 特殊攻击和特殊防御共用一个数值
- 超能力系过于强大（幽灵系技能有 bug）
- 暴击率基于速度属性
- 束缚/绑紧使对手无法行动
- 蓄力技能 bug：降低暴击率而非提高

## 停止游戏
1. 通过 POST /save 保存游戏并附带描述性名称
2. 使用 PKM:PROGRESS 更新内存
3. 告诉用户："游戏已保存为 [name]！说'play pokemon'继续。"
4. 终止服务器和隧道后台进程
